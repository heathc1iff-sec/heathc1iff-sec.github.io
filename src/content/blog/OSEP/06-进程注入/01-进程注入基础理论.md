# 进程注入基础理论

---

## 【Claude的学习建议】

> **进程注入是OSEP的核心技术之一！**
>
> 学完这章，你将理解：
> - 为什么攻击者要把代码注入到其他进程
> - 进程注入的完整流程
> - 如何选择合适的目标进程
>
> **给零基础同学的话**：
> 进程注入听起来很高级，但原理其实很简单：
> 1. 打开别人的进程
> 2. 在别人的进程里申请一块内存
> 3. 把你的代码写进去
> 4. 让那个进程执行你的代码
>
> 就像是"借用别人的身份做事"。

---

## 概述

进程注入允许将代码注入到其他进程中执行，可以绕过安全检测并实现持久化。

> 【Claude的通俗解释】
>
> **什么是进程注入？**
>
> 想象一下这个场景：
> - 你是一个"间谍"（shellcode）
> - 你现在在Word进程里
> - 用户关闭Word，你就"死"了
> - 但如果你能"潜入"到explorer.exe（Windows资源管理器）里
> - 用户永远不会关闭explorer，你就能一直存活
>
> 这就是进程注入的核心目的：**让你的代码在别的进程里运行**

## 为什么需要进程注入？

### 问题

1. **进程关闭** - 用户关闭应用会终止 shell
2. **网络检测** - 安全软件检测异常网络通信
3. **进程监控** - 某些进程被重点监控

### 解决方案

- 注入到长期运行的进程（如 explorer.exe）
- 注入到常有网络通信的进程（如 svchost.exe）
- 创建隐藏进程并注入

## 核心概念

### 进程 vs 线程

| 概念 | 说明 |
|------|------|
| 进程 | 运行应用程序的容器，有独立虚拟内存空间 |
| 线程 | 执行代码的单元，共享进程的内存空间 |

> 【Claude的深入解释】
>
> **进程和线程的关系**：
>
> ```
> 进程 = 一个"公司"
> ├── 有自己的办公楼（内存空间）
> ├── 有自己的资源（文件句柄、网络连接等）
> └── 可以有多个员工（线程）
>
> 线程 = 公司里的"员工"
> ├── 在公司的办公楼里工作（共享进程内存）
> ├── 每个员工做自己的任务（执行代码）
> └── 员工之间可以共享资源
> ```
>
> **为什么这很重要？**
>
> 进程注入的本质是：
> 1. 进入别人的"公司"（OpenProcess）
> 2. 在他们的办公楼里租一间办公室（VirtualAllocEx）
> 3. 把你的"员工"（shellcode）放进去（WriteProcessMemory）
> 4. 让你的员工开始工作（CreateRemoteThread）
>
> 因为线程共享进程的内存，所以你的shellcode可以访问目标进程的所有资源！

### 进程注入流程

```
1. OpenProcess - 打开目标进程
   ↓
2. VirtualAllocEx - 在目标进程分配内存
   ↓
3. WriteProcessMemory - 写入 shellcode
   ↓
4. CreateRemoteThread - 创建远程线程执行
```

> 【Claude的图解】
>
> ```
> 你的进程                          目标进程 (explorer.exe)
> ┌─────────────────┐              ┌─────────────────────────┐
> │                 │              │                         │
> │  shellcode      │              │   原有代码              │
> │  [fc 48 83...]  │              │                         │
> │                 │              │                         │
> │                 │   ────────>  │   ┌─────────────────┐   │
> │  OpenProcess    │   1.打开     │   │                 │   │
> │  VirtualAllocEx │   2.分配     │   │  新分配的内存   │   │
> │  WriteProcess   │   3.写入     │   │  [fc 48 83...]  │   │
> │  CreateRemote   │   4.执行     │   │  (你的shellcode)│   │
> │                 │              │   └─────────────────┘   │
> └─────────────────┘              └─────────────────────────┘
>
> 结果：你的shellcode在explorer.exe里运行了！
> 即使你的进程退出，shellcode还在explorer里继续运行
> ```

## 安全权限

### 访问权限

OpenProcess 需要的权限：
- `PROCESS_CREATE_THREAD` - 创建线程
- `PROCESS_QUERY_INFORMATION` - 查询信息
- `PROCESS_VM_OPERATION` - 内存操作
- `PROCESS_VM_WRITE` - 写入内存
- `PROCESS_VM_READ` - 读取内存

### 完整性级别

| 级别 | 说明 |
|------|------|
| System | 最高级别，系统进程 |
| High | 管理员进程 |
| Medium | 普通用户进程 |
| Low | 受限进程（如浏览器沙箱） |

**规则**: 只能注入到相同或更低完整性级别的进程

> 【Claude的重要提醒】
>
> **完整性级别是进程注入的"天花板"！**
>
> ```
> 你的进程级别          能注入的目标
> ─────────────────────────────────────
> System (系统)    →    所有进程
> High (管理员)    →    High, Medium, Low
> Medium (普通)    →    Medium, Low
> Low (受限)       →    只能注入 Low
> ```
>
> **实际意义**：
> - 如果你的shell是普通用户权限（Medium）
> - 你不能注入到svchost.exe（通常是System级别）
> - 但可以注入到explorer.exe（通常是Medium级别）
>
> **如何查看进程的完整性级别？**
> - 使用Process Explorer工具
> - 或者用PowerShell：`whoami /groups | findstr Level`

## 核心 Win32 API

### OpenProcess

```csharp
[DllImport("kernel32.dll", SetLastError = true)]
public static extern IntPtr OpenProcess(
    uint processAccess,
    bool bInheritHandle,
    int processId);
```

**参数**:
- `processAccess` - 访问权限
- `bInheritHandle` - 是否继承句柄
- `processId` - 目标进程 ID

### VirtualAllocEx

```csharp
[DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
public static extern IntPtr VirtualAllocEx(
    IntPtr hProcess,
    IntPtr lpAddress,
    uint dwSize,
    uint flAllocationType,
    uint flProtect);
```

**参数**:
- `hProcess` - 进程句柄
- `lpAddress` - 分配地址（0 = 系统选择）
- `dwSize` - 分配大小
- `flAllocationType` - 分配类型
- `flProtect` - 内存保护

### WriteProcessMemory

```csharp
[DllImport("kernel32.dll", SetLastError = true)]
public static extern bool WriteProcessMemory(
    IntPtr hProcess,
    IntPtr lpBaseAddress,
    byte[] lpBuffer,
    int nSize,
    out IntPtr lpNumberOfBytesWritten);
```

**参数**:
- `hProcess` - 进程句柄
- `lpBaseAddress` - 目标地址
- `lpBuffer` - 要写入的数据
- `nSize` - 数据大小
- `lpNumberOfBytesWritten` - 实际写入字节数

### CreateRemoteThread

```csharp
[DllImport("kernel32.dll")]
public static extern IntPtr CreateRemoteThread(
    IntPtr hProcess,
    IntPtr lpThreadAttributes,
    uint dwStackSize,
    IntPtr lpStartAddress,
    IntPtr lpParameter,
    uint dwCreationFlags,
    IntPtr lpThreadId);
```

**参数**:
- `hProcess` - 进程句柄
- `lpStartAddress` - 线程起始地址（shellcode 地址）
- 其他参数通常为 0

## 目标进程选择

> 【Claude的实战建议】
>
> **选择目标进程是一门艺术！**
>
> 好的目标进程应该满足：
> 1. **长期运行** - 不会被用户关闭
> 2. **网络通信正常** - 不会因为网络连接被怀疑
> 3. **权限足够** - 你能注入进去
> 4. **不被重点监控** - 杀软不会特别关注

### 推荐目标

| 进程 | 优势 | 风险 |
|------|------|------|
| explorer.exe | 长期运行，用户不会关闭 | 可能被监控 |
| svchost.exe | 常有网络通信 | 需要 SYSTEM 权限 |
| notepad.exe | 简单，易于测试 | 用户可能关闭 |
| iexplore.exe | 网络通信正常 | 可能被关闭 |

> 【Claude的进阶技巧】
>
> **不同场景的最佳选择**：
>
> | 场景 | 推荐目标 | 原因 |
> |------|----------|------|
> | 测试/学习 | notepad.exe | 简单，容易观察 |
> | 普通用户权限 | explorer.exe | 长期运行，权限匹配 |
> | 管理员权限 | svchost.exe | 网络通信不可疑 |
> | 需要网络 | 浏览器进程 | 网络流量正常 |
>
> **Meterpreter的migrate命令**：
> ```
> meterpreter > migrate -N explorer.exe
> ```
> 这个命令就是在做进程注入！把你的shell迁移到explorer进程里

### 查找进程 ID

```csharp
// 方法 1: 按名称查找
Process[] processes = Process.GetProcessesByName("explorer");
int pid = processes[0].Id;

// 方法 2: 遍历所有进程
foreach (Process p in Process.GetProcesses())
{
    if (p.ProcessName == "explorer")
    {
        int pid = p.Id;
        break;
    }
}
```

## 注意事项

1. **架构匹配** - 32位进程只能注入32位进程
2. **权限检查** - 确保有足够权限
3. **杀软检测** - 进程注入是常见检测点
4. **稳定性** - 注入失败可能导致目标进程崩溃
