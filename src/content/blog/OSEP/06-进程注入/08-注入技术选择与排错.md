# 注入技术选择与排错

## 定位

进程注入不应该靠背代码模板硬冲。考试里更重要的是：你拿到什么权限、目标是什么位数、需要稳定还是隐蔽、当前被什么防护拦住。这个文件把选择和排错整理成一张考场判断表。

---

## 技术选择总表

| 场景 | 优先考虑 | 为什么 |
|---|---|---|
| 只需要验证本机内存执行 | 本地 Shellcode Runner | 变量少，先验证 payload 和网络 |
| 当前进程不稳定，例如 Office | 远程进程注入 | 迁移到更稳定的用户进程 |
| 需要复杂功能和导出函数 | DLL 注入 | 开发简单，功能组织清晰 |
| 不想让 DLL 落地 | 反射 DLL 注入 | 直接内存加载，减少文件痕迹 |
| 需要伪装成合法新进程 | Process Hollowing | 进程名合法，但实现更复杂 |
| 只拿到脚本执行能力 | PowerShell/C# 反射加载 | 先恢复执行能力，再考虑注入 |

---

## 目标进程选择

优先选择同时满足这些条件的进程：

| 条件 | 原因 |
|---|---|
| 与 payload 位数一致 | 避免 x86/x64 入口和结构体问题 |
| 当前用户有访问权限 | `OpenProcess` 不会被权限直接拒绝 |
| 进程稳定长期存在 | 避免目标退出导致会话丢失 |
| 网络行为不突兀 | 回连更像正常业务流量 |
| 不是受保护进程 | 避免 PPL、EDR、自保护机制 |
| 所在会话合适 | 桌面交互、网络凭证、用户上下文更一致 |

不建议一上来就选：

```
├── lsass.exe：高价值但高风险，权限和防护要求高
├── 安全软件进程：容易触发自保护
├── 系统关键服务：崩溃影响大，证据也难解释
└── 不同架构进程：排错成本会飙升
```

---

## 经典注入链检查点

```
1. OpenProcess
   └── 我是否拿到足够权限的进程句柄？

2. VirtualAllocEx
   └── 远程内存是否申请成功，大小是否正确？

3. WriteProcessMemory
   └── 写入字节数是否等于 payload 长度？

4. VirtualProtectEx
   └── 内存权限是否从可写调整到可执行？

5. CreateRemoteThread
   └── 线程是否创建成功，入口地址是否正确？

6. 网络与结果
   └── 监听器、代理、防火墙、DNS 是否允许回连？
```

排错时记录每一步的返回值。不要只写“注入失败”，要写失败在第几步。

---

## 常见失败与处理

| 失败现象 | 优先怀疑 | 检查方向 |
|---|---|---|
| `OpenProcess` 返回空 | 权限不足或目标受保护 | 换同用户进程、提升权限、避开 PPL |
| `VirtualAllocEx` 失败 | 句柄权限不够或大小异常 | 检查 access mask、payload 长度 |
| 写入字节数不完整 | 内存地址或权限问题 | 检查返回值和 `GetLastError` |
| 线程创建成功但无回连 | 网络、payload、入口点 | 先本地 runner 验证 payload |
| 目标进程崩溃 | 架构错、shellcode 不稳定 | 换同架构 payload 和更稳目标 |
| 本地能跑远程不行 | 远程上下文差异 | 会话、权限、代理、环境变量 |
| 被 Defender/EDR 拦 | 行为或静态特征 | 先定位是写入、执行还是网络阶段 |
| 只在某台机器失败 | 环境差异 | 位数、补丁、策略、防护、代理 |

---

## 位数与会话排错

| 检查项 | 判断 |
|---|---|
| OS 位数 | 64 位系统不代表当前进程一定是 64 位 |
| Runner 位数 | 编译目标 x86/x64/AnyCPU 会影响行为 |
| Payload 位数 | 必须和执行进程匹配 |
| 目标进程位数 | 远程注入时尤其关键 |
| Session ID | 服务会话、用户桌面会话、RDP 会话不同 |

最小排错输出建议：

```
当前进程位数：
OS 位数：
当前 PID：
目标 PID：
目标进程名：
当前用户：
完整性级别：
Session ID：
```

## 报告证据

进程注入类步骤至少保留：

| 证据 | 说明 |
|---|---|
| 当前身份和权限 | 证明为什么能执行 |
| 目标进程信息 | PID、进程名、位数、用户 |
| 注入前后结果 | 回连、命令执行、权限上下文 |
| 失败分支 | 如果换过目标，说明原因 |
| 防护绕过说明 | 只写必要细节，避免报告像调试日志 |

□ 我有 x86/x64 两套 payload 和 runner 吗？
