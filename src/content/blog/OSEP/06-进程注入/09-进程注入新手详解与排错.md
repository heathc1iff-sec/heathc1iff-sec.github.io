# 进程注入新手详解与排错

## 对应课程范围

本文件对应 `6：进程注入迁移挖空.html`，用于把进程注入、DLL 注入、反射 DLL 注入和进程空洞化整理成初学者能理解的学习闭环。

进程注入不是一个单独命令，而是一类“把代码放进别的进程执行”的技术集合。

---

## 一句话理解

进程注入的本质是：

```
找到目标进程
  -> 获得操作权限
  -> 在目标进程内准备内存或模块
  -> 写入代码或路径
  -> 让目标进程执行
```

不同注入技术只是“写什么”和“怎么触发执行”不同。

---

## 核心概念

| 概念 | 说明 | 初学者重点 |
|---|---|---|
| 进程 | 正在运行的程序实例 | 每个进程有 PID、权限、架构、内存空间 |
| 句柄 | 操作系统给你的访问凭证 | 权限不够就不能写内存或开线程 |
| 远程内存 | 目标进程里的内存 | `VirtualAllocEx` 常用于分配 |
| 写入内存 | 把代码写进目标进程 | `WriteProcessMemory` 常用于写入 |
| 远程线程 | 让目标进程执行指定地址 | `CreateRemoteThread` 是典型方式 |
| DLL 注入 | 让目标进程加载 DLL | 常见思路是写入 DLL 路径并调用 `LoadLibrary` |
| 进程空洞化 | 创建暂停进程，替换映像后恢复 | 更复杂，排错也更多 |

## 经典远程线程注入流程

| 步骤 | API | 目的 | 常见失败 |
|---|---|---|---|
| 打开目标进程 | `OpenProcess` | 获取操作目标进程的句柄 | 权限不足、PID 错误、架构不匹配 |
| 分配远程内存 | `VirtualAllocEx` | 在目标进程准备空间 | 权限不足、大小错误 |
| 写入 payload | `WriteProcessMemory` | 把字节写进去 | 写入长度错误、句柄权限不足 |
| 修改权限 | `VirtualProtectEx` | 设置执行权限 | 权限常量错误 |
| 创建远程线程 | `CreateRemoteThread` | 触发执行 | 地址错误、EDR 拦截、架构错误 |

## 技术选择表

| 场景 | 优先考虑 | 原因 |
|---|---|---|
| 只需要验证概念 | 当前进程内执行 | 排错最少 |
| 有目标进程权限 | 远程线程注入 | 经典流程清晰 |
| 需要复用 DLL 能力 | DLL 注入 | 模块化，便于调试 |
| 不想 DLL 落地 | 反射 DLL 注入 | 减少文件痕迹 |
| 需要伪装新进程 | 进程空洞化 | 父进程和映像更复杂 |
| AppLocker 限制 EXE | DLL 或可信加载点 | 结合白名单绕过 |

## 常用检查命令

```powershell
Get-Process | Select-Object -First 10
Get-Process -Id <PID>
[Environment]::Is64BitProcess
[Environment]::Is64BitOperatingSystem
whoami /priv
```

```cmd
tasklist
tasklist /m
whoami /groups
```

这些命令用于判断目标进程、模块、权限和当前身份。

## 常见失败与排错

| 现象 | 优先排查 |
|---|---|
| `OpenProcess` 失败 | PID 是否存在、权限是否足够、是否受保护进程 |
| 写入失败 | 句柄权限、目标内存地址、payload 长度 |
| 创建线程后无效果 | 入口地址、内存权限、payload 架构 |
| 目标进程崩溃 | shellcode 架构、调用约定、退出方式 |
| 本地成功目标失败 | AV/EDR、权限、位数、目标进程选择 |
| DLL 注入失败 | DLL 路径、位数、导出函数、依赖库 |

## OSEP 考试重点

| 重点 | 你要能做到 |
|---|---|
| 解释 API 顺序 | 能说清每个 API 的目的 |
| 判断架构 | x86/x64 不匹配是最常见错误之一 |
| 选择目标进程 | 不要随便注入高价值或不稳定进程 |
| 记录证据 | 进程名、PID、当前身份、执行结果 |
| 联动绕过 | 注入常与 AV、AMSI、AppLocker 绕过结合 |
