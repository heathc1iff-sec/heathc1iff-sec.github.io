# JEA/JIT 考试速查与排错

## 对应课程小节

`18：ad内部攻击向量（JEA以及JIT策略）.html` 包含 JEA 理论、JEA 枚举与突破、JIT Administration 理论、JIT 枚举与利用。现有 `01-JEA-JIT利用.md` 已经覆盖基础流程，这份文件专门用于考试快速判断。

---

## 看到什么就想 JEA/JIT

| 信号 | 可能方向 |
|---|---|
| PowerShell 历史里出现 `-ConfigurationName` | JEA 端点 |
| `Get-PSSessionConfiguration` 有非默认配置 | JEA 端点 |
| 当前用户只能执行少数命令 | JEA 受限会话 |
| 可用命令包含 `Copy-Item`、`Start-Process`、`Invoke-*` | JEA 突破机会 |
| Web 管理站点能申请临时权限 | JIT |
| GPO 里出现临时本地管理员组 | JIT/PAM 相关 |
| 组名包含 `request`、`approve`、`jit`、`pam` | JIT 枚举方向 |

---

## JEA 判断链

```
发现 JEA 端点
  -> 当前用户是否能连接
  -> 进入后有哪些命令
  -> 语言模式是什么
  -> 命令参数是否有限制
  -> 是否能读/写敏感路径
  -> 是否能触发 DLL 劫持、文件覆盖、命令执行或凭证读取
```

---

## JEA 高风险命令

| 命令/能力 | 风险 |
|---|---|
| `Copy-Item` 无路径限制 | 可复制敏感文件或写入可加载位置 |
| `Start-Process` | 可能启动任意程序 |
| `Invoke-Expression` | 直接代码执行 |
| `Invoke-Command` | 远程执行扩展 |
| `New-Object` | COM/.NET 对象创建 |
| `Add-Type` | 编译并调用任意 .NET/Win32 |
| 可写服务目录 | DLL 劫持或替换组件 |
| 可读配置/备份 | 凭证发现 |

---

## JEA 排错

| 现象 | 排查 |
|---|---|
| 无法连接端点 | WinRM、端点名、权限、认证方式 |
| 进入后命令很少 | 这是正常现象，先 `Get-Command` 和看参数 |
| `Copy-Item` 失败 | 路径限制、Provider 限制、目标权限 |
| 写入成功但没有执行 | 需要服务重启、加载路径不对、架构不匹配 |
| DLL 劫持失败 | 目标不加载该 DLL、签名校验、目录权限 |
| 看不到配置文件 | 权限不足，尝试历史命令、GPO、端点枚举 |

---

## JIT 判断链

```
发现 JIT/PAM 线索
  -> 找请求入口和审批流程
  -> 找请求组、审批组和目标权限组
  -> 确认 GPO 把哪个组加到哪台机器的本地管理员
  -> 请求临时权限
  -> 刷新 Kerberos 票据
  -> 在时间窗口内访问目标
```

---

## JIT 常见坑

| 现象 | 排查 |
|---|---|
| 申请后权限没变 | 审批未通过、同步延迟、组策略未刷新 |
| `whoami /groups` 看不到组 | 旧票据未刷新，需要清票据或重新登录 |
| 本地管理员仍不可用 | GPO 没套到目标机器、组名不匹配 |
| 时间窗口过期 | 重新申请或缩短操作链 |
| 只能访问 Web 不能提权 | 找请求参数、审批人、目标组、后台接口 |

---

## JIT 三类角色速查

| 角色 | 你要确认什么 | 成功后的验证 |
|---|---|---|
| 请求者组 | 当前用户是否能发起申请，申请范围是什么 | Web/API/日志里出现申请记录 |
| 审批者组 | 谁能批准、能否绕过或影响审批流程 | 申请状态从 pending 变成 approved |
| 目标权限组 | 最终被加入哪个域组或本地组 | `whoami /groups`、本地 Administrators、目标资源访问 |

JIT 申请成功后要刷新 Kerberos 票据或重新登录，否则旧票据里没有新组成员关系。报告里记录申请时间、批准时间、票据刷新动作、权限窗口和目标主机访问结果。

---

## 和其他路径的联动

| JEA/JIT 发现 | 下一步 |
|---|---|
| JEA 可读文件 | 找凭证、配置、备份、SSH Key |
| JEA 可写目录 | DLL 劫持、计划任务、服务配置 |
| JIT 给本地管理员 | 立即横向到目标并抓凭证/Token |
| JIT 只给应用权限 | 看应用配置、数据库连接串、服务账户 |
| JEA/JIT 暂时不成立 | 转 ADCS、ACL、MSSQL、委派 |

## 证据清单

| 阶段 | 证据 |
|---|---|
| 发现 | 端点名、历史命令、GPO 或 Web 入口 |
| 权限 | 当前用户可连接或可申请 |
| 限制 | 可用命令、语言模式、参数限制 |
| 利用 | 文件读写、权限变化、组成员变化 |
| 影响 | 访问目标机器、读取敏感数据或获得更高身份 |
