# ADCS 证书攻击章节指南

## 这个目录是干什么的

ADCS 对应参考资料 `17：ad基于证书攻击（adcs）.html`，课程标题是 `Attacking Active Directory Certificate Services`。它不是单纯的“证书工具使用”，而是 AD 里一条非常重要的身份转换路径：

```text
普通域身份
  -> 枚举 CA 和证书模板
  -> 发现模板、权限或 HTTP 端点问题
  -> 申请或中继获得证书
  -> 用证书换取认证能力
  -> 访问 LDAP/WinRM/SMB/目标服务
  -> 继续提权、横向或 DCSync
```

初学者最容易卡在“我拿到 PFX 了，然后呢”。所以本目录的目标不是只列命令，而是帮你建立判断模型：谁能申请什么证书，这个证书为什么能代表某个身份，拿到后能访问哪里。

## 先建立的核心概念

| 概念 | 初学者理解 | 为什么重要 |
|---|---|---|
| CA | 颁发证书的服务 | 没有 CA，就没有 ADCS 攻击面 |
| 证书模板 | 规定谁能申请、申请什么用途、证书里能写什么身份 | 大多数 ADCS 风险都来自模板配置 |
| EKU | 证书用途，例如客户端认证 | 不是所有证书都能用于登录或冒充 |
| SAN/UPN | 证书里声明的主体身份 | 决定证书能否代表另一个用户 |
| 注册权限 | 谁能申请模板 | 普通用户可申请高价值模板时风险很高 |
| 审批要求 | 是否需要人工批准 | 需要审批通常不能直接利用 |
| PKINIT | 用证书向 Kerberos 换 TGT | 证书变成域认证能力的关键路径之一 |
| Schannel | TLS 客户端证书认证 | PKINIT 不通时可能还有 LDAP/证书认证路径 |
| HTTP 端点 | ADCS Web Enrollment 等服务 | 可能被 NTLM Relay 利用 |

把这些概念连起来看：模板决定证书内容和用途，CA 负责颁发证书，证书再通过 PKINIT 或 Schannel 变成访问能力。

## ADCS 正序判断链

```text
1. 我有没有域身份或能触发域内认证？
   没有身份时，先回到凭证、横向、MSSQL、Kiosk 或客户端入口。

2. 域里有没有 CA？
   记录 CA 名称、主机、证书服务角色、HTTP 端点。

3. 有哪些证书模板？
   记录模板名、启用状态、EKU、注册权限、审批要求、主题名控制。

4. 当前身份能申请什么？
   重点看 Domain Users、Authenticated Users、当前用户所在组。

5. 模板能不能代表高价值身份？
   关注 SAN/UPN、客户端认证、经理审批、授权签名、强映射限制。

6. 如果不能直接申请，能不能修改模板或对象？
   回到 ACL、WriteDACL、GenericAll、ESC4、影子凭证等路径。

7. 是否存在可中继 HTTP 端点？
   看 Web Enrollment / CES / CEP，以及 NTLM Relay 条件。

8. 拿到证书后怎么用？
   优先判断 PKINIT、Schannel、LDAP、WinRM、SMB 或转换为 ccache。
```

这条链可以帮你避免两个常见错误：看到 CA 就以为能打，拿到证书却不知道怎么变成访问能力。

## 初学者最该掌握的 4 类路径

| 路径 | 入口条件 | 成功后得到什么 | 重点文件 |
|---|---|---|---|
| 模板误配置直接申请 | 当前用户可注册危险模板，证书可用于客户端认证或可指定身份 | PFX、TGT、LDAP/服务访问能力 | `02-模板误配置与ESC判断.md`、`05-ADCS-ESC攻击决策树详解.md` |
| 修改模板后申请 | 当前用户对模板有写权限或能改关键属性 | 把原本不危险的模板改成可利用模板 | `02-模板误配置与ESC判断.md` |
| NTLM Relay 到 ADCS | 能强制目标认证，ADCS HTTP 端点可接收中继 | 机器或用户证书 | `03-NTLM中继到ADCS.md` |
| 证书转后续访问 | 已有 PFX/证书材料 | TGT、ccache、LDAP 修改、WinRM/SMB 访问 | `06-ADCS命令索引与附录映射.md` |

如果你还分不清 ESC 编号，先不要硬背。先按上表判断“我是直接申请、改模板、中继，还是已经拿到证书要用掉”。

## 拿到证书后的去向

| 你手里的材料 | 下一步判断 | 可能用途 |
|---|---|---|
| `.pfx` | 是否有密码，是否对应目标身份，是否能走 PKINIT | 换 TGT、生成 ccache、后续 Kerberos 认证 |
| 证书和私钥 | 是否能合成 PFX 或用于 Schannel | LDAP 证书认证、PassTheCert 类路径 |
| TGT/ccache | 是否能访问目标 SPN 或请求服务票据 | SMB、LDAP、WinRM、MSSQL、DCSync 前置 |
| 机器证书 | 机器账户能访问什么资源 | LDAP 修改、RBCD、目标主机横向、服务认证 |
| LDAP 访问 | 是否能修改对象属性或读取敏感信息 | 影子凭证、RBCD、组成员、ACL 链 |

这部分要和 `13-Windows凭证/06-Windows凭证新手闭环与材料去向.md` 一起看。证书不是终点，它只是另一种身份材料。

## 失败现象索引

| 现象 | 可能原因 | 优先看 |
|---|---|---|
| 枚举不到 CA | 当前 DNS/LDAP/域连通性问题，或域里没有 ADCS | `01-ADCS理论与枚举.md` |
| 有 CA 但没有危险模板 | 权限不足、模板未启用、审批严格、EKU 不适合 | `02-模板误配置与ESC判断.md` |
| 模板看起来危险但申请失败 | 注册权限不满足、CA 未发布模板、需要审批或签名 | `04-ADCS考试速查.md` |
| 申请成功但不能换 TGT | PKINIT 不支持、KDC/证书映射问题、时间/DNS 问题 | `06-ADCS命令索引与附录映射.md` |
| PFX 能用但访问目标失败 | 证书身份不等于目标权限，服务票据/SPN/协议不匹配 | `06-ADCS命令索引与附录映射.md` |
| NTLM Relay 没出证书 | EPA/HTTPS/模板选择/目标认证来源/中继条件不满足 | `03-NTLM中继到ADCS.md` |
| 拿到机器证书不知道做什么 | 先判断机器账户能否 LDAP 修改、RBCD 或访问本机资源 | `06-ADCS命令索引与附录映射.md` |
| 不知道算 ADCS 还是 ACL | 证书模板和 CA 属于本章；对象写权限和组路径回 `16-AD权限滥用` | `16-AD权限滥用/00-章节指南.md` |

## 学习目标

学完本目录后，你应该能回答：

1. 当前域里是否存在 ADCS，CA、模板、HTTP 端点分别在哪里。
2. 某个证书模板为什么危险，危险来自权限、用途、主题名、审批还是映射规则。
3. 为什么“能申请证书”和“能用证书认证成高价值身份”不是一回事。
4. 证书拿到后下一步该走 PKINIT、Schannel、WinRM/SMB，还是转向 LDAP 修改。
5. NTLM 中继到 ADCS 是否成立，失败时优先排查哪个条件。
6. 如何把 ADCS 路径写成“谁申请了什么证书、为什么能代表谁、产生什么影响”。
