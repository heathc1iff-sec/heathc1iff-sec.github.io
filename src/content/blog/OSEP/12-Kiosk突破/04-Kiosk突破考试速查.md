## Kiosk 的本质

Kiosk 环境不是“没有系统”，而是用户只被允许看见很窄的一层 UI。突破思路就是找 UI 到系统能力之间的缝：

```
浏览器 / 单应用界面
  -> 文件选择器 / URI Scheme / 打印 / 下载 / 帮助 / 配置
  -> 文件系统读写
  -> 命令执行或脚本执行
  -> 交互 Shell
  -> 权限提升
```

---

## 优先枚举

| 方向 | 例子 |
|---|---|
| 浏览器内部页 | `about:`、配置、下载、历史、扩展 |
| URI Scheme | `file://`、`irc://`、`mailto:`、自定义协议 |
| 文件对话框 | 地址栏、右键菜单、打开方式、路径跳转 |
| Profile | Firefox/Chrome 配置、书签、扩展、缓存 |
| 可写目录 | 用户目录、临时目录、配置目录 |
| 可执行组件 | 编辑器、脚本解释器、gtkdialog、终端模拟器 |
| 权限提升 | SUID、cron、窗口管理器、配置覆盖 |

## Linux Kiosk 判断链

```
能否打开浏览器内部页
  -> 能否读 file://
  -> 能否写文件
  -> 能否让某个程序解释该文件
  -> 能否形成交互命令执行
  -> 能否利用 cron/SUID/配置写入提权
```

常见机会点：

| 机会点 | 为什么 |
|---|---|
| Firefox Profile | 可创建新 Profile、改配置、找历史和缓存 |
| Scratchpad/保存功能 | 可能写入脚本或配置文件 |
| URI 协议处理 | 触发应用选择或外部程序 |
| gtkdialog | 可用 XML 定义简单交互界面 |
| Openbox/Xorg 配置 | 可能影响窗口/TTY/快捷键 |
| 符号链接 | 把可写路径导向高价值路径 |

---

## Windows Kiosk 判断链

```
单应用/浏览器限制
  -> 找文件对话框、打印、另存为、帮助、下载
  -> 进入文件系统视图
  -> 尝试 UNC、环境变量、shell: 路径、快捷方式
  -> 找可启动程序或脚本宿主
  -> 联动 AppLocker/凭证/横向移动
```

常见机会点：

| 机会点 | 说明 |
|---|---|
| `shell:` 路径 | 打开系统特殊目录 |
| UNC 路径 | 触发认证或访问攻击者共享 |
| 文件对话框地址栏 | 可能输入路径或程序 |
| 快捷方式 `.lnk` | 可能包装命令或路径 |
| 浏览器下载 | 落地文件到可控目录 |
| 辅助功能/热键 | 取决于系统配置和权限 |

## 模拟交互 Shell

Kiosk 场景里经常没有正常终端，课程重点不是“必须打开 cmd”，而是把有限输入输出变成可验证的命令执行。

| 限制 | 可尝试方向 | 记录重点 |
|---|---|---|
| 只能输入 URL/路径 | `file://`、UNC、协议处理器、下载目录 | 哪个入口触发了文件访问或认证 |
| 不能看到命令输出 | 写文件、DNS/HTTP 回连、时间延迟 | 输出落点、监听日志、时间戳 |
| 能写 Profile/配置 | 浏览器 Profile、书签、启动项、脚本配置 | 可写路径和下次触发条件 |
| 只能等计划任务触发 | cron hourly、启动脚本、定时器 | 当前时间、任务目录、触发后权限 |

考试可用总结：先证明“受限 UI -> 文件/网络/配置控制 -> 命令执行”这条链，不要一开始就追求完整交互。

## Cron Hourly 提权证据记录

| 证据 | 为什么要记 |
|---|---|
| 可写的 cron/脚本路径 | 证明入口条件 |
| 当前时间和计划触发间隔 | 解释为什么需要等待 |
| 写入前后的文件权限与内容摘要 | 证明可控性 |
| 触发后的 `id`/`whoami`/SUID 结果 | 证明权限变化 |
| 清理或恢复说明 | 报告里说明影响和复现边界 |

## 失败排查

| 现象 | 排查 |
|---|---|
| `file://` 被禁 | 换 URI、下载、打印、保存、Profile |
| 不能写文件 | 找临时目录、Profile、下载目录、缓存目录 |
| 写了文件不能执行 | 找解释器、协议处理器、计划任务、配置加载点 |
| 没有终端 | 自制交互窗口、反弹连接、利用编辑器执行 |
| 提权失败 | SUID、cron、服务、配置覆盖、可写目录权限 |

## 证据清单

| 阶段 | 证据 |
|---|---|
| 初始限制 | Kiosk 界面、可访问功能 |
| 突破 UI | URI、文件对话框、Profile 或配置入口 |
| 命令执行 | 当前用户、命令输出 |
| 提权 | 原权限、利用条件、新权限 |
| 影响 | 能访问的文件、凭证、网络或后续主机 |
