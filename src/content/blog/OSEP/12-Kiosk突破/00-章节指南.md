# Kiosk 突破章节指南

## 这个目录是干什么的

Kiosk 突破对应参考资料 `12：自助信息亭渗透.html`，课程标题是 `Kiosk Breakouts`。它讲的不是传统 Web 漏洞，也不是普通 Windows 提权，而是一个很特殊的场景：你面对的是一个被锁住的图形界面，用户看起来只能使用浏览器、信息亭应用或单一业务程序，但系统底层仍然有文件系统、协议处理器、配置文件、窗口管理器和权限边界。

对初学者来说，Kiosk 最容易被误解成“找一个快捷键打开终端”。这个理解太窄。真正的学习目标是把受限 UI 一步步转成可验证的能力：

```text
受限界面
  -> 找到可交互入口
  -> 证明能读文件
  -> 证明能写文件
  -> 找到可执行或可解释入口
  -> 获得命令执行
  -> 做本地枚举
  -> 进入提权、凭证或横向移动
```

只要你按这个链条复盘，就算一开始打不开终端，也不会完全没思路。

## 初学者先建立的 5 个概念

| 概念 | 你要理解成什么 | 卡住时怎么想 |
|---|---|---|
| UI 限制 | 只能点击某些窗口或网页，不代表系统能力不存在 | 我还能不能打开对话框、地址栏、下载框、打印框、帮助页 |
| 文件读取 | 能看到本地文件是突破链的第一层能力 | 能不能读 `/etc/passwd`、浏览器 profile、配置文件、历史记录 |
| 文件写入 | 能写入内容才可能进入执行、持久化或提权 | 下载、保存、scratchpad、书签、配置文件是否可写 |
| 执行入口 | 不一定是终端，也可能是解释器、协议处理器或应用选择器 | 哪些程序能被 UI 间接调用，参数能不能被控制 |
| 权限边界 | Kiosk 用户通常权限低，突破 UI 后仍要枚举提权 | 我现在只是有命令执行，还是已经有高权限 |

把 Kiosk 当成“能力升级题”会更稳：每一步都问自己现在多了什么能力，而不是只盯着一个 payload。

## 面对 Kiosk 时的第一轮枚举

第一次进入 Kiosk 场景，先记录所有能动的地方：

| 观察点 | 你要记录什么 | 可能通向哪里 |
|---|---|---|
| 键盘快捷键 | `Alt+Tab`、`Ctrl+L`、`Ctrl+O`、`Ctrl+P`、功能键是否有效 | 切窗口、地址栏、文件打开、打印、系统菜单 |
| 浏览器地址栏 | 是否能输入 `file://`、内部页、URI scheme | 本地文件读取、协议处理器、profile 配置 |
| 文件对话框 | 是否能浏览目录、右键、创建文件、输入 UNC | 文件读写、触发认证、打开程序 |
| 下载/保存功能 | 下载位置能否选择，文件名和扩展名能否控制 | 写入脚本、配置文件、快捷方式 |
| 帮助/关于/打印 | 是否能打开外部程序、系统对话框或路径 | 间接执行、路径跳转 |
| 应用选择器 | URI scheme 是否弹出“选择应用打开” | 指定解释器或系统程序 |
| 网络访问 | 能访问哪些端口、域名和内网地址 | 后续回连、代理、凭证捕获 |

记录时不要只写“失败”。要写清楚失败发生在哪一层：输入被拦、路径打不开、文件读不到、写入失败、程序不能执行、执行后没有回连。层次清楚，排错才不会乱。

## 从受限 UI 到命令执行的判断链

```text
1. 能不能改变输入？
   如果地址栏、搜索框、文件名、打印目标都不能控制，先找其他 UI 入口。

2. 能不能读取本地文件？
   能读文件说明沙箱或 UI 限制不等于文件系统完全隔离。

3. 能不能写入本地文件？
   能写入才可能准备脚本、配置、快捷方式或 profile。

4. 能不能让系统解释这个文件？
   关注浏览器 profile、URI scheme、gtkdialog、shell 路径、脚本宿主。

5. 能不能证明命令执行？
   先用低风险验证，例如写文件、回显、访问受控 HTTP，而不是一上来追求完整 shell。

6. 能不能稳定交互？
   如果没有交互 shell，就想办法做命令回显、文件回写、计划任务或 Web 请求确认。
```

这条链的价值是让你知道“下一步缺什么”。比如你能读文件但不能写文件，就不要急着研究提权；你能写文件但没有执行入口，就要回到协议处理器、应用选择器和配置加载点。

## Linux Kiosk 学习重点

Linux Kiosk 在课程里重点围绕 Porteus、Firefox、Openbox、profile、gtkdialog 和 cron 展开。学习时把它拆成四层：

| 层级 | 重点问题 | 推荐文件 |
|---|---|---|
| 浏览器层 | 能否打开内部页、profile、`file://`、URI scheme | `02-Porteus-Kiosk案例.md` |
| 文件层 | 能读哪些目录，能写哪里，写入后谁会加载 | `05-Kiosk从受限UI到命令执行闭环.md` |
| 执行层 | 是否能通过 gtkdialog、解释器、窗口管理器触发命令 | `02-Porteus-Kiosk案例.md` |
| 提权层 | cron、SUID、配置目录、TTY 是否能把低权限变高权限 | `02-Porteus-Kiosk案例.md`、`04-Kiosk突破考试速查.md` |

初学者常见误区是看到 Linux 就直接套普通提权清单。Kiosk 的入口不同：你可能没有终端，甚至没有稳定 shell，所以要先解决“如何输入命令、如何看到结果、如何保存文件”。

## Windows Kiosk 学习重点

Windows Kiosk 不一定有课程中的完整实操链，但知识库里已经把常见判断点整理出来。你主要学这几个方向：

| 方向 | 关注点 | 后续联动 |
|---|---|---|
| 文件对话框 | 地址栏、右键、打开方式、另存为、快捷方式 | `09-应用白名单绕过` |
| UNC 路径 | 是否会向攻击机发起认证请求 | `13-Windows凭证`、`15-MSSQL攻击` |
| `shell:` 路径 | 能否打开特殊目录或系统管理界面 | `14-横向移动` |
| 脚本宿主 | HTA、JScript、PowerShell、Office 是否可用 | `03-Office宏攻击`、`04-JScript攻击`、`08-AMSI绕过` |
| 白名单限制 | 能执行什么，不能执行什么 | `09-应用白名单绕过` |

Windows 方向最重要的是区分“UI 被锁”和“执行被策略锁”。如果能打开文件但不能运行程序，问题可能已经从 Kiosk 转到 AppLocker/WDAC/CLM；这时要回 `09-应用白名单绕过`，而不是继续在 Kiosk 里打转。

## 失败现象索引

| 现象 | 优先判断 | 推荐入口 |
|---|---|---|
| 快捷键全部没反应 | UI 层被严格限制，找浏览器内部入口或文件对话框 | `05-Kiosk从受限UI到命令执行闭环.md` |
| `file://` 打不开 | 浏览器策略或路径权限限制，换内部页、profile、下载/保存功能 | `02-Porteus-Kiosk案例.md` |
| 能读文件但不能保存 | 缺少写入点，找下载目录、scratchpad、书签、profile | `05-Kiosk从受限UI到命令执行闭环.md` |
| 能保存文件但不能运行 | 缺少解释/执行入口，检查 URI scheme、应用选择器、脚本宿主 | `01-Kiosk突破技术.md` |
| 命令执行无输出 | 先用写文件、HTTP 请求、DNS 请求证明执行 | `04-Kiosk突破考试速查.md` |
| 有执行但没回连 | 可能是网络过滤或防护拦截 | `10-网络过滤绕过`、`07-杀软绕过` |
| Linux 提权链断了 | 检查 cron 是否执行、权限位是否正确、路径是否可写 | `02-Porteus-Kiosk案例.md` |
| Windows 被策略阻止 | 进入 AppLocker/CLM/AMSI 判断 | `08-AMSI绕过`、`09-应用白名单绕过` |

## 学习目标

学完本章后，你应该能回答：

1. Kiosk 限制的是 UI、应用、文件系统、执行策略，还是权限。
2. 为什么文件读取、文件写入、命令执行是三层不同能力。
3. 浏览器内部页、URI scheme、文件对话框和 profile 分别能带来什么突破机会。
4. Linux Kiosk 中如何从 `file://`、profile、gtkdialog 走到命令执行。
5. Windows Kiosk 中如何利用文件对话框、UNC、`shell:` 路径和快捷方式继续枚举。
6. 为什么获得命令执行后还要继续做权限、网络和凭证枚举。
7. 如何把 Kiosk 过程写成报告里的证据链，而不是只写“绕过成功”。
