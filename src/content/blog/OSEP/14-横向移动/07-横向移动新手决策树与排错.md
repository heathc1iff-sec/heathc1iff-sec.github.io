# 横向移动新手决策树与排错

## 这篇文档解决什么问题

横向移动不是“拿到密码就 PsExec”。OSEP 里更重要的是判断：你手里的身份材料适合哪个协议、目标机器允许什么远程管理方式、失败到底是认证失败、权限不足、网络不通，还是防护拦截。

这篇文档把横向移动整理成一条新手决策树：

```text
我有什么材料
  -> 目标开放什么服务
  -> 材料能不能认证
  -> 认证后是否有远程执行/登录权限
  -> 选择 SMB/WMI/WinRM/RDP/DCOM/MSSQL/SSH
  -> 成功后重新枚举凭证和路径
```

只用于授权实验环境和 OSEP 备考。

---

## 横向移动的本质

横向移动要同时满足四件事：

| 条件 | 解释 |
|---|---|
| 有身份材料 | 密码、Hash、票据、Token、证书、SSH Key |
| 目标可达 | 网络、路由、防火墙、端口 |
| 认证方式匹配 | NTLM、Kerberos、证书、SSH、公钥 |
| 权限足够 | 本地管理员、远程管理组、RDP 权限、SQL 权限 |

缺任何一个条件，命令都可能失败。

---

## 总决策树

```text
准备横向移动
  |
  +-- 我有什么材料？
  |     |
  |     +-- 明文密码：优先 SMB/WinRM/RDP/MSSQL 验证
  |     +-- NTLM Hash：优先 SMB/WMI/PsExec，注意 NTLM 是否允许
  |     +-- Kerberos 票据：优先 Kerberos 服务，注意 SPN/时间/DNS
  |     +-- Token：先验证本地资源，再看是否能远程认证
  |     +-- SSH Key：走 Linux/DevOps 横向
  |
  +-- 目标开放什么？
  |     |
  |     +-- 445：SMB、PsExec、远程服务、文件复制
  |     +-- 135/高端口：WMI/DCOM
  |     +-- 5985/5986：WinRM/PowerShell Remoting
  |     +-- 3389：RDP
  |     +-- 1433：MSSQL
  |     +-- 22：SSH
  |
  +-- 认证成功了吗？
  |     |
  |     +-- 否：查凭证、域名、时间、协议、账号限制
  |     +-- 是：继续判断是否有执行或登录权限
  |
  +-- 有远程执行/登录权限吗？
        |
        +-- 管理员：WMI/PsExec/WinRM/DCOM
        +-- RDP 权限：RDP/Reverse RDP Proxy
        +-- SQL 权限：MSSQL 路线
        +-- 只有普通权限：枚举共享、配置、会话、AD 权限
```

## 先枚举目标服务

### Windows 常见端口

| 端口 | 服务 | 横向意义 |
|---|---|---|
| 445 | SMB | 文件共享、远程服务、PsExec、凭证验证 |
| 135 | RPC | WMI/DCOM 前置 |
| 5985 | WinRM HTTP | PowerShell Remoting |
| 5986 | WinRM HTTPS | 加密 Remoting |
| 3389 | RDP | 图形登录、RDP 会话、凭证风险 |
| 1433 | MSSQL | 数据库登录、xp_cmdshell、Linked Server |
| 88 | Kerberos | 票据相关 |
| 389/636 | LDAP/LDAPS | 域枚举、AD 对象操作 |

### 常用检查

```powershell
Test-NetConnection TARGET -Port 445
Test-NetConnection TARGET -Port 5985
Test-NetConnection TARGET -Port 3389
Test-NetConnection TARGET -Port 1433
```

```bash
nmap -Pn -p 445,135,3389,5985,5986,1433 TARGET
```

注意：端口开着只说明服务可达，不说明你有权限。

## 身份材料到协议的选择

| 材料 | 优先协议 | 说明 |
|---|---|---|
| 明文域账号 | SMB、WinRM、RDP、MSSQL、LDAP | 最灵活，先验证权限 |
| 本地管理员密码 | SMB、WMI、PsExec | 注意本地账号远程 UAC 限制 |
| NTLM Hash | SMB、WMI、PsExec | 不适合所有协议，注意 NTLM 限制 |
| Kerberos TGT | Kerberos 服务 | 需要 DNS、SPN、时间正确 |
| CIFS TGS | SMB 到特定主机 | 服务票据有目标限制 |
| Token | 本地/部分网络访问 | 网络认证可能受限制 |
| PFX/证书 | PKINIT/Schannel | 常接 ADCS 和 LDAP |
| SSH Key | SSH | Linux、DevOps、跳板机 |
| MSSQL 凭证 | TDS/MSSQL | 可能转代码执行或 SQL Link |

## Windows 横向路线

### SMB/PsExec

适合：

- 你有目标本地管理员权限。
- 445 可达。
- 允许创建服务或写 ADMIN$。

常见命令：

```bash
psexec.py DOMAIN/user:Password123!@TARGET
psexec.py -hashes :NTLMHASH DOMAIN/user@TARGET
```

排错：

| 现象 | 优先排查 |
|---|---|
| `STATUS_LOGON_FAILURE` | 凭证错误、域名错误 |
| `STATUS_ACCESS_DENIED` | 不是管理员、远程 UAC、共享权限 |
| ADMIN$ 不可写 | 权限不足、Admin share 禁用 |
| 服务创建失败 | 管理员权限、防护、服务控制管理器 |
| 连接但无 shell | payload 被拦、服务启动失败 |

### WMI

适合：

- 有管理员权限。
- RPC/WMI 可达。
- 希望少落地或使用系统管理接口。

常用命令：

```powershell
wmic /node:TARGET /user:DOMAIN\user /password:Password123! process call create "cmd.exe /c whoami > C:\Windows\Temp\wmi.txt"
```

```bash
wmiexec.py DOMAIN/user:Password123!@TARGET
wmiexec.py -hashes :NTLMHASH DOMAIN/user@TARGET
```

排错：

| 现象 | 排查 |
|---|---|
| RPC unavailable | 135/动态端口、防火墙 |
| Access denied | 管理员权限、UAC、WMI 权限 |
| 命令执行无输出 | 输出路径、权限、命令重定向 |
| 执行成功但无回连 | 网络过滤、AV、payload 架构 |

### WinRM

适合：

- 5985/5986 开放。
- 用户有远程管理权限。
- 想用 PowerShell Remoting。

命令：

```powershell
$cred = Get-Credential
Test-WSMan TARGET
Invoke-Command -ComputerName TARGET -Credential $cred -ScriptBlock { whoami }
Enter-PSSession -ComputerName TARGET -Credential $cred
```

排错：

| 现象 | 排查 |
|---|---|
| `Test-WSMan` 失败 | WinRM 未启用、防火墙 |
| 认证失败 | 凭证、域名、TrustedHosts、Kerberos |
| Access denied | 不在远程管理组或管理员组 |
| PowerShell 受限 | CLM、AMSI、AppLocker |

### RDP

适合：

- 需要图形界面。
- 用户有 RDP 登录权限。
- 目标允许交互式登录。

命令：

```cmd
cmdkey /generic:TERMSRV/TARGET /user:DOMAIN\user /pass:Password123!
mstsc /v:TARGET
```

注意：

| 点 | 说明 |
|---|---|
| RDP 会留下明显登录记录 | 考试要截图，但也要理解痕迹 |
| 需要允许远程桌面登录 | 管理员不一定能 RDP |
| 可能踢掉当前用户 | 谨慎操作 |
| RDP 凭证可能暴露 | 回看 `06-RDP与Fileless横向移动补充.md` |

### DCOM

适合：

- 有管理员权限。
- RPC 可达。
- 某些环境中 WMI/WinRM 不理想。

常见思路：

```powershell
$com = [activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application", "TARGET"))
$com.Document.ActiveView.ExecuteShellCommand("cmd.exe", $null, "/c whoami > C:\Windows\Temp\dcom.txt", "7")
```

排错重点：

- COM 对象是否存在。
- 远程激活权限。
- RPC 防火墙。
- 命令输出和执行路径。

## MSSQL 横向路线

当目标开放 1433 或你拿到数据库凭证时，不要只看数据库。MSSQL 可能提供：

- 服务账户身份。
- `xp_cmdshell` 操作系统命令执行。
- UNC 路径注入捕获 Net-NTLM。
- CLR 自定义程序集执行。
- Linked Server 横向移动。

入口：

```bash
mssqlclient.py DOMAIN/user:Password123!@TARGET -windows-auth
```

然后回看：`15-MSSQL攻击/07-MSSQL从凭证到代码执行学习闭环.md`。

## 横向成功后要做什么

很多初学者横向成功后会停下来截图。其实成功登录新主机只是新一轮循环开始。

新主机立刻做：

1. `whoami /all` 或 `id`。
2. 主机名、IP、域、路由。
3. 登录用户、会话、票据。
4. 防护状态。
5. 凭证材料。
6. 是否能访问更高价值资产。

记录模板：

| 新主机 | 进入方式 | 使用身份 | 当前权限 | 新发现 | 下一步 |
|---|---|---|---|---|---|
|  | WMI/WinRM/RDP/SSH/MSSQL |  |  |  |  |

## 失败排错总表

| 失败现象 | 更可能原因 | 下一步 |
|---|---|---|
| 端口不通 | 网络、防火墙、路由 | 换协议、查 pivot/隧道 |
| 认证失败 | 凭证错、域名错、认证方式不匹配 | 重新验证凭证材料 |
| 认证成功但 Access Denied | 权限不足、远程 UAC、组策略 | 查组、管理员、本地策略 |
| SMB 成功但 PsExec 失败 | 不能写 ADMIN$ 或不能创建服务 | 换 WMI/WinRM/DCOM |
| WinRM 不通 | 服务未启用或防火墙 | 换 WMI/SMB/RDP |
| RDP 拒绝 | 无 RDP 权限、NLA、策略限制 | 加组或换远程执行 |
| Kerberos 失败 | DNS、SPN、时间、票据过期 | 查 `klist`、时间、目标 FQDN |
| 执行成功但无回连 | 网络过滤、AV、payload 不匹配 | 回到 `10-网络过滤绕过` 和 `07-杀软绕过` |

## OSEP 考试相关重点

1. 横向移动要从“材料、协议、权限、网络”四个条件判断。
2. 不同材料适配不同协议，Hash、票据、Token、密码不能混用。
3. 成功认证不等于远程执行成功。
4. 每次横向成功都要重新枚举凭证、会话、网络和 AD 线索。
5. 报告要保留使用的身份、目标、协议、验证命令、成功结果和影响范围。
