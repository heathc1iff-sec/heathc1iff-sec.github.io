# AD 对象权限攻击决策树

## 对应课程范围

本文件补充 `16：ad间攻击向量.html` 中的 AD Object Security Permissions、GenericAll、WriteDACL、森林/信任与后续委派联动。

---

## 先把权限翻译成人话

| 权限 | 人话 | 常见动作 |
|---|---|---|
| GenericAll | 完全控制对象 | 改密码、加 SPN、写 RBCD、加组 |
| GenericWrite | 可写部分属性 | 写 SPN、登录脚本、RBCD/影子凭证视对象而定 |
| WriteDACL | 可改权限 | 给自己加 GenericAll 或 DCSync |
| WriteOwner | 可改所有者 | 先拿 Owner，再改 DACL |
| AddMember | 可加组成员 | 把自己加进高价值组 |
| ForceChangePassword | 可重置密码 | 接管用户 |
| AllExtendedRights | 扩展权限 | 可能包含 DCSync、重置密码等 |

---

## 对象类型决定打法

| 目标对象 | 优先打法 |
|---|---|
| 用户 | 改密码、加 SPN Kerberoast、影子凭证、登录脚本 |
| 组 | AddMember，把自己加入 |
| 计算机 | RBCD、影子凭证、读 LAPS、委派联动 |
| 域对象 | WriteDACL 到 DCSync |
| GPO | 修改策略，影响目标主机 |
| OU | 影响下级对象或 GPO 链接 |

---

## 决策树

```
发现我对某对象有权限
  -> 先确认对象类型
  -> 判断权限是否可直接利用
  -> 直接利用不行，能否升级成 GenericAll
  -> 能否转成凭证、票据、证书或本地管理员
  -> 利用后是否需要等待复制/GPO刷新/票据刷新
```

---

## 常见路径

| 发现 | 路径 |
|---|---|
| 对高价值用户 GenericAll | 重置密码或写影子凭证 |
| 对普通用户 GenericWrite | 加 SPN -> Kerberoast -> 破解 |
| 对组 AddMember | 加自己进组 -> 刷新 Token/重新登录 |
| 对计算机 GenericAll | RBCD -> S4U -> 目标机器本地管理员 |
| 对域对象 WriteDACL | 添加 DCSync 权限 -> secretsdump/mimikatz |
| 对模板/CA 相关对象可写 | 转 ADCS ESC4/模板滥用 |

---

## RBCD 判断重点

| 条件 | 说明 |
|---|---|
| 能创建机器账户 | `ms-DS-MachineAccountQuota` 大于 0 或已有机器账户 |
| 能写目标计算机的 RBCD 属性 | `msDS-AllowedToActOnBehalfOfOtherIdentity` |
| 能请求服务票据 | 需要机器账户密码/Hash |
| 目标服务可用 | 常用 CIFS/HOST/HTTP |

---

## 影子凭证判断重点

| 条件 | 说明 |
|---|---|
| 能写目标对象 `msDS-KeyCredentialLink` | 写入公钥凭据 |
| 域控支持 PKINIT | 否则证书无法换 TGT |
| 时间同步 | Kerberos 必须时间接近 |
| 目标身份价值 | 用户/机器权限决定后续能力 |

---

## 失败排查

| 现象 | 排查 |
|---|---|
| 写入成功但权限没变 | Token 未刷新、复制延迟、组策略未刷新 |
| RBCD 票据不能用 | SPN、DNS、时间、机器账户密码、目标服务 |
| DCSync 失败 | 权限没加到域根、ACE 错误、复制延迟 |
| 影子凭证失败 | PKINIT、KDC、证书、KeyCredential 格式 |
| BloodHound 显示可打但工具失败 | 当前路径权限、域名解析、工具版本 |
