## 通俗理解核心概念
### 编译型 vs 解释型语言
```plain
编译型语言（C、C++、C#）：
├── 先把代码"翻译"成机器能懂的文件（.exe）
├── 然后才能运行
├── 类比：先把中文书翻译成英文书，再给老外看
└── 优点：运行快，缺点：需要编译

解释型语言（Python、PowerShell、JavaScript）：
├── 边读边执行，不需要提前翻译
├── 类比：请一个翻译，边说边翻译给老外听
└── 优点：灵活，缺点：运行稍慢

为什么重要？
├── 渗透测试中两种都会用到
├── 编译型：写 Shellcode Runner、注入器
└── 解释型：写快速脚本、一次性工具
```

### 托管代码 vs 非托管代码
```plain
非托管代码（C、C++）：
├── 直接和操作系统打交道
├── 你要自己管理内存（申请、释放）
├── 类比：自己做饭，从买菜到洗碗都自己来
└── 更底层，更强大，但更容易出错

托管代码（C#、Java）：
├── 有一个"管家"帮你管理内存（.NET CLR）
├── 你不用操心内存问题
├── 类比：去餐厅吃饭，有人帮你服务
└── 更安全，更简单，但有些限制

为什么重要？
├── OSEP 主要用 C#（托管代码）
├── 但需要调用 Win32 API（非托管代码）
└── 需要学会两者之间的"桥梁"（P/Invoke）
```

### Win32 API - 核心
```plain
什么是 Win32 API？
├── Windows 提供的"服务接口"
├── 想让 Windows 做什么，就调用对应的 API
├── 类比：餐厅的菜单，想吃什么就点什么

常用的 API（必须记住）：
├── VirtualAlloc - 申请内存空间
├── VirtualProtect - 修改内存权限
├── CreateThread - 创建新线程
├── OpenProcess - 打开其他进程
├── WriteProcessMemory - 往其他进程写数据
└── CreateRemoteThread - 在其他进程创建线程

为什么重要？
├── 执行 Shellcode 需要这些 API
├── 进程注入需要这些 API
└── 几乎所有高级技术都基于这些 API
```

### Windows 注册表
```plain
什么是注册表？
├── Windows 的"配置数据库"
├── 存储系统和程序的各种设置
├── 类比：一本超级大的设置手册

重要的注册表位置：
├── HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
│   └── 开机自动运行的程序
├── HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
│   └── 当前用户登录时自动运行
└── HKLM\SYSTEM\CurrentControlSet\Services
    └── 系统服务配置

为什么重要？
├── 权限维持常用注册表
├── 很多配置可以通过注册表修改
└── 枚举系统信息需要读注册表
```

## 实践任务清单
### 任务1：用 Process Explorer 观察进程
```plain
步骤：
1. 下载并运行 Process Explorer
2. 找到 notepad.exe 进程
3. 双击查看详情：
   ├── Threads 标签 - 看有几个线程
   ├── TCP/IP 标签 - 看网络连接
   └── Handles 标签 - 看打开的文件/注册表

思考：
├── 为什么一个简单的记事本有这么多线程？
└── 它打开了哪些系统资源？
```

### 任务2：用 PowerShell 调用 Win32 API
```powershell
# 这是一个简单的例子：调用 MessageBox API

# 定义 API 签名
Add-Type @"
using System;
using System.Runtime.InteropServices;

public class Win32 {
    [DllImport("user32.dll", CharSet = CharSet.Auto)]
    public static extern int MessageBox(IntPtr hWnd, String text, String caption, int options);
}
"@

# 调用 API
[Win32]::MessageBox(0, "Hello from PowerShell!", "测试", 0)

# 运行这段代码，会弹出一个消息框
# 这就是 PowerShell 调用 Win32 API 的基本方法
```

### 任务3：探索注册表
```powershell
# 查看开机启动项
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"
Get-ItemProperty "HKCU:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"

# 查看已安装的程序
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\*" |
    Select-Object DisplayName, DisplayVersion

# 查看系统服务
Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Services\*" |
    Where-Object {$_.Start -eq 2} |  # 2 = 自动启动
    Select-Object PSChildName, ImagePath
```

### 任务4：写一个简单的 C# 程序调用 API
```csharp
// 在 Visual Studio 中创建一个 C# 控制台项目
// 这个程序会弹出一个消息框

using System;
using System.Runtime.InteropServices;

class Program
{
    // 声明要调用的 Win32 API
    [DllImport("user32.dll", CharSet = CharSet.Auto)]
    public static extern int MessageBox(IntPtr hWnd, String text, String caption, int options);

    static void Main(string[] args)
    {
        // 调用 API
        MessageBox(IntPtr.Zero, "Hello from C#!", "测试", 0);

        Console.WriteLine("按任意键退出...");
        Console.ReadKey();
    }
}
```

## 核心概念速记卡
```plain
┌─────────────────────────────────────────────────────┐
│ 进程 (Process)                                       │
│ = 正在运行的程序                                      │
│ = 有自己独立的内存空间                                │
│ = 可以包含多个线程                                    │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ 线程 (Thread)                                        │
│ = 进程中的执行单元                                    │
│ = 共享进程的内存空间                                  │
│ = CPU 调度的基本单位                                  │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ 虚拟内存 (Virtual Memory)                            │
│ = 每个进程都以为自己有 4GB 内存                       │
│ = 实际上是操作系统的"障眼法"                          │
│ = 进程之间的内存是隔离的                              │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ P/Invoke                                             │
│ = Platform Invocation Services                       │
│ = C# 调用 Win32 API 的方法                           │
│ = 用 [DllImport] 声明要调用的函数                    │
└─────────────────────────────────────────────────────┘
```
