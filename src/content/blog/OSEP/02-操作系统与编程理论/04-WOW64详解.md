# WOW64 详解

## 什么是 WOW64？

**WOW64**（Windows On Windows 64-bit）是 Windows 的一个子系统，允许 64 位 Windows 运行 32 位程序。

**为什么需要 WOW64？**
- 大量旧程序是 32 位的
- 企业不想重新开发所有软件
- 微软需要保持向后兼容性

---

## 32 位 vs 64 位

### 基本区别

| 特性 | 32 位 | 64 位 |
|------|-------|-------|
| 寄存器宽度 | 32 位 | 64 位 |
| 最大内存 | 4 GB | 16 EB（理论值） |
| 指针大小 | 4 字节 | 8 字节 |
| 寄存器名称 | EAX, EBX... | RAX, RBX... |

### 如何判断程序是 32 位还是 64 位？

**方法 1：任务管理器**
- 32 位程序后面会显示 "(32 位)"

**方法 2：PowerShell**
```powershell
# 检查进程
Get-Process notepad | Select-Object ProcessName,
    @{Name="Is64Bit";Expression={!$_.StartInfo.EnvironmentVariables["PROCESSOR_ARCHITEW6432"]}}
```

**方法 3：查看 PE 头**
- 32 位：Machine = 0x14c (IMAGE_FILE_MACHINE_I386)
- 64 位：Machine = 0x8664 (IMAGE_FILE_MACHINE_AMD64)

## WOW64 的工作原理

### 核心组件

WOW64 使用四个 64 位 DLL 来模拟 32 位环境：

| DLL | 作用 |
|-----|------|
| Ntdll.dll | 系统调用接口 |
| Wow64.dll | 核心模拟层，管理 32/64 位切换 |
| Wow64Win.dll | 处理 GUI 相关的系统调用 |
| Wow64Cpu.dll | 处理 CPU 状态切换 |

### 执行流程

```
32位程序调用 API
       ↓
32位 ntdll.dll（在 SysWOW64 中）
       ↓
Wow64.dll（转换层）
       ↓
64位 ntdll.dll（在 System32 中）
       ↓
Windows 内核
```

## 文件系统重定向

### 关键目录

| 目录 | 存放内容 |
|------|---------|
| C:\Windows\System32 | **64 位**系统文件和 DLL |
| C:\Windows\SysWOW64 | **32 位**系统文件和 DLL |

**注意**：名字很容易混淆！
- System**32** 存放 **64** 位文件
- SysWOW**64** 存放 **32** 位文件

### 文件系统重定向

当 32 位程序访问 `C:\Windows\System32` 时，WOW64 会自动重定向到 `C:\Windows\SysWOW64`。

**示例**：
```
32位程序请求: C:\Windows\System32\cmd.exe
实际访问的是: C:\Windows\SysWOW64\cmd.exe（32位版本）
```

### 禁用重定向

有时需要从 32 位程序访问真正的 System32：

```csharp
[DllImport("kernel32.dll")]
static extern bool Wow64DisableWow64FsRedirection(ref IntPtr ptr);

[DllImport("kernel32.dll")]
static extern bool Wow64RevertWow64FsRedirection(IntPtr ptr);

// 使用
IntPtr oldValue = IntPtr.Zero;
Wow64DisableWow64FsRedirection(ref oldValue);
// 现在可以访问真正的 System32
// ...
Wow64RevertWow64FsRedirection(oldValue);  // 恢复重定向
```

## 注册表重定向

### 32 位程序的注册表视图

32 位程序访问注册表时，某些键会被重定向到 `Wow6432Node` 子键。

**示例**：
```
32位程序访问: HKLM\SOFTWARE\Microsoft
实际访问的是: HKLM\SOFTWARE\WOW6432Node\Microsoft
```

### 受影响的注册表键

主要是 `HKLM\SOFTWARE` 下的键会被重定向。

## 渗透测试中的重要性

### 1. Shellcode 必须匹配架构

| 目标进程 | 需要的 Shellcode |
|---------|-----------------|
| 32 位进程 | 32 位 shellcode |
| 64 位进程 | 64 位 shellcode |

**错误示例**：
```
向 64 位 notepad.exe 注入 32 位 shellcode → 崩溃
```

### 2. 进程注入的限制

| 注入器 | 可注入的目标 |
|--------|-------------|
| 32 位注入器 | 只能注入 32 位进程 |
| 64 位注入器 | 只能注入 64 位进程 |

**解决方案**：
- 准备两个版本的注入器
- 或使用 Heaven's Gate 技术（从 32 位代码调用 64 位 API）

### 3. DLL 注入的限制

| DLL 架构 | 可加载到 |
|---------|---------|
| 32 位 DLL | 32 位进程 |
| 64 位 DLL | 64 位进程 |

### 4. 工具选择

| 目标 | 工具选择 |
|------|---------|
| 32 位目标 | 使用 32 位版本的 Mimikatz 等 |
| 64 位目标 | 使用 64 位版本 |

## 实用命令

### 检查系统架构

```powershell
# PowerShell
[Environment]::Is64BitOperatingSystem  # 系统是否64位
[Environment]::Is64BitProcess          # 当前进程是否64位
```

### 检查进程架构

```powershell
# 列出所有32位进程
Get-Process | Where-Object { $_.Path -like "*SysWOW64*" -or $_.Path -like "*Program Files (x86)*" }
```

### C# 中检查架构

```csharp
// 检查当前进程
if (IntPtr.Size == 4)
    Console.WriteLine("32位进程");
else if (IntPtr.Size == 8)
    Console.WriteLine("64位进程");

// 检查操作系统
if (Environment.Is64BitOperatingSystem)
    Console.WriteLine("64位操作系统");
```

## 本节要点总结

| 概念 | 要点 |
|------|------|
| WOW64 | 让 64 位 Windows 运行 32 位程序 |
| System32 | 存放 64 位文件（名字误导） |
| SysWOW64 | 存放 32 位文件 |
| 文件重定向 | 32 位程序访问 System32 会被重定向 |
| 注册表重定向 | 32 位程序访问 SOFTWARE 会被重定向到 Wow6432Node |
| 架构匹配 | Shellcode、DLL、注入器必须匹配目标架构 |
