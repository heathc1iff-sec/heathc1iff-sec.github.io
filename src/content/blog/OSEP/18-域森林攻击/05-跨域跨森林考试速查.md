# 跨域跨森林考试速查

## 对应课程范围

本文件对应 `16：ad间攻击向量.html` 中的 Active Directory Forest Theory、Trust in a Forest、Extra SID、Printers、Trust Between Forests、Linked SQL Servers in the Forest。

---

## 先判断信任方向

| 信任方向 | 含义 |
|---|---|
| Inbound | 对方信任我方，我方用户可能访问对方资源 |
| Outbound | 我方信任对方，对方用户可能访问我方资源 |
| Bidirectional | 双向互信 |

考试里不要只记录“有信任”，一定要记录方向、类型、是否跨森林、是否有 SID Filtering。

---

## 枚举优先级

1. 当前域 SID、域名、DC。
2. 当前森林根域和子域。
3. Trust Direction 和 Trust Type。
4. 对方域的 DC、DNS、可访问端口。
5. Enterprise Admins、Domain Admins、高价值组。
6. 是否能枚举对方 LDAP。
7. 是否存在 SQL Link、委派、ADCS、共享资源跨域。

---

## 常见跨域路径

| 发现 | 路线 |
|---|---|
| 子域拿到 krbtgt | Extra SID / SID History 到森林根 |
| 拿到 Trust Key | 构造跨域票据 |
| 跨森林 SQL Link | 通过 MSSQL 权限链进入另一森林 |
| 打印机/强制认证 | 诱导跨域认证，结合委派或 Relay |
| 双向信任 + 可用凭证 | 直接枚举对方域资源 |
| 对方域 ADCS 暴露 | 证书攻击跨边界 |

---

## Extra SID 判断

```
控制子域 krbtgt
  -> 获取父域 SID
  -> 构造包含 Enterprise Admins SID 的票据
  -> 访问森林根资源
```

卡点：

| 现象 | 排查 |
|---|---|
| 票据生成但不能访问 | SID 错、RID 错、SID Filtering |
| 找不到父域 SID | DNS/LDAP/信任枚举失败 |
| 访问服务失败 | SPN、服务名、目标主机名 |

---

## 跨森林注意点

跨森林比同森林更容易被 SID Filtering、选择性认证、名称解析和防火墙卡住。

| 检查 | 为什么 |
|---|---|
| DNS 能否解析对方域 | Kerberos 和 LDAP 都依赖 |
| 是否选择性认证 | 有信任不代表能访问所有主机 |
| SID Filtering | 影响 Extra SID |
| 可访问端口 | LDAP/SMB/Kerberos/MSSQL/RDP |
| 凭证适用范围 | 当前域凭证不一定能访问对方 |

---

## SQL Link 跨森林

Linked SQL Servers 常常绕过你对 AD 信任的直觉，因为 SQL 身份链可能已经配置好了跨边界访问。

记录每一跳：

| 记录 | 说明 |
|---|---|
| 源 SQL | 当前实例 |
| 目标 SQL | 链接实例 |
| 远端身份 | 链接过去后是谁 |
| 权限 | 是否 sysadmin |
| 可执行能力 | 查询、xp_cmdshell、CLR、UNC |
| 所属域 | 是否跨域/跨森林 |

| 阶段 | 证据 |
|---|---|
| 信任枚举 | 信任方向、类型、目标域 |
