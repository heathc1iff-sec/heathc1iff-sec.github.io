# 委派攻击考试速查

## 对应课程范围

本文件串联 `16：ad间攻击向量.html`、`18：ad内部攻击向量（JEA以及JIT策略）.html` 和 `组合起来的渗透流程.html` 中的无约束委派、约束委派、RBCD、S4U 攻击。

---

## 三类委派一句话

| 类型 | 人话 |
|---|---|
| 无约束委派 | 用户把 TGT 交给服务，服务可代表用户访问几乎任意服务 |
| 约束委派 | 服务只能代表用户访问指定 SPN |
| RBCD | 目标资源决定谁可以代表用户访问自己 |

---

## 枚举后看什么字段

| 字段/现象 | 说明 |
|---|---|
| `TRUSTED_FOR_DELEGATION` | 无约束委派 |
| `TRUSTED_TO_AUTH_FOR_DELEGATION` | 协议转换，约束委派常见 |
| `msDS-AllowedToDelegateTo` | 约束委派目标 SPN |
| `msDS-AllowedToActOnBehalfOfOtherIdentity` | RBCD |
| SPN | 服务账户身份和可请求服务 |
| 机器账户 Hash | 机器委派路径的关键材料 |

---

## 无约束委派判断链

```
发现无约束委派主机/账户
  -> 是否能控制该主机/账户
  -> 是否能诱导高价值用户或 DC 认证
  -> 是否能抓到 TGT
  -> 注入 TGT
  -> DCSync 或访问目标资源
```

卡点：

| 现象 | 排查 |
|---|---|
| 抓不到票据 | 没有高价值认证、触发失败、权限不足 |
| 票据不可用 | 过期、时间不同步、注入会话不对 |
| DCSync 失败 | 抓到的不是 DC/高权限身份 |

---

## 约束委派判断链

```
发现约束委派账户
  -> 是否拿到该账户密码/Hash/TGT
  -> 目标 SPN 是什么
  -> 是否支持协议转换
  -> S4U2Self + S4U2Proxy
  -> 使用服务票据访问目标服务
```

关键点：

| 点 | 说明 |
|---|---|
| 需要控制委派账户 | 没有凭证就不能直接打 |
| SPN 决定目标服务 | CIFS/LDAP/HOST/HTTP 影响后续能力 |
| `altservice` | 某些场景可换同主机服务类型 |
| 票据格式 | kirbi/ccache 需要转换 |

---

## RBCD 判断链

```
能写目标机器 RBCD 属性
  -> 准备可控机器账户
  -> 写入允许委派的安全描述符
  -> 用可控机器账户请求 S4U
  -> 冒充高权限访问目标服务
```

优先条件：

| 条件 | 说明 |
|---|---|
| 能创建机器账户 | 常见默认配额为 10 |
| 能写目标计算机属性 | ACL 路径或 GenericAll |
| 目标服务可访问 | CIFS 常用于落地和执行 |
| 时间/DNS 正常 | Kerberos 强依赖 |

## 委派失败排查总表

| 现象 | 排查 |
|---|---|
| `KRB_AP_ERR_SKEW` | 时间同步 |
| `KDC_ERR_S_PRINCIPAL_UNKNOWN` | SPN 写错或 DNS/FQDN 问题 |
| 票据有但访问失败 | 服务类型不匹配、权限不足 |
| RBCD 写入失败 | 没有写属性权限或安全描述符错误 |
| 约束委派不能 impersonate | 缺少协议转换或账户未被允许 |
| Impacket 和 Rubeus 结果不同 | 票据格式、域名、SPN、工具参数 |
