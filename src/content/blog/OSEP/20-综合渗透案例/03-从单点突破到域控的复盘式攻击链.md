# 从单点突破到域控的复盘式攻击链

## 文档目标

这篇文档不是某个固定靶机的 walkthrough，而是教你如何复盘一条 OSEP 攻击链。初学者要学会把每一步都拆成：

```
我有什么
  -> 我能验证什么
  -> 我能得到什么
  -> 失败后换什么
```

---

## 攻击链总览

| 阶段 | 目标 |
|---|---|
| 入口 | 获得第一次执行或凭证 |
| 稳定 | 确认身份、网络、防护 |
| 提权 | 获得本地更高权限或更高价值材料 |
| 凭证 | 收集密码、hash、ticket、token、证书 |
| 横向 | 扩大主机访问范围 |
| 域提权 | ACL、委派、ADCS、MSSQL、跨域 |
| 证据 | 保留报告需要的信息 |

---

## 阶段一：初始入口

| 入口 | 成功后立刻记录 |
|---|---|
| Office/JScript | 当前用户、主机名、出网路径 |
| Web RCE | 用户、目录、权限、网络 |
| MSSQL | SQL 身份、服务器权限、执行能力 |
| 凭证登录 | 协议、权限、目标 |
| Kiosk | 可执行能力、文件访问、用户上下文 |

入口失败时，不要只改 payload。先判断是执行、AV、AMSI、AppLocker 还是网络。

---

## 阶段二：主机内枚举

| 枚举项 | 目的 |
|---|---|
| 用户和组 | 判断权限 |
| 防护状态 | 判断执行限制 |
| 网络接口/路由 | 判断内网位置 |
| 域信息 | 找 DC、DNS、域名 |
| 进程和服务 | 找凭证和提权点 |
| 文件系统 | 找配置、密钥、脚本 |
| 票据和 token | 找可复用身份 |

---

## 阶段三：提权与凭证获取

| 发现 | 下一步 |
|---|---|
| 本地管理员 | LSASS/MiniDump、SAM、Token |
| 普通用户 | 配置文件、浏览器、计划任务、服务弱点 |
| Linux shell | SSH key、sudo、keytab、ccache |
| SQL 权限 | xp_cmdshell、Link、凭证表 |
| Token | 模拟访问资源 |

---

## 阶段四：域枚举和路径选择

| 线索 | 方向 |
|---|---|
| BloodHound ACL 边 | ACL 滥用 |
| 委派字段 | Kerberos 委派 |
| 证书服务 | ADCS |
| SQL Link | MSSQL 跨边界 |
| 跨域信任 | 域森林攻击 |
| JEA/JIT | 受限管理或临时权限 |
| Linux Kerberos | Linux 到 AD 横向 |

---

## 阶段五：横向移动

| 条件 | 路线 |
|---|---|
| 本地管理员 + SMB | PsExec/服务 |
| WinRM 可用 | PowerShell Remoting |
| RDP 凭证 | RDP |
| Kerberos 票据 | 对应 SPN 服务 |
| MSSQL 权限 | SQL 执行或 Link |
| SSH key | SSH |

失败时优先查：端口、协议、权限、本地管理员、UAC、目标防护、凭证是否过期。

---

## 阶段六：域控目标

常见目标不是“必须登录 DC”，而是证明你拥有域级影响：

| 方式 | 证据 |
|---|---|
| DCSync | 导出高权限 hash |
| 域控访问 | 当前身份、目标路径、flag |
| ADCS 高权限证书 | 证书身份、TGT、LDAP 操作 |
| ACL 域级权限 | 权限修改和验证 |
| 跨域控制 | 信任方向和访问证明 |

## OSEP 重点

1. 每一步都要有验证。
2. 每个失败都要归类。
3. 不要把工具输出当结论，要能解释影响。
4. 报告证据要同步收集，不要最后补。
