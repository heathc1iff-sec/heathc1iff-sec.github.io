# 综合渗透案例 - 新手章节指南

## 这个目录是干什么的？

这是整个课程的"毕业设计"。前面学的都是单个技术，这里教你如何把所有技术串联起来，完成一次完整的渗透测试。

---

## 为什么要学这个？

```
综合案例的价值：
├── 单个技术只是"零件"
├── 需要学会组装成"机器"
├── 真实渗透测试是连贯的过程
├── 考试也是综合性的
└── 这是检验学习成果的最好方式

学习目标：
├── 理解完整的攻击链
├── 学会根据情况选择技术
├── 培养渗透测试思维
└── 为考试做准备
```

综合案例的学习目标不是背 walkthrough，而是训练每 30 分钟能复盘一次：“我有什么、我验证了什么、我还缺什么、下一步去哪”。

## 完整渗透测试流程

### 阶段1：初始访问

```
目标：获得目标网络中一台机器的访问权限

常用方法：
├── 钓鱼邮件 + Office 宏
├── 钓鱼邮件 + HTA/JScript
├── 社会工程学
├── 外部漏洞利用
└── 物理访问

需要绕过：
├── 邮件网关（HTML Smuggling）
├── 杀毒软件（加密、混淆）
├── AMSI（内存补丁）
└── 应用白名单（LOLBAS）
```

### 阶段2：本地枚举

```
目标：了解当前机器和用户的情况

枚举内容：
├── 当前用户权限
├── 本地用户和组
├── 已安装的软件
├── 网络配置
├── 域信息（如果加入域）
└── 敏感文件

常用工具：
├── Seatbelt - 自动化枚举
├── PowerView - 域枚举
├── 手动命令
└── BloodHound 数据收集
```

### 阶段3：权限提升

```
目标：获得更高的权限（管理员/SYSTEM）

常用方法：
├── 本地漏洞利用
├── 服务配置错误
├── 计划任务滥用
├── 注册表权限问题
├── UAC 绕过
└── 凭证提取

工具：
├── PowerUp - 自动化检查
├── WinPEAS - 枚举脚本
└── 手动检查
```

### 阶段4：凭证提取

```
目标：获取可用于横向移动的凭证

提取位置：
├── LSASS 内存
├── SAM 数据库
├── 凭证管理器
├── 浏览器密码
├── 配置文件
└── Kerberos 票据

工具：
├── Mimikatz
├── Rubeus
└── 自定义工具
```

### 阶段5：横向移动

```
目标：访问网络中的其他机器

常用方法：
├── Pass-the-Hash
├── Pass-the-Ticket
├── WMI
├── WinRM
├── PsExec
├── DCOM
└── RDP

选择依据：
├── 可用的凭证类型
├── 目标机器的配置
├── 隐蔽性要求
└── 网络限制
```

### 阶段6：域渗透

```
目标：获得域管理员权限

攻击路径：
├── ACL 滥用
├── Kerberos 委派
├── MSSQL 攻击
├── 组策略滥用
├── DCSync
└── 金票/银票

工具：
├── BloodHound - 分析攻击路径
├── PowerView - 枚举
├── Rubeus - Kerberos 攻击
└── Mimikatz - 凭证操作
```

### 阶段7：持久化

```
目标：保持访问权限

常用方法：
├── 金票
├── 银票
├── 计划任务
├── 服务
├── 注册表
├── WMI 事件订阅
└── 域后门

注意：
├── 考试中可能不需要
├── 但实际渗透测试很重要
└── 了解即可
```

## 实战思维导图

```
初始访问
    │
    ├── 成功 → 本地枚举
    │              │
    │              ├── 是管理员？
    │              │      │
    │              │      ├── 是 → 凭证提取
    │              │      │
    │              │      └── 否 → 权限提升 → 凭证提取
    │              │
    │              └── 凭证提取
    │                      │
    │                      ├── 有域凭证？
    │                      │      │
    │                      │      ├── 是 → 横向移动
    │                      │      │
    │                      │      └── 否 → 继续枚举
    │                      │
    │                      └── 横向移动
    │                              │
    │                              ├── 到达高价值目标？
    │                              │      │
    │                              │      ├── 是 → 域渗透
    │                              │      │
    │                              │      └── 否 → 继续横向移动
    │                              │
    │                              └── 域渗透
    │                                      │
    │                                      └── 获得域管理员 → 完成
    │
    └── 失败 → 尝试其他方法
```

## 考试技巧

### 时间管理

```
48 小时考试建议：
├── 前 4 小时：初始访问和枚举
├── 4-12 小时：权限提升和横向移动
├── 12-24 小时：域渗透
├── 24-36 小时：继续攻击 + 写报告
├── 36-48 小时：完善报告
└── 留出休息时间！
```

### 笔记准备

```
提前准备好：
├── 所有代码模板
├── 常用命令速查
├── 工具使用说明
├── 绕过技术清单
└── 故障排除指南
```

### 遇到困难时

```
如果卡住了：
├── 重新枚举，可能遗漏了什么
├── 尝试不同的技术
├── 检查是否有其他攻击路径
├── 休息一下，换个思路
└── 不要在一个点上花太多时间
```
