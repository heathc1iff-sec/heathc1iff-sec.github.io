# WinDbg 调试器使用指南

## 什么是 WinDbg

WinDbg（Windows Debugger）是微软提供的强大调试工具，可以在汇编级别检查和修改代码执行。

### 为什么使用 WinDbg

| 特性 | WinDbg | Immunity Debugger |
|------|--------|-------------------|
| 32 位支持 | ✓ | ✓ |
| 64 位支持 | ✓ | ✗ |
| 内核调试 | ✓ | ✗ |
| 符号支持 | 优秀 | 一般 |

## 安装 WinDbg

### 方法 1：Windows SDK

1. 下载 Windows SDK
2. 安装时选择 "Debugging Tools for Windows"

### 方法 2：Microsoft Store

1. 搜索 "WinDbg Preview"
2. 安装新版 WinDbg

## 基本操作

### 附加到进程

1. 打开 WinDbg
2. File → Attach to a Process（或按 F6）
3. 选择目标进程
4. 点击 OK

```
附加后，进程会暂停执行
```

### 界面布局


## 断点操作

### 设置断点

```
bp <地址或函数名>
```

**示例**：

```
0:005> bp kernel32!WriteFile
```

在 WriteFile 函数入口设置断点。

### 断点类型

| 命令 | 类型 | 说明 |
|------|------|------|
| bp | 软件断点 | 在指定地址设置断点 |
| bu | 未解析断点 | 模块加载时设置 |
| ba | 硬件断点 | 访问/写入时触发 |

### 管理断点

```
bl              ; 列出所有断点
bc <编号>       ; 清除指定断点
bc *            ; 清除所有断点
bd <编号>       ; 禁用断点
be <编号>       ; 启用断点
```

## 执行控制

### 继续执行

```
g               ; 继续执行直到下一个断点
```

### 单步执行

```
p               ; 单步执行（不进入函数）
t               ; 单步执行（进入函数）
```

**示例**：

```
0:000> p
KERNELBASE!WriteFile:
00007fff`cff400b0 48895c2410      mov     qword ptr [rsp+10h],rbx

0:000> p
KERNELBASE!WriteFile+0x5:
00007fff`cff400b5 4889742418      mov     qword ptr [rsp+18h],rsi
```

### 执行到返回

```
pt              ; 执行到当前函数返回
```

## 查看内存

### 内存显示命令

| 命令 | 格式 | 说明 |
|------|------|------|
| db | 字节 | 显示字节和 ASCII |
| dw | 字 | 显示 16 位值 |
| dd | 双字 | 显示 32 位值 |
| dq | 四字 | 显示 64 位值 |
| dc | 双字+ASCII | 显示 32 位值和 ASCII |
| da | ASCII | 显示 ASCII 字符串 |
| du | Unicode | 显示 Unicode 字符串 |

### 示例

```
0:000> dd rsp
00000063`4c93e8c8  9a465c0e 00007ff6 0000003f 00000063
00000063`4c93e8d8  3a156590 00000230 00000004 00000000

0:000> dc rsp
00000063`4c93e8c8  9a465c0e 00007ff6 0000003f 00000063  .\F.....?...c...
00000063`4c93e8d8  3a156590 00000230 00000004 00000000  .e.:0...........

0:000> dq rsp
00000063`4c93e8c8  00007ff6`9a465c0e 00000063`0000003f
00000063`4c93e8d8  00000230`3a156590 00000000`00000004
```

### 限制显示数量

使用 L 标志指定数量：

```
0:000> dd rsp L4
00000063`4c93e8c8  9a465c0e 00007ff6 0000003f 00000063
```

## 修改内存

### 编辑命令

| 命令 | 说明 |
|------|------|
| eb | 编辑字节 |
| ew | 编辑字 (16位) |
| ed | 编辑双字 (32位) |
| eq | 编辑四字 (64位) |

### 示例

```
0:000> dd rsp L1
00000063`4c93e8c8  9a465c0e

0:000> ed rsp 0

0:000> dd rsp L1
00000063`4c93e8c8  00000000
```

## 反汇编

### 反汇编命令

```
u <地址>        ; 反汇编指定地址
u               ; 继续反汇编
uf <函数>       ; 反汇编整个函数
```

### 示例

```
0:000> u rip L5
KERNELBASE!WriteFile+0xf:
00007fff`cff400bf 57              push    rdi
00007fff`cff400c0 4883ec60        sub     rsp,60h
00007fff`cff400c4 498bd9          mov     rbx,r9
00007fff`cff400c7 4c8bda          mov     r11,rdx
00007fff`cff400ca 488bf9          mov     rdi,rcx
```

## 符号操作

### 加载符号

```
.symfix                 ; 设置微软符号服务器
.sympath+ <路径>        ; 添加符号路径
.reload                 ; 重新加载符号
```

### 查看模块

```
lm                      ; 列出所有模块
lm m kernel32           ; 查看特定模块
```

### 查看导出函数

```
x kernel32!Write*       ; 列出匹配的函数
```

## 实战：追踪 AMSI

### 1. 附加到 PowerShell

```
1. 打开 PowerShell
2. 记录进程 ID
3. WinDbg → Attach to Process → 选择 PowerShell
```

### 2. 设置断点

```
0:005> bp amsi!AmsiScanBuffer
0:005> g
```

### 3. 触发断点

在 PowerShell 中输入任意命令，断点会被触发。

### 4. 分析参数

```
0:000> r
; 查看 RCX, RDX, R8, R9 中的参数

0:000> du rdx
; 显示被扫描的缓冲区内容
```

### 5. 单步执行

```
0:000> p
; 逐步执行，观察函数行为
```

## 调试技巧

### 1. 条件断点

```
bp kernel32!WriteFile ".if (poi(rsp+8) == 0x100) {} .else {gc}"
```

只在特定条件下中断。

### 2. 日志断点

```
bp kernel32!WriteFile ".printf \"WriteFile called\\n\"; gc"
```

记录调用但不中断。

### 3. 查看调用栈

```
k               ; 显示调用栈
kp              ; 显示参数
kv              ; 显示详细信息
```
