# 客户端初始访问考试速查

## 对应课程范围

本文件串联 `1：office钓鱼.html`、`2：日历钓鱼.html`、`5：客户端反射代码攻击.html` 中的初始访问内容：Office 宏、PowerShell Shellcode Runner、HTML Smuggling、代理感知、日历钓鱼、客户端反射代码执行。

---

## 初始访问的目标

考试里客户端攻击不是为了“做一个漂亮 payload”，而是为了稳定进入下面任一状态：

| 状态 | 价值 |
|---|---|
| 能执行一次命令 | 可以验证入口和环境 |
| 能下载并执行脚本 | 可以开始主机枚举 |
| 能建立回连 | 可以进入后渗透 |
| 能捕获 NTLM | 可以破解或中继 |
| 能获取用户上下文 | 可以开始域枚举和凭证链 |

---

## 入口选择表

| 场景 | 首选入口 | 备选 |
|---|---|---|
| 目标可打开 Office 文档 | VBA 宏、文档替换、代理感知下载 | HTML Smuggling |
| 目标更可能点会议邀请 | ICS 日历钓鱼 | UNC 触发 Responder |
| Office 宏受限 | JScript、HTA、HTML Smuggling | XSL/MSHTA、第三方执行 |
| PowerShell 可用 | PowerShell Shellcode Runner、反射加载 | WMI 解链 |
| PowerShell 被拦 | JScript + DotNetToJScript、C# Loader | AppLocker 绕过路径 |
| 出网需要代理 | 代理感知 WebClient/Meterpreter | DNS/域前置 |

---

## Office / 日历 / JScript 入口选择矩阵

| 目标限制 | 优先入口 | 为什么 | 失败后切换 |
|---|---|---|---|
| 宏可启用，Office 版本明确 | VBA + Win32 API / PowerShell Shellcode Runner | 课程主线最直接 | 宏被杀时转 AV 绕过或 WMI 解链 |
| 宏被禁但用户会打开附件 | HTML Smuggling / JScript | 避开宏启用步骤 | 转 DotNetToJScript 或 HTA |
| 用户更可能处理会议邀请 | ICS 日历钓鱼 | 适合凭证捕获和链接诱导 | 捕获失败转 Office/JScript |
| PowerShell 被 AMSI 拦 | JScript + 反射 C# / DotNetToJScript | 减少直接 PowerShell 暴露 | 转 AMSI 绕过或 AppLocker 路线 |
| AppLocker/CLM 明显存在 | MSHTA/XSL/InstallUtil/Workflow/第三方执行 | 找可信执行器或受信任加载点 | 转网络/凭证/AD 路径 |
| 出网必须走代理 | 代理感知 VBA/C#/.NET | 先解决回连和下载 | 转 DNS 隧道/域前置/内网落地 |

考试可用总结：入口层、加载层、能力层分开选。入口失败时换入口；加载失败时换反射/程序集加载；回连失败时先查网络过滤。

---

## Office 宏判断链

```
能否启用宏
  -> Office 位数与 payload 位数是否匹配
  -> 是否能调用 Win32 API
  -> 是否需要代理感知
  -> 是否被 AMSI/AV 拦
  -> 是否需要 WMI 解链或反射加载
```

优先排查：

| 现象 | 排查 |
|---|---|
| 宏不运行 | 宏设置、Protected View、Mark-of-the-Web、文件来源 |
| Office 崩溃 | Shellcode 架构、API 声明、内存保护、EXITFUNC |
| 本地成功目标失败 | Office 位数、代理、AV、语言环境 |
| 有进程无回连 | 出网、监听器、证书/UA、代理配置 |

---

## 日历钓鱼判断链

日历钓鱼在考试里更像“凭证入口”而不是“执行入口”。

```
目标会处理 ICS
  -> 描述/链接/UNC 是否能触发访问
  -> 是否能捕获 NetNTLMv2
  -> Hash 能否破解
  -> 不能破解时是否能中继
  -> 拿到凭证后进入域枚举
```

记录证据：

| 证据 | 说明 |
|---|---|
| ICS 内容 | 说明触发点 |
| Responder/监听输出 | 证明认证来源 |
| 破解或中继结果 | 证明影响 |
| 后续访问 | 证明凭证价值 |

---

## 载荷设计最小原则

| 原则 | 说明 |
|---|---|
| 先验证执行 | 用最小命令确认宏/脚本能跑 |
| 再验证出网 | 下载一个小文件或访问监听器 |
| 再上完整载荷 | 不要一开始就塞复杂 Shellcode |
| 分层替换 | 载体、加载器、能力模块分开 |
| 记录失败原因 | AV、AMSI、代理、位数、策略分别记录 |
