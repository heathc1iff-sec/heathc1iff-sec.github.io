# AV 绕过决策树

## 对应课程范围

本文件串联 `7：基于编码加密或者修改特征值针对静态杀软绕过.html` 和现有 `07-杀软绕过` 目录，重点把签名定位、编码/加密、行为绕过、VBA Stomping、WMI 解链转成考试决策树。

---

## 先判断被什么拦

| 现象 | 更可能的原因 |
|---|---|
| 文件落地即被删 | 静态签名、云查杀、Mark-of-the-Web |
| 执行后被杀 | 行为检测、内存扫描、父子进程链 |
| 下载被拦 | URL/域名/代理/内容扫描 |
| PowerShell 报 AMSI | AMSI 内容扫描 |
| 只有某些路径能运行 | AppLocker/WDAC |

## 决策树

```
文件落地被杀？
  -> 先定位静态签名
  -> 加密/编码 payload
  -> 拆字符串和重构敏感 API
  -> 换编译器/模板/资源段

执行后被杀？
  -> 看父子进程链
  -> 加延迟/环境检查
  -> WMI 解链或换宿主
  -> 减少可疑 API 组合

脚本内容被杀？
  -> AMSI 绕过或换脚本宿主
  -> 混淆关键字符串
  -> 反射加载

策略不让运行？
  -> 进入 AppLocker/CLM 决策树
```

---

## 静态签名处理顺序

| 顺序 | 动作 | 目标 |
|---|---|---|
| 1 | 确认是整个文件还是局部字节触发 | 避免盲目改 |
| 2 | 二分定位签名区间 | 缩小修改范围 |
| 3 | 判断签名来自 shellcode、字符串、导入表还是资源 | 选择处理方式 |
| 4 | 加密或重写对应部分 | 保持功能可用 |
| 5 | 回归测试 | 确认绕过没有破坏载荷 |

---

## 静态签名定位记录表

| 记录项 | 写什么 |
|---|---|
| 样本版本 | 原始 payload、加密版、改字符串版、改 API 版 |
| 触发位置 | 全文件、资源段、字符串、shellcode、导入表、宏流 |
| 定位方法 | 二分、分段替换、删除测试、重新编译对比 |
| 修改动作 | 加密、重命名、拆分、动态解析、重写逻辑 |
| 功能回归 | 是否仍能执行、回连、退出不崩溃 |

---

## 行为检测处理顺序

| 行为 | 缓解方向 |
|---|---|
| Office 直接启动 PowerShell | WMI 解链、HTA/JScript、延迟加载 |
| 立即分配 RWX 内存 | 分阶段内存保护、先 RW 后 RX |
| 短时间网络回连 | 延迟、代理感知、正常 UA/证书 |
| 进程注入敏感组合 | 换目标进程、减少权限、调整 API |
| Dump LSASS | 离线策略、最小化动作、换凭证路径 |

## VBA/Office 特别注意

| 检测面 | 处理 |
|---|---|
| 宏关键字 | 字符串拆分、变量化、编码 |
| Shellcode 数组 | 加密后运行时解密 |
| Win32 API 声明 | 名称拆分、延迟解析、换调用方式 |
| 文档内容异常 | 文档替换、预文本、Office 安全机制 |
| 父进程链 | WMI 解链、脚本宿主中转 |

---

## 不要做的事

| 行为 | 问题 |
|---|---|
| 无限换 msfvenom encoder | 现代 AV 通常不吃这一套 |
| 只改扩展名 | 签名和行为不会消失 |
| 不做功能回归 | 很容易绕过检测但 payload 坏了 |
| 一上来就塞大 payload | 排错困难 |
| 所有绕过都放一个文件 | 单点失败，难替换 |

## 考前最小训练

1. 准备一个最小可运行 payload，先确认功能正常。
2. 记录未修改样本在哪个阶段被拦。
3. 只改一个变量，例如字符串、shellcode 编码、父进程链或延迟。
4. 每改一次都做功能回归。
5. 把成功版本和失败版本差异写进本页的记录表。
