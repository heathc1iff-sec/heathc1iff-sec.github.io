# 杀软检测原理与绕过基础

---

## 【Claude的学习建议】

> **杀软绕过是OSEP的核心技能！**
>
> 在真实的渗透测试中，你的payload几乎100%会被杀软检测到。
> 如果不会绕过杀软，你的攻击就无法成功。
>
> **学习目标**：
> 1. 理解杀软是如何检测恶意软件的
> 2. 知道有哪些绕过方法
> 3. 能够定位和修改被检测的特征
>
> **给零基础同学的话**：
> 杀软就像是"安检门"，它会检查你的代码是否"可疑"。
> 我们的目标是让代码看起来"正常"，骗过安检门。

---

## 杀软检测机制

> 【Claude的通俗解释】
>
> **杀软是怎么发现恶意软件的？**
>
> 想象杀软是一个"警察"，它有三种抓坏人的方法：
>
> ```
> 方法1：看照片（签名检测）
> ├── 警察有一本"通缉犯相册"
> ├── 看到一个人，就翻相册对比
> ├── 照片一样 = 抓！
> └── 缺点：换个发型就认不出了
>
> 方法2：观察行为（启发式检测）
> ├── 警察不看长相，看行为
> ├── 有人鬼鬼祟祟、东张西望
> ├── 行为可疑 = 抓！
> └── 缺点：可能抓错好人
>
> 方法3：问总部（云检测）
> ├── 警察拍照发给总部
> ├── 总部用AI分析
> ├── 总部说有问题 = 抓！
> └── 缺点：没网就不行
> ```

### 1. 签名检测

**原理**:
- 基于文件哈希（MD5/SHA-1）
- 基于特定字节序列
- 匹配已知恶意软件特征

**特点**:
- 速度快
- 误报率低
- 容易绕过

### 2. 启发式检测

**原理**:
- 在沙箱中模拟执行
- 分析行为模式
- 检测可疑操作

**特点**:
- 可检测未知恶意软件
- 资源消耗大
- 可能产生误报

### 3. 云检测

**原理**:
- 上传文件到云端分析
- 利用 AI/ML 技术
- 实时更新检测规则

**特点**:
- 检测率高
- 需要网络连接
- 成本较高

## 在线扫描服务

### VirusTotal

- 扫描 50+ 杀软引擎
- **缺点**: 会分享样本给杀软厂商
- 不适合测试自定义工具

### AntiScan.Me

- 扫描 26 个杀软引擎
- **优点**: 不分享样本
- 免费 3 次扫描

## 定位签名位置

### Find-AVSignature 脚本

**原理**: 将文件分割成多个片段，逐个扫描定位签名位置

### 使用方法

```powershell
# 导入脚本
Import-Module .\Find-AVSignature.ps1

# 第一轮：大范围扫描
Find-AVSignature -StartByte 0 -EndByte max -Interval 10000 `
    -Path C:\Tools\met.exe -OutPath C:\Tools\avtest1 -Verbose -Force

# 扫描分割文件
.\clamscan.exe C:\Tools\avtest1

# 第二轮：缩小范围
Find-AVSignature -StartByte 10000 -EndByte 20000 -Interval 1000 `
    -Path C:\Tools\met.exe -OutPath C:\Tools\avtest2 -Verbose -Force

# 第三轮：精确定位
Find-AVSignature -StartByte 18000 -EndByte 19000 -Interval 100 `
    -Path C:\Tools\met.exe -OutPath C:\Tools\avtest3 -Verbose -Force
```

### 扫描结果分析

```
met_0.bin: OK           # 空文件
met_10000.bin: OK       # 前 10000 字节无签名
met_20000.bin: FOUND    # 签名在 10000-20000 之间
```

## 常见检测点

### Meterpreter 特征

1. **Shellcode 模式** - 特定字节序列
2. **API 调用序列** - VirtualAlloc + CreateThread
3. **网络特征** - 反向连接模式
4. **字符串** - 配置信息、URL 等

### 检测规则示例

```
Win.Trojan.MSShellcode-7
```

## 绕过策略概述

> 【Claude的实战指南】
>
> **绕过杀软的核心思路**：
>
> ```
> 杀软检测的是"特征"，我们要做的就是"消除特征"
>
> 方法1：改变外表（修改签名）
> ├── 找到被检测的字节
> ├── 改成别的值
> └── 功能不变，但"长相"变了
>
> 方法2：穿上伪装（加密/编码）
> ├── 把恶意代码加密
> ├── 运行时再解密
> └── 杀软看到的是"乱码"
>
> 方法3：隐身术（内存执行）
> ├── 代码不存到硬盘
> ├── 直接在内存中运行
> └── 杀软扫描硬盘时什么都找不到
> ```
>
> **OSEP中最常用的方法**：
> 1. 自定义shellcode runner（不用现成工具）
> 2. 加密shellcode（运行时解密）
> 3. 内存执行（不落地）
> 4. 进程注入（借用合法进程）

### 1. 修改签名字节

- 定位签名位置
- 修改或替换特征字节
- 保持功能不变

### 2. 加密/编码

- XOR 加密
- Base64 编码
- 自定义编码

### 3. 混淆

- 代码混淆
- 字符串混淆
- 控制流混淆

### 4. 打包/加壳

- UPX 等打包工具
- 自定义加壳
- 运行时解密

### 5. 内存执行

- 避免文件落地
- 反射加载
- 进程注入

## 测试环境准备

### 禁用实时保护

测试时需要禁用实时扫描，只使用按需扫描：

1. 打开杀软设置
2. 关闭实时保护
3. 使用命令行扫描

### ClamAV 命令行扫描

```powershell
cd 'C:\Program Files\ClamAV\'
.\clamscan.exe C:\Tools\target_folder
```

## 注意事项

1. **测试环境隔离** - 使用虚拟机
2. **样本保护** - 不上传到 VirusTotal
3. **持续更新** - 杀软规则会更新
4. **多引擎测试** - 不同杀软检测不同
