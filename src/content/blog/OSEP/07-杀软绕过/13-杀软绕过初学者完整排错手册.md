# 杀软绕过初学者完整排错手册

## 这篇文档解决什么问题

当 payload 被杀、被删、无回连时，初学者最容易做的是“继续换免杀方法”。这篇文档的目标是让你先判断失败发生在哪一层，再决定下一步。

---

## 第一步：先判断失败发生在哪一层

| 阶段 | 现象 | 更可能的问题 |
|---|---|---|
| 下载前 | 浏览器/脚本请求不到文件 | 网络、代理、URL、证书 |
| 下载时 | 文件下载被拦 | URL 分类、内容扫描、静态签名 |
| 落地后 | 文件刚出现就被删 | 静态签名、云查杀 |
| 执行时 | 进程启动后被杀 | 行为检测、内存扫描 |
| 脚本执行前 | PowerShell 报恶意内容 | AMSI |
| 启动前 | 提示策略阻止 | AppLocker/WDAC |
| 回连时 | 执行成功但无 session | 网络过滤、监听器、payload 配置 |

---

## 文件落地前被删：静态签名排查

### 判断方法

1. 不执行文件，只保存到磁盘。
2. 看是否立即被删除或隔离。
3. 改文件名不解决问题时，说明可能是内容签名。
4. 用签名定位工具或二分法缩小触发区域。

### 处理方向

| 触发点 | 处理 |
|---|---|
| Shellcode 字节 | 加密，运行时解密 |
| 明显字符串 | 拆分、编码、运行时拼接 |
| 导入表 | 动态解析 API |
| 资源段 | 移除或重建资源 |
| 公开模板 | 重写结构，避免直接套工具输出 |

---

## 文件落地后运行被杀：行为检测排查

### 常见行为检测点

| 行为 | 为什么危险 |
|---|---|
| Office -> PowerShell | 父子进程链明显 |
| 立即创建 RWX 内存 | shellcode runner 特征明显 |
| 快速回连 | C2 行为明显 |
| 注入系统进程 | 行为高危 |
| 访问 LSASS | 凭证访问高危 |

### 排查顺序

1. 先用最小命令确认执行能力。
2. 再测试内存分配，不启动网络。
3. 再测试网络连接。
4. 最后组合完整 payload。

---

## 宏能打开但 payload 不执行

| 现象 | 排查 |
|---|---|
| 宏入口不触发 | `AutoOpen`、`Document_Open`、宏设置、Protected View |
| PowerShell 不启动 | 引号、路径、执行策略、父进程限制 |
| Web 服务没有请求 | URL、代理、DNS、出站端口 |
| 有请求但没 session | payload 架构、handler、LHOST/LPORT、EXITFUNC |
| Office 崩溃 | x86/x64、API 声明、shellcode 损坏 |

---

## C# Runner 编译成功但无回连

| 检查项 | 说明 |
|---|---|
| 编译平台 | x86/x64/AnyCPU 是否匹配 |
| shellcode 类型 | 与 handler payload 一致 |
| LHOST/LPORT | 是否能从目标访问 |
| 内存权限 | RW -> RX 是否正确 |
| 线程退出 | EXITFUNC 是否合适 |
| 当前用户 | 是否有网络和执行权限 |

---

## 为什么编码器不是万能解

msfvenom encoder 主要改变 payload 字节形态，但现代检测不只看原始字节。

| 检测方式 | 编码器能否解决 |
|---|---|
| 简单静态签名 | 可能能 |
| 行为检测 | 通常不能 |
| AMSI 内容扫描 | 通常不能 |
| 父子进程链 | 不能 |
| 网络特征 | 不能 |

所以编码器只是工具，不是免杀策略。

---

## 修改签名后程序坏了怎么办

1. 回退到上一个可运行版本。
2. 只改一个触发点。
3. 用最小 payload 做回归。
4. 分别验证执行、回连、退出。
5. 记录“绕过成功但功能失败”的版本，不要覆盖它。

## 什么时候转向 AMSI 或 AppLocker

| 现象 | 转向 |
|---|---|
| `malicious content` | `08-AMSI绕过` |
| `ConstrainedLanguage` | `08-AMSI绕过`、`09-应用白名单绕过` |
| 策略阻止运行 | `09-应用白名单绕过` |
| 执行成功无回连 | `10-网络过滤绕过` |
