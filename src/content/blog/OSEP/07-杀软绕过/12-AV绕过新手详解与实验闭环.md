# AV 绕过新手详解与实验闭环

## 对应课程范围

本文件对应 `7：基于编码加密或者修改特征值针对静态杀软绕过.html`，并补充说明 `8：windows防御机制绕过（amsi+clm）.html` 当前疑似重复导出的情况。

AV 绕过不是背“某个免杀技巧”，而是判断样本在哪个阶段被检测，然后只改最小必要部分。

---

## 一句话理解

AV 绕过的本质是：

```
定位被拦阶段
  -> 判断检测类型
  -> 最小化修改
  -> 功能回归
  -> 记录成功和失败差异
```

初学者不要一上来就混淆、加密、换模板全做一遍。那样即使成功，也不知道到底是哪一步起作用。

---

## 检测类型

| 类型 | 触发时机 | 常见表现 | 处理方向 |
|---|---|---|---|
| 静态签名 | 文件落地或扫描时 | 文件被删、下载被拦 | 定位签名、加密、拆字符串、改结构 |
| 启发式检测 | 执行前后 | 可疑 API、可疑结构 | 重构逻辑、减少明显特征 |
| 行为检测 | 运行时 | 进程被杀、回连被阻断 | 延迟、解链、换宿主、降低异常行为 |
| 内存扫描 | payload 已进入内存后 | 执行中断、线程异常 | 调整内存权限、分阶段、换加载方式 |
| 云查杀 | 文件刚出现或上传后 | 本地短暂存在后被删 | 改哈希、减少公开模板特征 |
| AMSI | 脚本内容执行前 | PowerShell/JScript 报恶意内容 | 转 `08-AMSI绕过` |
| AppLocker/WDAC | 程序启动前 | 策略阻止 | 转 `09-应用白名单绕过` |

## 实验记录模板

| 记录项 | 内容 |
|---|---|
| 样本版本 | 原始、编码、加密、改字符串、改 API、换宿主 |
| 被拦阶段 | 下载、落地、打开、执行、回连 |
| 检测类型 | 静态、行为、AMSI、AppLocker、网络 |
| 修改动作 | 只写本次改了什么 |
| 功能回归 | 是否还能执行、是否回连、是否崩溃 |
| 结论 | 这次修改解决了什么，没有解决什么 |

## 静态签名定位思路

1. 先确认文件是否“落地即杀”。
2. 如果是，尝试缩小触发区域，例如字符串、shellcode 字节、资源、导入表。
3. 每次只替换一部分。
4. 找到触发点后，再决定加密、编码、拆分还是重写。
5. 修改后必须重新验证功能。

如果改完能绕过但 payload 坏了，这不算完成。

## 行为检测排错

| 行为 | 为什么可疑 | 初学者处理方向 |
|---|---|---|
| Office 直接启动 PowerShell | 父子进程链明显 | WMI 解链、脚本宿主中转、降低动作 |
| 立即分配 RWX 内存 | 常见 shellcode 特征 | 先 RW 写入，再改 RX |
| 快速外连陌生地址 | C2 行为明显 | 代理感知、正常 UA、延迟 |
| 注入敏感进程 | 目标进程选择异常 | 换低风险目标，先本地验证 |
| 访问 LSASS | 凭证访问高危 | 优先离线 dump 或替代凭证路线 |

## OSEP 考试重点

| 重点 | 要求 |
|---|---|
| 会定位 | 不盲目换工具，先判断阶段 |
| 会记录 | 每个改动都要能解释 |
| 会回归 | 绕过检测后功能必须还在 |
| 会切换 | AV、AMSI、AppLocker、网络过滤要分清 |
| 会降级 | 先用最小动作验证执行，再上完整链 |
