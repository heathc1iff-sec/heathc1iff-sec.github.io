# 出网受限决策树

## 对应课程范围

本文件串联 `10：绕过网络过滤器.html` 的 DNS Filters、Web Proxies、IDS/IPS、Full Packet Capture、HTTPS Inspection、Domain Fronting、DNS Tunneling。

---

## 先判断是哪种出网问题

| 现象 | 更可能 |
|---|---|
| DNS 解析失败 | DNS 过滤、内网 DNS 限制 |
| HTTP 可出但 payload 不回 | 代理、User-Agent、URL 分类 |
| HTTPS 证书异常 | HTTPS Inspection |
| 回连后立刻断 | IDS/IPS、证书/流量特征 |
| 只有 53/80/443 可出 | 严格出口控制 |
| 只能访问白名单域名 | 域名分类或代理白名单 |

---

## 决策树

```
先测 DNS
  -> 能解析外部域名吗
  -> 能指定 DNS 服务器吗
  -> 是否能做 DNS 隧道

再测 HTTP/HTTPS
  -> 是否需要代理
  -> SYSTEM 和用户代理是否不同
  -> User-Agent 是否被拦
  -> 是否存在 HTTPS Inspection

再测流量特征
  -> 默认 Meterpreter 证书是否被拦
  -> URI/UA/证书是否像正常业务
  -> 是否需要域前置或自定义证书
```

---

## 代理环境检查

| 上下文 | 检查点 |
|---|---|
| 普通用户 | IE/WinHTTP/浏览器代理 |
| SYSTEM | 通常没有用户代理配置，需要读取其他用户或设置默认代理 |
| PowerShell | `WebClient` 默认代理、显式代理 |
| Meterpreter | Payload 和 handler 都要配置对应代理/UA |
| 自定义 C# | `WebRequest.DefaultWebProxy`、凭证、认证代理 |

---

## HTTPS Inspection 判断

| 发现 | 说明 |
|---|---|
| 证书颁发者是企业 CA | 流量被中间解密 |
| 自签名/默认 Meterpreter 证书被拦 | 需要自定义证书或模拟真实证书 |
| TLS 握手失败 | 协议版本、证书、SNI、代理 |
| 浏览器能访问但 payload 不行 | UA、代理认证、TLS 指纹差异 |

---

## Full Packet Capture 下的操作约束

| 现象 | 验证 | 风险 | 下一步 |
|---|---|---|---|
| 所有出网都有完整载荷记录 | 小文件、多协议探测都被记录 | 明文命令、固定 URI、固定 User-Agent 会暴露 | 降低频率，使用短请求，优先代理感知 HTTPS |
| DNS 请求全量记录 | 随机子域、长 TXT 查询明显 | DNS 隧道容易形成高熵特征 | 只做低速控制或改用允许的 HTTP/HTTPS |
| 大文件下载失败或被告警 | 分片下载仍被截断 | 工具和 Payload 体积过大 | 分层加载，先传最小能力模块 |
| 目标能访问互联网但回连失败 | 只允许浏览器/代理出网 | 直连 C2 暴露且不通 | 使用系统代理、域前置或反向 RDP/Chisel |

---

## HTTPS Inspection 后的路线切换

| 发现 | 说明 | 下一步 |
|---|---|---|
| 企业 CA 替换证书 | TLS 被中间设备解密 | 让工具信任企业 CA，或改走浏览器/系统代理路径 |
| SNI/Host 不一致被拦 | 域前置失败概率高 | 换允许域名、DNS 隧道或代理认证 |
| 证书校验错误但浏览器正常 | 工具未使用系统证书库 | 导入企业 CA，或使用 PowerShell/.NET 默认代理与证书链 |
| HTTPS 可出但下载内容被改写 | 内容检查或 DLP | 换小块、换编码、换静态资源形态，记录失败证据 |

---

## DNS 隧道适用条件

| 条件 | 说明 |
|---|---|
| 目标能解析外部域名 | DNS 查询能出网 |
| 你控制权威域名 | 能把子域查询导向隧道服务器 |
| 流量低速可接受 | DNS 隧道速度慢 |
| 目标允许 TXT/CNAME/MX 等查询 | dnscat2 等工具依赖查询类型 |

不适合：

| 情况 | 原因 |
|---|---|
| 大文件传输 | 速度慢且显眼 |
| 长期高频 C2 | 容易被检测 |
| DNS 严格代理化 | 可能看不到真实外部查询 |

---

## 域前置适用条件

| 条件 | 说明 |
|---|---|
| CDN/前置域可用 | SNI 和 Host 分离仍成立 |
| 目标允许访问前置域 | 白名单或高信誉域名 |
| 后端能接收真实 Host | CDN 配置正确 |
| TLS/SNI/Host 能被正确设置 | 客户端和 payload 支持 |
