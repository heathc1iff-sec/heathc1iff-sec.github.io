# 网络过滤新手详解与出网排错

## 对应课程范围

本文件对应 `10：绕过网络过滤器.html`，用于把 DNS 过滤、Web 代理、IDS/IPS、HTTPS 检查、域前置和 DNS 隧道整理成初学者排错地图。

---

## 一句话理解

网络过滤绕过的本质是：

```
先判断哪条路能出去
  -> DNS 是否通
  -> HTTP/HTTPS 是否通
  -> 是否需要代理
  -> TLS/证书是否被检查
  -> 再选择 C2、下载、隧道或域前置
```

不要看到无回连就立刻换 payload。很多时候 payload 是好的，网络路径不对。

---

## 常见过滤层

| 层级 | 它管什么 | 常见现象 |
|---|---|---|
| DNS 过滤 | 域名解析 | 域名解析失败、特定域名被拦 |
| Web 代理 | HTTP/HTTPS 出站 | 需要认证、直接连接失败 |
| IDS/IPS | 流量内容和行为 | 连接被重置、特征流量被拦 |
| TLS 检查 | HTTPS 证书和内容 | 证书异常、SNI/域名不匹配 |
| URL 分类 | 域名信誉和分类 | 新域名、可疑路径被拦 |
| 出站端口 | 允许的端口 | 只有 80/443/53 可用 |

## 出网排错顺序

| 顺序 | 检查 | 目的 |
|---|---|---|
| 1 | IP 是否可达 | 区分路由问题和协议问题 |
| 2 | DNS 是否可解析 | 判断域名过滤 |
| 3 | 端口是否开放 | 判断出站端口 |
| 4 | HTTP/HTTPS 是否需要代理 | 判断代理认证和系统代理 |
| 5 | TLS 是否异常 | 判断 HTTPS 检查 |
| 6 | 特定路径是否被拦 | 判断 URL/内容过滤 |
| 7 | C2 特征是否被拦 | 判断工具流量特征 |

## 常用检查命令

```powershell
Resolve-DnsName example.com
Test-NetConnection example.com -Port 443
Invoke-WebRequest https://example.com -UseBasicParsing
netsh winhttp show proxy
```

```cmd
nslookup example.com
ping example.com
curl -I https://example.com
```

在受限环境里，`ping` 失败不代表 HTTP 失败。ICMP 经常被禁，所以要分协议检查。

## 代理判断

| 现象 | 说明 | 下一步 |
|---|---|---|
| 浏览器能访问，命令行不行 | 浏览器使用用户代理配置 | 查 HKCU 代理 |
| SYSTEM 下不能访问 | SYSTEM 没有用户代理 | 查 WinHTTP 代理 |
| 返回 407 | 代理需要认证 | 判断当前凭证是否可用 |
| HTTPS 报证书异常 | TLS 被检查或证书链不信任 | 使用合法证书或调整通信方式 |

---

## DNS 隧道判断

| 条件 | 判断 |
|---|---|
| DNS 能出网 | `nslookup` 能解析外部域名 |
| 能控制域名 | 可以设置 NS 记录到自己的服务端 |
| 查询能到服务端 | 服务端能看到子域名查询 |
| 数据量可接受 | DNS 隧道慢，不适合大文件 |
| 检测风险可接受 | 高频长子域名很明显 |

DNS 隧道是备用通道，不是默认首选。能走正常 HTTPS 且稳定时，通常先解决代理和 TLS。

---

## 域前置判断

| 条件 | 说明 |
|---|---|
| CDN 支持相关行为 | 不是所有 CDN 都支持 |
| 前置域名被允许 | 目标环境允许访问高信誉域名 |
| Host/SNI/证书关系正确 | 配错会直接失败 |
| C2 支持该通信方式 | 工具侧也要兼容 |

域前置重点不是命令，而是理解前置域名、真实后端、TLS/SNI、HTTP Host 的关系。

## OSEP 考试重点

| 重点 | 要求 |
|---|---|
| 会分层排错 | DNS、端口、代理、TLS、内容过滤分别检查 |
| 会记录证据 | 每次失败的错误码、协议、目标、时间 |
| 会切换路径 | HTTPS、代理、域前置、DNS 隧道 |
| 会先小后大 | 先请求小文本，再上完整 C2 |
| 会联动初始访问 | Office/JScript 回连失败时先查网络 |
