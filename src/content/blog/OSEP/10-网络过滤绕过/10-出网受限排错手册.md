# 出网受限排错手册

## 先判断哪一层被拦

| 层 | 问题 | 常见现象 |
|---|---|---|
| DNS | 域名解析 | 域名解析失败 |
| TCP | 端口连接 | 连接超时或拒绝 |
| HTTP | Web 请求 | 返回代理或过滤页面 |
| HTTPS | TLS/SNI/证书 | 证书错误、握手失败 |
| 代理 | 认证或配置 | 浏览器能出，payload 不能出 |
| IDS/IPS | 流量特征 | 连接被重置 |

---

## DNS 能解析但不能连接

可能原因：

1. 出站端口被拦。
2. 目标 IP 被防火墙拦。
3. 需要走代理。
4. DNS 解析到错误地址。

检查：

```powershell
Resolve-DnsName example.com
Test-NetConnection example.com -Port 80
Test-NetConnection example.com -Port 443
```

---

## IP 能连但域名不能连

可能原因：

| 原因 | 说明 |
|---|---|
| DNS 过滤 | 域名被分类或拦截 |
| 内部 DNS 限制 | 只能解析内部域 |
| SNI/Host 检查 | 直接 IP 可连但 HTTPS 域名失败 |

---

## HTTP 能出但 HTTPS 不行

排查：

1. TLS 版本是否兼容。
2. 证书是否被信任。
3. 是否存在 HTTPS inspection。
4. SNI 是否正确。
5. 代理是否只允许特定域名。

---

## 浏览器能出但 payload 不能出

这是 OSEP 里很常见的情况。

| 原因 | 检查 |
|---|---|
| 浏览器使用用户代理 | 查 HKCU 代理 |
| payload 不读取代理 | 换代理感知通信 |
| SYSTEM 没有代理 | 查 WinHTTP 代理 |
| User-Agent 被拦 | 模拟正常 UA |
| TLS 指纹异常 | 换通信方式 |

---

## 用户能出但 SYSTEM 不能出

| 用户上下文 | 代理来源 |
|---|---|
| 普通用户 | HKCU Internet Settings |
| SYSTEM | WinHTTP 或机器级配置 |

检查：

```powershell
netsh winhttp show proxy
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings"
```

---

## 代理认证导致失败

| 现象 | 判断 |
|---|---|
| HTTP 407 | 代理需要认证 |
| 浏览器弹认证 | 当前用户未自动认证 |
| 命令行失败 | 工具不支持代理认证 |

处理方向：

1. 使用支持代理的下载方式。
2. 复用当前用户上下文。
3. 使用系统代理配置。
4. 改用 DNS 或其他允许通道。

---

## DNS 隧道失败排查

| 检查 | 目的 |
|---|---|
| 域名 NS 记录 | 是否指向你的服务端 |
| 服务端是否收到查询 | 判断流量是否出网 |
| 子域名长度 | 避免过长被丢弃 |
| 查询频率 | 避免过于异常 |
| 防火墙策略 | 是否只允许内部 DNS |

---

## 域前置失败排查

| 检查 | 说明 |
|---|---|
| CDN 是否支持 | 很多现代 CDN 已限制传统域前置 |
| SNI | TLS 握手使用的域名 |
| Host Header | HTTP 层路由使用的域名 |
| 证书 | 是否匹配前置域名 |
| C2 配置 | 工具参数是否对应 |
