# Module 10: 网络过滤绕过

---

## 【Claude的学习建议】

> **网络过滤绕过是高级渗透测试的必备技能！**
>
> 在企业环境中，即使你拿下了内网的一台电脑，你的C2流量还要穿过层层防护才能出去。
> 如果不会绕过网络过滤，你的shell就是"哑巴"——能进去但出不来。
>
> **学习目标**：
> 1. 理解企业网络边界有哪些防护
> 2. 掌握DNS隧道、域前置等绕过技术
> 3. 知道什么场景用什么技术
>
> **给零基础同学的话**：
> 想象你在一个戒备森严的监狱里（企业内网），你想给外面的同伙（C2服务器）传递消息：
> - 直接喊话？会被狱警（防火墙）拦截
> - 写信？会被检查（IDS）
> - 怎么办？把消息藏在合法的东西里！
>
> 这就是网络过滤绕过的核心思想：**把恶意流量伪装成正常流量**

---

## 第一节：网络过滤概述

### 为什么需要绕过网络过滤？

在企业环境中，即使我们成功获取了内部系统的访问权限，我们的 C2 (Command & Control) 流量仍然需要穿越多层网络防御才能到达外部。

```
内部主机 → DNS过滤 → 内部防火墙 → SSL检查 → Web代理/IDS → 外部防火墙 → 互联网
```

> 【Claude的重要概念】
>
> **什么是C2流量？**
>
> C2 = Command & Control = 命令与控制
>
> ```
> 攻击者的电脑（C2服务器）
>        ↑↓ C2流量（命令和结果）
> 被控制的电脑（受害者）
>
> C2流量包含：
> ├── 攻击者发送的命令（如：执行whoami）
> └── 受害者返回的结果（如：CORP\admin）
> ```
>
> 如果C2流量被阻断，攻击者就无法控制受害者电脑了！

## 一、企业网络边界架构

### 1.1 典型的网络边界防御

```
┌─────────────────────────────────────────────────────────────┐
│                    互联网边界架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  内部设备                                                    │
│     │                                                       │
│     ▼                                                       │
│  ┌─────────────┐                                            │
│  │  DNS 过滤器  │ ← 域名黑名单/分类过滤                       │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 内部防火墙   │ ← IP/端口过滤                              │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ SSL 检查设备 │ ← 解密/重加密 HTTPS 流量                   │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐     ┌─────────────┐                        │
│  │ Web 代理/IDS │ ←→ │ 全包捕获设备 │                        │
│  └──────┬──────┘     └─────────────┘                        │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 外部防火墙   │                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│      互联网                                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 各防御层功能

| 防御层 | 功能 | 检测方式 |
|--------|------|----------|
| DNS 过滤器 | 阻止恶意域名解析 | 域名黑名单、分类过滤 |
| 内部防火墙 | 限制出站流量 | IP/端口规则 |
| SSL 检查 | 检查加密流量 | 中间人解密 |
| Web 代理 | 过滤 HTTP/HTTPS | URL 过滤、内容检查 |
| IDS/IPS | 检测恶意流量 | 签名匹配、深度包检测 |
| 全包捕获 | 取证分析 | 流量记录 |
| 外部防火墙 | 最后一道防线 | IP/端口规则 |

## 二、DNS 过滤器

### 2.1 工作原理

DNS 过滤器检查 DNS 请求，阻止对恶意域名的解析。

```
客户端 DNS 请求
       │
       ▼
┌─────────────────┐
│   DNS 过滤器    │
│                 │
│ 检查域名是否在: │
│ - 黑名单中      │
│ - 恶意分类中    │
│ - 新注册域名    │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
  允许      阻止
    │         │
    ▼         ▼
 正常解析   返回 Sinkhole IP
            或无响应
```

### 2.2 常见 DNS 过滤服务

| 服务 | 说明 |
|------|------|
| OpenDNS (Cisco Umbrella) | 企业级 DNS 过滤 |
| Google Safe Browsing | 恶意网站检测 |
| VirusTotal | 多引擎域名检测 |
| IPVoid | 域名信誉查询 |

### 2.3 Sinkhole 技术

当域名被阻止时，DNS 服务器可能：
- 返回空响应
- 返回 Sinkhole IP（假 IP）
- 重定向到警告页面
- 重定向到监控设备

```bash
# 使用 Google DNS 解析（未过滤）
nslookup www.example.com 8.8.8.8

# 使用 OpenDNS 解析（可能被过滤）
nslookup www.example.com 208.67.222.222
```

## 三、Web 代理

### 3.1 工作原理

Web 代理作为中间人，转发并检查 HTTP/HTTPS 流量。

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  客户端   │ ──→ │ Web 代理  │ ──→ │ 目标服务器│
│          │ ←── │          │ ←── │          │
└──────────┘     └──────────┘     └──────────┘
                      │
                      ▼
               ┌──────────────┐
               │ 检查内容:     │
               │ - URL 过滤   │
               │ - 分类检查   │
               │ - 内容检查   │
               │ - User-Agent │
               └──────────────┘
```

### 3.2 代理检查项目

| 检查项 | 说明 |
|--------|------|
| URL | 完整 URL 路径检查 |
| 域名分类 | 网站类别（如：赌博、恶意软件） |
| User-Agent | 浏览器类型检查 |
| HTTP 头 | 各种 HTTP 头字段 |
| 内容 | 响应内容检查 |

### 3.3 常见代理产品

- Symantec Bluecoat
- Cisco Web Security
- Zscaler
- Squid (开源)

## 四、IDS/IPS 传感器

### 4.1 IDS vs IPS

| 特性 | IDS | IPS |
|------|-----|-----|
| 部署方式 | 旁路监听 | 串联部署 |
| 阻断能力 | 只告警 | 可阻断 |
| 性能影响 | 低 | 较高 |
| 实时性 | 延迟告警 | 实时阻断 |

### 4.2 检测方式

```
网络流量
    │
    ▼
┌─────────────────┐
│  数据包重组     │ ← 处理 IP 分片
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  深度包检测     │ ← 检查应用层数据
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  签名匹配       │ ← 与已知恶意模式比对
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
  正常      告警/阻断
```

### 4.3 常见 IDS/IPS 产品

- Snort (开源)
- Suricata (开源)
- Cisco Firepower
- Palo Alto Networks

## 五、HTTPS 检查

### 5.1 工作原理

HTTPS 检查设备作为中间人，解密并重新加密流量。

```
┌──────────┐     ┌──────────────┐     ┌──────────┐
│  客户端   │ ←→ │ HTTPS 检查设备│ ←→ │ 目标服务器│
└──────────┘     └──────────────┘     └──────────┘
                        │
                        ▼
                 ┌──────────────┐
                 │ 1. 解密流量   │
                 │ 2. 检查内容   │
                 │ 3. 重新加密   │
                 └──────────────┘
```

### 5.2 实现方式

1. **企业 CA 证书**：客户端信任企业 CA
2. **动态证书生成**：为每个域名生成证书
3. **透明代理**：客户端无感知

### 5.3 检测 HTTPS 检查

```powershell
# 检查证书颁发者
# 如果是企业 CA 而非网站原始 CA，说明存在 HTTPS 检查
```

## 七、绕过策略概述

### 7.1 总体思路

```
目标: 让我们的流量看起来像正常流量

策略:
1. 使用合法域名 → 绕过 DNS 过滤
2. 正确的 User-Agent → 绕过代理检查
3. 修改流量特征 → 绕过 IDS/IPS
4. 使用加密 → 绕过内容检查
5. 域前置 → 隐藏真实目标
6. DNS 隧道 → 绕过所有 HTTP 过滤
```

### 7.2 各技术适用场景

| 技术 | 适用场景 | 复杂度 |
|------|----------|--------|
| 域名信誉管理 | DNS 过滤 | 低 |
| 代理感知 Payload | Web 代理 | 低 |
| 自定义证书 | IDS/IPS | 中 |
| 域前置 | 多层过滤 | 高 |
| DNS 隧道 | 严格过滤环境 | 高 |
