# PowerShell反射 - 新手章节指南

## 这个目录是干什么的？

PowerShell 反射技术是 OSEP 的核心技能之一。它让你能够在内存中直接执行代码，而不需要把文件保存到磁盘，从而绕过很多安全检测。

---

## 为什么要学这个？

```
PowerShell 反射的威力：
├── 代码完全在内存中执行
├── 不会在磁盘上留下痕迹
├── 可以绕过很多杀软检测
├── 是高级渗透测试的必备技能

现实应用：
├── 执行 Mimikatz 而不落地
├── 加载自定义工具到内存
├── 绕过应用白名单
└── 实现无文件攻击
```

---

## 通俗理解核心概念

### 什么是反射（Reflection）？

```
反射 = 程序在运行时"审视"和"操作"自己的能力

类比：
├── 普通程序 = 按照剧本演戏
├── 反射 = 演员可以临时修改剧本
└── 可以在运行时动态加载和执行代码

在 PowerShell 中的应用：
├── 动态创建类型
├── 动态调用方法
├── 动态加载 DLL
└── 绕过静态检测
```

### Add-Type vs 纯反射

```
Add-Type 方法：
├── 简单易用
├── 把 C# 代码编译成临时 DLL
├── 会在磁盘上留下痕迹
└── 可能被杀软检测

纯反射方法：
├── 更复杂
├── 完全在内存中操作
├── 不会留下磁盘痕迹
└── 更难被检测

选择建议：
├── 学习阶段用 Add-Type（简单）
└── 实战中用纯反射（隐蔽）
```

### DelegateType 反射

```
什么是 DelegateType？
├── 一种函数指针的包装
├── 可以指向任何函数
├── 用于动态调用 Win32 API

为什么需要它？
├── 直接调用 API 会被检测
├── 通过 Delegate 间接调用
├── 更难被静态分析
└── 是高级绕过技术的基础
```

## 实践任务清单

### 任务1：理解 Add-Type 方法

```powershell
# 使用 Add-Type 调用 Win32 API
# 这是最简单的方法

$code = @"
using System;
using System.Runtime.InteropServices;

public class Win32 {
    [DllImport("user32.dll")]
    public static extern int MessageBox(IntPtr hWnd, String text, String caption, int type);
}
"@

Add-Type $code
[Win32]::MessageBox(0, "Hello from PowerShell!", "测试", 0)
```

### 任务2：使用反射获取类型信息

```powershell
# 反射的基本用法：获取类型信息

# 获取 String 类型的所有方法
[String].GetMethods() | Select-Object Name

# 获取 Process 类型的所有属性
[System.Diagnostics.Process].GetProperties() | Select-Object Name

# 动态调用方法
$str = "hello"
$method = [String].GetMethod("ToUpper", [Type]::EmptyTypes)
$result = $method.Invoke($str, $null)
Write-Host $result  # 输出: HELLO
```

### 任务3：使用反射调用 Win32 API

```powershell
# 纯反射方式调用 Win32 API（不使用 Add-Type）

# 获取 GetProcAddress 和 GetModuleHandle
$systemAssembly = [AppDomain]::CurrentDomain.GetAssemblies() |
    Where-Object { $_.GlobalAssemblyCache -and $_.Location.Split('\\')[-1] -eq 'System.dll' }

$unsafeNativeMethods = $systemAssembly.GetType('Microsoft.Win32.UnsafeNativeMethods')

$getModuleHandle = $unsafeNativeMethods.GetMethod('GetModuleHandle')
$getProcAddress = $unsafeNativeMethods.GetMethod('GetProcAddress', [Type[]]@([System.Runtime.InteropServices.HandleRef], [String]))

# 获取 kernel32.dll 的句柄
$kernel32 = $getModuleHandle.Invoke($null, @("kernel32.dll"))
$kernel32Handle = New-Object System.Runtime.InteropServices.HandleRef -ArgumentList @($null, $kernel32)

# 获取 VirtualAlloc 的地址
$virtualAllocAddr = $getProcAddress.Invoke($null, @($kernel32Handle, "VirtualAlloc"))

Write-Host "VirtualAlloc 地址: $virtualAllocAddr"
```

### 任务4：理解 Delegate 类型

```powershell
# 创建一个 Delegate 类型来调用函数

$delegateType = [System.Linq.Expressions.Expression].Assembly.GetType('System.Runtime.CompilerServices.RuntimeHelpers').GetMethod('GetUninitializedObject').Invoke($null, @([System.MulticastDelegate]))
```

## 学习要点总结

```
反射技术的核心步骤：
1. 获取程序集（Assembly）
2. 获取类型（Type）
3. 获取方法（Method）
4. 调用方法（Invoke）

为什么反射能绕过检测：
├── 不使用明文的 API 名称
├── 动态构建调用
├── 代码在运行时才确定
└── 静态分析无法检测
```
