# 应用白名单失败排错手册

## 运行被拒绝时先看哪里

当程序或脚本被拒绝运行时，先不要换 LOLBin。先确认规则来源：

| 检查 | 目的 |
|---|---|
| 错误提示 | 看是 AppLocker、WDAC 还是权限问题 |
| 事件日志 | 确认被哪类规则阻止 |
| 当前用户和组 | 规则可能按用户/组应用 |
| 文件路径 | 判断路径规则 |
| 文件签名 | 判断发布者规则 |
| 文件哈希 | 判断哈希规则 |

---

## AppLocker 事件日志怎么看

常见日志：

```powershell
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/EXE and DLL" -MaxEvents 10
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/MSI and Script" -MaxEvents 10
```

看三个信息：

| 字段 | 作用 |
|---|---|
| 被阻止的文件 | 确认对象 |
| 规则集合 | EXE、DLL、Script、MSI |
| 用户 SID/账户 | 确认规则应用对象 |

---

## 规则类型意味着什么

| 规则类型 | 含义 | 绕过思路 |
|---|---|---|
| 路径 | 允许或拒绝某些目录 | 找可写允许目录 |
| 哈希 | 精确匹配文件 | 改文件会失效，也可能绕过旧哈希 |
| 发布者 | 按签名厂商/产品/版本 | 找同签名可信程序或加载点 |

---

## 为什么管理员能跑，普通用户不能跑

可能原因：

1. 规则按用户或组不同。
2. 管理员能写入受信任目录。
3. 管理员能绕过某些策略。
4. 普通用户触发 CLM，管理员没有。

考试里要记录当前用户，不要把管理员测试结果直接套到普通用户。

---

## 为什么复制到可信目录仍然失败

| 原因 | 说明 |
|---|---|
| 目录并不在允许规则中 | 你以为可信，规则不认 |
| 文件类型被其他规则拦 | EXE 允许不代表脚本允许 |
| 哈希/发布者规则优先 | 路径不是唯一判断 |
| MOTW 或 SmartScreen | 下载来源影响执行 |
| 权限不足 | 文件没有真正写入或执行权限 |

---

## 为什么 DLL 路线不生效

| 原因 | 排查 |
|---|---|
| 目标程序没有加载该 DLL | Procmon 看加载路径 |
| 位数不匹配 | x86 程序加载 x86 DLL |
| 导出函数不对 | 检查 DLL 导出 |
| 搜索顺序不成立 | 当前目录、系统目录、PATH |
| DLL 规则也严格 | 事件日志确认 |

---

## 为什么 InstallUtil/MSHTA/XSL 没触发

| 路线 | 常见问题 |
|---|---|
| InstallUtil | .NET 版本、安装器类、路径规则、参数错误 |
| MSHTA | 脚本规则、AMSI、HTA 内容格式、出网 |
| XSL | 工具版本、脚本扩展禁用、远程 XSL 加载失败 |
| Workflow Compiler | PowerShell/CLM、路径、编译限制 |

---

## 从失败现象反推下一步

| 现象 | 下一步 |
|---|---|
| EXE 被拦，脚本可跑 | JScript/MSHTA |
| 脚本被拦，DLL 可加载 | DLL 搜索顺序/第三方加载点 |
| PowerShell CLM | Custom Runspace 或非 PowerShell |
| 第三方程序可运行 | 反编译找加载点 |
| 执行成功但被杀 | AV/AMSI |
| 执行成功无回连 | 网络过滤 |
